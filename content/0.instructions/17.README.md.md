Github's take About READMEs
::

### Structure

1. **Project Description**: A brief overview of what the project does and why it exists.
2. **Installation Instructions**: Steps for getting the project up and running on a local machine.
3. **Usage**: How to use the project, often with code examples.
4. **Contributing**: Guidelines for contributing to the project (e.g., creating issues, submitting pull requests).
5. **License**: The project’s licensing information to define its terms of use.

### Why It’s Important
- **Onboarding**: Helps new developers get started quickly.
- **Documentation**: Provides a central place for important project info.
- **Collaboration**: Explains how others can contribute to the project.
