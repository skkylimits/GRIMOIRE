The `.gitignore` file tells Git which files or directories to ignore in version control, preventing unnecessary or sensitive files (like build outputs, logs, or environment configs) from being tracked and shared in the repository.

## ::callout

icon: i-vscode-icons:file-type-git
target: \_blank
to: https://docs.github.com/en/get-started/getting-started-with-git/ignoring-files

---

[Learn more about how the `.gitignore` file works.](https://docs.github.com/en/get-started/getting-started-with-git/ignoring-files)
::

```[.gitignore]
# Nuxt dev/build outputs
.output
.data
.nuxt
.nitro
.cache
dist

# Node dependencies
node_modules

# Logs
logs
*.log

# Misc
.DS_Store
.fleet
.idea

# Local env files
.env
.env.*
!.env.example

# VSC
.history
```
