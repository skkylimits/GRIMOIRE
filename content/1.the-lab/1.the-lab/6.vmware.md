## Install Windows

- **Download het Windows 11 ISO-bestand**:
   - Ga naar de [Microsoft Windows 11 downloadpagina](https://www.microsoft.com/en-us/software-download/windows11).
   - Kies "Download Windows 11 Disk Image (ISO)" en sla het bestand op je computer op.

## **Stap 1: VMware Player Starten**
1. Open **VMware Player**.
2. Klik op **"Create a New Virtual Machine"**.

## **Stap 2: Kies de Configuratie**
1. Selecteer **Custom (Advanced)** om volledige controle over de configuratie te hebben.
2. Klik op **Next**.

### **Virtual Machine Hardware Compatibility**
1. Kies de VMware-versie die gelijk is aan of hoger is dan je huidige versie van VMware (bijvoorbeeld VMware Workstation 16.x of hoger).
2. Klik op **Next**.

## **Stap 3: Kies de Installatiemedia**
1. Selecteer **Installer Disc Image File (ISO)**.
2. Zoek en selecteer het gedownloade Windows 11 ISO-bestand.
3. Controleer of de Windows 11 ISO correct wordt gedetecteerd:
   - Je zou **Windows 11 x64** moeten zien.
4. Klik op **Next**.

## **Stap 4: Naam en Locatie**
1. Geef je VM een naam of laat deze standaard staan.
2. Kies een locatie op je harde schijf om de VM-bestanden op te slaan (standaard is aanbevolen).
3. Klik op **Next**.

## **Stap 5: Encryption Type**
1. Kies de optie voor encryptie:
   - Selecteer **Encrypt all the files**.
2. Voer een 8-cijferige pincode in.
3. **Uncheck** de optie **Use Credential Manager**.
4. Klik op **Next**.

### **UEFI en Secure Boot**
1. Zorg ervoor dat **UEFI Firmware** is geselecteerd.
2. Vink **Enable Secure Boot** aan (vereist voor Windows 11).
3. Klik op **Next**.

## **Stap 6: CPU en Geheugen Instellen**
1. **Processors**:
   - Kies minimaal 2 processors, aanbevolen is **3**.
2. **Cores**:
   - Stel in op **3 cores per processor**.
3. **RAM**:
   - Toewijzing: **16,238 MB** (aanbevolen).
4. Klik op **Next**.

## **Stap 7: Netwerk en Controller**
1. Selecteer **Use NAT** als netwerkoptie.
2. Klik op **Next**.

### **I/O Controller Type**
1. Kies **LSI Logic SAS** (aanbevolen).
2. Klik op **Next**.

### **Disk Type**
1. Kies **NVMe** als schijftype (aanbevolen voor prestaties).
2. Klik op **Next**.

## **Stap 8: Schijfinstellingen**
1. Kies **Create a New Virtual Disk**.
2. **Grootte**:
   - Minimaal: 64 GB.
   - Aanbevolen: **128 GB of meer**.
3. Kies **Store virtual disk as a single file**.
4. Klik op **Next**.

### **Disk Bestandsnaam**
1. Laat de standaard bestandsnaam staan (de naam van de virtuele machine).
2. Klik op **Next**.

## **Stap 9: Hardwarecontrole**
Controleer de hardware-instellingen:
1. Bevestig dat:
   - Het Windows 11 ISO-bestand is geselecteerd bij **CD/DVD Drive**.
   - Netwerkadapter is ingesteld op **NAT**.
   - **UEFI en Secure Boot** zijn ingeschakeld.
2. Klik op **Finish**.

## **Stap 10: VM Starten**
1. Selecteer je nieuwe VM in VMware Player.
2. Klik op **Power on this virtual machine**.
3. Volg de Windows 11 installatieprocedure.

### **Problemen Oplossen**
Als je problemen ondervindt (bijvoorbeeld EFI Network Timeout of bootproblemen), volg dan de volgende stappen:

1. **Firmware Instellingen Aanpassen**:
   - Start de VM en druk onmiddellijk op **ESC** om het bootmenu te openen.
   -
2. **.VMX-bestand Bewerken**
   1. Open het `.vmx`-bestand van de VM met een teksteditor.
   2. Controleer of de volgende instellingen correct zijn:

   ```plaintext
   ethernet0.present = "true"
   ethernet0.connectionType = "nat"
   ```

## **Stap 11: Windows Installatie Beginnen**
1. Nadat de virtuele machine is gestart, volg de Windows 11 installatieprocedure:
   - **Taal, Tijd en Toetsenbord**: Selecteer de gewenste taal, tijdzone en toetsenbordindeling en klik op **Next**.
   - Klik op **Install Now** om de installatie te starten.
   - **Productcode**: Voer een productcode in als je deze hebt, of klik op **I don't have a product key** om door te gaan zonder activatie.
   - **Windows Versie**: Kies de gewenste Windows 11 versie en klik op **Next**.
   - **Licentievoorwaarden**: Accepteer de licentievoorwaarden door het selectievakje aan te vinken en klik op **Next**.
   - **Installatietype**: Kies **Custom: Install Windows only (advanced)** voor een schone installatie.

2. **Schijfselectiescherm**:
   - Selecteer de aangemaakte virtuele schijf (de schijf die je hebt gemaakt tijdens de VM-configuratie) om Windows 11 te installeren.
   - Klik op **Next** om de installatie te starten.

## **Stap 12: Windows 11 Configuratie Voltooien**
1. **Is dit de juiste regio?**:
   - Selecteer de juiste regio en klik op **Yes**.

2. **Is dit het juiste toetsenbordindeling?**:
   - Bevestig de toetsenbordindeling en klik op **Yes**.
   - Indien gewenst, kun je een extra toetsenbordindeling toevoegen door op **Add layout** te klikken, anders kies je **Skip**.

3. **Instellen van de naam van je apparaat** (optioneel):
   - Voer een naam in voor je apparaat en klik op **Next**. Je VM zal opnieuw opstarten als je een naam hebt ingevoerd.

4. **Kies een manier om in te loggen**:
   - **Voor persoonlijk gebruik**: Kies **Set up for personal use** en klik op **Next**.
   - **Organisatie**: Als je deel uitmaakt van een organisatie, kies **Set up for work or school**.

5. **Work Account en Alternatieve Aanmeldopties**:
   - Nadat je hebt gekozen voor **work account**, vraagt Windows om een Microsoft-account.
   - Klik op **Sign-in options** om alternatieve aanmeldopties te krijgen.
   - Kies vervolgens voor **Offline account** om verder te gaan zonder een Microsoft-account.

6. **Gebruikersnaam Invoeren**:
   - Voer een gebruikersnaam in voor het lokale account. Dit kan bijvoorbeeld je naam of een algemene naam zoals **User** zijn.
   - Klik op **Next**.

7. **Wachtwoord Instellen**:
   - Voer een wachtwoord in voor je account en klik op **Next**.
   - Bevestig het wachtwoord door het opnieuw in te voeren en klik op **Next**.

8. **Beveiligingsvragen**:
   - Om je account extra te beveiligen, zal Windows je vragen om **drie beveiligingsvragen** in te stellen.
   - Kies een vraag uit de lijst voor elke beveiligingsvraag en voer het antwoord in. Dit helpt om je wachtwoord te herstellen als je het ooit vergeet.
   - Klik op **Next** na het invoeren van elk antwoord.

9. **Privacyinstellingen Configureren**:
   - Windows 11 zal je een reeks privacyopties laten zien, zoals:
     - **Locatie gebruiken** voor beter afgestemde diensten.
     - **Find my device** om je apparaat te vinden als het kwijt raakt.
     - **Diagnostische gegevens** voor probleemoplossing.
     - **Inkt- en typgewoonten** voor betere tekstsuggesties.
     - **Advertentie-ID** voor gepersonaliseerde advertenties.
   - Schakel de opties naar wens in of uit en klik op **Accept**.

10. **Windows Configuratie Voltooien**:
    - Windows zal verdere configuraties uitvoeren, zoals het toepassen van instellingen en het voorbereiden van je bureaublad. Dit kan enkele minuten duren.
    - Het scherm zal de boodschap **This might take a few minutes** weergeven.

11. **Bureaublad Weergeven**:
    - Na het voltooien van de configuratie wordt je naar het Windows 11-bureaublad gebracht.
    - Je hebt nu een volledig functioneel Windows 11-systeem dat klaar is voor gebruik in VMware.

12. **Even wachten...**:
    - Windows zal verdere configuraties voltooien. Dit kan enkele minuten duren, waarbij de boodschap **This might take a few minutes** getoond wordt.

13. **Bureaublad Weergeven**:
    - Na de configuratie word je naar het bureaublad van Windows 11 gebracht. Je hebt nu een volledig geïnstalleerd en werkend Windows 11 in VMware.

### **Installatie VMware Tools**

1. **Start installatie**:
   - Ga in VMware naar **VM > Install VMware Tools**.

2. **Open schijf**:
   - Open **Verkenner** in de VM en dubbelklik op de schijf **VMware Tools**.

3. **Start installatie**:
   - Run **setup64.exe** (of **setup.exe** voor 32-bit).

4. **Kies installatieoptie**:
   - Selecteer **Complete** en klik op **Install**.

5. **Herstart**:
   - Klik op **Finish** en herstart de VM.

#### **Wat kun je nu doen met VMware Tools?**

- **Volledig schermmodus**: Naadloze aanpassing van schermresolutie.
- **Gedeeld klembord**: Kopiëren/plakken tussen de host en VM.
- **Drag & Drop**: Bestanden slepen tussen de host en VM.
- **Muisintegratie**: Geen handmatige "muis vastpakken" meer nodig.
- **Betere prestaties**: Verbeterde grafische en netwerksnelheid.
- **Gedeelde mappen**: Toegang tot host-mappen vanuit de VM. (gebruik \downloads\)

## Loading Virtual Machine Files: HDD vs NVMe M.2

### Performance Differences

#### NVMe M.2 Benefits:
1. **Faster Read/Write Speeds:** NVMe drives achieve speeds of 3,000–7,000 MB/s compared to HDD’s 50–150 MB/s.
2. **Improved Random Access:** NVMe handles random read/write operations more efficiently, critical for virtual machine (VM) performance.
3. **Higher IOPS:** NVMe offers superior Input/Output Operations Per Second, ideal for multitasking and multiple VMs.
4. **System Responsiveness:** Faster application loads and smoother VM operation.
5. **Durability:** NVMe’s wear-leveling technology prolongs lifespan compared to HDDs under heavy workloads.

#### HDD Limitations:
1. **Slower Performance:** Lower read/write speeds and slower random access.
2. **Mechanical Seek Time:** Performance bottlenecks due to physical disk movements.

#### Recommended Use Cases
1. Use **NVMe M.2** if:
   - Running modern operating systems or disk-intensive applications.
   - Working with multiple VMs.
   - Prioritizing speed and system responsiveness.

2. Use **HDD** if:
   - Budget constraints are a factor.
   - Hosting lightweight or infrequently accessed VMs.

## File Storage on System Drive (C:) vs Partitioning

### Temporary Storage on C:\
1. **Advantages:**
   - Quick setup without disk modifications.
   - Easy access to VM files (e.g., `C:\VM`).
2. **Disadvantages:**
   - Fragmentation risks on HDDs.
   - Limited space can affect OS and VM performance.

### Transitioning to a Partition
1. **Steps to Move VMs Later:**
   - Shut down the VM.
   - Copy files to the new partition (e.g., `D:\VMs`).
   - Update VMware to reference the new location.
   - Test the VM and delete old files.
2. **Fragmentation Handling:**
   - Defragment the C: drive before moving files.
   - Use Windows' **Optimize Drives** for NVMe SSDs (enables TRIM).

## Addressing Fragmentation on NVMe SSDs

### Why Fragmentation Isn’t a Concern:
1. **No Mechanical Parts:** NVMe has no moving components, so file fragmentation doesn’t impact performance.
2. **TRIM Command:** Optimizes data block management and maintains performance.
3. **Wear-Leveling:** Prevents localized wear by evenly distributing writes.

### Best Practices:
1. Ensure **TRIM is enabled** (default in Windows):
   - Run in Command Prompt: `fsutil behavior query DisableDeleteNotify`. Output should be `DisableDeleteNotify = 0`.
2. Periodically run **Optimize Drives** to perform TRIM.
3. Avoid overfilling the drive; keep 10–20% free space.
4. Use pre-allocated VM disk space for better efficiency.

### Do Not:
- Perform traditional defragmentation on NVMe SSDs; it’s unnecessary and may cause wear.

## Conclusion
- Use NVMe M.2 for optimal performance, especially with multiple or resource-intensive VMs.
- Starting with `C:\VM` is fine temporarily, but migrating to a dedicated partition is recommended for better organization.
- Fragmentation is not a concern for NVMe drives; focus on TRIM and sufficient free space instead.

By following these practices, you can ensure smooth performance and longevity for your virtual machines and storage setup.
