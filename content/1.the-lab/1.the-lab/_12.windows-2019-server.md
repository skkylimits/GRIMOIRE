This lab focuses on setting up a Windows enterprise environment, which includes a Windows Server 2019 (acting as a Domain Controller) providing Active Directory Domain Services, and a Windows 10 host. The lab simulates common user activities across a simple Windows domain to help understand enterprise network features and practice skills in data acquisition, disk forensic, and memory forensic.

## Setup Instructions

This lab builds on a basic virtual lab setup by adding a Windows Server 2019 Domain Controller and joining a  [`Windows 10 workstation`](/the-lab/the-lab/windows-10-workstation)
 to the domain.


**Prerequisite**: To gain an understanding of virtualization software and best practices, it is highly recommended to read the [Virtualization Primer](#).

## 1 Download and Install VirtualBox and a Windows 10 VM
1. **Install VirtualBox** and a Windows 10 VM as described in the Basic Lab section.
2. In VirtualBox, create a "NATNetwork" to allow VM-to-VM communication as well as internet access via the host system:
   - In VirtualBox, go to **Settings** -> **Network** and click the **+** sign to add a new "NATNetwork".
   - Attach this network to the Windows 10 VM network interface and all future VMs.

## 2 Install a Windows Server 2019 VM

This guide describes how to install Windows Server 2019 from an ISO. Alternatively, download a VHD from the Microsoft Evaluation site, attach it to a new VM, and skip most of the installation steps.

1. **Download the ISO**: Get the Windows Server 2019 Essentials ISO from the [Microsoft Evaluation Center](https://www.microsoft.com/en-us/evalcenter/).
2. **Create a new VM** in VirtualBox:
   - Select **Type**: Windows and **Version**: Windows 2019 (64-bit).
   - Ensure the VM has at least **2 GB RAM** and create a virtual hard disk (preferably **dynamic** and **VMDK** for compatibility with forensic tools).
   - Assign at least **30 GB** for the disk size.
3. Before starting the VM, open its settings and configure:
   - **Storage**: Attach the downloaded Windows 2019 ISO to the VM’s optical drive.
   - **Networking**: Attach "NATNetwork" to the network adapter.
4. **Start the VM** and follow the Windows Server installation instructions:
   - No need to provide a product key when prompted.
   - Select "**Custom**" install since there is no pre-existing Windows installation.
   - Set a password for the built-in **Administrator** account.
5. After installation, use the VirtualBox menu to select **Input -> Keyboard -> Insert Ctrl+Alt+Del** to enter your password.
6. **Allow PC discovery** when prompted after startup.
7. **Install VirtualBox Guest Additions** (optional but recommended) to enable full-screen display, shared folders, copy & paste, etc.
8. **Change hostname** (important!):
   - Go to **Settings -> System -> About**.
   - Click **Rename this PC** to update the hostname. Do not change it after setting up Active Directory.
9. **Take a snapshot** once the VM is set up and powered off.

## 3 Install Active Directory Services

To set up a domain environment, Active Directory Services need to be installed on the Windows Server.

1. Open **Windows Server Manager** from the Start menu.
2. Click **Manage** -> **Add Roles and Features**.
3. Hit **Next** until you reach **Server Roles**. Select **Active Directory Domain Services**.
4. After installation, click **Promote this server to a Domain Controller**, which appears as a notification in the Server Manager (click refresh if it's not there yet).
5. In the **Deployment Configuration Window**:
   - Select **Add a new forest** and enter your desired domain name, followed by `.local`. For example: `BCS.local`.
   - Define a **Directory Services Restore Mode (DSRM)** password.
   - Click **Next** on all following windows until the prerequisite checks pass, then click **Install**.
6. The system will reboot. The login prompt will now show your username in `DOMAIN/Administrator` format.

## 4 Create Domain Users and Administrators

Initially, there is one admin account called **Administrator** in the domain. Now, we will add additional users.

1. In **Server Manager**, click **Tools -> Active Directory Users and Computers**.
2. In the new window, click on your domain (`BCS.local`) to open its **Organizational Units (OUs)**, such as **Users**.
3. Open the **Users** folder, which contains various security groups and the **Administrator** user.
4. To create additional users (e.g., **Alice**):
   - Right-click **Users** -> **New -> User**.
   - Enter the **first and last name**. Use the first name as the **username**.
   - Set a **password**. Important: Check the box for **Password never expires** for testing purposes.
5. To assign a user to **Domain Admins**:
   - Double-click the user.
   - Select the **Member Of** tab and click **Add**.
   - Type **Domain Admins** in the object names field and click **Check Names**.

## 5 Add the Windows 10 Workstation to the Domain

1. **Update the Workstation's Computer Name**: Ensure your workstation has a unique name in the domain.
2. Ensure the workstation can **ping the Domain Controller IP**.
3. **Add the Domain Controller IP** as the preferred DNS server:
   - Right-click the **Ethernet adapter** icon in the taskbar and click **Open Network & Internet settings**.
   - Select **Change adapter options**.
   - Right-click the adapter and select **Properties**.
   - Select **Internet Protocol Version 4 (TCP/IPv4)** and click **Properties**.
   - Enable **Use the following DNS server address** and enter the Domain Controller's IP.
4. **Join the Domain**:
   - Open **Windows Settings -> Accounts -> Access work or school**.
   - Click the **[+] Connect** button.
   - Click **Join this device to a local Active Directory domain**.
   - Enter the domain name (e.g., `BCS.local`).
   - Enter **domain admin credentials** when prompted.
   - Assign the desired domain user (e.g., **Alice**) to this workstation and grant **Administrator** privileges.
   - Reboot the workstation. The login prompt will show the username in the format `DOMAIN\user`.
5. To validate, go to **Server Manager** on the Domain Controller -> **Tools -> Active Directory Users and Computers**:
   - Open your domain (`BCS.local`) -> **Computers**, which should show the newly joined workstation.

## 6 Finish the Setup

- Once the VMs are fully configured, **power them down** and take **snapshots**. This allows you to easily revert to a fresh environment.
