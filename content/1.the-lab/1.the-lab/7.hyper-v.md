This document outlines the steps to create a Hyper-V virtual machine with an unattended Windows installation using a Generation 1 VM.

## **Prerequisites**

### **Tools and Resources Needed**
1. **Windows Installation ISO**
   - Obtain the official ISO from the [Microsoft website](https://www.microsoft.com/software-download/).

2. **OSCDIMG Tool**
   - Part of the Windows ADK (Assessment and Deployment Kit).

3. **Unattend.xml Generator**
   - Use the [Schneegans Unattend Generator](https://schneegans.de/windows/unattend-generator/) to create or modify the `unattend.xml` file.

4. **Windows ADK Installation**
   - Download the Windows ADK from the [Microsoft ADK page](https://learn.microsoft.com/en-us/windows-hardware/get-started/adk-install).
   - During installation, ensure the **Deployment Tools** feature is selected.

## **Steps to Create the Bootable ISO with Unattend.xml**

### **Step 1: Install OSCDIMG**
1. **Download and Install the Windows ADK**:
   - Visit [Microsoft ADK](https://learn.microsoft.com/en-us/windows-hardware/get-started/adk-install).
   - Select the Deployment Tools feature during installation.

2. **Locate OSCDIMG**:
   - Typically installed in the following path:
     ```
     C:\Program Files (x86)\Windows Kits\10\Assessment and Deployment Kit\Deployment Tools\amd64\Oscdimg\oscdimg.exe
     ```

3. **Add OSCDIMG to System PATH (Optional)**:
   - Add the folder containing `oscdimg.exe` to the PATH environment variable to use it without specifying the full path.
   - Alternatively, directly call it with the full path.

### **Step 2: Mount the Windows ISO**
1. Right-click on the Windows ISO and select **Mount**.
2. Note the drive letter assigned to the mounted ISO (e.g., `D:`).

### **Step 3: Copy ISO Contents**
1. Create a working directory for the ISO contents:
   ```cmd
   mkdir C:\WinISO
   ```
2. Copy all files from the mounted ISO to the working directory:
   ```cmd
   xcopy D:\* C:\WinISO /E /H /K
   ```

### **Step 4: Add the Unattend.xml File**
1. Generate or modify your `unattend.xml` file using the [Schneegans Unattend Generator](https://schneegans.de/windows/unattend-generator/).
   - Adjust settings to:
     - Set BIOS mode.
     - Ignore Windows 11 requirements (if applicable).
   - Save the file.
2. Place the `unattend.xml` file into the `root` folder of your working directory and rename it to:
   ```
   autounattend.xml
   ```

### **Step 5: Rebuild the ISO**
Use the `oscdimg` tool to create a bootable ISO with the unattended file included:
```cmd
"C:\Program Files (x86)\Windows Kits\10\Assessment and Deployment Kit\Deployment Tools\amd64\Oscdimg\oscdimg.exe" -u2 -m -bC:\WinISO\boot\etfsboot.com C:\WinISO C:\CustomWindows.iso
```
- **Options Explained**:
  - `-u2`: Specifies UDF file system version 2.0.
  - `-m`: Ignores file size limits.
  - `-b`: Points to the boot sector file for a bootable ISO.

## **Steps to Create and Configure the Hyper-V VM**

### **Step 1: Create a Generation 1 Virtual Machine**
1. Open **Hyper-V Manager**.
2. Click **New > Virtual Machine** and follow the wizard:
   - Name: `LILITH` (or your preferred name).
   - Generation: **Generation 1** (BIOS mode).
   - Memory: Allocate at least **2 GB** of RAM.
   - Hard Disk: Create a new **VHD** of at least 20 GB.
   - Attach the newly created ISO as the boot media.

### **Step 2: Configure VM Settings**
1. **Attach the ISO**:
   - Open VM **Settings**.
   - Under **IDE Controller > DVD Drive**, attach the newly created `CustomWindows.iso`.

2. **Boot Order**:
   - Go to **Firmware**.
   - Ensure **DVD Drive** is at the top of the boot order.

3. **Disable Secure Boot (Optional)**:
   - If issues persist, disable Secure Boot by unchecking **Enable Secure Boot** under **Security**.

### **Step 3: Start the VM**
1. Power on the VM.
2. The installation should proceed automatically using the `autounattend.xml` file without requiring manual input.

## **Tips for Debugging Issues**
1. **Unattend.xml Not Recognized**:
   - Ensure the file is correctly named `autounattend.xml` and placed in the `root` folder.
   - Validate the XML file using Windows System Image Manager.

2. **ISO Not Booting**:
   - Verify the `oscdimg` command includes the correct boot sector file (`etfsboot.com`).
   - Ensure the VM is set to boot from the DVD Drive.

3. **Generation 2 Not Working**:
   - Generation 2 requires UEFI-compatible ISO and Secure Boot configuration. Use Generation 1 if compatibility issues arise.

By following these steps, you can create a fully unattended Windows installation ISO and deploy it in a Hyper-V virtual machine.

## Default Locations

**Hyper-V**
`C:\ProgramData\Microsoft\Windows\Hyper-V`

**VHDX**
`C:\ProgramData\Microsoft\Windows\Virtual Hard Disks`

## Remove VM

```powershell
Get-VM
```

- Delete from Hyper-V Manager
- delete from `C:\ProgramData\Microsoft\Windows\Virtual Hard Disks`
- `Remove-VM -Name "VMName" -Force` in powershell
- To clean up associated files (configuration, VHDs, snapshots):
  - Replace C:\Path\To\VMFiles with the appropriate folders:
      - Configuration: `C:\ProgramData\Microsoft\Windows\Hyper-V`
      - Hard Disks: `C:\ProgramData\Microsoft\Windows\Virtual Hard Disk`

### **Checkpoints: Ensure Proper Permissions**

It's possible that you can't create checkpoints at the point of installation. Make sure the system is not holding the files hostage.

Simple reboot the machine and try again. Else make sure that:

1. **Add Administrator to Hyper-V Administrators**:
   - Press `Windows + X` and select **Computer Management**.
   - In the left panel, expand **Local Users and Groups**.
   - Select **Groups**, then double-click **Hyper-V Administrators**.
   - Click **Add** and enter the admin username.
   - Click **OK** to save changes.

2. **Verify VM Permissions**:
   - Navigate to the VM configuration folder (default: `C:\ProgramData\Microsoft\Windows\Hyper-V`).
   - Ensure the following accounts have **Full Control**:
     - `NT VIRTUAL MACHINE\<GUID>`
     - `SYSTEM`
   - Add permissions if missing.

3. **Verify Checkpoint Storage Permissions**:
   - Open Hyper-V Manager and check the checkpoint storage path under **Hyper-V Settings > Checkpoints**.
   - Ensure permissions are correct on this folder.

4. **Verify Disk Accessibility**:
   - Navigate to the VHDX file location (default: `C:\Users\Public\Documents\Hyper-V\Virtual Hard Disks` or custom).
   - Ensure the `NT VIRTUAL MACHINE\<GUID>` account has access.
