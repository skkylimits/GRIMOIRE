If **authentication** is the key that opens the door, **authorization** decides where you can go and what you can see inside.

## Authentication

Authentication is the process of verifying a person's identity.

> For example, when making a purchase with a credit card, you may need to show an ID to prove you are the cardholder. Similarly, when accessing a computer, entering a username and password verifies your identity.

### Authentication Factors

- **Something You Know:** Passwords, PINs, security questions
- **Something You Have:** ID cards, USB keys, phones
- **Something You Are:** Fingerprints, facial recognition, retinal scans

![authentication-types](/content/1.the-lab/2.core-principles/5.AuthN-AuthZ/authentication-types.png)

### Single-Factor Authentication

Uses only one type of authentication, like a password or fingerprint, which is simpler but less secure.

### Multifactor Authentication

[MFA](https://docs.azure.cn/en-us/entra/identity/authentication/concept-mfa-howitworks) Combines two or more authentication types for enhanced security. For instance, entering a password and then a code sent to your phone.

The following additional forms of verification:

- **Microsoft Authenticator app**
- **Windows Hello for Business**
- **FIDO2 security key**
- **OATH hardware token (preview)**
- **OATH software token**
- **SMS**
- **Voice call**

**Multifactor authentication is recommended for robust security.**

#### **Password**

::caution
NIST Password Guidelines:

- **[Minimum Length](https://pages.nist.gov/800-63-3/sp800-63b.html#appA)**: Passwords should be at least 15 characters long.
- **[Maximum Length](https://pages.nist.gov/800-63-3/sp800-63b.html#verifimpers)**: Systems should allow passwords up to 64 characters.
- **[Character Composition](https://pages.nist.gov/800-63-3/sp800-63b.html#appA)**: Avoid enforcing specific character types (e.g., uppercase, numbers) to reduce user burden.
- **[Password Hints](https://pages.nist.gov/800-63-3/sp800-63b.html#sec5)**: Do not use password hints that could aid in guessing passwords.
- **[Password Expiration](https://pages.nist.gov/800-63-FAQ/#q-b05)**: Do not require periodic password changes **unless there's evidence of compromise**.
- **[Compromised Passwords](https://pages.nist.gov/800-63-3/sp800-63b.html#sec5)**: Check new passwords against known compromised lists.
- **Password Storage**: Store passwords securely using salted and hashed methods.
- **Password Policies**: Implement policies to block weak or easily guessable passwords.
::

- **[Your Pa$$word doesn't matter](https://techcommunity.microsoft.com/blog/microsoft-entra-blog/your-paword-doesnt-matter/731984)**: Are common but weak, especially in single-factor auth—easy to remember often means easy to hack. [Eliminate](https://docs.azure.cn/en-us/entra/identity/authentication/concept-password-ban-bad) the weak ones.

	- **Global banned password list**: Microsoft Entra ID Protection maintains an auto-updated [global banned password list](https://docs.azure.cn/en-us/entra/identity/authentication/concept-password-ban-bad#global-banned-password-list) of weak or compromised passwords like P@$$w0rd and its variations, blocking their use. Users are notified to choose stronger passwords when attempting weak ones, as Microsoft Entra blocks millions of common weak passwords using real-world data and fuzzy-matching.

		::note
		Variations are created using an algorithm that transposes text case and letters to numbers such as "1" to an "l". Variations on Password1 might include Passw0rd1, Pass0rd1, and others. These passwords are then checked and added to the global banned password list. The global banned password list is automatically applied to all users in a Microsoft Entra tenant and can't be disabled.
		::

	- **Custom banned password list**: The global banned password list combines with [custom banned password](https://docs.azure.cn/en-us/entra/identity/authentication/tutorial-configure-custom-password-protection)—to block weak and organization-specific passwords during changes or resets, enhancing security.
		- Brand names
		- Product names
		- Locations
		- Company-specific internal terms
		- Abbreviations that have specific company meaning

		::note
		The custom banned password list allows up to 1,000 terms and isn’t meant for very large lists. To maximize its effectiveness, understand how passwords are evaluated before adding terms to efficiently block weak passwords and variants.
		::

		![enable-configure-custom-banned-passwords-cropped](/content/1.the-lab/2.core-principles/13.AuthN-AuthZ/enable-configure-custom-banned-passwords-cropped.png)

	- **Hybrid**: Admins can integrate Microsoft Entra password protection with on-premises Active Directory to [enforce banned password policies](https://docs.azure.cn/en-us/entra/identity/authentication/concept-password-ban-bad-on-premises) on all password changes.

	- **Password Spraying**: Microsoft Entra password protection combines global and custom banned password lists to block common weak passwords used in spray attacks, adding extra security alongside account [smart lockout policies](https://learn.microsoft.com/en-us/entra/identity/authentication/howto-password-smart-lockout#manage-microsoft-entra-smart-lockout-values),  in accordance to NIST guidelines that [no longer recommend password expiration..](https://pages.nist.gov/800-63-FAQ/#q-b05) and following the [NIST Password Guidelines](https://pages.nist.gov/800-63-3/sp800-63b.html) should provide a advanced level of security.

::caution
The use of passwords should be supplemented or replaced with more secure authentication methods. Eliminate them preferably.
::

#### **SMS**

* **[SMS](https://learn.microsoft.com/en-us/entra/identity/authentication/howto-authentication-sms-signin) (Text Message)**: Can be used as a primary sign-in method (replacing username and password) or as secondary verification (MFA/SSPR) via code sent to the phone.

* **[Voice Call](https://learn.microsoft.com/en-us/entra/identity/authentication/concept-authentication-phone-options)**: Users can verify identity via automated voice calls during SSPR or MFA by pressing # when prompted.

	::warning
	Voice calls aren’t supported as a primary authentication method.
	::

#### **OATH**

* **OATH Software Token**: [OAuth](https://learn.microsoft.com/en-us/entra/identity/authentication/concept-authentication-oath-tokens) apps generate time-based codes.
* **OATH Hardware Token**: Physical devices display rotating codes.

	::warning
	OATH tokens are only supported as secondary authentication for SSPR or Microsoft Entra MFA.
	::

	::caution
	Because OATH tokens generate time-based codes, the device’s clock must be synced—otherwise, OTPs won’t match.
	::

#### **Passwordless**

Many organizations aim to eliminate passwords by using secure [passwordless](https://learn.microsoft.com/en-us/entra/identity/authentication/concept-authentication-passwordless) methods like biometrics with Windows Hello or FIDO2 keys, which Microsoft Entra ID supports to simplify sign-in and reduce attack risks.

#### Windows Hello for Business

PIN or biometrics tied to a device with a cryptographic key, enabling strong two-factor or passwordless sign-in. PIN or biometric input triggers the private key to cryptographically sign data sent to the identity provider for verification, protecting against credential theft.

![windows-hello](/content/1.the-lab/2.core-principles/13.AuthN-AuthZ/windows-hello.png)

#### FIDO2 Security Keys

Hardware keys (USB, NFC, Bluetooth) that provide secure, unphishable passwordless authentication and single sign-on (SSO) by leveraging public key cryptography enabling users to  sign in to Microsoft Entra ID or Microsoft Entra hybrid joined Windows 10 devices and get single-sign on to their cloud and on-premises resources.

::caution
Supported by most websites. FIDO2 security keys are a great option for enterprises who are very security sensitive or have scenarios or employees who aren't willing or able to use their phone as a second factor.
::

![FIDO-Security-Keys](/content/1.the-lab/2.core-principles/13.AuthN-AuthZ/FIDO-Security-Keys.png)

### SSPR

[Self-service password reset](https://docs.azure.cn/en-us/entra/identity/authentication/concept-sspr-howitworks) (SSPR) in Microsoft Entra ID lets users reset or change their passwords without admin or help desk support, cutting IT costs and boosting productivity.

![registration](/content/1.the-lab/2.core-principles/13.AuthN-AuthZ/registration.png)

::warning
Admin accounts must use two verification methods and are auto-enabled for SSPR.
::

**Benefits:**

* Reduces help desk calls and downtime.
* Admins can update settings without disrupting users.
* Provides audit logs compatible with SIEM tools.
* Supports [password write-back](https://docs.azure.cn/en-us/entra/identity/authentication/tutorial-enable-sspr) to on-premises Active Directory.

**Requirements:**

* Microsoft Entra ID license
* SSPR enabled by an admin
* Registration with one or more auth methods (two+ recommended)

**Supported Authentication Methods:**

* Authenticator app (notification/code)
* Email
* Mobile/office phone
* Security questions (only for SSPR, not sign-in; not allowed for admins)

::caution
Admins can configure email alerts for SSPR events to notify (privileged)users and admins of password resets. For admin accounts, all global admins are notified to ensure added awareness.
::

| User Type        | Minimum License Needed                          |
| ---------------- | ----------------------------------------------- |
| Cloud-only users | Microsoft Entra ID Free                         |
| Hybrid users     | Microsoft Entra ID P1 or P2                     |
| Admin features   | P1/P2 recommended for full control & monitoring |

::warning
SSPR

> When a user attempts to reset a password to something that's on the global or custom banned password list, they see one of the following error messages:

- **_Unfortunately, your password contains a word, phrase, or pattern that makes your password easily guessable. Please try again with a different password._**
- **_Unfortunately, you can't use that password because it contains words or characters that have been blocked by your administrator. Please try again with a different password_**

### Types of Attacks

- **Brute Force Attack:** Attackers try numerous username and password combinations until they succeed.
- **Dictionary Attack:** A subset of brute force attacks using a list of common words or phrases.
- **Credential Stuffing:** Reusing stolen credentials from one breach to access other accounts.
- **Keylogging:** Malicious software captures keystrokes to steal credentials.
- **Social Engineering:** Manipulates individuals into revealing personal information or performing actions that compromise security.

### Social Engineering

- **Phishing:** Fake emails or websites trick users into revealing credentials.
- **Pretexting:** Attackers pose as trusted entities to extract secure information.
- **Baiting:** Fake offers or rewards lure victims into divulging personal details.

**Prevent attacks with education and multifactor authentication.**

## Authorization

When you authenticate a user, you need to decide where they can go, and what they're allowed to see and touch. This process is called authorization.

> Suppose you want to spend the night in a hotel. The first thing you do is go to reception to start the "authentication process." After the receptionist has verified who you are, you're given a keycard and can go to your room. Think of the keycard as the authorization process. The keycard will only let you open the doors and elevators you're permitted to access, such as for your hotel room.

## Authorization Security Techniques

Authorization controls what an authenticated user can access. Key techniques include:

### Zero Trust
- **Verify Explicitly:** Every request is authenticated and authorized before access.
- **Use Least Privileged Access:** Limit permissions minimaly. Only access to what they need to.
- **Assume Breach:** Plan for breaches and minimize attacker movement within the network.

### Conditional Access

As the name implies, conditional access involves access with conditions. One way to think about conditional access is with if/then statements. If something is true, you're granted access, but if it's false, you're denied.

> Let’s see how this would work in an IT scenario. Increasingly, people are working from home. Because of this, they might be using their personal computer to access work-related content. With conditional access, an organization might grant access for an authenticated user to a confidential system, such as payroll, only if it's made through secure corporate computers located at their headquarters. If the authenticated user tries to access the payroll system from a personal computer at home, they would be blocked.

### Least Privileged Access
Users are given the minimum permissions needed for their tasks, reducing potential damage if a breach occurs.

> For example, when you board an airplane, you have access to the main cabin area to get to your seat, but no passenger is allowed in to the cockpit. Also, if you're traveling with a coach-class ticket, you only have access to that section. To improve security, each person can only access the areas they need to.

### Lateral Movement Prevention
Adds layers of security to limit an attacker’s ability to move between systems and access more resources.

> Think of an office building where a criminal gets past the security of the main reception area. They can then generally move around the rest of the building, accessing different floors and offices. It's important to provide added layers of security to protect against intrusion in sensitive areas.
