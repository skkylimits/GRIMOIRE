## Shared Responsibility Model

In on-premises environments, the organization handles all security and compliance. With cloud services, responsibility is shared between the cloud provider and the customer, depending on the service model:

* **SaaS**: The cloud provider manages most security, while the customer is responsible for data, accounts, and devices.
* **PaaS**: The provider handles infrastructure and operating systems; the customer manages applications and data.
* **IaaS**: The provider manages physical infrastructure; the customer controls software components and data.
* **On-premises**: The organization is responsible for everything, including physical security and encryption.

![3-shared-responsibility-model](/content/1.the-lab/2.core-principles/11.defense-in-depth/3-shared-responsibility-model.png)

::caution
In all cases, the customer owns and protects:


- Information and data
- Devices (mobile and PCs)
 - Accounts and identities
::

## Defense in Depth

Defense in depth uses multiple layers of security to slow and prevent attacks, ensuring that if one layer is breached, others will protect data. Example layers include:

* **Physical security**: Restricting access to authorized personnel.
* **Identity & access controls**: Using MFA and conditional access to secure infrastructure.
* **Perimeter security**: DDoS protection to prevent large-scale attacks.
* **Network security**: Network segmentation and access controls to limit communication.
* **Compute layer security**: Securing access to virtual machines by closing ports.
* **Application security**: Protecting applications from vulnerabilities.
* **Data security**: Managing access and encrypting sensitive data.
