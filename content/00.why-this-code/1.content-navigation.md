> We had to implement this custom navigation logic because **Nuxt Content v2** changed how navigation items are handled. Unlike v1, v2 **does not automatically support filtering by `standalone` or `stripped`**, nor does it provide a built-in way to dynamically adjust navigation based on the current route. Additionally, key functions like **`mapContentNavigation` and `findNavByPath` were removed**, requiring us to manually traverse the navigation structure. Since the default behavior renders **all objects in the `/content` directory**, we needed a way to selectively display only relevant items. Our approach ensures that navigation remains structured, follows expected UX patterns, and avoids clutter from unintended pages.

## **🔍 Understanding `findValidNavPath` in Vue**
This script dynamically **filters and displays navigation items** in a Nuxt app based on the current route.
It ensures that only **valid navigation items** (`standalone` or `stripped`) are displayed.

---

## **Why We Chose This Approach**

Our goal was to create a **dynamic, scalable, and context-aware** navigation system for Nuxt that:
1. **Matches the current route path** to navigation items.
2. **Filters out stripped directories** while promoting their children.
3. **Handles standalone nodes separately**, ensuring they display correctly.
4. **Adapts dynamically to route changes** without requiring manual updates.

---

### **Why These Functions?**

#### **🔍 `findNavItemByPath(path, navItems)`**
- Recursively searches for a navigation item by its path.
- Ensures we correctly locate any given page within the nested structure.
- Essential for **traversing deeply nested navigation trees**.

#### **📌 `findValidNavPath(currentPath, navItems)`**
- Uses `findNavItemByPath` to get the **closest valid navigation node**.
- If `standalone`, returns **only that node**.
- If `stripped`, promotes its **children to top-level items**.
- If neither is found, **moves up one level** and retries.
- Guarantees that navigation **remains intuitive** without unnecessary nesting.

#### **🛠 `filteredNavigation` (computed property)**
- Calls `findValidNavPath()` to get the correct navigation items.
- If `standalone`, returns **only that item** (prevents unrelated links from showing).
- If `stripped`, **returns only its children**, keeping the hierarchy clean.
- Automatically updates whenever the **route changes**, ensuring a **seamless UX**.

#### **🔄 `watchEffect()` for Debugging**
- Runs every time the route updates.
- Logs the **valid navigation path found** for better debugging.
- Helps verify that the **correct items are being rendered** at all times.

---

### **Why This Matters**
✅ **Avoids displaying unnecessary directories** that are stripped.
✅ **Prevents orphaned navigation items** by promoting child elements.
✅ **Scales well with deeply nested content structures**.
✅ **Automatically adapts to new content** without requiring manual updates.

This approach ensures that navigation remains **clean, dynamic, and contextually accurate**, even as content evolves. 🚀

## **📌 Step-by-step Breakdown**
### **1️⃣ Import & Setup**
```ts
import type { ContentNavigationItem } from '@nuxt/content'

const route = useRoute()
const navigation = inject<Ref<ContentNavigationItem[]>>('navigation')
```
- `route = useRoute()` → Gets the **current page route**.
- `navigation = inject(...)` → **Injects** the global navigation structure (a list of `ContentNavigationItem[]`).

---

### **2️⃣ Function: Find a Navigation Item by Path**
```ts
function findNavItemByPath(path: string, navItems: ContentNavigationItem[]): ContentNavigationItem | undefined {
	console.log('Searching for path:', path)

	for (const item of navItems) {
		console.log('Checking item:', item.path)

		if (item.path === path) {
			console.log('✅ Found match:', item)
			return item
		}

		if (item.children) {
			console.log('🔄 Searching children of:', item.path)
			const found = findNavItemByPath(path, item.children)
			if (found) {
				console.log('✅ Found match in children:', found)
				return found
			}
		}
	}

	console.log('❌ No match found for:', path)
	return undefined
}
```
### **🛠 What This Does**
1. **Iterates through all navigation items** (`navItems`).
2. If it **finds an exact match**, it **returns the item**.
3. If the item has **children**, it **recursively searches deeper**.
4. If **nothing is found**, it logs and returns `undefined`.

---

### **3️⃣ Function: Find the Valid Navigation Path**
```ts
function findValidNavPath(currentPath: string, navItems: ContentNavigationItem[]) {
	console.log('🔍 Finding valid navigation path for:', currentPath)

	let currentNav = findNavItemByPath(currentPath, navItems)
	let searchPath = currentPath

	while (currentNav) {
		// Check if currentNav is a valid standalone or stripped node
		if (currentNav.standalone) {
			console.log('✅ Found standalone node:', currentNav)
			return { node: currentNav, type: 'standalone', path: searchPath }
		}

		if (currentNav.stripped) {
			console.log('✅ Found stripped node:', currentNav)
			return { node: currentNav, type: 'stripped', path: searchPath }
		}

		// Move up one level in the path
		const parentPath = searchPath.split('/').slice(0, -1).join('/')
		if (!parentPath) break // Stop if there's no more parent path

		console.log('⬆️ No standalone/stripped found, checking parent path:', parentPath)
		searchPath = parentPath
		currentNav = findNavItemByPath(searchPath, navItems)
	}

	console.log('❌ No valid standalone/stripped navigation node found.')
	return null
}
```
### **🛠 What This Does**
1. **Finds the navigation item** matching `currentPath`.
2. If it **has `standalone` or `stripped`**, **return it immediately**.
3. If **not found**, move **up one level** (`/parent-path/child` → `/parent-path`).
4. **Repeat** until we find a valid `standalone` or `stripped` node.
5. If we **reach the top level** and **still don't find anything**, return `null`.

---

### **4️⃣ Compute the Filtered Navigation**
```ts
const filteredNavigation = computed(() => {
	if (!navigation?.value) return []

	const validNav = findValidNavPath(route.path, navigation.value)

	if (validNav) {
		if (validNav.type === 'standalone') {
			// Only return the standalone node itself
			return [validNav.node]
		}
		else if (validNav.type === 'stripped') {
			// Return the children of the stripped node as top-level items
			return validNav.node.children || []
		}
	}

	return []
})
```
### **🛠 What This Does**
1. **Computes the correct navigation items** to show.
2. Calls `findValidNavPath(route.path, navigation.value)`.
3. If `standalone` → **Display only that node**.
4. If `stripped` → **Display its children** as top-level items.
5. If no valid node → Return an **empty array** (`[]`).

---

### **5️⃣ Watch for Route Changes**
```ts
watchEffect(() => {
	if (navigation?.value) {
		const validNav = findValidNavPath(route.path, navigation.value)
		console.log('🔍 Valid Navigation Path for:', route.path, validNav)
	}
})
```
### **🛠 What This Does**
- **Runs every time the route changes**.
- **Logs** what the function returns for debugging.

---

### **6️⃣ Template for Rendering Navigation**
```vue
<template>
	<UContainer>
		<UPage>
			<template #left>
				<UPageAside>
					<UContentNavigation
						type="single"
						highlight
						:navigation="filteredNavigation"
					/>
				</UPageAside>
			</template>

			<slot />
		</UPage>
	</UContainer>
</template>
```
### **🛠 What This Does**
1. **Wraps content** in `<UContainer>`.
2. **Creates a sidebar** (`<UPageAside>`) for navigation.
3. **Uses `filteredNavigation`** to display the correct items.
