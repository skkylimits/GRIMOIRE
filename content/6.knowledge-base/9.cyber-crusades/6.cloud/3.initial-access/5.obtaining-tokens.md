## 2. Requesting Token

::caution
- Both **Az PowerShell** and **AzureAD modules** allow the use of **access tokens** for authentication.
- Tokens contain all claims, including **MFA and Conditional Access policies**, making them useful for bypassing security controls.
::

### **Azure CLI**
**For Azure Resource Manager (ARM):**
```sh
az account get-access-token
```

**For Microsoft Graph (Entra ID Access):**
```sh
az account get-access-token --resource-type ms-graph
```

### **Azure PowerShell**

**Supported Endpoints**

| **Azure Service**           | **Command** |
|----------------------------|------------|
| **Microsoft Graph (MSGraph)** | `Get-AzAccessToken -ResourceTypeName MSGraph` |
| **Azure AD Graph (Deprecated)** | `Get-AzAccessToken -Resource "https://graph.windows.net"` |
| **Azure Resource Manager (ARM)** | `Get-AzAccessToken -ResourceTypeName Arm` |
| **Analysis Services** | `Get-AzAccessToken -ResourceTypeName AnalysisServices` |
| **Key Vault** | `Get-AzAccessToken -ResourceTypeName KeyVault`  |
| **Data Lake** | `Get-AzAccessToken -ResourceTypeName DataLake`  |
| **Batch Services** | `Get-AzAccessToken -ResourceTypeName Batch` |
| **Attestation** | `Get-AzAccessToken -ResourceTypeName Attestation` |
| **Azure Storage** | `Get-AzAccessToken -ResourceTypeName Storage` |
| **Synapse Analytics** | `Get-AzAccessToken -ResourceTypeName Synapse` |

> You can use the full resource url aswell, like https://graph.microsoft.com instead of MSGraph

**Using an existing session (ARM):**
- If a user is logged in, retrieve their access token via:
```powershell
$token = (Get-AzAccessToken).Token
```

**For Microsoft Graph (Entra ID Access):**
```powershell
$msgraphtoken = (Get-AzAccessToken -ResourceTypeName MSGraph).Token
```

**Use multiple access tokens for different services:**
```powershell
Connect-AzAccount -AccountId test@qhps.nl -AccessToken $token -MicrosoftGraphAccessToken $msgraphtoken
```

**Decoded:**

```json [HEADER: ALGORITME & TOKEN TYPE]
{
	"typ": "JWT",
	"nonce": "wzNti6EgoPGec_0DODG4NFZPEj70L8cjaF2_wd4jscM",
	"alg": "RS256",
	"x5t": "YTceO5IJyyqR6jzDS5iAbpe42Jw",
	"kid": "YTceO5IJyyqR6jzDS5iAbpe42Jw"
}
```

This is a **JWT (JSON Web Token) header**, which provides metadata about the token. Here's a breakdown:

- **`"typ": "JWT"`** → Specifies the token type as **JWT** (JSON Web Token).
- **`"nonce"`** → A unique random value to prevent replay attacks, often used in authentication flows.
- **`"alg": "RS256"`** → Specifies the **RS256** algorithm (**RSA with SHA-256**) for digital signatures.
- **`"x5t"` & `"kid"`** → These are key identifiers:
  - **`"x5t"`** → Thumbprint of the signing certificate (X.509).
  - **`"kid"`** → Key ID, used to identify the correct public key for verification.

**What Does This Mean?**

This JWT header suggests the token is signed using **RS256** and includes key identifiers to locate the correct public key for verification. It’s typically used in **OAuth 2.0 / OpenID Connect** for secure authentication.

## **4. Replaying & CAE**

**Replaying an access token** means using an already obtained token to authenticate and perform actions **without re-authenticating**. However, once the token expires, replaying it won’t work.

### **Azure PowerShell**
```powershell
Connect-AzAccount -AccountId <UserId> -AccessToken <stolen_or_saved_access_token>
```
If the token is still valid, it will authenticate successfully. If expired, it will fail.

No method found yet.

### **Microsoft Graph**

No method found yet.

::foldable
#title
Can IP-Based Restrictions Block Access Token Requests?

#content
- **No**, if an authentication request is allowed from an IP, an access token **will be issued**.
- However, **Conditional Access (CA) policies can block authentication before a token is granted**.
- **Token replay attacks** can be mitigated by **CAE**, which enforces real-time revocation of access tokens.
::

::foldable
#title
What are the limitations?

#content
- **Access tokens can be replayed unless CAE is enforced.** `ClaimsChallengeRequiredException`
- Some tokens may be invalid if **Conditional Access Policies (CAPs) prevent authentication.**
- If authentication fails **before** issuing a token, the attack is stopped **before it starts**.

::

## 5. Refreshing

You can't obtain a refresh token from an access token alone because refresh tokens are only issued during the initial authentication flow, requiring user interaction to grant consent. This is why tools like **365Stealer** and **Evilginx** are useful—they intercept login sessions to capture refresh tokens for persistent access.

## 6. Disconnect

To clear the access tokens, always use:

### **Azure CLI**
```powershell
az logout
```

### **Azure PowerShell**
```powershell
Disconnect-AzAccount
```

### **Microsoft Graph**
```powershell
Disconnect-MgGraph
```
