## **In Production**

We've seen a lot of enumeration, but doing this in a **production environment** will drive you insane. There are far more objects to deal with, and the approach needs to be efficient.

We initially used **Graph** to map attack paths in Azure before **AzureHound** was even a thing. While it deserves an honorary mention, it’s being removed from the class soon since it’s no longer maintained. **RoadTools** and **AzureHound** are what we’ll focus on.

## **ROADTools vs. Azure PowerShell**

- The **Azure AD Module** is an official **Microsoft tool** and **doesn’t use** the **1.61-internal API** version.
- **RoadTools** uses this special API version, allowing deeper enumeration.
- RoadTools is essentially a **collection of API calls** that extracts valuable data in a structured way.

## **No Activity Logs**

> Previously, attackers could use **AAD Graph** for enumeration without triggering **Graph activity logs**. Running **AzureHound** with AAD Graph wouldn’t generate logs, but using **MS Graph** would.

Now, the **`1.61-internal API`** takes it a step further:

- Even a **normal authenticated user** can enumerate **Conditional Access policies**, which typically require **Security Reader** or **Global Reader** roles.
- Using **MS Graph**, you'd leave **a lot** of logs.
- However, new research shows ways to **enumerate Entra ID without using Graph API.**

For example, **IbizaR**, an internal API in **Entra**, allows enumeration without leaving **Graph activity logs**. There's also an implementation by **DolphinSec** that I plan to test.

## **AAD Graph Deprecation**

When **AAD Graph is deprecated**, **RoadTools will still work,** but with limitations:

1. You **likely won’t** be able to enumerate **Conditional Access policies.**
2. Enumeration will **leave Graph activity logs** unless another method is used.

::important
Attackers will always find **new ways** to enumerate **Entra ID** without Graph API.
::

## **RoadRecon**

**RoadTools** contains a tool called **RoadRecon**, which is used to enumerate **Entra ID**. However, note that:

::note
 **RoadTools/RoadRecon does NOT enumerate Azure resources.** It focuses only on **Entra ID.**
::

Enumeration using RoadRecon includes three steps:
1. Authentication
2. Data Gathering
3. Data Exploration

**Key points to keep in mind:**
- It uses a specific version of the **`AAD Graph API (1.61-internal)`** instead of **MS Graph.**
- This version provides **far more** information than standard AAD Graph or MS Graph.
- Running **RoadRecon plugin policies** lets us **enumerate Conditional Access policies** as a **low-privileged user**—something that normally requires **Security Reader** or **Global Reader**.
- Running **RoadRecon GUI** provides a snapshot of users, groups, applications, service principals, devices, admin units, etc.

### 1. **Clone The Repo**
   - You cloned the repository into the `roadtools` directory:
     ```bash
     git clone https://github.com/dirkjanm/ROADtools.git
     cd roadtools/
     ```

### 2. **.venv**
   - We can either use pyenv directly in vscode or make sure [`pyenv`](/syntax/python/pyenv) and `pyenv-virtualenv` are installed and configured in bash for managing Python versions and virtual environments with the CLI:
     ```bash
     # Install pyenv and pyenv-virtualenv if not already installed
     curl https://pyenv.run | bash

     # Set up shell config
     export PATH="$HOME/.pyenv/bin:$PATH"
     eval "$(pyenv init --path)"
     eval "$(pyenv init -)"
     eval "$(pyenv virtualenv-init -)"

     # Restart shell or open a new terminal session
     ```

   - Created a new virtual environment using [`pyenv`](/syntax/python/pyenv) for Python 3.12 inside /roadtools:
     ```bash
     pyenv install 3.12.0
     pyenv virtualenv 3.12.0 roadtools-env
     pyenv activate roadtools-env
     ```

### 3. **Python Dependencies**
   - Inside the `roadtools` directory, you installed the necessary Python dependencies with `pip`:
     ```bash
     cd roadrecon/
     pip install -e .
     ```
  - Run inside anhywhere `/roadtools` to see if you've installed roadrecon correctly:
     ```bash
     roadrecon -h
     ```
   - This was done for the `roadrecon` module (and possibly others like `roadlib` if applicable).

### 4. **Set up the Frontend**
   - Navigated to the `frontend` directory:
     ```bash
     cd roadrecon/frontend/
     ```

   - Installed the necessary `npm` dependencies:
     ```bash
     npm install
     ```

   - Built the frontend:
     ```bash
     npm run build
     ```

### 5. **Authenticate**
   - After setting up the environment and installing dependencies, you authenticated using username/password, access token, refresh token, PRT, or device code flow  with the following command:
     ```bash
     roadrecon auth -u <user@domain.com> -p <password>
     ```

### 6. **Gather**
   - You gathered information (e.g., data collection or scanning):
     ```bash
     roadrecon gather
     ```

### 7. **GUI on Port 5000**
   - To launch the GUI, you ran the following command, which likely started a Flask web server:
     ```bash
     roadrecon gui
     ```

   - The application was then accessible on `http://127.0.0.1:5000`.

## **The Attackers View**

Roles in Entra ID

- **Application Roles:** Assigned at the **application level** (e.g., granting Operations Group access to the **Finance Management System App**).
- **Directory Roles:** Apply **tenant-wide** (e.g., **Global Administrator, Security Reader**).

Example:

- The **Operations Group** can access the **Finance Management System App** as a **user**—others in the tenant **cannot.**
- **Admin App Simulation** has extensive permissions over **Mark Baldwin**, making it useful for **illicit consent grant attacks.**

## **Defensive Considerations**

::caution
If you connected your graph to your Microsoft Sentinal **An attacker running AzureHound/ROADTools 1,000 times could bankrupt an organization** due to excessive API calls.
::

- Enumeration using **Graph API** logs activity, making detection possible. But don’t stress over Graph activity logs— even as a defender, there are plenty of other ways to detect an attacker.
- The **layered defense approach**. The moment a sign-in takes place, you step in as a defender—every action gets logged. That’s exactly why Graph exists.

::warning
However, **blocking Graph API entirely isn’t possible**—only the Graph **module** can be blocked, not API calls.
::
