- **Azure CLI (`az CLI`)** is a command-line interface for managing Azure resources. It can be used on Windows, Linux, or via Cloud Shell.

> This file will teach you how to use the Azure CLI to enumerate the following resources.

| **Resource**        | **Description**                                                                 |
|---------------------|---------------------------------------------------------------------------------|
| **Virtual Machines** | Compute resources that allow you to run an operating system and applications in a virtualized environment on Azure. |
| **App Services**     | A platform-as-a-service (PaaS) for building, hosting, and scaling web applications and APIs with minimal infrastructure management. |
| **Function Apps**    | Serverless compute service for running event-driven code without worrying about the underlying infrastructure. |
| **Storage Accounts** | Scalable cloud storage solutions for storing various types of data such as files, blobs, tables, and queues. |
| **Key Vaults**       | A secure service to store and manage sensitive information like secrets, encryption keys, and certificates. |

---

::important
The Microsoft Graph API is ideal for Red Team operations focused on identity and access management, allowing detailed enumeration of Azure AD users, groups, and roles, making it valuable for social engineering and targeting high-value individuals.

In contrast, the Azure CLI excels at managing and interacting with Azure resources such as VMs, storage, and networking, providing a more infrastructure-focused approach for resource enumeration and potential privilege escalation.

While Graph API is best for understanding Azure AD and Microsoft 365 environments, Azure CLI is better suited for managing and manipulating Azure infrastructure
::

## Setup

### **Connecting to Entra ID with Azure CLI**

1. **Login to Azure CLI**:
   To use Azure CLI, you need to connect to Entra ID first. This will open a login page using the default browser.
   ```bash
   az login
   ```

2. **Login with Credentials**:
   You can log in using your username and password from the command line.
   ```bash
   az login -u test@defcorphq.onmicrosoft.com -p SuperVeryEasytoGuessPassword@1234
   ```

3. **Login Without Subscription**:
   If the user has no permissions on the subscription, use the following command:
   ```bash
   az login -u test@defcorphq.onmicrosoft.com -p SuperVeryEasytoGuessPassword@1234 --allow-no-subscriptions
   ```

4. **Configure Azure CLI**:
   You can configure Azure CLI to set default behavior like output type, location, and resource group.
   ```bash
   az configure
   ```

### **Searching for Popular Commands**
Azure CLI allows you to search for popular commands based on user telemetry:

- **Search for commands related to VMs**:
   ```bash
   az find "vm"
   ```

- **Search for commands within `az vm`**:
   ```bash
   az find "az vm"
   ```

- **Search for subcommands and parameters within `az vm list`**:
   ```bash
   az find "az vm list"
   ```

### **Formatting Output**

- **List all users in Entra ID and format output as a table**:
   ```bash
   az ad user list --output table
   ```

- **List only the `userPrincipalName` and `givenName` for all users and format output as a table**:
   ```bash
   az ad user list --query "[].[userPrincipalName,displayName]" --output table
   ```

- **Rename properties and list the `userPrincipalName` and `givenName` for all users**:
   ```bash
   az ad user list --query "[].{UPN:userPrincipalName, Name:displayName}" --output table
   ```

- **View query examples**:
   You can add `--query-examples` at the end of any command to see examples of JMESPath queries.
   ```bash
   az ad user show list --query-examples
   ```

## **Entra ID**

### **Tenant and Subscription Details**

- **Get details of the current tenant**:
   ```bash
   az account tenant list
   ```

- **Get details of the current subscription**:
   ```bash
   az account subscription list
   ```

- **List the current signed-in user**:
   ```bash
   az ad signed-in-user show
   ```

### **AAD Users**

- **List all users**:
   ```bash
   az ad user list
   ```

- **List only the `displayName` of all users and output as a table**:
   ```bash
   az ad user list --query "[].[displayName]" -o table
   ```

- **List specific user attributes (e.g., userPrincipalName)**:
   ```bash
   az ad user show --id test@defcorphq.onmicrosoft.com
   ```

- **Search for users containing the word "admin" in their display name (case-sensitive)**:
   ```bash
   az ad user list --query "[?contains(displayName,'admin')].displayName"
   ```

- **Search for users containing the word "admin" in their display name (case-insensitive)** using PowerShell:
   ```bash
   az ad user list | ConvertFrom-Json | %{$_.displayName -match "admin"}
   ```

### **AAD Groups**

- **List all groups**:
   ```bash
   az ad group list
   ```

- **List all groups with the display name**:
   ```bash
   az ad group list --query "[].[displayName]" -o table
   ```

- **Enumerate a specific group using display name or object ID**:
   ```bash
   az ad group show -g "VM Admins"
   az ad group show -g 783a312d-0de2-4490-92e4-539b0e4ee03e
   ```

- **Search for groups containing the word "admin" in their display name** (case-sensitive):
   ```bash
   az ad group list --query "[?contains(displayName,'admin')].displayName"
   ```

- **Search for groups containing the word "admin" (case-insensitive)** using PowerShell:
   ```bash
   az ad group list | ConvertFrom-Json | %{$_.displayName -match "admin"}
   ```

- **List groups synced from on-prem**:
   ```bash
   az ad group list --query "[?onPremisesSecurityIdentifier!=null].displayName"
   ```

- **List groups that are from Entra ID**:
   ```bash
   az ad group list --query "[?onPremisesSecurityIdentifier==null].displayName"
   ```

### **AAD Apps (Applications)**

- **Get all application objects registered in the current tenant**:
   ```bash
   az ad app list
   ```

- **Get details about a specific application using the application ID or object ID**:
   ```bash
   az ad app show --id a1333e88-1278-41bf-8145-155a069ebed0
   ```

- **Search for applications containing the word "app" in their display name**:
   ```bash
   az ad app list --query "[?contains(displayName,'app')].displayName"
   ```

- **List apps that have password credentials**:
   ```bash
   az ad app list --query "[?passwordCredentials != null].displayName"
   ```

- **List apps that have key credentials (certificate authentication)**:
   ```bash
   az ad app list --query "[?keyCredentials != null].displayName"
   ```

### **AAD Service Principals**

- **Enumerate Service Principals (Enterprise Applications)**:
   ```bash
   az ad sp list --all
   ```

- **Get details about a service principal using service principal ID or object ID**:
   ```bash
   az ad sp show --id cdddd16e-2611-4442-8f45-053e7c37a264
   ```

- **Get service principals based on the display name**:
   ```bash
   az ad sp list --all --query "[?contains(displayName,'app')].displayName"
   ```

- **Get the owner of a service principal**:
   ```bash
   az ad sp owner list --id cdddd16e-2611-4442-8f45-053e7c37a264 --query "[].[displayName]" -o table
   ```

- **List apps that have password credentials**:
   ```bash
   az ad sp list --all --query "[?passwordCredentials != null].displayName"
   ```

- **List apps that have key credentials (certificate authentication)**:
   ```bash
   az ad sp list --all --query "[?keyCredentials != null].displayName"
   ```

## az CLI

### **Virtual Machines**

- **List all Virtual Machines**:
   ```bash
   az vm list --output table
   ```

- **List virtual machines in a specific resource group**:
   ```bash
   az vm list --resource-group <your-resource-group-name> --output table
   ```

- **List virtual machines in a specific location**:
   ```bash
   az vm list --query "[?location=='eastus'].{Name:name, Location:location}" --output table
   ```

### **App Services**

#### **List All App Services**
To list all the App Services (Web Apps) in your Azure subscription:

```bash
az webapp list --output table
```

This command will display all App Services in a tabular format.

#### **List App Services in a Specific Resource Group**
To list App Services in a specific resource group:

```bash
az webapp list --resource-group <your-resource-group-name> --output table
```

Replace `<your-resource-group-name>` with the name of the resource group you're interested in.

#### **List App Services in a Specific Location**
To list App Services in a specific Azure region (e.g., "eastus"):

```bash
az webapp list --query "[?location=='eastus'].{Name:name, Location:location}" --output table
```

This query filters the App Services to show only those in the "eastus" location.

#### **Show Details of a Specific App Service**
To get details about a specific App Service using its name and resource group:

```bash
az webapp show --name <your-app-service-name> --resource-group <your-resource-group-name>
```

Replace `<your-app-service-name>` with the name of the App Service and `<your-resource-group-name>` with the resource group name.

### **Function Apps**

- **List all Function Apps**:
   ```bash
   az functionapp list --output table
   ```

- **List function apps in a specific resource group**:
   ```bash
   az functionapp list --resource-group <your-resource-group-name> --output table
   ```

- **List function apps in a specific location**:
   ```bash
   az functionapp list --query "[?location=='eastus'].{Name:name, Location:location}" --output table
   ```

### **Storage Accounts**

To list all the **Storage Accounts** in your Azure subscription:

```bash
az storage account list --output table
```

This will list the storage accounts in a table format. You can also filter the results further by adding a query or specifying a particular resource group. For example:

- **List storage accounts in a specific resource group:**

  ```bash
  az storage account list --resource-group <your-resource-group-name> --output table
  ```

- **List storage accounts in a specific location:**

  ```bash
  az storage account list --query "[?location=='eastus'].{Name:name, Location:location}" --output table
  ```

### **Key Vaults**

To list all **Key Vaults** in your Azure subscription:

```bash
az keyvault list --output table
```

This will show all the key vaults in your subscription in table format. Similar to storage accounts, you can filter key vaults by resource group or location.

- **List key vaults in a specific resource group:**

  ```bash
  az keyvault list --resource-group <your-resource-group-name> --output table
  ```

- **List key vaults in a specific location:**

  ```bash
  az keyvault list --query "[?location=='eastus'].{Name:name, Location:location}" --output table
  ```
