---
title: Az Powershell Attack Path
Entra ID enumeration focuses on understanding identities, roles, and permissions within an organization. The goal is to identify privileged accounts, misconfigurations, and potential lateral movement opportunities. Here's a structured attack path:
navigation: false
---

## 1. **What resources exist?**
**Why?**
- Identify the **users**, **groups**, and **devices** in the environment to understand the identity landscape.
- Discover **app registrations** and **enterprise applications** to know what services are present and available for use.

**Commands:**
```powershell
Get-AzADUser                   # List all users in the directory
Get-AzADGroup                  # List all groups in the directory
Get-AzADDevice                 # List all devices registered in Entra ID
Get-AzADApplication             # List all app registrations
Get-AzADServicePrincipal       # List all enterprise applications (service principals)
```

## 2. **Who has access?**
**Why?**
- Discover who has access to **critical resources** (e.g., **privileged users**, service accounts, or members of sensitive groups).
- Enumerate group memberships, especially for high-privilege roles like **Global Admin** and **User Admin**.
- Identify devices that have been granted access to these resources.

**Commands:**
```powershell
Get-AzADGroupMember -ObjectId <GroupID>   # List members of specific groups
Get-AzRoleAssignment                    # List role assignments for users and service principals
Get-AzADUser -Filter "UserType eq 'Member'"  # Find users assigned to roles
```

## 3. **What apps have permissions?**
**Why?**
- Identify which **apps** (app registrations or service principals) have permissions to access **Azure resources** or **critical data**.
- Check if any **apps** have excessive privileges, such as the ability to read emails, reset passwords, or access key vaults.

**Commands:**
```powershell
Get-AzADServicePrincipal      # List enterprise apps (service principals)
Get-AzADServicePrincipal | Select DisplayName, AppId, AppRoleAssignments  # Check app role assignments
Get-AzADApplication           # List app registrations
```

## 4. **Where can we pivot next? (VMs, Apps, Storage, Secrets)**
**Why?**
- Identify potential **lateral movement** or **pivoting** opportunities.
- Focus on areas such as **VMs**, **Azure Storage**, and **Secrets** (key vaults).
- Look for misconfigurations in permissions that could give access to **sensitive data** or **critical resources** (e.g., apps with access to key vaults or VMs with weak security).

**Commands:**
```powershell
Get-AzADDevice                 # Find devices that could be used for lateral movement
Get-AzKeyVault                 # Check for Azure Key Vaults and their access policies
Get-AzVM                       # Enumerate virtual machines (VMs) in the environment
Get-AzStorageAccount           # List Azure Storage Accounts
```
