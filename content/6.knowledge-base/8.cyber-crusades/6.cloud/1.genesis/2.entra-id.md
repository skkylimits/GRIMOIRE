## **Entra ID (Azure AD)**

Entra ID uses oAuth 2.0 and OpenID Connect to provide secure access to applications and resources. It is a cloud-based identity and access management solution that provides secure access to applications and resources. Entra ID is a product offering within Azure, Microsoft’s cloud platform.

- **Entra ID** is **Microsoft’s cloud-based identity and access management service**, formerly known as **Azure AD**.
- **Entra ID** can be used to access both:
  - **External resources** like Azure Portal, Office 365, etc.
  - **Internal resources** like on-premises applications.
- Provides **secure remote access** for **AD-integrated apps**, **devices**, and **identity governance** for **AD accounts**.

## **Entra ID vs Azure**

- **Entra ID** is **not** Azure.
- **Entra ID** is a **product offering** within Azure.
- **Azure** is Microsoft’s **cloud platform**, whereas **Entra ID** is the **enterprise identity service** within Azure.

## Entra ID vs. On-Prem AD

- The only real similarity between Entra ID and on-prem AD is that both manage identities and access.
- While they share some terminology, **Entra ID works differently**—trying to compare it directly to on-prem AD can be misleading.
- **Entra ID is not a cloud version of Active Directory.** Instead, **Azure AD Domain Services** provides "domain controller as a service" for those who need traditional AD features in the cloud.
- You can integrate on-prem AD with Entra ID to create a **hybrid identity** solution.

## **Terminology**

- **Tenant** – An instance of **Entra ID**, representing a **single organization**.
- **Entra ID Directory** – A **dedicated directory** for each tenant, used to perform identity and access management for resources.
- **Subscriptions** – Used to **pay for services**. Multiple subscriptions can exist within a directory.
- **Core Domain** – The **default domain** `<tenant>.onmicrosoft.com`. Custom domain names can also be defined.
