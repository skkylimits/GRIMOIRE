## I OSINT.md

> At the end of [this](/knowledge-base/cyber-crusades/cloud/recon-and-discovery/osint) module you'll be able to enumerate the following using OSINT techniques.

If we only know the **domain name** or **email addresses** of the target organization (e.g., **defcorphq**), we can extract useful information, including:

- Whether the target organization uses an **Azure tenant & Entra ID**.
	- _Check if Entra ID is being utilized for managing identities and access._ `NameSpaceType`
- **Tenant ID**.
- **Tenant name**.
- **Authentication type** (e.g., **Federation** or not).
- **Domains** associated with the tenant.
- **Azure services** used by the target organization.
- **Research & Validate email IDs** of users in the organization.
	_Ensure the email addresses for the organization are properly configured and valid._
   - Admin Email: `admin@defcorphq.com`
   - Support Email: `support@defcorphq.com`
   - User Email: `user@defcorphq.com`

## IV azure-powershell.md

> Az PowerShell provides a powerful way to interact with Entra ID and Azure resources. Targeted enumeration avoids detection, unlike broad tools that trigger security alerts. At the end of [this](/knowledge-base/cyber-crusades/cloud/enumeration/azure-powershell) file you will be able to enumerate:

| **Resource**                                | **Command**                                                                 |
|---------------------------------------------|-----------------------------------------------------------------------------|
| **All Resources Visible to the User** 🌐    | `Get-AzResource`                                                            |
| **All Azure Roles Assigned to the User** 🔑 | `Get-AzRoleAssignment -SignInName test@defcorphq.onmicrosoft.com`           |
| **Virtual Machines** 🖥️                     | `Get-AzVM`                                                                  |
| **App Services** 🌍                         | `Get-AzWebApp`                                                              |
| **Function Apps** 🔧                        | `Get-AzWebApp` (since Function Apps are part of App Services)               |
| **Storage Accounts** 💾                     | `Get-AzStorageAccount`                                                      |
| **Key Vaults** 🔒                           | `Get-AzKeyVault`                                                            |

## V azure-cli.md

> [This](/knowledge-base/cyber-crusades/cloud/enumeration/azure-cli) file helps identify Azure resources within the `tenant`, which can be useful for **attack path analysis**, **inventory management**, and **security auditing**. By the end of this, you will be able to enumerate using az cli using the credentials of `UserId`:

| **Resource**        | **Description**                                                                 | **Command**                                      |
|---------------------|---------------------------------------------------------------------------------|------------------------------------------------|
| **Virtual Machines** 🖥️  | Compute resources that allow you to run an operating system and applications in a virtualized environment on Azure. | `az vm list --output table` |
| **App Services** 🌐  | A platform-as-a-service (PaaS) for building, hosting, and scaling web applications and APIs with minimal infrastructure management. | `az webapp list --output table` |
| **Function Apps** ⚡  | Serverless compute service for running event-driven code without worrying about the underlying infrastructure. | `az functionapp list --output table` |
| **Storage Accounts** 📦  | Scalable cloud storage solutions for storing various types of data such as files, blobs, tables, and queues. | `az storage account list --output table` |
| **Key Vaults** 🔑  | A secure service to store and manage sensitive information like secrets, encryption keys, and certificates. | `az keyvault list --output table` |
