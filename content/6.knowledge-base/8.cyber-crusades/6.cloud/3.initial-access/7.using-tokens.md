But you still want to take something away from this class
stay away from user identities. They are very closely observed. Chances of detection are high.
::

::caution
You can use refresh token to get new access tokens. **Refresh tokens** allow you to obtain new access tokens once they expire. However, accessing user identities or extracting tokens from user machines is not recommended due to high detection risks.
::

- You won't always be able to get tokens manually unless you have an interactive session or can use a tool that mimics the authentication process, that's why you need to steal them.

## **Using Stolen Tokens**

### **Using Tokens in CLI Tools (AzureAD & Mg Modules)**
- **AzureAD Module:**
  ```powershell
  Connect-AzureAD -AccountId user@domain.com -AadAccessToken eyJ0eXA...
  ```
- **Microsoft Graph Module:**
  ```powershell
  Connect-MgGraph –AccessToken ($Token | ConvertTo-SecureString -AsPlainText -Force)
  ```

### **Using Tokens with APIs**
- **Two main API endpoints:**
  - **Azure Resource Manager (ARM):** `https://management.azure.com`
  - **Microsoft Graph:** `https://graph.microsoft.com`

#### **Example: Using a Token with Azure Resource Manager (ARM API)**
- List all subscriptions:
  ```powershell
  $Token = 'eyJ0eXAi..'
  $URI = 'https://management.azure.com/subscriptions?api-version=2020-01-01'
  $RequestParams = @{
   Method = 'GET'
   Uri = $URI
   Headers = @{ 'Authorization' = "Bearer $Token" }
  }
  (Invoke-RestMethod @RequestParams).value
  ```

#### **Example: Using a Token with Microsoft Graph API**
- List all users:
  ```powershell
  $Token = 'eyJ0eXAi..'
  $URI = 'https://graph.microsoft.com/v1.0/users'
  $RequestParams = @{
   Method = 'GET'
   Uri = $URI
   Headers = @{ 'Authorization' = "Bearer $Token" }
  }
  (Invoke-RestMethod @RequestParams).value
  ```
