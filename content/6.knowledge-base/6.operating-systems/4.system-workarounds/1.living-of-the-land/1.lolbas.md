title: Living off the Land
description: Learn about Living off the Land (LOLBAS) tactics in cybersecurity, including examples, use cases, and defense strategies against these stealthy attacks.
---

**Living off the Land (LOLBAS)** is a cybersecurity term that describes a tactic where attackers use legitimate tools and features that are already present on a target system to carry out malicious activities. Instead of deploying custom malware or external tools, attackers exploit the software, scripts, and processes that are trusted and commonly used in the environment.

### Why Attackers Use LOLBAS Tactics:
- **Stealth**: Using legitimate tools avoids detection by traditional security mechanisms like antivirus or intrusion detection systems, which might flag custom malware but not standard tools.
- **Trust**: Security solutions and system administrators often trust native tools, making their malicious use harder to distinguish.
- **Portability**: Attackers don't need to introduce new files or binaries, reducing the risk of leaving traces.

---

### Examples of LOLBAS Tools and Techniques:
1. **Windows Built-In Tools**:
   - **PowerShell**: Used for executing scripts, downloading payloads, or performing reconnaissance.
   - **WMIC (Windows Management Instrumentation Command-line)**: Exploited to gather system information or execute commands remotely.
   - **PsExec (Sysinternals)**: Enables remote execution of commands.
   - **Task Scheduler**: Used to establish persistence by scheduling malicious tasks.
   - **BITSAdmin**: Misused to download or upload malicious files.

2. **Native Processes**:
   - **rundll32.exe**: Runs DLL files and is often used to execute malicious payloads.
   - **mshta.exe**: Executes HTML Applications (HTA) files and is used to execute malicious JavaScript or VBScript.

3. **Third-Party or Admin Tools**:
   - **Sysinternals Suite**: Tools like Process Explorer or PsExec can be misused for malicious purposes.
   - **Remote Desktop Protocol (RDP)**: Attackers exploit RDP to maintain access.

4. **Other Techniques**:
   - Abusing legitimate cloud services (e.g., Google Drive or OneDrive) to exfiltrate data.
   - Using tools like `certutil` for file downloads or decoding malicious payloads.

---

### How LOLBAS Tactics Are Used in Cyber Attacks:
- **Initial Access**: Attackers use phishing emails to exploit tools like macros in Microsoft Office.
- **Privilege Escalation**: Misuse system binaries to escalate privileges.
- **Lateral Movement**: Employ tools like PsExec or RDP to move within the network.
- **Persistence**: Schedule malicious tasks or modify registry keys.
- **Data Exfiltration**: Use trusted protocols (e.g., HTTP, HTTPS) to exfiltrate data while avoiding detection.

---

### Defense Against LOLBAS Attacks:
1. **Endpoint Detection and Response (EDR)**:
   - Use EDR tools that analyze behavioral patterns rather than solely relying on known signatures.

2. **Restrict Execution**:
   - Limit the use of PowerShell, macros, and other tools to authorized users only.
   - Disable unnecessary services like WMIC or Remote Desktop if not needed.

3. **Application Whitelisting**:
   - Implement strict policies for approved software and scripts.

4. **Logging and Monitoring**:
   - Enable detailed logging for tools like PowerShell, Task Scheduler, and other built-in utilities.
   - Monitor for abnormal usage patterns, such as PowerShell commands with encoded scripts.

5. **User Training**:
   - Educate employees about phishing and social engineering techniques.

---

By blending into the environment, LOLBAS tactics make cyberattacks more sophisticated and challenging to detect. Defenders need proactive monitoring and advanced threat detection techniques to counter these methods effectively.
