## VBoxManage

**VBoxManage - Command-Line Control for Oracle VM VirtualBox**

**Overview:**
VBoxManage is the command-line interface for Oracle VM VirtualBox, offering full control over VirtualBox, including features not available in the GUI.

### Setup Instructions

**1. Run VBoxManage:**
- Open Command Prompt and navigate to the VirtualBox folder:
```cmd [cmd]
cd "C:\Program Files\Oracle\VirtualBox"
```

**2. Optional - Add to PATH:**
- Add VirtualBox to your system PATH for easier access:
```cmd [cmd]
SET PATH=%PATH%;C:\Program Files\Oracle\VirtualBox
```

Then:

```cmd [cmd]
VBoxManage
```

### **Troubleshooting Tips**

1. **Permission Errors:**
   - Restart VirtualBox in **Admin Mode** if you encounter:
```cmd
VERR_ACCESS_DENIED
```

2. **Environment Issues:**
   - Confirm VirtualBox folder is correctly added to the PATH variable.
   - Restart the terminal or system if changes don’t take effect.

## Memory Acquisition

1. list all VMs:

```cmd [cmd]
VBoxManage list vms
```

2. Extract Memory Dump:

```cmd [cmd]
VBoxManage debugvm {vm-uuid} dumpvmcore --filename win10-memory.raw
```

VBoxManage debugvm {vm-uuid} dumpvmcore --filename C:\Users\%user%\Desktop\win10-memory.raw

### Shasum Verification

#### Linux

Install shasum
```shell [shell]
sudo apt install perl
```

Verify installation
```shell [shell]
shasum --version
```

Generate hash
```shell [shell]
shasum win10-memory.raw -> win10-memory-hash.txt
```

See generated hash in `win10-memory-hash.txt`
```shell [shell]
cat win10-memory-hash.txt
```

::warning
We need to make sure that this hash, always matches the original one when sharing this with others.
::

#### Wndows

```cmd [cmd]
certUtil -hashfile win10-memory.raw -> win10-memory-hash.txt
```

See generated hash in `win10-memory-hash.txt`
```cmd [cmd]
type win10-memory-hash.txt
```

## Disk Acquisition

::note
To ensure that Windows writes the latest changes and updates to the disk, we need to unpause the VM. Afterward, we should power it down to properly save the data and maintain system integrity.
::

### Extract Disk Image:

#### **Manual: Extracting UUID and Exporting VHD**

1. **Access the Tools Menu:**
   - In the left menu, click on **Tools** → **Media**.

2. **Locate the Machine:**
   - Find the machine that was just powered down.

3. **Extract the UUID:**
   - Look for the UUID of the machine (the one you just powered down) and copy it.

4. **Export the VHD:**
   - **Click on "Copy"** to initiate the export process.
   - **Select VHD** as the export format.
   - Choose **Dynamically Allocated** for storage settings.
   - **Select the Destination Location** where you want to save the exported file.

5. **Complete the Process:**
   - Press **Finish** to complete the export and save the VHD file.

This process ensures that you can extract and export your virtual machine's VHD while preserving the machine's settings and state.

#### **Command-Line: Extracting Disk Image**

List hdds
```cmd [cmd]
vboxmanage list hdds
```

Export VHD
```cmd [cmd]
vboxmanage clonemedium disk {disk-uuid} --format VHD win10-disk.vhd
```

Hash the VHD
```
certUtil -hasfile win10-disk.vhd > win10-disk-hash.txt`
```

Verify the hasg
```
type win10-disk-hash.txt
```

On Unix/Linux to see all hashes if were in the same directory
```
cat *.txt
```
