| **Function**          | **Description**                                                                   |
| --------------------- | --------------------------------------------------------------------------------- |
| `on(names, callback)` | Subscribes to one or more events. Stores the callback to be triggered later.      |
| `off(names)`          | Unsubscribes from one or more events. Removes callbacks from the event list.      |
| `trigger(name, args)` | Triggers the specified event(s), calling all associated callbacks with arguments. |
| `resolveNames(names)` | Cleans up and splits event names into an array of valid strings.                  |
| `resolveName(name)`   | Parses a single event name into `{ value, namespace }`.                           |



## 🎬 Imagine This Whole App Is Like a Giant Movie Set

You’ve got:

* 🎥 A **camera guy** (Camera.js)
* 🎨 A **painter** (Renderer.js)
* 🖥️ A **screen designer** (UI.js)
* 📐 A **stage manager** (Sizes.js)
* 🧠 A **director** (Experience.js)

They all need to *know* when the window resizes, right? Because the stage (your screen) just got bigger or smaller!

But you don’t want **every single person** to keep checking every second if the screen size changed. That’s super wasteful and messy.

## 🪧 STEP 1: Someone Watches for Resize (Sizes.js)

In `Sizes.js`:

```js
window.addEventListener('resize', () => {
	this.resize()
	setTimeout(() => this.trigger('resize'))
})
```

### 🔍 What this means:

* The **stage manager** (Sizes.js) is watching the window.
* When the window changes size, he:

  1. **Updates his clipboard** with the new size (`this.resize()`)
  2. Then **shouts** "RESIZE!" across the set by calling `this.trigger('resize')`.

👉 `trigger('resize')` is how he broadcasts the event.

## 🧠 STEP 2: The Director (Experience.js) Is Listening

```js
this.sizes.on('resize', () => {
	this.resize()
})
```

The **director** hears the resize shout, and then yells:

> "Hey camera guy! Painter! Screen designer! Resize yourselves!"

And then the director calls:

```js
resize() {
  this.camera.resize()
  this.renderer.resize()
  this.ui.resize()
}
```

Every team does their part:

* Camera repositions.
* Renderer adjusts canvas.
* UI rearranges.

🎉 Everyone updates themselves in perfect sync.

## 🧠 STEP 3: It’s the Same With the Clock (Time.js + RAF)

You also have a **clock guy** (`Time.js`) who ticks every animation frame:

```js
this.time.on('RAF', () => {
	this.update()
})
```

Every frame, `Time.js` goes:

> "Hey everyone! One more frame just passed!"

It does this by calling:

```js[Time.js]
this.trigger('RAF')
```

inside its `RAF()` method. That’s how it **emits the event**, so everyone listening (like the director) knows it's time to update.

Then the **director** again yells:

```js
update() {
  this.camera.update()
  this.renderer.update()
  this.world.update()
  this.ui.update()
}
```

And all the teams animate smoothly every frame.

## 🏁 TL;DR:

Let’s say your screen is a **stage**, and everything happening is a **live play**. Here’s the order of action:

| Who?                                 | What they do                                                 |
| ------------------------------------ | ------------------------------------------------------------ |
| `window.addEventListener`            | Tells the **stage manager** (Sizes.js) the stage has resized |
| `Sizes.trigger('resize')`            | The manager shouts “resize!” to everyone who cares           |
| `Experience.sizes.on('resize', ...)` | The director hears it and calls resize on the crew           |
| `Camera/Renderer/UI`                 | Each part does its job to adapt to the new stage size        |

Same deal for **animations** with the **time guy**.
