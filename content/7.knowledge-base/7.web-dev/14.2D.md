In this project, our UI system functions like a well-coordinated **movie set**. Each component is an actor or a crew member, contributing to the overall experience. And just like in any film production, every crew member has a specific role, and they all rely on one another to make the show a success.

### **🎬 Movie Set Setup - UI Initialization**

In our movie, the `UI` class is the **director**, calling the shots and ensuring that the show runs smoothly. When everything is loaded and ready to go, it’s time to roll the cameras and start filming!

* **First, we wait for the resources**: Like waiting for the set to be built and props to be delivered, the `Experience.js` class makes sure that all resources are loaded.
* **Then, the show begins**: Once the resources are ready, the director (`UI.js`) calls action and initializes all the components, one by one.
* **Each component has a specific task**: From showing the landing page to handling scroll actions, each crew member (component) gets their moment to shine.

### **🌳 The UI Crew's Hierarchy - Who Works with Who?**

### **📽️ The Flow of Events - How the Movie Unfolds**

1. **Waiting for Resources**: The set isn’t ready until the `Experience.js` class loads everything.
2. **Lights, Camera, Action!**: Once everything is in place, the `UI.js` director calls the `loaded` event, and all components are instantiated.
3. **Resize Handling**: Just like adjusting the camera for the perfect shot, each component will respond to resizing the window and recalibrate accordingly.
4. **On-Set Updates**: During the show, the `update()` function keeps things running smoothly. It ensures that animations, scroll interactions, and user inputs are up to date.

### **🎞️ Event Handling - How the Crew Communicates**

Here’s how the different components talk to each other during the show:

* **Resource Events**: Once resources are loaded, the `UI.js` director lets the crew know and everything starts (via the `loaded` event).
* **Resize Events**: When the scene (window) changes, the stage manager (`Sizes.js`) adjusts the set, ensuring the entire experience fits the new dimensions.
* **Update Events**: Every frame, the director calls "action!" through the `update()` function, updating scroll actions, animations, and all real-time interactions.
