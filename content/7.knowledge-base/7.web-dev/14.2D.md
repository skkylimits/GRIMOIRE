### **рЯУљпЄП The Flow of Events - How the Movie Unfolds**

1. **Waiting for Resources**: The set isnвАЩt ready until the `Experience.js` class loads everything.
2. **Lights, Camera, Action!**: Once everything is in place, the `UI.js` director calls the `loaded` event, and all components are instantiated.
3. **Resize Handling**: Just like adjusting the camera for the perfect shot, each component will respond to resizing the window and recalibrate accordingly.
4. **On-Set Updates**: During the show, the `update()` function keeps things running smoothly. It ensures that animations, scroll interactions, and user inputs are up to date.

### **рЯОЮпЄП Event Handling - How the Crew Communicates**

HereвАЩs how the different components talk to each other during the show:

* **Resource Events**: Once resources are loaded, the `UI.js` director lets the crew know and everything starts (via the `loaded` event).
* **Resize Events**: When the scene (window) changes, the stage manager (`Sizes.js`) adjusts the set, ensuring the entire experience fits the new dimensions.
* **Update Events**: Every frame, the director calls "action!" through the `update()` function, updating scroll actions, animations, and all real-time interactions.
