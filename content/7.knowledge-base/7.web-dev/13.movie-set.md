## `style.css`

> Global stylesheet to set up base styles, variables, fonts, and utilities across the app.

Responsible for:

* Importing modular CSS files like `intro.css`, `menu.css`, `contact.css`, etc.
* Setting up global variables (`--primary`, `--text`, etc.)
* Defining font faces (`BlenderProBold`, `BlenderProBook`)
* Providing base styles for HTML elements (`h1`, `a`, `section`, etc.)
* Custom cursors, hover effects, utility classes, and media queries

Useful when adding additional stylesheets for new features, layouts, or page sections—everything stays modular but shares a consistent foundation.

## `index.js`

```js
import Experience from './Experience/Experience'
import './style.css'

const experience = new Experience(document.querySelector('canvas.webgl'))
```

* **Loads global styles**
* **Initializes** the main `Experience` class querying a `<canvas>` element** in your HTML with class `.webgl`

## Movie Set

We’re totally rolling with the movie set analogy from now on 🎬✨

> And YES — Making a list of everyone on the set (your classes and components) and what they do is one of the *smartest* things you can do to understand your project deeply.

It’s like making a cast and crew list for your movie, so you know:

* Who’s doing what
* Who talks to who
* Who reacts to what events (like “resize” or “frame update”)


## 🎭 Your Experience.js Cast & Crew (So Far)

Here’s what we’ve seen already:

| Role                           | File / Class      | What They Do                                                           |
| ------------------------------ | ----------------- | ---------------------------------------------------------------------- |
| 🎬 **Director**                | `Experience.js`   | The big boss. Starts the show, coordinates everyone.                   |
| 👨‍🎥 **Camera Guy**           | `Camera.js`       | Controls where the viewer looks. Zooms, pans, moves.                   |
| 🎨 **Painter**                 | `Renderer.js`     | Draws what the camera sees on the canvas. Updates every frame.         |
| 🖥️ **Screen Designer**        | `UI.js`           | Handles buttons, overlays, anything 2D on screen.                      |
| 📐 **Stage Manager**           | `Sizes.js`        | Keeps track of the screen size. Shouts "resize!" when needed.          |
| ⏰ **Clock Guy**                | `Time.js`         | Ticks every frame. Shouts "RAF!" to update animations.                 |
| 🧠 **Event System**            | `EventEmitter.js` | Like walkie-talkies between crew. Lets people shout and others listen. |
| 🗺️ **Navigator**              | `Waypoints.js`    | Handles locations the user can move/teleport to.                       |
| ✋ **Gesture Coach**            | `Gestures.js`     | Interprets user touch/mouse gestures.                                  |
| 👇 **Ray Guy (Laser Pointer)** | `Raycasting.js`   | Shoots invisible rays to see what’s under the cursor or gaze.          |
| 🚀 **Teleport Master**         | `Teleport.js`     | Handles moving from place to place, based on waypoints.                |
| 🎬 **Set Designer**            | `World/World.js`  | Creates and updates 3D objects in the world.                           |
| 📦 **Resource Manager**        | `Recources.js`    | Loads and gives access to textures, models, etc.                       |
| 🎒 **Backpack (Assets List)**  | `Sources.js`      | Just a list of what needs to be loaded — used by Resources.            |
| 🛠️ **Debug Console**          | `Debug.js`        | Opens up tools for tweaking things during development.                 |

## ✅ Why This Is So Powerful

* You can see **how they communicate** (via `on()` and `trigger()` thanks to EventEmitter).
* You understand **who depends on who**.
* It’s way easier to debug because you know **who to blame** if something breaks 😅
* When you add a new feature, you can ask:

  > “Which crew member should handle this?”

## 2. 🌳 Family Tree Style

This shows the hierarchy — who creates who.

```
Experience.js (Director)
├── Sizes.js (Stage Manager) ─▶ triggers "resize"
│   ├── Camera.js (Camera Guy) ─▶ listens to "resize"
│   ├── Renderer.js (Painter) ─▶ listens to "resize"
│   └── UI.js (Screen Designer) ─▶ listens to "resize"
├── Time.js (Clock Guy) ─▶ triggers "RAF"
│   ├── Camera.js (Camera Guy) ─▶ listens to "RAF"
│   ├── Renderer.js (Painter) ─▶ listens to "RAF"
│   ├── World.js (Set Designer) ─▶ listens to "RAF"
│   └── UI.js (Screen Designer) ─▶ listens to "RAF"
├── Resources.js (Asset Loader)
│   └── Sources.js (Asset List)
├── World.js (Set Designer) ─▶ listens to "RAF"
├── Gestures.js (Gesture Coach) ─▶ triggers "scroll-up", "scroll-down", "swipe-left", "swipe-right", "touch-start", "touch-down", "touch-up"
├── Waypoints.js (Navigator) ─▶ listens for teleport-related events
├── Teleport.js (Teleport Master) ─▶ listens for waypoints to teleport to
└── Raycasting.js (Ray Guy) ─▶ triggers hover and click interactions

```

🧠 Everything is created by the **Director**, and most of them are **listening** to events like `resize` or `RAF`.

## 3. 🔁 Flowchart Style

Here’s a simplified diagram of how **communication flows**:

```
[Window Resize] ─▶ [Sizes.js] ─▶ triggers "resize"
                                     │
                                     ├─▶ Camera.resize()
                                     ├─▶ Renderer.resize()
                                     └─▶ UI.resize()

[requestAnimationFrame] ─▶ [Time.js] ─▶ triggers "RAF"
                                           │
                                           ├─▶ Camera.update()
                                           ├─▶ Renderer.update()
                                           ├─▶ World.update()
                                           └─▶ UI.update()

[Mousewheel / Keyboard] ─▶ [Gestures.js] ─▶ triggers "scroll-down", "scroll-up", "scroll"
                                           │
                                           ├─▶ Trigger scrolling actions based on user input
                                           └─▶ Update scroll states

[Touch Start / Touch End] ─▶ [Gestures.js] ─▶ triggers "touch-start", "swipe-left", "swipe-right", "touch-down", "touch-up"
                                           │
                                           ├─▶ Trigger actions based on swipe and touch gestures
                                           └─▶ Update touch states

[Hover Interaction] ─▶ [Raycasting.js] ─▶ triggers hover/click actions
                                          │
                                          ├─▶ Set cursor (pointer/default)
                                          ├─▶ Trigger click events
                                          └─▶ Handle hover interactions
```

👂 Components **listen** to events like `"resize"` or `"RAF"` and **do something** when they hear it.
