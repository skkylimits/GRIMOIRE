### 🔍 1️⃣ Waar protocollen wonen

| Soort               | Wat het beheert                                          | Hoofdbron                                            |
| :------------------ | :------------------------------------------------------- | :--------------------------------------------------- |
| 📘 **IETF (RFC’s)** | De technische specificaties van alle internetprotocollen | [datatracker.ietf.org](https://datatracker.ietf.org) |
| 🗂️ **IANA**        | De nummering, poorten, parameters & registries           | [iana.org/protocols](https://www.iana.org/protocols) |
| 🧱 **IEEE**         | Fysieke en datalink-standaarden (Ethernet, Wi-Fi)        | [ieee802.org](https://www.ieee802.org)               |
| 🌐 **W3C**          | Weblaag — HTML, CSS, WebSockets, HTTP specs              | [w3.org](https://www.w3.org)                         |
| 📞 **ITU-T**        | Telecom & transport (bijv. VoIP, SS7, modemstandaarden)  | [itu.int](https://www.itu.int)                       |

---

### 🧭 2️⃣ Hoe je protocollen vindt

| Stap | Actie                          | Tool                                                        |
| :--- | :----------------------------- | :---------------------------------------------------------- |
| 1    | Zoek op **naam → RFC**         | `site:ietf.org "<protocol name>" RFC`                       |
| 2    | Zoek op **RFC → document**     | `iana.org/go/rfcXXXX` of `datatracker.ietf.org/doc/rfcXXXX` |
| 3    | Zoek op **parameters/poorten** | IANA registry voor dat protocol                             |
| 4    | Zoek op **updates**            | Bekijk “Updated by / Obsoletes” sectie in RFC               |
| 5    | Controleer **standaardstatus** | “Internet Standard”, “Proposed Standard”, “Draft”           |

Zoek je naar DNS of SMTP?
Gebruik:

```
site:ietf.org "Simple Mail Transfer Protocol" RFC
site:ietf.org "Domain Names - Implementation and Specification" RFC
```

::

### ⚙️ 3️⃣ Praktische workflow

```text
Protocolnaam → RFC-nummer → IANA-registries → updates → implementatie
```

Bijvoorbeeld voor **SMTP**:

```
SMTP → RFC 5321 → IANA SMTP Registry → RFC 5321bis (draft) → Postfix config
```

Voor **DNS**:

```
DNS → RFC 1035 → IANA DNS Parameters → RFC 8499 (terminologie) → dig/nslookup
```

---

### 🧩 4️⃣ Wat je leert in dit hoofdstuk

* Hoe je *de juiste* RFC vindt (zonder te verdwalen in drafts)
* Wat de statuslabels betekenen (Internet Standard vs Proposed)
* Hoe IANA registries en IETF RFC’s elkaar aanvullen
* Hoe je protocollen traceert die elkaar “updaten” of “vervangen”
* Hoe je experimentele protocollen herkent (bijv. QUIC vóór standaardisatie)

### 💡 Bonus-sectie

---

title: RFC Lifecycle
icon: i-octicon:git-branch
--------------------------

1. `Draft` → idee van een werkgroep
2. `Proposed Standard` → eerste goedgekeurde versie
3. `Internet Standard` → volwassen en breed geïmplementeerd
4. `Historic / Obsolete` → verouderd of vervangen
   ::
