## 🎯 Waarom TCP Leren?

**TCP (Transmission Control Protocol)** is het meest gebruikte transport protocol op het internet. Als security engineer, malware analyst of network troubleshooter **moet** je TCP op expert-level begrijpen.

### Waarom Dit Belangrijk Is

**90% van applicaties gebruikt TCP:**
- Web traffic (HTTP/HTTPS)
- Email (SMTP, IMAP, POP3)
- File transfers (FTP, SFTP, SMB)
- Databases (MySQL, PostgreSQL, MSSQL)
- **Malware C2 communicatie**
- **Data exfiltration**
- **Remote access tools**

**90% van performance problemen zit in TCP, niet in het netwerk:**
- Window size bottlenecks
- Retransmissions
- Flow control issues
- Application delays
- Misconfiguraties (MSS, MTU)

::note
**Reality check:** Network teams kijken naar Ethernet/Switches/Routers (Layer 1-3). Application teams kijken naar hun code (Layer 7). **TCP (Layer 4) valt ertussen** - niemand "bezit" deze laag. Dit is jouw kans om expert te worden waar anderen blind zijn!
::

## 💡 De 3 Core Concepts

Er zijn **3 dingen** die je absoluut moet masteren:

### 1. The Handshake (SYN → SYN/ACK → ACK)

**Wat gebeurt er:**
- Client en server negotiëren parameters
- MSS (Maximum Segment Size)
- Window Scaling
- SACK (Selective Acknowledgement)
- RTT (Round Trip Time) wordt gemeten

**Waarom belangrijk:**
- Handshake bepaalt performance voor hele sessie
- Misconfiguraties hier = slow performance
- TTL analysis toont network path
- Window scaling ontbreekt = throughput ceiling

### 2. Sequence Numbers

**Wat zijn sequence numbers:**
- Elk byte krijgt een nummer
- Sender tracks welke bytes verstuurd zijn
- Receiver kan gaps/holes detecteren
- Enables reliable, ordered delivery

**Waarom belangrijk:**
- Lost packets zijn direct zichtbaar
- Out-of-order delivery detecteren
- Retransmissions begrijpen
- Data completeness verifiëren

### 3. Acknowledgement Numbers

**Wat zijn ACK numbers:**
- Receiver zegt: "Ik heb alles tot hier ontvangen"
- Cumulative acknowledgements
- Next expected sequence number

**Waarom belangrijk:**
- ACK = data is veilig aangekomen
- Geen ACK = data timeout → retransmit
- Duplicate ACKs = missing segment
- Fast retransmit trigger

::important
**Master these 3 concepts** en je bent al verder dan 80% van network engineers. Alles daarna bouwt hierop voort!
::

## 🔍 Real-World Impact

### Security & Malware Analysis

**C2 Communication Detection:**
- Malware gebruikt vaak TCP voor beaconing
- Analyze handshakes naar onbekende IPs
- Detect data exfiltration via TCP length patterns
- Identify encryption via payload entropy

**Incident Response:**
- Timeline reconstruction via sequence numbers
- Data volume calculations
- Connection duration analysis
- Lateral movement tracking

### Performance Troubleshooting

**Case: Backup Taking 8 Hours**
- Should take 30 minutes
- **Root cause:** RWND bottleneck (buffer stalls)
- **Evidence:** 19ms delays, thousands of times
- **Fix:** Increase receive buffer size

**Case: 21-Second File Access Delay**
- User complains of slow file server
- **Root cause:** MSS mismatch (1460 vs 1432)
- **Evidence:** Retransmission timeouts (3s + 6s + 12s)
- **Fix:** MTU/MSS configuration on router

::tip
Deze zijn **echte cases** die je in latere hoofdstukken volledig gaat analyseren met Wireshark!
::

## ⚡ Quick TCP Facts

**Protocol Properties:**
- **Layer:** Transport (Layer 4)
- **Reliability:** Guaranteed delivery
- **Ordering:** Packets arrive in-order
- **Flow Control:** Yes (windowing)
- **Congestion Control:** Yes (CWND)
- **Header Size:** 20-60 bytes

**Comparison with UDP:**

| Feature | TCP | UDP |
|---------|-----|-----|
| **Reliability** | ✅ Guaranteed | ❌ Best effort |
| **Ordering** | ✅ In-order | ❌ May reorder |
| **Connection** | ✅ Connection-oriented | ❌ Connectionless |
| **Overhead** | Higher (headers, ACKs) | Lower (minimal) |
| **Speed** | Slower (reliability cost) | Faster (no guarantees) |
| **Use Cases** | HTTP, FTP, SSH | DNS, VoIP, Gaming |
