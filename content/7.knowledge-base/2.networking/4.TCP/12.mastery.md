## ✅ Level 1: Beginner Skills

### 1.1 TCP Header Fundamentals

- [ ] Ik kan de TCP header velden benoemen
- [ ] Ik weet wat Source Port en Destination Port betekenen
- [ ] Ik begrijp wat Sequence Numbers zijn
- [ ] Ik begrijp wat Acknowledgement Numbers zijn
- [ ] Ik kan TCP flags herkennen (SYN, ACK, PSH, FIN, RST)
- [ ] Ik weet wat Window Size betekent
- [ ] Ik kan Wireshark openen en een pcap laden

**Practice:**
```
Open een pcap in Wireshark
Find een TCP packet
Expand TCP header
Identify alle fields
```

**Target tijd:** 2 minuten

### 1.2 Handshake Basics

- [ ] Ik weet dat handshake bestaat uit 3 packets
- [ ] Ik kan SYN, SYN/ACK, ACK herkennen
- [ ] Ik weet dat MSS in handshake wordt negotiate
- [ ] Ik weet wat Window Scaling doet
- [ ] Ik kan de handshake vinden met filter `tcp.flags.syn==1`

**Practice:**
```
Filter: tcp.flags.syn == 1
Find first 3 packets
Identify SYN → SYN/ACK → ACK
```

**Target tijd:** 1 minuut

### 1.3 Basic Filters

- [ ] Ik kan filteren op TCP: `tcp`
- [ ] Ik kan filteren op port: `tcp.port == 443`
- [ ] Ik kan filteren op IP: `ip.addr == X.X.X.X`
- [ ] Ik kan filters combineren met `&&` en `||`
- [ ] Ik kan filters verwijderen (clear button)

**Practice:**
```
Filter alle HTTPS traffic: tcp.port == 443
Filter specific IP:         ip.addr == 192.168.1.1
Combine beiden
```

**Target tijd:** 30 seconden per filter

### 1.4 Basic Packet Navigation

- [ ] Ik kan door packets navigeren (pijltjestoetsen)
- [ ] Ik kan packet details uitklappen
- [ ] Ik kan een stream volgen (Follow TCP Stream)
- [ ] Ik kan terug naar packet list na Follow Stream

**Practice:**
```
Rechtsklik packet → Follow → TCP Stream
Bekijk conversation
Sluit stream view
```

**Target tijd:** 30 seconden

## ✅ Level 2: Intermediate Skills

### 2.1 Handshake Mastery

- [ ] Ik kan MSS values lezen in SYN/SYN-ACK
- [ ] Ik kan Window Scaling factor identificeren
- [ ] Ik kan calculated window size berekenen (value × 2^scale)
- [ ] Ik kan SACK Permitted detecteren
- [ ] Ik kan iRTT aflezen (initial RTT)
- [ ] Ik kan IP TTL interpreteren (hops berekenen)
- [ ] Ik kan handshake issues herkennen (missing options)

**Practice:**
```
Analyze handshake completley binnen 60 seconden:
☑ MSS client + server
☑ Window scaling
☑ SACK enabled?
☑ iRTT value
☑ TTL → hop count
```

**Target tijd:** 60 seconden

### 2.2 Sequence/ACK Tracking

- [ ] Ik begrijp: SEQ + Length = Next SEQ
- [ ] Ik kan Next Sequence Number berekenen (mental math)
- [ ] Ik kan ACK numbers verifiëren (klopt dit?)
- [ ] Ik kan gaps detecteren (missing data)
- [ ] Ik begrijp relative vs raw sequence numbers
- [ ] Ik kan sequence errors herkennen in Expert Info

**Practice:**
```
Take 3 consecutive data packets:
1. Calculate next SEQ manually
2. Verify met Wireshark "Next Sequence Number"
3. Check of ACK klopt
```

**Target tijd:** 2 minuten voor 3 packets

### 2.3 Retransmission Detection

- [ ] Ik kan retransmissions herkennen (rood in packet list)
- [ ] Ik kan filteren: `tcp.analysis.retransmission`
- [ ] Ik kan duplicate ACKs vinden
- [ ] Ik kan fast retransmit vs RTO timeout onderscheiden
- [ ] Ik kan retransmission rate schatten

**Practice:**
```
Filter: tcp.analysis.retransmission
Count packets
Calculate: retransmits / total packets
Classify: fast vs timeout (check delta time)
```

**Target tijd:** 3 minuten

### 2.4 Essential Columns

- [ ] Ik kan kolommen toevoegen (Apply as Column)
- [ ] Ik heb deze kolommen configured:
  - [ ] `tcp.len` (TCP Segment Length)
  - [ ] `tcp.seq` (Sequence Number)
  - [ ] `tcp.ack` (Acknowledgement Number)
  - [ ] `tcp.window_size` (Window Size)
  - [ ] `tcp.time_delta` (Time Delta)
- [ ] Ik kan kolommen sorteren (click header)

**Practice:**
```
Configure alle 5 kolommen
Sort op tcp.len (grootste packets eerst)
Sort op tcp.time_delta (langste delays eerst)
```

**Target tijd:** 2 minuten

## ✅ Level 3: Advanced Skills

### 3.1 RWND vs CWND

- [ ] Ik begrijp verschil tussen RWND en CWND
- [ ] Ik kan RWND zien in packet headers
- [ ] Ik kan CWND deduceren uit burst patterns
- [ ] Ik weet: Effective Window = min(RWND, CWND)
- [ ] Ik kan RWND bottleneck herkennen
- [ ] Ik kan CWND bottleneck herkennen
- [ ] Ik begrijp slow start vs congestion avoidance

**Practice:**
```
Open Time-Sequence Graph
Identify:
  - RWND (green line)
  - CWND (burst pattern)
  - Welke is bottleneck?
```

**Target tijd:** 5 minuten

### 3.2 Delay Classification

- [ ] Ik kan RTT meten (handshake iRTT)
- [ ] Ik kan application delay herkennen
- [ ] Ik kan network delay herkennen
- [ ] Ik kan TCP delay (retransmission) herkennen
- [ ] Ik kan Time Reference gebruiken
- [ ] Ik kan delta time interpreteren
- [ ] Ik weet: ACK komt snel = server heeft data, response laat = app delay

**Practice:**
```
Given: Client request at 0.000, Server ACK at 0.130, Response at 0.157

Calculate:
  - Network RTT: 130ms
  - App delay: 27ms
```

**Target tijd:** 1 minuut

### 3.3 Time-Sequence Graph

- [ ] Ik kan Time-Sequence Graph openen
- [ ] Ik kan steep line interpreteren (good throughput)
- [ ] Ik kan flat sections interpreteren (stalls)
- [ ] Ik kan staircase pattern herkennen (RWND bottleneck)
- [ ] Ik kan overlapping lines herkennen (retransmissions)
- [ ] Ik kan RWND line analyseren (green line)

**Practice:**
```
Open graph voor stream met performance issue
Identify binnen 2 minuten:
  - RWND touching data? (flow control)
  - Retransmissions? (loss)
  - Long stalls? (application)
```

**Target tijd:** 2 minuten

### 3.4 Case Study Analysis

## 📚 Appendix: Quick Reference

### Essential Filters

```wireshark
# Handshake
tcp.flags.syn == 1

# Retransmissions
tcp.analysis.retransmission
tcp.analysis.fast_retransmission

# Windows
tcp.window_size < 5000
tcp.analysis.zero_window

# Delays
tcp.time_delta > 0.02

# SACK
tcp.options.sack

# Specific stream
tcp.stream == 0
```

### Essential Kolommen

```
tcp.len
tcp.seq
tcp.ack
tcp.window_size
tcp.time_delta
tcp.analysis.ack_rtt
```

### Essential Graphs

```
Statistics → TCP Stream Graph → Time-Sequence (tcptrace)
Statistics → TCP Stream Graph → Throughput
Statistics → I/O Graphs
```

### Essential Formulas

```
Next SEQ = SEQ + Length
BDP = Bandwidth × RTT
Window = window_size_value × 2^scale_factor
Throughput ≤ Window / RTT
```
