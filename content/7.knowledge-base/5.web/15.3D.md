## **🎬 3D World System - The Sequel** 🌍🎥

If our 2D UI was a movie set, then the 3D world is the **blockbuster sequel**—bigger, more immersive, and packed with dynamic performances. At the heart of this experience is the `World.js` file, which acts like a **Scene Director**, managing complex interactions between 3D environments, animated characters, and thematic rooms.

### **🌟 Behind the Scenes - How the 3D World Boots Up**

Once the stage (i.e., resources) is set by `Experience.js`, the `World.js` director springs into action. Here’s how it plays out:

1. **Resources First**: The entire cast waits until the set is loaded (`resources.on('loaded')`).
2. **Scene Setup Begins**:

   * **Environment effects** are initialized first.
   * **Landing room** and **desktop props** are placed.
   * The **main character** is fully assembled:

     * `Body.js` places the model
     * `Animations.js` wires up the motion system
     * `Face.js` and `Intervals.js` add facial micro-animations
   * **Lab** is built, tube and bubbles included.
   * Finally, the **Contact scene** is staged and animated.
3. **Update Loop**: Every frame, real-time updates are pushed, including animations, particle effects, and ambient behavior.

### **🎭 Static vs Reactive Roles**

Not every script in the 3D world is meant to move, animate, or react. Some are like props or background lighting—important, but passive. Here's a breakdown:

| **Type**         | **Files / Classes**                                                                      | **Notes**                                                                  |
| ---------------- | ---------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| 🪵 **Static**    | `Environment.js`, `Desktops.js`, `Tube.js`, `Lab.js`, `ContactScene.js`, `Background.js` | Don't respond to user input or time, but may use shaders or lighting.      |
| ⚡ **Reactive**   | `World.js`, `Animations.js`, `Intervals.js`, `ContactAnimation.js`, `Room.js`, `Face.js` | Tied to animation frames, UI visibility, time, or idle-based interactions. |
| 🧍 **Placement** | `Body.js`                                                                                | Places and orients the 3D model in scene. Doesn't animate on its own.      |

This helps separate the **"actors" on stage** from the **"set dressing"**—making the system easier to reason about when debugging or planning new features.

### ✅ Character System Lifecycle Summary

| Step            | File            | Role                                                                 |
| --------------- | --------------- | -------------------------------------------------------------------- |
| Load model      | `Body.js`       | Adds `characterModel` to the scene, sets base position & orientation |
| Setup animation | `Animations.js` | Loads all animation clips into `AnimationMixer`, defines transitions |
| Intro behavior  | `Animations.js` | Triggers `fall-down → wave → idle` sequence using GSAP               |
| Expressions     | `Intervals.js`  | Loops idle facial/body gestures over time                            |
