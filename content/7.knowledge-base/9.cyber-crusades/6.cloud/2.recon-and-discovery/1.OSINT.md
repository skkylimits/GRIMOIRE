### Name & Federation

Get if Azure tenant is in use, tenant name and Federation

```bash
https://login.microsoftonline.com/getuserrealm.srf?login=[USERNAME@DOMAIN]&xml=1
```

Output(In firefox. Open developer tools to get it formatted properly):

```xml
<RealmInfo Success="true">
	<State>4</State>
	<UserState>1</UserState>
	<Login>info@defcorphq.onmicrosoft.com</Login>
	<NameSpaceType>Managed</NameSpaceType>
	<DomainName>defcorphq.onmicrosoft.com</DomainName>
	<IsFederatedNS>false</IsFederatedNS>
	<FederationBrandName>Defense Corporation</FederationBrandName>
	<CloudInstanceName>microsoftonline.com</CloudInstanceName>
	<CloudInstanceIssuerUri>urn:federation:MicrosoftOnline</CloudInstanceIssuerUri>
</RealmInfo>
```

#### Understanding the request

- **Success="true"**: Indicates that the request was successful.
- **State**: Represents the authentication state of the domain. Here are the possible values:
    - **1**: **Unknown** – The state of the domain or user is unknown.
    - **2**: **Managed Domain** – The domain is a standard Azure AD-managed domain. Users authenticate directly with Azure AD.
    - **3**: **Federated Domain** – The domain is federated with an external identity provider, such as ADFS or another SAML-based service.
    - **4**: **Federated Domain with Password Hash Synchronization** – The domain is federated, but password hashes are also synchronized to Azure AD, allowing Azure AD to authenticate users even if the federation provider is unavailable.
    - **5**: **Azure AD B2B Collaboration** – The domain uses Azure AD B2B collaboration for external users, granting access to users from other organizations.
    - **6**: **Azure AD B2C** – The domain is an Azure AD B2C tenant, used for managing external customer identities.
- **UserState**: Represents the state of the user. Here are the possible values:
    - **1**: **Valid user** – The user is valid and can be authenticated.
    - **2**: **Invalid user** – The user is not valid and cannot be authenticated.
    - **3**: **Locked out** – The user is locked out, often due to multiple failed login attempts.
    - **4**: **Disabled** – The user's account has been disabled by an administrator.
    - **5**: **Pending** – The user's account is in a pending state, likely due to a pending action such as activation or verification.
    - **6**: **Password expired** – The user's password has expired and needs to be updated.
    - **7**: **Password reset** – The user has recently reset their password and may need additional verification.
    - **8**: **Password invalid** – The user's password is invalid or incorrect, often after failed login attempts.
- **Login**: The email address associated with the account (e.g., `info@defcorphq.onmicrosoft.com`).
- **NameSpaceType**: Indicates the type of domain configuration:
    - **Managed**: The domain is fully managed by Azure AD. All users authenticate directly with Azure AD.
    - **Federated**: The domain is federated with an external identity provider, such as Active Directory Federation Services (ADFS) or another SAML-based service.
    - **External** (rare): Refers to a third-party identity system that is not directly federated but is still external to Azure AD. This type is less commonly seen in typical enterprise environments.
- **DomainName**: The domain name of the tenant (e.g., `defcorphq.onmicrosoft.com`).
- **IsFederatedNS**: Indicates whether the domain is federated with another identity provider. `false` means the domain is not federated.
- **FederationBrandName**: The name of the organization if the domain is federated. This is `Defense Corporation` in the case of a federated domain, but it is not used if the domain is not federated.
- **CloudInstanceName**: Specifies the cloud service instance being used (e.g., `microsoftonline.com`).
- **CloudInstanceIssuerUri**: URI representing the federation provider, used to authenticate users (e.g., `urn:federation:MicrosoftOnline`).

::note
The domain is marked as **"Federated"** (State `4`), which typically indicates a **Federated Domain** with **Password Hash Synchronization (PHS)** enabled. This hybrid configuration allows **users to authenticate** with **Azure AD** even if the **federation provider** (like **ADFS**) is unavailable.

However, the **NameSpaceType** is **Managed**, meaning that the domain is primarily managed by **Azure AD** for its authentication and identity services. The combination of these settings—**State `4`** and **NameSpaceType `Managed`**—shows a hybrid setup where **Password Hash Synchronization** acts as a fallback to ensure continuous access, even in case of federation downtime.
::

## O365Creeper

::note
In a real worl scenario, you will have a list of emails. **O365Creeper** is used to check if an email ID belongs to an Azure AD tenant by querying the GetCredentialType API.

- Tools like **Hunter.io** and **theHarvester** would be used to gather email lists before validation with O365Creeper.
::

**O365Creeper** (https://github.com/LMGsec/o365creeper) is used to check if an email ID belongs to an Azure AD tenant by querying the GetCredentialType API.

Make sure you have python running on your system. If not click [here](/syntax/python) to follow instructions.

The script is writtin for python2. I converted it to python3.

::warning
Microsoft has implemented some rate-limiting on this service. Don't request to many emails at once.
::

```python
#!/usr/bin/python3

# Created by Mikey Rojas
# https://mikeyrojas.com

# 2 februari, 2025

# This tool will query the Microsoft Office 365 web server to determine
# if an email account is valid or not. It does not need a password and
# should not show up in the logs of a client's O365 tenant.

# Note: Microsoft has implemented some throttling on this service, so
# quick, repeated attempts to validate the same username over and over
# may produce false positives. This tool is best ran after you've gathered
# as many email addresses as possible through OSINT in a list with the
# -f argument.

# This has been forked from the original script by @Korey McKinley. It was writting in python2.
# I've updated it to python3 and added the ability to output valid email addresses to a file.

import requests as req
import argparse
import re
import time

parser = argparse.ArgumentParser(description='Enumerates valid email addresses from Office 365 without submitting login attempts.')
parser.add_argument('-e', '--email', help='Single email address to validate.')
parser.add_argument('-f', '--file', help='List of email addresses to validate, one per line.')
parser.add_argument('-o', '--output', help='Output valid email addresses to the specified file.')
args = parser.parse_args()

url = 'https://login.microsoftonline.com/common/GetCredentialType'

def main():
    if args.file is not None:
        with open(args.file, encoding="utf-8") as file:
            for line in file:
                s = req.session()
                email = line.strip()
                body = '{"Username":"%s"}' % email
                request = req.post(url, data=body)
                response = request.text
                valid = re.search('"IfExistsResult":0', response)
                invalid = re.search('"IfExistsResult":1', response)

                if invalid:
                    print(f"{email} - INVALID")
                if valid and args.output is not None:
                    print(f"{email} - VALID")
                    with open(args.output, 'a+', encoding="utf-8") as output_file:
                        output_file.write(email + '\n')
                else:
                    if valid:
                        print(f"{email} - VALID")

    elif args.email is not None:
        email = args.email
        body = '{"Username":"%s"}' % email
        request = req.post(url, data=body)
        response = request.text
        valid = re.search('"IfExistsResult":0', response)
        invalid = re.search('"IfExistsResult":1', response)

        if invalid:
            print(f"{email} - INVALID")
        if valid and args.output is not None:
            print(f"{email} - VALID")
            with open(args.output, 'w', encoding="utf-8") as output_file:
                output_file.write(email + '\n')
        else:
            if valid:
                print(f"{email} - VALID")

if __name__ == "__main__":
    main()
```

## Azure Security Assessments

- Request two accounts: **a normal user** and **a reader role on the root management group**.
- Without proper visibility, security recommendations will be incomplete. You cannot protect the resources or reccomend that you can not even see.
- If the client grants **Global Reader**, that is ideal.

::warning
- Identifying application owners can take **months** in large organizations.
- Understanding **tenant structure** and **service usage** is key to improving security.
- Always validate findings and ensure reported services belong to the client.
::
