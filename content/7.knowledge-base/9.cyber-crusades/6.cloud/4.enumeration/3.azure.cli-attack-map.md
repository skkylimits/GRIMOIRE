## **1. What resources exist?**
**Why?**
- Identify **users, groups, and devices** in Entra ID (Azure AD) to understand the identity landscape.
- Discover **app registrations and service principals** that may have privileged access.

**Commands:**
```bash
az ad user list --output table            # List all users in Entra ID
az ad group list --output table           # List all groups in Entra ID
az ad group member list --group <GroupID> # List members of a specific group
az ad user show --id <UserEmail>          # Show details of a specific user
az ad device list --output table          # List all devices registered in Entra ID
az ad app list --output table             # List all app registrations
az ad sp list --output table              # List all enterprise applications (service principals)
```

## **2. Who has access?**
**Why?**
- Identify **privileged users**, service accounts, and **sensitive groups** (e.g., **Global Admin, User Admin**).
- Enumerate **role assignments** to determine who has control over resources.

**Commands:**
```bash
az role assignment list --output table           # List all role assignments
az role definition list --output table           # List role definitions (privileges assigned to roles)
az ad group member list --group "Global Admins" # List members of the Global Admin group
az ad user list --query "[?userType=='Member']" --output table  # Find users assigned to roles
```

## **3. What apps have permissions?**
**Why?**
- Identify **applications and service principals** with access to sensitive resources.
- Look for **excessive permissions** on apps that could lead to data breaches.

**Commands:**
```bash
az ad sp list --output table                         # List all service principals (enterprise apps)
az ad sp show --id <AppID>                          # Show details of a specific service principal
az ad app list --query "[?passwordCredentials!=null]" --output table  # List apps with password credentials
az ad app list --query "[?keyCredentials!=null]" --output table       # List apps using certificate authentication
```

## **4. Where can we pivot next? (VMs, Apps, Storage, Secrets)**
**Why?**
- Look for **lateral movement** opportunities via **VMs, Azure Storage, Key Vaults, and App Services**.
- Find **misconfigurations** that grant unintended access to sensitive resources.

**Commands:**
```bash
az vm list --output table                  # Enumerate all virtual machines (VMs)
az storage account list --output table     # List all storage accounts
az storage container list --account-name <StorageAccount>  # List containers in a storage account
az keyvault list --output table            # List all Azure Key Vaults
az keyvault show --name <VaultName>        # Show details of a specific Key Vault
az webapp list --output table              # List all Azure App Services
```
