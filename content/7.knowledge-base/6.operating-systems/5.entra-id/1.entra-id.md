## **Who uses Microsoft Entra ID?**


- **IT admins** – Control & secure apps/accounts, and link systems like Windows Server and M365.
- **App developers** – Add login features (SSO) and build apps with secure company data.
- **Microsoft 365, Azure, and Dynamics users** – Automatically use Entra ID — it’s built in!

## Microsoft Entra Licenses

- **P1** – Basic identity features like SSO, MFA, and password reset.
- **P2** – Advanced tools for risk detection and identity protection.
- **Entra Suite** – P1, P2, and additional Entra tools for full management.

## Make a chapter for ACL/RBAC/ABAC/PBAC

### ACL

In Microsoft Entra, you might see ACL-like behavior via group memberships or application-specific permissions, but ACLs are generally managed within the application or resource, not directly in Entra.

### RBAC

::caution
Best practice is to grant users the least privilege needed. For example, assign the user administrator role to someone who manages users—not global administrator—to limit potential damage from compromised accounts.
::

Managing access using roles is known as role-based access control (RBAC)

#### Built-in Roles

Microsoft Entra ID offers built-in roles—predefined sets of permissions for common tasks. Examples include:

* **Global administrator:** Full access to all administrative features.
* **User administrator:** Manages users, groups, support tickets, and service health.
* **Billing administrator:** Handles purchases, subscriptions, and support.

These roles have fixed permissions that cannot be changed.

#### Custom Roles

**Custom roles** in Microsoft Entra offer flexibility beyond built-in roles by letting you select specific permissions from a preset list.

Creating and using custom roles involves two steps:

1. **Define** a custom role by choosing permissions.
2. **Assign** the role to users or groups at a specific scope—either organization-wide or limited to certain resources (like a single app).

::caution
Custom roles require Microsoft Entra ID P1 or P2 licenses.
::

## Categories

Microsoft Entra roles manage access across Microsoft 365 services and fall into three categories:

- **Microsoft Entra-specific roles:** Manage resources within Microsoft Entra only (e.g., User Administrator).
- **Service-specific roles:** Manage features in specific Microsoft 365 services like Exchange, Intune, SharePoint, and Teams.
- **Cross-service roles:** Grant permissions across multiple services, such as Security Administrator and Compliance Administrator.

### Microsoft Entra RBAC Vs. Azure RBAC

- **Microsoft Entra RBAC:**

  * Controls access to Microsoft Entra resources (users, groups, applications)
  * Manages roles and permissions specific to identity management

- **Azure RBAC:**

  * Controls access to Azure resources (virtual machines, storage, etc.)
  * Manages roles and permissions related to Azure infrastructure

### ABAC

Conditional Access policies using user/device/environment attributes

### PBAC

Conditional Access + entitlement management + access reviews (policy-driven access control)
