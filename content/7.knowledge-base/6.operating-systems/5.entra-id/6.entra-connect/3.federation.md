Implement Federation
::

![federated-identity](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/3.federation/federated-identity.png)

**Federation** means setting up trust between different systems so they can share sign-ins and permissions.

Microsoft Entra Connect helps set this up. It lets users sign in to cloud apps using their regular work password—and if they’re on the company network, they won’t need to type it in again.

::important
With Microsoft Entra ID, you can connect your on-premises setup (like AD FS or PingFederate) so that all logins happen on your own servers. This gives you full control over security and access.
::

::tip
You can enable Password Hash Sync as a backup if your AD FS setup fails.
::

## Security

* **All authentication stays on-premises**—nothing is validated in the cloud.
* Useful if regulations or policies require **no credentials or hashes in the cloud**.

## Customization & Control

* Lets you implement **custom authentication flows**, like:
	* Third-party MFA
	* Smart card auth
	* Conditional access based on network location or device

## SSO across complex environments

* Best for **multi-domain, multi-forest**, or **B2B** scenarios with heavy customization.
* Enables **true SSO**—users on corporate networks don’t re-enter credentials at all.

## Failover

* You can use **PHS as a backup** to Federation. If AD FS goes down, users can still authenticate via Entra ID.

## Downsides

* **More complex to deploy and maintain**
* **Requires on-prem infrastructure** (like AD FS servers)
* **Higher TCO** (cost, time, and effort)
