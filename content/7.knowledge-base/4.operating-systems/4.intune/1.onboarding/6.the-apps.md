## 📱 Understanding App Personas: Matching Apps to Users and Devices

Een van de eerste stappen in effectief applicatiebeheer is het begrijpen van **app personas**. Dit betekent weten wie wat nodig heeft om hun werk te doen en ervoor zorgen dat de juiste apps de juiste devices bereiken.

::warning
**Common Issue:** Wat ik vaak zie is dat de meeste organisaties dit **niet in kaart hebben gebracht**.

Maar als je Part 3 hebt gelezen, ben je goed op weg naar een sterke design architecture en dit is een groot deel daarvan.
::

## 🎯 Core Apps vs. Role-Specific Apps

### 🔹 Core Apps

De **must-haves** die iedereen over de hele linie moet hebben:

**Voorbeelden:**
- Office Suite
- VPN clients
- Endpoint security software

::note
**Strategie:** Deployen naar elk device, ongeacht de use-case, houdt de baseline functionality **uniform**.
::

### 🔹 Role-Based Apps

Verschillende job roles vereisen vaak verschillende sets tools:

**Voorbeelden:**
- **Design team** → Adobe Creative Suite
- **Finance** → Gespecialiseerde accounting software

::tip
**Mapping:** Het in kaart brengen van deze roles helpt je meer **targeted app deployment policies** te creëren.
::

### 🔹 Specialized Apps

Niche tools die alleen een select group nodig heeft:

**Voorbeelden:**
- Development software
- Industry-specific applicaties

→ Kunnen worden uitgerold **on an as-needed basis**

## ✅ Required vs. Available Apps: When to Push and When to Let Users Choose

### 🔒 Required Apps: The Essentials

Sommige apps zijn gewoon **non-negotiable**. Deze worden direct naar users' devices gepusht om ervoor te zorgen dat ze hebben wat ze nodig hebben, wanneer ze het nodig hebben, zonder interventie.

#### 🛡️ Security Software

Alles gerelateerd aan device security of compliance moet worden deployed als **required**.

::important
Je wilt niet dat je gebruikers **opt out** van endpoint protection!
::

**Common Apps:**
- CrowdStrike
- CarbonBlack
- BeyondTrust
- SentinelOne
- TeamViewer

#### 💼 Productivity Tools

Core productivity apps zoals de Microsoft Office Suite, die essentieel zijn voor daily tasks.

**Common Apps:**
- Microsoft 365 Apps (Office)
- G Suite (Google Docs)
- Snagit
- Zoom
- Slack
- Webex
- Google Chrome

**✅ The Benefit:** Je verzekert dat **critical applications** altijd installed en up to date zijn.

### 🏪 Available Apps: Empowering User Choice

Available apps verschijnen in de **Company Portal** (Intune's self-service app store), waardoor gebruikers ze **as needed** kunnen installeren.

Dit is perfect voor apps die **useful maar niet mission-critical** zijn.

#### 🔧 Niche Productivity Tools

Apps die alleen een portion van het bedrijf nodig heeft:

**Common Apps:**
- Visio
- Project
- Visual Studio
- Adobe Acrobat

#### ⚙️ Optional Tools

Apps die productivity verbeteren maar niet door iedereen zijn vereist:

**Common Apps:**
- Notepad++
- VLC
- 7Zip

**✅ The Benefit:** Gebruikers hebben de **flexibility** om te installeren wat ze nodig hebben zonder dat IT hun devices overload.

## 🚀 Different Methods to Deploy Apps Through Intune

We hebben app deployment behoorlijk behandeld op het channel, dus je weet dat er **vele manieren** zijn om dit te doen.

::tip
**Design Consideration:** Als onderdeel van design zou je op zijn minst moeten beginnen met nadenken over **welke apps sense maken** om te deployen met welke methode.
::

### 1️⃣ Win32 App Deployment

| Aspect | Details |
|--------|---------|
| **When to Use** | Ideaal voor complexe apps of apps die custom installation scripts vereisen, zoals `.exe` installers of apps met unique configuration requirements |
| **✅ Pros** | Volledige controle over installation parameters, ondersteunt detection rules, biedt comprehensive installation monitoring |
| **❌ Cons** | Meer effort upfront om te packagen en maintainen |

### 2️⃣ Microsoft Store for Business (Legacy)

| Aspect | Details |
|--------|---------|
| **When to Use** | ❌ **Don't** |
| **✅ Pros** | None |
| **❌ Cons** | Not applicable |

::warning
**Deprecated:** Microsoft Store for Business is legacy en moet worden vermeden.
::

### 3️⃣ Microsoft Store App (New)

| Aspect | Details |
|--------|---------|
| **When to Use** | Voor free-to-use en community apps die je wilt deployen met minimal effort |
| **✅ Pros** | Vereisen geen packaging omdat de apps worden gesourced van Microsoft Store of Winget |
| **❌ Cons** | Voor packages van Winget moet je **due diligence** doen om de integrity van de app te verifiëren (community tool met minimal security measures) |

### 4️⃣ Line-of-Business (LOB) Apps

| Aspect | Details |
|--------|---------|
| **When to Use** | Voor custom-developed apps of specific line-of-business applicaties die moeten worden deployed als `.msi` files |
| **✅ Pros** | Simpel voor straightforward `.msi`-based apps |
| **❌ Cons** | Geen controle over requirements of detection. Kan conflicteren bij deployen alongside Win32 apps |

### 5️⃣ Web Apps

| Aspect | Details |
|--------|---------|
| **When to Use** | Voor apps die primair via een browser draaien maar quick access vanaf het device nodig hebben |
| **✅ Pros** | Fast setup en minimal configuration vereist |
| **❌ Cons** | Beperkt tot browser-based functionality; niet ideaal voor offline use |

## 🔄 Patch Management: Keeping Apps Up to Date Without Surprises

Deployen van apps is één ding — ze **up to date houden** is waar het tricky wordt.

**Here's how to handle app patching effectively:**

### 📊 Version Control and Updates

#### 🎯 Plan for Staged Rollouts

Net zoals met Windows updates, overweeg een **phased approach** voor app updates.

::tip
**Best Practice:** Start met een kleine, controlled group voordat je company-wide uitrolt.
::

#### 📈 Monitor for Issues

Gebruik Intune's **built-in monitoring tools** om deployments in de gaten te houden en snel installation failures of version conflicts te identificeren.

#### 💾 Maintain Older Versions

Heb altijd een **rollback plan**.

::important
**Critical:** Het bijhouden van oudere versies van apps on standby kan de dag redden wanneer een update meer een probleem blijkt te zijn dan een oplossing.
::

### ⚠️ Intune Limitation

::warning
**Belangrijk om te weten:** Intune heeft **geen geautomatiseerd patching system** voor 3rd party apps.

Als je een grote catalog van applicaties deployt en beheert, is het zeker de moeite waard om te kijken naar een platform **specifiek ontworpen voor automated app management**.
::

**Aanbeveling:** Check out de video over het **ZeroTouch.AI platform** om te leren over app automation.

### 📢 Communicating Updates

Houd gebruikers **in the loop** door hen te notificeren over komende updates.

**Voordelen:**
- ✅ Dit vermindert de **surprise factor**
- ✅ Laat hen weten dat je **improvements** maakt (niet gewoon random dingen verandert for fun)

## 💡 Final Thoughts: Apps as the Heart of User Productivity

Het beheren van applicaties in Intune betekent het balanceren van:
- **User needs**
- **Security requirements**
- **IT sanity**

### 🎯 Success Formula

Door de volgende stappen te nemen, zet je je organisatie op voor **smoother operations** en minder 3 a.m. calls (als je het type bent dat je telefoon aan houdt):

1. ✅ **Map out app personas**
2. ✅ **Decide tussen required en available apps**
3. ✅ **Choose de juiste deployment method**
4. ✅ **Handle patch management met care**

::tip
**The Good News:** Zolang je door proper design gaat en dingen uitplant, kun je het **easy houden** zodat IT tijd kan besteden aan meer belangrijke issues…

Zoals het beheren van **Windows en driver updates**. Yeah — we'll talk about that next.
::

## 📋 Quick Reference: Deployment Method Selector

| App Type | Recommended Method | Reasoning |
|----------|-------------------|-----------|
| **Complex installers (.exe)** | Win32 | Volledige controle + custom scripts |
| **Simple installers (.msi)** | LOB | Quick deployment voor straightforward apps |
| **Community apps** | Microsoft Store App | Minimal packaging effort |
| **Browser-based tools** | Web Apps | Fast setup, geen install nodig |
| **Legacy** | ❌ Store for Business | Deprecated - vermijden |

## 🏆 App Management Best Practices

| Practice | Impact |
|----------|--------|
| **📊 Map app personas** | Targeted deployment, minder waste |
| **🔒 Required apps voor security** | Compliance guaranteed |
| **🏪 Available apps voor choice** | User empowerment + minder IT overhead |
| **🔄 Staged rollouts** | Risk mitigation bij updates |
| **💾 Rollback planning** | Quick recovery bij issues |
| **📢 Communication** | User satisfaction + awareness |
