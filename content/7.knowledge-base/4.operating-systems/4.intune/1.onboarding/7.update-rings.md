## 🔄 Why Move from WSUS to Windows Autopatch?

Als je al een tijdje in IT zit, ben je waarschijnlijk bekend met **WSUS**. Het is lange tijd de go-to geweest voor Windows updates, met gecentraliseerde controle over wat wordt geïnstalleerd en wanneer.

::note
**Maar** met meer organisaties die overstappen naar remote en cloud-based management, begint WSUS zijn **age te tonen**.
::

**Enter Windows Autopatch**, dat een cloud-friendly, hands-off manier biedt om updates te handelen zonder on-premises infrastructure te vereisen.

### 🎯 Why Windows Autopatch Can Make Your Life Easier

#### 🌐 Flexibility for Hybrid and Remote Devices

**No more relying on on-prem connections.**

Windows Autopatch en Intune beheren updates **wherever there's internet**, wat perfect is voor today's hybrid en remote work setups.

#### 🛠️ Less Maintenance

**No more WSUS servers** om te managen, patchen en troubleshooten.

::tip
**Windows Autopatch** taps into Microsoft's cloud, waardoor die workload van je plate wordt gehaald.
::

#### ⚙️ Control Without Micromanagement

**Update rings** laten je deployments staggeren, waardoor een meer betrouwbaar update-proces ontstaat **zonder elke patch te micromanagen**.

## 🎯 The Power of Update Rings: A Gradual, Controlled Rollout

Met WSUS zou je **manually elke update approven**, en deze in waves uitrollen over de organisatie.

In Intune geven **update rings** je vergelijkbare controle maar met **meer automation en flexibility**.

::important
Je kunt updates structureren in **rings**, devices assignen aan groups voor een **gradual rollout** die disruption minimaliseert en issues **isolated** houdt tot kleinere groups.
::

### 📊 Effective Update Ring Structure

#### 1️⃣ Pilot Ring

**Start met een kleine group test devices.**

Deze zijn je **eerste linie van defensie**, en helpen je issues te catchen voordat ze de bredere organisatie bereiken.

::tip
**Best Practice:** Selecteer IT-savvy gebruikers of test machines voor deze ring.
::

#### 2️⃣ Fast Ring

**Na updates de pilot phase passeren**, move naar een slightly larger group van **early adopters**.

**Doel:** Ze helpen je verzekeren dat updates stabiel zijn op een breder scala aan devices.

#### 3️⃣ Broad Ring

**Finally**, de overige devices krijgen updates.

::note
Met pilot en fast rings die je een **head start** geven op testing, zou de broad ring relatief **smooth sailing** moeten zijn.
::

### 🏆 Benefits van Deze Approach

| Voordeel | Impact |
|----------|--------|
| **🎯 Reduced risk** | Widespread issues worden voorkomen |
| **🔧 Tailored experience** | Updates aangepast aan verschillende user needs |
| **📱 Device flexibility** | Verschillende device configuraties ondersteund |
| **🛡️ Issue isolation** | Problemen blijven beperkt tot kleinere groups |

## ⏰ Configuring Update Rings: Timing, Deferrals, and Active Hours

Een major perk van update rings is dat ze je **update timing laten managen** zonder constant approvals.

Je kunt **deferrals en active hours** instellen om te verzekeren dat updates **op het juiste moment** gebeuren voor je organisatie.

### 📅 Deferral Policies

Set deferrals op:

#### 🔒 Quality Updates (Security Patches)

**Aanbeveling:** Aim for a **short deferral**, zoals een paar dagen of een week.

::important
**Security patches** moeten snel worden uitgerold om de organisatie beschermd te houden.
::

#### 🆕 Feature Updates (Major OS Releases)

**Aanbeveling:** Je kunt een **longer deferral period** instellen, giving time voor testing.

**Voordeel:** Dit geeft je ruimte om compatibility te testen met bestaande apps en workflows.

### 🕐 Active Hours

Set **active hours** om updates te vermijden tijdens high-traffic work hours.

**Benefits:**
- ✅ Minimaliseert disruptions
- ✅ Verzekert dat devices **niet rebooten** in het middle van iets critieks
- ✅ Verbetert user experience

::tip
**Result:** Je onderhoudt een level of control die **familiar aanvoelt** als WSUS maar met minder manual work, waardoor updates **predictable en user-friendly** blijven.
::

## 🚀 Boost Efficiency with Delivery Optimization for On-Site Updates

Voor on-site environments is **Delivery Optimization** in Intune een **game-changer** voor bandwidth management.

::important
**Delivery Optimization** allows devices to download updates from **local peers**, waardoor de load op je network wordt gereduceerd en het proces voor iedereen wordt versneld.
::

### 🔄 How Delivery Optimization Works

In plaats van dat elk device updates **directly van Microsoft** download, enables Delivery Optimization devices om **content met elkaar te sharen**.

**Mechanisme:**
1. Devices op hetzelfde network kunnen update files pullen van **nearby peers**
2. Dit betekent **minder downloads van het internet**
3. **Faster updates** overall

### ⚙️ Setting Up Delivery Optimization in Intune

#### 1️⃣ Configure Download Mode

In Intune, set de Delivery Optimization download mode naar **"Group" of "LAN"**, wat devices op hetzelfde network enabled om update files te sharen.

#### 2️⃣ Monitor Network Performance

Delivery Optimization minimaliseert internet bandwidth usage, maar je kunt het effect monitoren met Intune's **built-in reports**.

::tip
**Check de metrics** om te zien:
- Hoeveel bandwidth is saved
- Of adjustments nodig zijn
::

#### 3️⃣ Adjust Based on Office Size

Als je meerdere office locations hebt, kun je Delivery Optimization settings **per location** adjusten.

| Office Size | Recommendation |
|-------------|----------------|
| **Large offices** | Meer devices = meer benefit van peer sharing |
| **Small offices** | Default settings werken meestal fine |
| **Remote workers** | Minimal peer benefit, maar geen nadelen |

### 🏆 The Result

- ✅ **Faster updates**
- ✅ **Less strain** op je internet connection
- ✅ **Smoother experience** voor gebruikers
- ✅ **All with minimal effort** aan jouw kant

## 🔐 Compliance Policies: Making Sure Devices Are Up-to-Date

Intune's **compliance policies** zijn je best friend voor ensuring devices **stay updated**.

Compliance policies allow je om **requirements** te stellen voor OS en patch levels, zodat je makkelijk kunt zien of devices achterblijven.

### 📊 Monitor Compliance with Reports

Intune's reporting geeft je een **clear view** van:
- ✅ Welke devices **compliant** zijn
- ❌ Welke devices **attention** nodig hebben

**Voordeel:** Het is een efficiënte manier om **stragglers** te catchen.

### 🔒 Conditional Access for Security

Als een device niet voldoet aan update compliance, kan **Conditional Access** het blokkeren van sensitive resources **totdat het up to date is**.

::important
**Win-Win:** Het is een geweldige manier om:
- Data **secure** te houden
- Gebruikers te **motiveren** om compliant te blijven
::

### 🎯 Focus on Overall Health

Het gebruiken van compliance policies laat je focussen op **overall update health** zonder constant elk device **individually** te checken.

## 🧪 Testing and Fine-Tuning the Update Process

Moving away van WSUS betekent **niet** abandoning testing.

::warning
**If anything**, de transitie naar Windows Autopatch maakt het **even more important** om het update-proces te monitoren en fine-tunen.
::

### 📈 Use Intune's Reporting Tools

Monitor **update status en compliance** across je rings om issues **early** te catchen.

**Action:** Als een update faalt in de pilot ring, **hold off** op verder uitrollen.

### 🔄 Adjust Rings Based on Feedback

Pay attention to feedback van gebruikers in de **pilot en fast rings**.

::tip
**Best Practice:** Als issues crop up, maak **adjustments** voordat je procedeert naar de broad ring.
::

## 📋 Quick Reference: Update Ring Strategy

| Ring | Timing | Size | Purpose |
|------|--------|------|---------|
| **🧪 Pilot** | First (0-3 days deferral) | Small (~5-10 devices) | Catch issues early |
| **⚡ Fast** | Second (3-7 days deferral) | Medium (~10-20% fleet) | Validate broader compatibility |
| **🌐 Broad** | Last (7-14 days deferral) | Large (remaining fleet) | Full deployment |
