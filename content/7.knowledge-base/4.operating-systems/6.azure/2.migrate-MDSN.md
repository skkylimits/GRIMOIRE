## 🚀 Introductie

Soms krijg je een **Visual Studio (MSDN)** abonnement via je werk, maar wil je de **Azure Dev/Test credits gebruiken in je eigen tenant** — bijvoorbeeld voor labs, Intune testing of offensive security research.

Deze gids laat zien hoe je een subscription **veilig** en **zonder tenant-adminrechten** migreert naar je eigen tenant.

We volgen hier exact het proces zoals we het in de praktijk tegenkwamen, inclusief obstakels en oplossingen.

## 🎯 Doel

We willen dit bereiken:

```
WORK TENANT (QUBE) → DEV TENANT (sp33ltu1n)
Visual Studio Enterprise – MPN subscription
```

Zodat je je credits kunt gebruiken in jouw eigen omgeving.


## 🧭 Stapsgewijze Uitleg

### 0️⃣Prerequisites

**1. Microsoft 365 Developer Program**

Word lid van het programma
👉 [https://developer.microsoft.com/en-us/microsoft-365/dev-program](https://developer.microsoft.com/en-us/microsoft-365/dev-program)

**2. Developer Tenant activeren**

Activeer een **gratis dev-tenant** voor je aangemaakt.

**2. Visual Studio (MSDN) licentie verkrijgen**

Open een ticket bij je werkgever en vraag om **MSDN-licentierechten**.
Na toekenning:

1. Ga naar 👉 [https://my.visualstudio.com](https://my.visualstudio.com)
2. Zoek naar:

![activate](/content/7.knowledge-base/4.operating-systems/6.azure/2.migrate-MDSN/activate.png)

3. Klik op **Activate**

Vanaf dit moment is je Visual Studio Enterprise subscription actief en klaar om te migreren naar je dev-tenant.


### 1️⃣ Check of jouw MDSN-account bestaat in de dev-tenant

In de **sp33ltu1n** tenant:

Ga naar:

```
Entra → Users → All Users
```

Zoek:

```
[MSDN-account]@qube.nl
```

* Staat hij erin → ga door
* Staat hij er niet → klik **Invite external user**

Dit maakt je MDSN-account bekend in de target tenant → harde voorwaarde voor migratie.

::warning
Alleen tijdelijk.

> Dit is nodig om de role assignment te kunnen uitvoeren.

Geef de guest tijdelijk **Global Administrator**. Anders kan je straks niet een ander account owner maken.
::

### 2️⃣ Ga naar je MDSN-tenant en open de subscription

Login met je MSDN-account:

```
https://portal.azure.com
```

Open:

```
Home → Subscriptions → Visual Studio Enterprise – MPN
```


### 3️⃣ Selecteer je dev-tenant

Menu:

```
Subscription → Overview → Change Directory
```

Je krijgt nu een lijst van tenants waarin jouw MDSN-account bestaat.

**Als jouw dev-tenant NIET zichtbaar is:**

* guest-user bestaat niet
* of de uitnodiging is niet geaccepteerd



### 4️⃣ ️Selecteer je dev-tenant

Kies:

```
sp33ltu1n.onmicrosoft.com
```

Klik **Change**.

Azure verhuist nu de subscription.
Dit duurt:

⏱ 2–10 minuten
(max 30 minuten)

::caution
Zorg ervoor dat je de juiste tenant selecteert.  
  
- Controleer tenant id https://portal.azure.com/#view/Microsoft_AAD_IAM/ActiveDirectoryMenuBlade/~/Overview
::


### 5️⃣ Log in op jouw dev-tenant **met je MDSN-account (guest user)**

Na de migratie is **jouw guest-account** nog steeds de enige **Owner** van de subscription.

Daarom moet je:

1. Naar de dev-tenant gaan


```
https://portal.azure.com/#@sp33ltu1n.onmicrosoft.com
```

2. Inloggen met je **MDSN-account** (guest):

   ```
   [MSDN-account]@qube.nl
   ```

3. Vervolgens klik je op:

```
Subscriptions →  Visual Studio Enterprise – MPN
```


### 6️⃣ Maak **aazsp3** de permanente Owner

Nu je MDSN-account (guest) de subscription *in de dev-tenant* kan openen:

Ga naar:

```
Subscriptions → VS Enterprise MPN → Access Control (IAM)
```

Klik:

```
Add → Add role assignment
```

Selecteer:

```
Owner
```

Selecteer het account:

```
aazsp3@sp33ltu1n.onmicrosoft.com
```

(Je guest-account ziet members, omdat hij tijdelijk GA is)

Bij conditions:

```
Allow user to assign all roles (highly privileged)
```

> ✔ Hiermee wordt **aazsp3** de primaire Owner in jouw eigen tenant.
> ✔ Vanaf dit moment heb jij volledige unrestricted controle.

::caution
Ik heb voor simplicity het aazsp3 account owner gemaakt. Je hebt natuurlijk hierin de vrije hand.
::

### 7️⃣ Log in met **aazsp3** en controleer of alles werkt

Ga opnieuw naar:

```
https://portal.azure.com
```

Open:

```
Subscriptions → VS Enterprise MPN
```

Je zou nu moeten zien:

```
Owner: aazsp3@sp33ltu1n.onmicrosoft.com
Credits: Active (€145)
```


### 8️⃣ Cleanup — verwijder tijdelijke adminrechten

Nu alles werkt en **aazsp3** confirmed Owner is:

#### 1. Verwijder Global Admin van je guest user:

```
Entra → Users → Guest (MDSN account) → Assigned roles → Remove
```

#### 2. (Optioneel) verwijder guest user volledig

```
Entra → Users → Delete
```
