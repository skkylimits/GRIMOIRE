**Kerberos Authentication (Windows)**

Kerberos version 5 is a secure authentication protocol used in Windows Server environments. It verifies user and host identities, primarily within Active Directory (AD) domains.

::note
Before Kerberos, NTLM required servers to contact a domain controller for every authentication. Kerberos replaces this with reusable session tickets, letting servers validate clients directly—only contacting the domain controller when needed for PAC checks.
::

## AuthN Protocol

**Kerberos** is a core **authentication protocol**, not a *sign-in method* you configure in Entra ID like `Password Hash Sync` (PHS), `Passthrough Authentication` (PTA), or Federation. Those are integration methods between Entra ID and on-prem AD.

## Practical Application

* **Integrated with AD:** The Key Distribution Center (KDC) is part of the domain controller and uses AD to manage credentials.
* **Single Sign-On (SSO):** Users sign in once and access multiple resources without re-entering credentials.
* **Delegated Authentication:** Services can act on behalf of users to access resources across systems.
* **Mutual Authentication:** Both client and server verify each other's identities.
* **Efficiency:** Uses renewable session tickets instead of repeated domain controller requests, unlike NTLM.
* **Interoperability:** Based on standard Kerberos v5, supporting cross-platform compatibility.
