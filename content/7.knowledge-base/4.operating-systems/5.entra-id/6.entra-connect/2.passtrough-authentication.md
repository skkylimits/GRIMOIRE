## PTA

Pass-through Authentication **allows users to sign in to both on-premises and cloud apps using the same password by validating credentials directly against the on-premises Active Directory in real-time**.

::tip
Use PTA if you must enforce on-premises policies at sign-in and prefer no passwords in the cloud.
::
![pta1](/content/7.knowledge-base/4.operating-systems/5.entra-id/6.entra-connect/2.passtrough-authentication/pta1.png)

## Setup Requirements

* Install lightweight PTA agents on one or more on-premises servers (recommended: 3 for high availability).
* Agents require network access to domain controllers and outbound internet access.
* Agents must be installed inside the internal network (not in a perimeter/DMZ network).
* Enable PTA via Microsoft Entra Connect.

## High Availability

* **Highly available**: Multiple agents can be installed on multiple on-premises servers for load balancing and failover.

## Maintenance

* **No management overhead**: The agent automatically receives improvements and bug fixes.
* **Easy to deploy & administer**:

  * No complex on-premises deployments or network configuration.
  * Only a lightweight agent needs to be installed on-premises.
* **Free to use**: No extra licensing required.

## User Experience

* **Consistent user experience**: Users sign in with the same password for both on-premises and cloud apps.
* **Reduced helpdesk calls**: Fewer password-related issues.
* **Cloud self-service**: Users can reset passwords in the cloud, syncing back to on-premises AD.
* **Seamless Single Sign-On** (SSO): Smooth sign-in on Entra hybrid joined devices inside corporate networks.

## Features

* **Real-time validation**: Passwords are checked directly against on-premises AD at sign-in.
* **Supports all modern authentication apps**: Works with web and Microsoft Office applications.
* **Multi-forest support**: Works in environments with AD trusts and proper routing.

## Security

* **Enforces on-premises policies:** Account lockout, expiration, sign-in hours, and password policies apply instantly.
* **Protects against brute force attacks:** Malicious password attempts are filtered in the cloud.
* **Secure communication:** Encrypted, certificate-based authentication with automatic renewal.
* **No password hashes stored in the cloud:** Password validation happens only on-premises, minimizing cloud exposure.
* **Better control for compliance:** Meets strict security requirements needing on-premises authentication checks.
* **Supports complex MFA setups:** Works with on-premises or third-party MFA solutions.
* **The agent only makes outbound connections from within your network**: Therefore, there is no requirement to install the agent in a perimeter network, also known as a DMZ.
* **Supports Conditional Access and MFA:** Integrates with Microsoft Entra Conditional Access for extra security.

## Considerations

* Deploy at least three PTA agents for redundancy and high availability.
* PTA requires unrestricted network access to domain controllers.
* The agent only makes outbound connections, so it’s safer and simpler to manage network security.

## Failover

* Use `Password Hash Synchronization`as a **manual backup** in case PTA agents or on-premises AD are unavailable.

::caution
Failover to `PHS` must be switched manually using Microsoft Entra Connect.
::
