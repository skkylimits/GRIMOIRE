## 🚀 Waarom Windows Begrijpen?

Als penetration tester is gedegen kennis van Windows en Linux essentieel. De meeste systemen die je tegenkomt — on-premise of in de cloud — draaien op deze besturingssystemen.

::important
Het begrijpen van **hoe je deze systemen aanvalt én verdedigt** is cruciaal voor effectieve security assessments.
::

## 📜 Windows Geschiedenis

Microsoft introduceerde Windows op **20 november 1985** als een grafische shell voor MS-DOS.

**Belangrijke mijlpalen:**
- **Windows 95** — Eerste volledige DOS/Windows integratie, Internet Explorer geïntroduceerd
- **Windows NT 3.1 Advanced Server (1993)** — Eerste Windows Server release
- **Windows 2000** — Introductie van Active Directory, Microsoft Management Console (MMC)
- **Windows Server 2003** — Server roles, ingebouwde firewall, Volume Shadow Copy Service
- **Windows Server 2008** — Hyper-V, failover clustering, Server Core, grote AD-verbeteringen
- **Windows Server 2019** — Kubernetes support, Linux containers, geavanceerde security features

::caution
Oudere Windows-versies zoals Server 2008 en 2012 hebben **end-of-life** bereikt en ontvangen geen security updates meer (behalve via extended support contracten).
::

## 🖥️ Toegang tot Windows Systemen

### Local Access

**Local access** is de meest voorkomende manier om een computer te gebruiken — via keyboard, muis en scherm. Organisaties bouwen security policies rond dedicated workspaces en bedrijfseigen apparatuur.

### Remote Access

**Remote access** betekent toegang tot een computer over het netwerk. Dit is essentieel voor:
- IT-beheer en sysadmin-taken
- MSPs & MSSPs (Managed Service Providers)
- Software development teams
- Security teams

**Veelgebruikte remote access technologieën:**
- Virtual Private Networks (VPN)
- Secure Shell (SSH)
- File Transfer Protocol (FTP)
- Virtual Network Computing (VNC)
- Windows Remote Management (WinRM)
- **Remote Desktop Protocol (RDP)** ← Focus van deze module

## 🔐 Remote Desktop Protocol (RDP)

RDP gebruikt een **client/server architectuur**:
- Client specificeert het **target IP-adres** of hostname
- Server is de Windows machine waar RDP is geactiveerd
- RDP luistert standaard op **poort 3389**

::note
**Analogie:**
Netwerk subnet = straat in een stad
IP-adres = huisnummer
Logical port = raam/deur om binnen te komen
::

### RDP vanaf Windows

Gebruik de ingebouwde **Remote Desktop Connection** (`mstsc.exe`):

1. Start `mstsc.exe`
2. Voer het target IP-adres in
3. Authenticeer met credentials


![UsingRemoteDesktopConnection](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/1.introductie-tot-windows/UsingRemoteDesktopConnection.gif)

::tip
Je kunt connection profiles opslaan als `.rdp` bestanden — deze zijn interessant tijdens penetration tests!
::

![SavingRDPConnections](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/1.introductie-tot-windows/SavingRDPConnections.gif)

### RDP vanaf Linux

Gebruik **xfreerdp** voor RDP-toegang vanaf Linux:

```bash
xfreerdp /v:<targetIp> /u:htb-student /p:Password
```

**Handige xfreerdp opties:**
- `/cert:ignore` — Certificaat warnings negeren
- `/dynamic-resolution` — Resolutie aanpassen aan venstergrootte
- `/drive:share,/tmp` — Drive redirection voor file transfers

::caution
**Standaard staat RDP niet aan** op Windows systemen. Het moet expliciet geactiveerd worden op de target machine.
::
