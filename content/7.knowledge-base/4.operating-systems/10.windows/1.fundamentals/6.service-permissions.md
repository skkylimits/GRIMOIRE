## 🔐 Service Permissions en Security

### Service Accounts: Het Risico van User Context

Services draaien standaard binnen de context van de huidige ingelogde gebruiker. Dit kan ernstige operationele problemen veroorzaken.

::caution
**Voorbeeld scenario**: Als we zijn ingelogd als Bob op een server tijdens de DHCP-installatie, dan wordt die service geconfigureerd om te draaien als Bob, tenzij anders aangegeven.
::

**Wat kan er misgaan?**
- Bob verlaat de organisatie of wordt ontslagen
- Bob's account wordt uitgeschakeld als onderdeel van het exit-proces
- Alle services die draaien onder Bob's account **falen bij opstarten**

**Impact voorbeeld (DHCP)**:
- DHCP (Dynamic Host Configuration Protocol) is verantwoordelijk voor het leasen van IP-adressen
- Als deze service stopt op een Windows DHCP server, krijgen clients geen IP-adres
- Dit betekent **downtime en productiviteitsverlies**

::important
**Best Practice**: Creëer individuele user accounts om kritieke netwerkservices te draaien. Deze worden **service accounts** genoemd.
::

### Security Risico's van Service Directories

We moeten ook alert zijn op:
- **Service permissions** - De rechten van de service zelf
- **Directory permissions** - De rechten van de directories waar executables draaien

::warning
Het is mogelijk om het pad naar een executable te vervangen met een malicious DLL of executable bestand als de NTFS-rechten zwak geconfigureerd zijn.
::

## 🔍 Services Onderzoeken met services.msc

Zoals besproken in de processes en services sectie, kunnen we **services.msc** gebruiken om bijna elk detail van alle services te bekijken en te beheren.

![wuauserv](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/6.service-permissions/wuauserv.png)

### Windows Update Service Voorbeeld (wuauserv)

**Belangrijke properties**:

**Service Name** (`wuauserv`)
- Zeer nuttig bij het gebruik van command-line tools

**Path to Executable**
- Het volledige pad naar het programma en commando dat uitgevoerd wordt wanneer de service start
- Als de NTFS-rechten zwak zijn, kan een aanvaller de originele executable vervangen

::caution
Als de NTFS permissions van de destination directory zwak zijn geconfigureerd, kan een aanvaller de originele executable vervangen met één die gemaakt is voor malicious doeleinden.
::

### 🎭 Service Account Privileges

**LocalSystem Account**:
- Meeste services draaien standaard met **LocalSystem privileges**
- Dit is het hoogste toegangsniveau op een individueel Windows OS

![local-system](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/6.service-permissions/local-system.png)

::tip
Niet alle applicaties hebben Local System account-level permissions nodig. Voer research uit per applicatie wanneer je nieuwe applicaties overweegt te installeren in een Windows-omgeving.
::

**Principle of Least Privilege**:
- Identificeer applicaties die kunnen draaien met de **minst mogelijke privileges**
- Align met het **principle of least privilege**

### 📋 Notable Built-in Service Accounts

Windows heeft verschillende ingebouwde service accounts:

| **Account** | **Beschrijving** |
|-------------|------------------|
| **LocalService** | Beperkte privileges, netwerk toegang als anonymous |
| **NetworkService** | Beperkte privileges, netwerk toegang als computer account |
| **LocalSystem** | Hoogste privileges, volledige toegang tot systeem |

::note
We kunnen ook nieuwe accounts creëren en deze gebruiken voor het exclusieve doel van het draaien van een service.
::

### 🔄 Recovery Tab: Een Attack Vector

De recovery tab stelt configuratie van stappen in wanneer een service faalt.

::warning
**Attack Vector**: Deze service kan worden ingesteld om een programma te draaien na de eerste failure. Dit is nóg een vector die een aanvaller kan gebruiken om malicious programma's te draaien door een legitieme service te misbruiken.
::

Het **sc** (Service Control) commando kan worden gebruikt om services te configureren en beheren vanuit de command line.

```cmd

        START_TYPE         : 3   DEMAND_START

        LOAD_ORDER_GROUP   :
        TAG                : 0
        DISPLAY_NAME       : Windows Update
        DEPENDENCIES       : rpcss

```

**Remote Query**:
```cmd
C:\Users\htb-student>sc //hostname or ip of box query ServiceName
```

### Service Start/Stop

```cmd
C:\Users\htb-student> sc stop wuauserv

[SC] OpenService FAILED 5:

Access is denied.
```

::note
We worden geweigerd toegang zonder het uit te voeren binnen een administrative context. Als we een command prompt draaien met elevated privileges, wordt de actie toegestaan.
::

::

## 🔐 Security Descriptors met sc sdshow

Een andere handige manier om service permissions te onderzoeken is via het **sdshow** commando.

```cmd
C:\WINDOWS\system32> sc sdshow wuauserv

D:(A;;CCLCSWRPLORC;;;AU)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SY)S:(AU;FA;CCDCLCSWRPWPDTLOSDRCWDWO;;;WD)
```

### 📖 SDDL: Security Descriptor Definition Language

Op het eerste gezicht ziet de output er chaotisch uit, maar er is een betekenis achter deze "madness".

**Belangrijke concepten**:
- Elk **named object** in Windows is een **[securable object](https://docs.microsoft.com/en-us/windows/win32/secauthz/securable-objects)**
- Als het securable is in een Windows OS, heeft het een **[security descriptor](https://docs.microsoft.com/en-us/windows/win32/secauthz/security-descriptors)**
- Security descriptors identificeren:
  - Object's owner
  - Primary group
  - **DACL** (Discretionary Access Control List)
  - **SACL** (System Access Control List)

::note
Een **DACL** wordt gebruikt voor het controleren van toegang tot een object, en een **SACL** wordt gebruikt voor accounting en logging van access attempts.
::

### 🔍 SDDL Breakdown

```text
D:(A;;CCLCSWRPLORC;;;AU)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SY)
```

**Lees in deze volgorde**:

```text
D: (A;;CCLCSWRPLORC;;;AU)
```

| **Component** | **Betekenis** | **Uitleg** |
|---------------|---------------|------------|
| `D:` | DACL permissions | De volgende characters zijn DACL permissions |
| `AU` | Security Principal | Authenticated Users |
| `A;;` | Access Allowed | Toegang is toegestaan |
| `CC` | SERVICE_QUERY_CONFIG | Query naar SCM voor service configuratie |
| `LC` | SERVICE_QUERY_STATUS | Query naar SCM voor huidige status |
| `SW` | SERVICE_ENUMERATE_DEPENDENTS | Enumerate lijst van dependent services |
| `RP` | SERVICE_START | Start de service |
| `LO` | SERVICE_INTERROGATE | Query service voor huidige status |
| `RC` | READ_CONTROL | Query de security descriptor van de service |

### 📝 Access Control Entry Structuur

**Pattern**:
```text
;;CCLCSWRPLORC;;;
```
- Elke set van 2 characters tussen de puntkomma's = acties die mogen worden uitgevoerd

**Security Principal**:
```text
;;;AU
```
- Characters na de laatste set puntkomma's = security principal (User/Group)

**Allowed/Denied**:
```text
A;;
```
- Character direct na opening parentheses = Allowed (A) of Denied (D)

::important
Deze security descriptor voor Windows Update (wuauserv) heeft **drie sets** van access control entries omdat er drie verschillende security principals zijn, elk met specifieke permissions.
::

### 🎯 Voordelen van Get-ACL

**Ten opzichte van `sc` command**:
- ✅ Specifieke account permissions in **easy-to-read format**

- ✅ **SID** van elke security principal (User/Group) is aanwezig
- ✅ Betere scripting mogelijkheden
