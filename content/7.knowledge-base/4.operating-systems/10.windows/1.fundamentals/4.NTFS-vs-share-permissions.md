## 🎯 Waarom Windows het Primaire Malware Doelwit Is

Microsoft bezit meer dan **70% van de wereldwijde markt** voor desktop besturingssystemen. Dit verklaart waarom de meeste malware-auteurs kiezen om malware voor Windows te schrijven en waarom velen Windows als minder veilig beschouwen dan andere besturingssystemen.

::note
**Vanuit een business perspectief** is het logisch voor malware-auteurs om resources te besteden aan Windows malware — het is een **high-value target**.
::

::important
Het idee dat een OS immuun is voor malware is een **technische denkfout**. Als software geschreven kan worden voor een besturingssysteem, dan kan een virus ook geschreven worden voor dat besturingssysteem. Een virus is per definitie software met **malicious intent** en kan voor elk OS geschreven worden.
::

Veel malware-varianten voor Windows kunnen zich verspreiden over het netwerk via **network shares met permissive permissions**. De beruchte **EternalBlue vulnerability** blijft tot op de dag van vandaag unpatched Windows systemen met SMBv1 teisteren en biedt vaak toegang voor ransomware om organisaties plat te leggen.

## 🔗 Server Message Block (SMB) Protocol

Het **Server Message Block protocol (SMB)** wordt in Windows gebruikt om shared resources zoals files en printers te verbinden. Het wordt gebruikt in grote, middelgrote en kleine enterprise-omgevingen.

![smb_diagram](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/4.NTFS-vs-share-permissions/smb_diagram.png)

::note
**SMB Werking:**
- **Client** stuurt een SMB request naar de **server**
- **Server** antwoordt en geeft toegang tot file systems en printers
- De server toont een file directory aan de client
::

::tip
**Visuele concepten:** Neem altijd de tijd om diagrammen en visualisaties grondig te begrijpen. Een afbeelding kan duizend woorden waard zijn, maar wordt vaak verleidelijk om over te slaan.
::

## 🔐 NTFS vs. Share Permissions

Gelden op het lokale systeem waar files worden gehost.

NTFS permissions en share permissions worden vaak als hetzelfde gezien. **Dit is niet waar**, maar ze worden vaak op dezelfde gedeelde resource toegepast. Laten we kijken naar de individuele permissions die kunnen worden ingesteld om toegang te beveiligen/verlenen tot een network share gehost op een Windows systeem met NTFS.

### Share Permissions

Gelden bij netwerk toegang via SMB.

| Permission | Beschrijving |
|-----------|--------------|
| **Full Control** | Users mogen alle acties uitvoeren van Change en Read permissions, en permissions wijzigen voor NTFS files en subfolders |
| **Change** | Users mogen files en subfolders lezen, bewerken, verwijderen en toevoegen |
| **Read** | Users mogen file en subfolder contents bekijken |

### NTFS Basic Permissions

| Permission | Beschrijving |
|-----------|--------------|
| **Full Control** | Users mogen files en folders toevoegen, bewerken, verplaatsen, verwijderen en NTFS permissions wijzigen voor alle toegestane folders |
| **Modify** | Users krijgen permission of denial om files en folders te bekijken en wijzigen. Dit omvat het toevoegen of verwijderen van files |
| **Read & Execute** | Users krijgen permission of denial om file contents te lezen en programma's uit te voeren |
| **List folder contents** | Users krijgen permission of denial om een lijst van files en subfolders te bekijken |
| **Read** | Users krijgen permission of denial om file contents te lezen |
| **Write** | Users krijgen permission of denial om wijzigingen naar een file te schrijven en nieuwe files aan een folder toe te voegen |
| **Special Permissions** | Een verscheidenheid aan geavanceerde permission opties |

### NTFS Special Permissions

| Permission | Beschrijving |
|-----------|--------------|
| **Full control** | Users krijgen permission of denial om files en folders toe te voegen, bewerken, verplaatsen, verwijderen en NTFS permissions te wijzigen voor alle toegestane folders |
| **Traverse folder / execute file** | Users krijgen permission of denial om een subfolder binnen een directory structuur te bereiken, zelfs als de user geen toegang heeft tot de parent folder. Users kunnen ook permission krijgen om programma's uit te voeren |
| **List folder/read data** | Users krijgen permission of denial om files en folders in de parent folder te bekijken. Users kunnen ook permission krijgen om files te openen en bekijken |
| **Read attributes** | Users krijgen permission of denial om basic attributes van een file of folder te bekijken. Voorbeelden: system, archive, read-only, hidden |
| **Read extended attributes** | Users krijgen permission of denial om extended attributes van een file of folder te bekijken. Attributes verschillen per programma |
| **Create files/write data** | Users krijgen permission of denial om files binnen een folder te maken en wijzigingen aan een file te maken |
| **Create folders/append data** | Users krijgen permission of denial om subfolders binnen een folder te maken. Data kan worden toegevoegd aan files maar bestaande content kan niet worden overschreven |
| **Write attributes** | Users krijgen permission of denial om file attributes te wijzigen. Deze permission geeft geen toegang tot het maken van files of folders |
| **Write extended attributes** | Users krijgen permission of denial om extended attributes van een file of folder te wijzigen. Attributes verschillen per programma |
| **Delete subfolders and files** | Users krijgen permission of denial om subfolders en files te verwijderen. Parent folders worden niet verwijderd |
| **Delete** | Users krijgen permission of denial om parent folders, subfolders en files te verwijderen |
| **Read permissions** | Users krijgen permission of denial om permissions van een folder te lezen |
| **Change permissions** | Users krijgen permission of denial om permissions van een file of folder te wijzigen |
| **Take ownership** | Users krijgen permission of denial om ownership van een file of folder over te nemen. De owner van een file heeft volledige permissions om alle permissions te wijzigen |

## 🧩 Hoe NTFS en Share Permissions Samenwerken

::important
**Belangrijke Verschillen:**
- **NTFS permissions** — Gelden op het systeem waar de folder en files worden gehost
- **Share permissions** — Gelden wanneer de folder via SMB wordt benaderd, typisch vanaf een ander systeem over het netwerk
::

::note
**Lokale toegang vs. Netwerk toegang:**
- Iemand die **lokaal** is ingelogd (of via RDP) hoeft alleen NTFS permissions te overwegen
- Iemand die **over het netwerk** via SMB toegang heeft, moet zowel share als NTFS permissions overwegen
- **De meest restrictieve permission wint** - Permission layering
::

Folders aangemaakt in NTFS **erven permissions van parent folders** standaard. Het is mogelijk om inheritance uit te schakelen om custom permissions in te stellen op parent en subfolders.

NTFS permissions geven administrators **veel meer granulaire controle** over wat users kunnen doen binnen een folder of file.

## 🛠️ Een Network Share Maken

Om een solide fundamenteel begrip te krijgen van SMB en de relatie met NTFS, gaan we een network share maken op een Windows 11 target box.

::tip
**Ideale Lab Setup:**
- Eén monitor voor geschreven content
- Eén monitor voor interactie met systemen
- Alternatief: Smartphone of tablet voor content, één scherm voor lab werk
::

### Share Configuratie in Enterprise Omgevingen

In de meeste grote enterprise-omgevingen worden shares gemaakt op:
- **Storage Area Network (SAN)**
- **Network Attached Storage device (NAS)**
- **Dedicated partitie** op drives via Windows Server

::warning
**Red Flag:** Als je shares op een **desktop operating system** tegenkomt, is het ofwel:
- Een klein bedrijf zonder dedicated infrastructure
- Een **beachhead system** gebruikt door een penetration tester of aanvaller om data te verzamelen en exfiltreren
::

### Folder Maken via GUI

1. **Rechtsklik op Desktop** → New → Folder
2. Geef de folder een naam (bijv. "DATA")

![create-folder](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/4.NTFS-vs-share-permissions/create-folder.png)

### Advanced Sharing Configureren

1. **Rechtsklik op folder** → Properties → Sharing tab → Advanced Sharing
2. Vink **"Share this folder"** aan
3. **Share name** — Wordt automatisch ingesteld op folder naam
4. **User limit** — Limiteer aantal gelijktijdige verbindingen

![advanced-sharing](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/4.NTFS-vs-share-permissions/advanced-sharing.png)

::tip
**Best Practice:** In een productie-omgeving stel je dit aantal in op basis van het aantal users dat regelmatig toegang nodig heeft tot de gedeelde resource.
::

### Share Permissions ACL

Vergelijkbaar met NTFS permissions bestaat er een **Access Control List (ACL)** voor shared resources. Dit is de **SMB permissions list**.

![ACL](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/4.NTFS-vs-share-permissions/ACL.png)

::important
**Dubbele Permissioning:**
Bij shared resources gelden **ZOWEL** SMB als NTFS permissions voor elke resource die in Windows wordt gedeeld.
::

De ACL bevat **Access Control Entries (ACEs)**, die typisch bestaan uit:
- **Users** (security principals)
- **Groups** (security principals)

Dit zijn geschikte mechanismen voor het beheren en tracken van toegang tot shared resources.

### Standaard Share Permissions

De standaard ACE toont vaak:
- **Everyone** group met **Read** permission

## 🛡️ Windows Defender Firewall Overwegingen

::warning
**Blokkade-oorzaak:** De **Windows Defender Firewall** kan toegang tot de SMB share blokkeren, vooral bij verbindingen vanaf Linux-gebaseerde systemen die niet deel uitmaken van dezelfde workgroup.
::

### Workgroup vs. Domain Authenticatie

| Type | Authenticatie Database |
|------|----------------------|
| **Workgroup** | Lokale **SAM database** op het specifieke Windows systeem |
| **Windows Domain** | Gecentraliseerde **Active Directory** database |

::important
**Kritische Kennis:** Bij het inloggen en authenticeren met een Windows systeem moet je weten waar het account is gehost om correct te kunnen verbinden.
::

### Windows Defender Firewall Profiles

De verschillende inbound en outbound rules zijn gekoppeld aan verschillende firewall profiles:

- **Public** — Voor openbare netwerken (meest restrictief)
- **Private** — Voor privé netwerken (thuisnetwerk)
- **Domain** — Voor domain-joined systemen

::tip
**Best Practice:** Enable **predefined rules** of voeg **custom exceptions** toe in plaats van de firewall volledig uit te schakelen. Helaas wordt de firewall vaak volledig uitgeschakeld voor gemak of door gebrek aan begrip.
::

::note
**Centraal Beheer:** Firewall rules op desktop systemen kunnen centraal worden beheerd in een Windows Domain-omgeving via **Group Policy**.
::

## 🔗 Share Mounten vanaf Linux

Zodra de juiste inbound firewall rules zijn ingeschakeld, kunnen we succesvol verbinden met de share.

::important
**Permission Layering:** We kunnen alleen verbinden met de share omdat het user account (htb-student) in de **Everyone** group zit. De share permissions voor Everyone zijn ingesteld op **Read**, wat betekent dat we alleen files kunnen **lezen** op deze share.
::

Zodra een verbinding is gemaakt, kunnen we een **mount point** maken van de Pwnbox naar het Windows 10 file system. Dit is waar **NTFS permissions naast share permissions** gelden.

### CIFS Mount Commando

```bash
skkylimits@htb[/htb]$ sudo mount -t cifs -o username=htb-student,password=Academy_WinFun! //ipaddoftarget/"Company Data" /home/user/Desktop/
```

::tip
**Troubleshooting:** Als dit commando niet werkt, controleer dan de syntax. Als de syntax correct is, moet **cifs-utils** mogelijk worden geïnstalleerd:

```bash
sudo apt-get install cifs-utils
```
::

## 📊 Shares Monitoren op Windows

### Net Share Commando

`net share`, Computer Management, Event Viewer.

Het `net share` commando toont alle shared folders op het systeem.

```cmd
C:\Users\htb-student> net share

Share name   Resource                        Remark

-------------------------------------------------------------------------------
C$           C:\                             Default share
IPC$                                         Remote IPC
ADMIN$       C:\WINDOWS                      Remote Admin
Company Data C:\Users\htb-student\Desktop\Company Data

The command completed successfully.
```

::caution
**Security Concern:** De **C$ share** wordt automatisch aangemaakt bij Windows installatie. Dit betekent dat iemand met de juiste toegang **het volledige C:\ drive** van elk Windows systeem op het netwerk remote kan benaderen!
::

### Computer Management

**Computer Management** is een andere tool om shared resources te identificeren en monitoren:

1. Open **Computer Management** (compmgmt.msc)
2. Navigeer naar **Shared Folders**
3. Bekijk **Shares**, **Sessions**, en **Open Files**

::tip
**Incident Response:** Bij het assisteren met een SMB-gerelateerde breach zijn dit uitstekende plekken om te controleren en te begrijpen hoe de breach kan zijn gebeurd en wat achtergebleven kan zijn.
::

### Event Viewer

**Event Viewer** is een goede plek om acties uitgevoerd op Windows te onderzoeken.

::note
**Logging Concept:** Een log is als een dagboekentry voor een computer, waar de computer alle uitgevoerde acties en talrijke bijbehorende details noteert.
::

We kunnen logs bekijken voor elke actie bij:
- Toegang tot de Windows 10 target box
- Het maken, bewerken en benaderen van de shared folder

**Event Viewer Location:** Security logs → Event ID 5059 (Microsoft Windows security auditing)
