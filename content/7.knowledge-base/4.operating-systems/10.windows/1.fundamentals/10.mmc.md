## 🎛️ Microsoft Management Console (MMC)

De **Microsoft Management Console (MMC)** is een framework dat **snap-ins** (administrative tools) groepeert om hardware, software en netwerkcomponenten binnen een Windows host te beheren.

::important
**MMC Core Information:**
- **Beschikbaar sinds** Windows Server 2000
- **Draait op** alle Windows versies (client en server)
- **Executable:** `mmc.exe` (locatie: `C:\Windows\System32\mmc.exe`)
- **Purpose:** Centralized management interface voor diverse administrative tools
::

### MMC Concept: Snap-ins

MMC werkt met het concept van **snap-ins**, waarmee administrators een **customized console** kunnen maken met alleen de administrative tools die nodig zijn voor specifieke management taken.

**Snap-in voordelen:**
- **Flexibility** — Combineer alleen de tools die je nodig hebt
- **Customization** — Maak role-specific management consoles
- **Distribution** — Deel custom `.msc` files met andere administrators
- **Local & Remote** — Beheer zowel lokale als remote systemen
- **Consistency** — Gestandaardiseerde interface voor diverse tools

## 🚀 MMC Openen en Gebruiken

### MMC Starten

**Meerdere methoden:**

```cmd
# Via Start Menu
Type "mmc" in Start menu search

# Via Run dialogue
Win + R → type "mmc"

# Via Command Line
mmc.exe
```

::note
**Eerste Keer Openen:**
Wanneer je MMC voor de eerste keer opent, zie je een **blanco console** zonder snap-ins. Dit is het startpunt voor het bouwen van je custom management console.
::

### Snap-ins Toevoegen

**Stap 1: Add/Remove Snap-ins Menu Openen**

Navigeer naar: **File → Add or Remove Snap-ins** (of druk `Ctrl + M`)

**Stap 2: Snap-ins Selecteren**

In het "Add or Remove Snap-ins" venster zie je:
- **Available snap-ins** (links) — Alle beschikbare administrative tools
- **Selected snap-ins** (rechts) — Snap-ins toegevoegd aan de current console

**Stap 3: Management Scope Kiezen**

Bij het toevoegen van snap-ins word je gevraagd:
- **Local computer** — Beheer alleen deze machine
- **Another computer** — Beheer een remote systeem via netwerk
- **Allow changing the selected computer when launching from command line** — Flexibiliteit voor scripting

**Stap 4: Console Opslaan**

Nadat je snap-ins hebt toegevoegd:
1. **File → Save As**
2. **Default locatie:** `%AppData%\Microsoft\Windows\Start Menu\Programs\Administrative Tools\`
3. **Bestandsextensie:** `.msc` (Microsoft Saved Console)

::tip
**Custom Console Naming:**
Geef je saved console een **beschrijvende naam** zoals:
- `NetworkManagement.msc` — Voor netwerk-gerelateerde snap-ins
- `SecurityAudit.msc` — Voor security monitoring tools
- `ServerAdmin.msc` — Voor server management
::

## 📦 Veelgebruikte MMC Snap-ins

| Snap-in | Beschrijving | Executable | Primary Use Case |
|---------|--------------|------------|------------------|
| **Computer Management** | All-in-one management console | `compmgmt.msc` | System management, disk management, services |

| **Disk Management** | Disk partitioning en volume management | `diskmgmt.msc` | Create/delete partitions, assign drive letters |
| **Event Viewer** | System, security en application event logs | `eventvwr.msc` | Log analysis, troubleshooting, security monitoring |
| **Shared Folders** | Manage network shares en sessions | `fsmgmt.msc` | Share management, session monitoring |
| **Local Users and Groups** | User en group account management | `lusrmgr.msc` | Local account administration |
| **Performance Monitor** | System performance monitoring | `perfmon.msc` | Performance analysis, bottleneck detection |
| **Services** | Windows service management | `services.msc` | Start/stop/configure services |
| **Task Scheduler** | Scheduled task automation | `taskschd.msc` | Task creation, automation, persistence |
| **Windows Firewall** | Firewall configuration | `wf.msc` | Firewall rules, network protection |
| **Certificates** | Certificate store management | `certmgr.msc` | Certificate management, PKI |
| **Group Policy Editor** | Local Group Policy management | `gpedit.msc` | Policy configuration (Pro/Enterprise only) |
| **Active Directory Users and Computers** | AD user/computer management | `dsa.msc` | Domain administration (requires AD DS) |
| **Active Directory Sites and Services** | AD replication topology | `dssite.msc` | Site/subnet management (requires AD DS) |
| **DNS** | DNS server management | `dnsmgmt.msc` | DNS zones, records (requires DNS role) |
| **DHCP** | DHCP server management | `dhcpmgmt.msc` | IP address management (requires DHCP role) |

| **Hyper-V Manager** | Virtual machine management | `virtmgmt.msc` | VM administration (requires Hyper-V) |

::note
**Domain vs. Standalone:**
Sommige snap-ins (zoals Active Directory tools) zijn **alleen beschikbaar** op:
- **Domain Controllers** (voor AD-gerelateerde snap-ins)
- **Systems met specifieke roles** geïnstalleerd (IIS, DNS, DHCP, Hyper-V)
- **Windows Pro/Enterprise** editions (Group Policy Editor niet beschikbaar op Home edition)
::

## 🔧 Custom MMC Console Maken

### Voorbeeld: Security Monitoring Console

**Scenario:** Maak een custom console voor security monitoring taken.

**Snap-ins om toe te voegen:**
1. **Event Viewer** — Security log analysis
2. **Services** — Monitor security-related services
3. **Local Users and Groups** — User account auditing
4. **Windows Firewall with Advanced Security** — Firewall rule review
5. **Certificates** — Certificate store auditing
6. **Task Scheduler** — Scheduled task persistence detection

**Stappen:**
```
1. Open MMC (mmc.exe)
2. File → Add or Remove Snap-ins
3. Voeg bovenstaande snap-ins toe
4. Configure voor "Local computer" of "Another computer"
5. File → Save As → "SecurityMonitoring.msc"
6. Bewaar in Administrative Tools directory
```

**Gebruik:**
```cmd
# Direct openen
mmc "C:\Path\To\SecurityMonitoring.msc"

# Of via Start Menu
Start → Administrative Tools → SecurityMonitoring
```

### Voorbeeld: Remote Server Management Console

**Scenario:** Beheer meerdere remote servers vanuit één console.

**Configuration:**
1. Voeg snap-ins toe zoals **Computer Management**, **Services**, **Event Viewer**
2. Kies **"Another computer"** bij elke snap-in
3. Geef de remote server naam of IP op
4. Herhaal voor meerdere servers (elke server krijgt eigen snap-in instance)
5. Save as `RemoteServerManagement.msc`

::important
**Remote Management Vereisten:**
- **Network connectivity** naar target systeem
- **Administrative credentials** voor remote system
- **Firewall rules** configured voor remote management (RPC, WMI)
- **Remote Registry service** enabled op target (voor sommige snap-ins)
::

## 🎯 MMC voor Offensive Security

MMC en bijbehorende snap-ins kunnen worden gebruikt tijdens **penetration testing** en **red team operations** voor reconnaissance en privilege escalation.

### Reconnaissance via MMC

**1. Computer Management (compmgmt.msc)**
```
System Tools → Event Viewer → Security logs (logon events)
System Tools → Local Users and Groups (user enumeration)
System Tools → Shared Folders → Shares (SMB share discovery)
Storage → Disk Management (volume information)
Services and Applications → Services (service enumeration)
```

**2. Event Viewer (eventvwr.msc)**
```
Windows Logs → Security (Event ID 4624, 4625, 4672)
Windows Logs → System (service failures, crashes)
Applications and Services Logs → Microsoft → Windows → PowerShell/Operational (script execution)
```

**3. Services (services.msc)**
```
# Identify services running as SYSTEM
# Check for misconfigured service permissions
# Identify unquoted service paths
# Find services with weak file permissions
```

**4. Task Scheduler (taskschd.msc)**
```
# Enumerate scheduled tasks
# Identify tasks running as SYSTEM or privileged users
# Check for misconfigured task permissions
# Persistence location reconnaissance
```

**5. Local Users and Groups (lusrmgr.msc)**
```
# Enumerate local users
# Identify users in Administrators group
# Check for disabled accounts
# Password policy reconnaissance
```

### Privilege Escalation via MMC Snap-ins

**Scenario 1: Unquoted Service Path**

Via **Services (services.msc)**:
1. Open Services snap-in
2. Identify service met unquoted path: `C:\Program Files\Vulnerable App\service.exe`
3. Check service **startup type** (Automatic) en **run as** (SYSTEM)
4. Check **write permissions** op intermediate directories
5. Place malicious executable in path

**Scenario 2: Weak Service Permissions**

Via **Services (services.msc)** + **sc.exe**:
1. Identify service with weak permissions (modifiable by current user)
2. Modify service binary path: `sc config VulnService binPath= "C:\payload.exe"`
3. Restart service voor execution as SYSTEM

**Scenario 3: Scheduled Task Hijacking**

Via **Task Scheduler (taskschd.msc)**:
1. Enumerate scheduled tasks running as SYSTEM or high-privilege user
2. Check task **action** (executable path)
3. Check **write permissions** op executable
4. Replace executable met malicious binary
5. Wait voor scheduled execution

::warning
**Defensive Detection:**
Monitor voor:
- **Unusual MMC usage** in non-administrative contexts
- **Remote MMC connections** van workstations
- **Service configuration changes** (Event ID 7040, 7045)
- **Scheduled task creation/modification** (Event ID 4698, 4702)
- **Privilege escalation attempts** via service/task manipulation
::

## 🛡️ Defensive Monitoring van MMC Activiteit

### Event IDs voor MMC Tracking

| Event ID | Log | Beschrijving | Relevantie |
|----------|-----|--------------|------------|
| **4688** | Security | Process Creation | Track `mmc.exe` execution met command line parameters |
| **4663** | Security | Object Access | Monitor access to sensitive snap-in files (`.msc`) |
| **7040** | System | Service state change | Detect service configuration modifications |
| **7045** | System | Service installed | Detect new service creation |
| **4698** | Security | Scheduled task created | Detect task persistence attempts |
| **4702** | Security | Scheduled task updated | Detect task modification |
| **4719** | Security | System audit policy change | Detect audit policy tampering |

### Sysmon Detection

**Sysmon Event IDs relevant voor MMC:**

| Event ID | Beschrijving |
|----------|--------------|
| **1** | Process Creation — Track `mmc.exe` met command line args |
| **10** | Process Access — Detect MMC accessing sensitive processes (LSASS) |
| **11** | File Created — Monitor `.msc` file creation in suspicious locations |
| **13** | Registry Value Set — Track service/task configuration changes via registry |

**Suspicious MMC Patterns:**
```
# MMC with custom .msc from unusual location
mmc.exe "C:\Users\Public\malicious.msc"

# MMC with specific snap-ins via command line
mmc.exe services.msc -computer=REMOTE_TARGET

# MMC from non-administrative users
mmc.exe (executed by standard user)
```

### Defensive Hardening

::important
**MMC Hardening Measures:**

**Access Control:**
- Restrict **MMC.exe execution** via AppLocker/WDAC policies
- Limit **snap-in access** via Group Policy (restrict access to specific `.msc` files)
- Implement **least privilege** — Only administrators should have unrestricted MMC access

**Audit Configuration:**
- Enable **Process Creation Auditing** (Event ID 4688) with command line logging
- Enable **Object Access Auditing** voor `.msc` files in Administrative Tools directory
- Monitor **Service Control Manager** events (7040, 7045)
- Enable **Task Scheduler** event auditing (4698, 4699, 4700, 4701, 4702)

**Network Restrictions:**
- Block **remote MMC connections** tussen workstations via firewall rules
- Restrict **RPC/DCOM** ports (135, dynamic) used by MMC voor remote management
- Use **Jump Boxes/PAWs** (Privileged Access Workstations) voor administrative tasks

**Snap-in Restrictions via Group Policy:**
```
Computer Configuration → Policies → Administrative Templates → Windows Components → Microsoft Management Console
- Restrict users to the explicitly permitted list of snap-ins (Enabled)
- User Configuration → Same path → Restrict snap-in access
```
::

## 🎯 Offensive vs. Defensive Cheat Sheet

**Offensive (Educational):**
```cmd
# Enumerate services for privilege escalation
services.msc
# Look for: unquoted paths, weak permissions, SYSTEM services

# Enumerate scheduled tasks
taskschd.msc
# Look for: tasks as SYSTEM, modifiable task actions

# Enumerate local users/groups
lusrmgr.msc
# Look for: admin accounts, password policies

# Review event logs for credentials
eventvwr.msc → Security → Event ID 4624, 4648
# Look for: cleartext credentials, password spraying targets

# Identify shares for lateral movement
compmgmt.msc → Shared Folders → Shares
# Look for: writable shares, sensitive data exposure
```

**Defensive:**
```powershell
# Monitor MMC execution
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4688} | Where {$_.Properties[5].Value -like "*mmc.exe*"}

# Audit service changes
Get-WinEvent -FilterHashtable @{LogName='System'; ID=7040,7045}

# Audit scheduled task creation
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4698,4702}

# Review unusual .msc file access
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4663} | Where {$_.Message -like "*.msc*"}

# Sysmon: Track MMC process creation
Get-WinEvent -FilterHashtable @{LogName='Microsoft-Windows-Sysmon/Operational'; ID=1} | Where {$_.Properties[4].Value -like "*mmc.exe*"}
```

## 💡 Best Practices

::tip
**Administrative Efficiency:**
1. **Create role-specific consoles** — Network admin, security admin, server admin
2. **Distribute `.msc` files** via GPO Preferences voor team consistency
3. **Document custom consoles** — Beschrijf welke snap-ins en waarom
4. **Use descriptive naming** — Duidelijke console namen voor quick identification
5. **Regular updates** — Review en update custom consoles bij role changes

**Security Considerations:**
1. **Least privilege access** — Restrict MMC usage tot administrative accounts
2. **Audit MMC activity** — Enable comprehensive logging
3. **Network segmentation** — Limit remote MMC to management networks
4. **AppLocker/WDAC** — Whitelist legitimate `.msc` files
5. **Privileged Access Workstations** — Use dedicated systems voor administrative tasks
::
