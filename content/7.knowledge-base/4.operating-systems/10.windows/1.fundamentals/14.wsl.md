## 🐧 Windows Subsystem for Linux (WSL)


[WSL2](https://learn.microsoft.com/en-us/windows/wsl/)

**WSL** is een feature dat **Linux binaries natively** laat draaien op Windows 10 en Windows Server 2019+. Het werd oorspronkelijk ontworpen voor developers die **Bash, Ruby en native Linux command-line tools** (zoals `sed`, `awk`, `grep`) direct op hun Windows workstation nodig hadden.

::important
**WSL Core Information:**
- **Beschikbaar sinds:** Windows 10 (Build 14393) en Windows Server 2019
- **Current versies:** WSL 1 (translation layer) en WSL 2 (real Linux kernel)
- **Purpose:** Run Linux environments zonder dual-boot of virtual machine overhead
- **Primary users:** Developers, DevOps engineers, security professionals, penetration testers
::

### WSL Evolution: Versie 1 vs. Versie 2

**WSL 1 (2016):**
- **Translation layer** — Syscalls worden vertaald van Linux naar Windows NT kernel
- **Geen echte Linux kernel** — Emulatie laag voor compatibility
- **File system performance** — Direct access tot Windows NTFS (sneller voor cross-OS file access)
- **Compatibility** — Beperkte compatibility met sommige Linux applicaties

**WSL 2 (Mei 2019):**
- **Real Linux kernel** — Lightweight virtual machine met echte Linux kernel
- **Hyper-V integration** — Gebruikt subset van Hyper-V features
- **Volledige syscall compatibility** — 100% Linux kernel compatibility
- **Verbeterde performance** — Snellere file I/O binnen Linux filesystem
- **Docker support** — Native Docker container support

::note
**WSL 2 is de aanbevolen versie** voor de meeste use cases vanwege:
- Volledige system call compatibility
- Betere performance voor Linux-native workloads
- Docker Desktop integration
- Echte Linux kernel features (eBPF, io_uring, etc.)
::

## 🚀 WSL Installatie

### WSL Enablen via PowerShell

**Windows 10/11 (Modern methode):**
```powershell
# Install WSL met default Linux distributie (Ubuntu)
wsl --install

# Install specifieke distributie
wsl --install -d Ubuntu-22.04
```

**Windows 10 (Legacy methode voor WSL 1):**
```powershell
# Enable WSL feature (Administrator PowerShell vereist)
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

# Voor WSL 2: Enable Virtual Machine Platform
Enable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform

# Set WSL 2 als default
wsl --set-default-version 2
```

### Linux Distributie Installeren

**Methode 1: Microsoft Store**
- Open **Microsoft Store**
- Zoek naar "Linux" of specifieke distro (Ubuntu, Debian, Kali Linux, etc.)
- Click **Install**

**Methode 2: Command Line (Manual Download)**
```powershell
# Download Ubuntu 22.04 (voorbeeld)
Invoke-WebRequest -Uri https://aka.ms/wslubuntu2204 -OutFile Ubuntu.appx -UseBasicParsing

# Install de .appx
Add-AppxPackage .\Ubuntu.appx

# Launch distributie voor eerste setup
ubuntu2204.exe
```

**Beschikbare distributies:**
- Ubuntu (18.04, 20.04, 22.04, 24.04)
- Debian
- Kali Linux (populair voor penetration testing)
- openSUSE
- Alpine Linux
- Oracle Linux
- Fedora Remix

::tip
**Kali Linux in WSL:**
Voor penetration testing en security research is **Kali Linux** een populaire keuze:
```powershell
wsl --install -d kali-linux
```
Kali bevat **600+ security tools** pre-installed, perfect voor offensive security werk.
::

## 💻 WSL Gebruiken

### Bash Shell Starten

WSL installeert **Bash.exe** waarmee je een Bash shell kunt spawnen:

```powershell
# Start default WSL distributie
wsl

# Of via bash command
bash

# Start specifieke distributie
wsl -d Ubuntu-22.04
wsl -d kali-linux
```

### Linux Directory Structure

Eenmaal in de Bash shell heb je de **volledige Linux directory structure**:

```bash
PS C:\htb> wsl
user@WS01:/mnt/c/htb$ ls /

bin   dev  home  lib    lib64   media  opt   root  sbin  srv  tmp  var
boot  etc  init  lib32  libx32  mnt    proc  run   snap  sys  usr
```

### Windows Volumes Access

Windows volumes zijn toegankelijk via de **`/mnt/` directory**:

```bash
# Access C: drive
cd /mnt/c

# Access D: drive
cd /mnt/d

# List all Windows volumes
ls /mnt/
```

::important
**Seamless Integration:**
De `/mnt/` directory maakt de transitie tussen WSL en Windows **naadloos**:
- **Lezen/schrijven** van Windows bestanden vanuit Linux
- **Uitvoeren** van Windows executables vanuit Linux (met `.exe` extensie)
- **Interoperability** tussen Linux en Windows tools
::

### Linux Commands Uitvoeren

```bash
# System informatie
uname -a
# Output: Linux WS01 5.15.90.1-microsoft-standard-WSL2 #1 SMP x86_64 GNU/Linux

# Package updates (Ubuntu/Debian)
sudo apt update && sudo apt upgrade -y

# Install tools
sudo apt install nmap netcat-traditional python3-pip

# File operations
grep -r "password" /mnt/c/Users/*/Documents/
find /mnt/c/ -name "*.txt" -type f

# Network reconnaissance
nmap -sV 192.168.1.0/24
```

### Windows Executables vanuit WSL

Je kunt **Windows executables** aanroepen vanuit WSL:

```bash
# Execute Windows command
cmd.exe /c dir

# PowerShell command
powershell.exe -Command "Get-Process"

# Open Windows Explorer in current directory
explorer.exe .

# Notepad
notepad.exe file.txt
```
