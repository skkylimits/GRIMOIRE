## 🔧 Windows Services

Services zijn een belangrijk component van het Windows besturingssysteem. Ze maken het mogelijk om **long-running processen** te creëren en beheren. Windows services kunnen automatisch starten bij system boot **zonder user interventie** en blijven draaien in de achtergrond, zelfs nadat de user uitlogt.

::note
**Service Gebruik:**
- Applicaties kunnen als service worden geïnstalleerd (bijv. network monitoring applicatie op een server)
- Services zijn verantwoordelijk voor veel functies: networking, system diagnostics, user credentials, Windows updates, etc.
::

### Service Control Manager (SCM)

Windows services worden beheerd via het **Service Control Manager (SCM)** systeem, toegankelijk via de **`services.msc`** MMC add-in.

Deze add-in biedt een **GUI interface** voor interactie met en beheer van services en toont informatie over elke geïnstalleerde service:

| Veld | Beschrijving |
|------|--------------|
| **Name** | Service naam |
| **Description** | Omschrijving van service functionaliteit |
| **Status** | Running, Stopped, of Paused |
| **Startup Type** | Manual, Automatic, of Automatic (Delayed Start) |
| **Log On As** | User account waaronder de service draait |

### Services Beheren via Command Line

Services kunnen ook worden beheerd via de command line met **`sc.exe`** of PowerShell cmdlets zoals **`Get-Service`**.

```powershell
PS C:\htb> Get-Service | ? {$_.Status -eq "Running"} | select -First 2 |fl

Name                : AdobeARMservice
DisplayName         : Adobe Acrobat Update Service
Status              : Running
DependentServices   : {}
ServicesDependedOn  : {}
CanPauseAndContinue : False
CanShutdown         : False
CanStop             : True
ServiceType         : Win32OwnProcess

Name                : Appinfo
DisplayName         : Application Information
Status              : Running
DependentServices   : {}
ServicesDependedOn  : {RpcSs, ProfSvc}
CanPauseAndContinue : False
CanShutdown         : False
CanStop             : True
ServiceType         : Win32OwnProcess, Win32ShareProcess
```

### Service Statussen en Types

**Service Statussen:**
- **Running** — Service is actief
- **Stopped** — Service is gestopt
- **Paused** — Service is gepauzeerd
- **Starting** — Service wordt opgestart
- **Stopping** — Service wordt gestopt

**Startup Types:**
- **Manual** — Service moet handmatig worden gestart
- **Automatic** — Service start automatisch bij boot
- **Automatic (Delayed Start)** — Service start met vertraging na boot

### Service Categorieën

Windows heeft **drie categorieën** van services:

| Categorie | Beschrijving |
|-----------|--------------|
| **Local Services** | Services die met beperkte privileges draaien |
| **Network Services** | Services met netwerk toegang maar beperkte lokale privileges |
| **System Services** | Services die met de hoogste privileges draaien |

::warning
**Privilege Escalation Vector:**
Services kunnen meestal alleen worden gemaakt, gewijzigd en verwijderd door users met **administrative privileges**. Misconfigurations rond service permissions zijn een **veelvoorkomend privilege escalation vector** op Windows systemen.
::

::important
**Kritieke System Services:**
Sommige kritieke system services kunnen **niet worden gestopt en herstart zonder system reboot**. Als we een file of resource updaten die door een van deze services wordt gebruikt, moeten we het systeem herstarten.
::

## 🎯 Kritieke System Services

| Service | Beschrijving |
|---------|--------------|
| **smss.exe** | Session Manager SubSystem — Verantwoordelijk voor het beheren van sessions op het systeem |
| **csrss.exe** | Client Server Runtime Process — De user-mode component van het Windows subsystem |
| **wininit.exe** | Start het Wininit .ini bestand dat alle wijzigingen bevat die worden gemaakt aan Windows bij herstart na software installatie |
| **logonui.exe** | Faciliteert user login op een PC |
| **lsass.exe** | Local Security Authentication Server — Verifieert de validiteit van user logons op een PC of server. Genereert het proces verantwoordelijk voor authenticatie voor de Winlogon service |
| **services.exe** | Beheert het starten en stoppen van services |
| **winlogon.exe** | Verantwoordelijk voor het afhandelen van de secure attention sequence, het laden van een user profile bij logon, en het vergrendelen van de computer bij screensaver |
| **System** | Een achtergrond system proces dat de Windows kernel draait |
| **svchost.exe with RPCSS** | Beheert system services die draaien vanuit DLLs zoals "Automatic Updates", "Windows Firewall" en "Plug and Play". Gebruikt de Remote Procedure Call (RPC) Service |
| **svchost.exe with Dcom/PnP** | Beheert system services die draaien vanuit DLLs. Gebruikt Distributed Component Object Model (DCOM) en Plug and Play (PnP) services |

::tip
Een volledige lijst van Windows components, inclusief key services, is beschikbaar in de officiële Microsoft documentatie.
::

## ⚙️ Windows Processes

Processes draaien in de achtergrond op Windows systemen. Ze draaien ofwel:
- **Automatisch** als onderdeel van het Windows besturingssysteem
- **Gestart door geïnstalleerde applicaties**

### Application vs. System Processes

::note
**Application Processes:**
Processes geassocieerd met geïnstalleerde applicaties kunnen vaak worden beëindigd **zonder ernstige impact** op het besturingssysteem.
::

::caution
**Critical System Processes:**
Bepaalde processen zijn **kritiek** en als ze worden beëindigd, stoppen bepaalde OS-componenten met correct functioneren. Voorbeelden:
- Windows Logon Application
- System
- System Idle Process
- Windows Start-Up Application
- Client Server Runtime
- Windows Session Manager
- Service Host
- **Local Security Authority Subsystem Service (LSASS)**
::

## 🔐 Local Security Authority Subsystem Service (LSASS)

**`lsass.exe`** is het proces verantwoordelijk voor het **afdwingen van de security policy** op Windows systemen.

### LSASS Functionaliteit

**Wanneer een user probeert in te loggen:**
1. LSASS **verifieert de logon attempt**
2. Creëert **access tokens** gebaseerd op user permission levels
3. LSASS is ook verantwoordelijk voor **user account password changes**

::important
**Event Logging:**
Alle events geassocieerd met dit proces (logon/logoff attempts, etc.) worden gelogd in de **Windows Security Log**.
::

### LSASS als High-Value Target

::warning
**Security Concern:**
LSASS is een **extreem high-value target** omdat er verschillende tools bestaan om **zowel cleartext als hashed credentials** te extraheren die in geheugen worden opgeslagen door dit proces.

**Bekende Tools:**
- **Mimikatz** — Extraheert credentials uit LSASS geheugen
- **ProcDump** — Dumpt LSASS proces voor offline analyse
- **SafetyKatz** — Mimikatz variant met evasion technieken
::

::tip
**Defensive Perspective:**
- Monitor voor **abnormale toegang tot lsass.exe** (Event ID 10: Process Access)
- Implementeer **Credential Guard** om LSASS te beschermen via virtualization-based security
- Gebruik **Windows Defender Application Control (WDAC)** om toegang tot LSASS te beperken
- Enable **LSA Protection** (RunAsPPL) om LSASS als Protected Process te draaien
::

De **SysInternals Tools suite** is een set van portable Windows applicaties die kunnen worden gebruikt om Windows systemen te administreren, meestal **zonder installatie**.

### Toegang tot Sysinternals Tools

**Download opties:**
1. **Microsoft website** — Directe download van individuele tools
2. **Live network share** — Direct uitvoeren vanaf `\\live.sysinternals.com\tools`

## 📊 Task Manager

**Windows Task Manager** is een krachtige tool voor het beheren van Windows systemen. Het biedt informatie over running processen, system performance, services, startup programs, logged-in users en meer.

### Task Manager Openen

**Meerdere manieren:**
- **Rechtsklik** op taskbar → Task Manager
- **Toetsencombinatie:** `Ctrl + Shift + Esc`
- **Security menu:** `Ctrl + Alt + Del` → Task Manager
- **Start menu:** Type "Task Manager"
- **Command line:** `taskmgr` in CMD of PowerShell

![task-manager](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/5.services-and-processes/task-manager.png)

### Task Manager Tabs

| Tab | Beschrijving |
|-----|--------------|
| **Processes** | Lijst van running applicaties en achtergrond processen met CPU, memory, disk, network en power usage voor elk proces |
| **Performance** | Grafieken en data voor CPU utilization, system uptime, memory usage, disk, networking en GPU usage. Bevat link naar **Resource Monitor** voor meer gedetailleerde view |
| **App history** | Resource usage voor de current user account per applicatie over een periode |
| **Startup** | Applicaties geconfigureerd om te starten bij boot, inclusief **impact op startup process** |
| **Users** | Logged in users en de processen/resource usage geassocieerd met hun sessie |
| **Details** | Naam, process ID (PID), status, username, CPU en memory usage voor elke running applicatie |
| **Services** | Naam, PID, beschrijving en status van elke geïnstalleerde service. Toegang tot Services add-in (`services.msc`) |

::note
**Resource Monitor:**
Kan geopend worden vanaf de Performance tab en biedt een **veel meer diepgaande view** van de current CPU, Memory, Disk en Network resource usage.
::

## 🧠 Resource Monitor

**Resource Monitor** is een advanced diagnostic tool in Windows die **real-time inzicht** geeft in CPU, Memory, Disk en Network activiteit. Het biedt veel dieper detail dan Task Manager en wordt vaak gebruikt voor performance troubleshooting, malware analysis en process forensics.

### Resource Monitor openen

**Manieren om het te starten:**

* **Via Task Manager:** Performance tab → *Open Resource Monitor*
* **Run dialog:** `resmon`
* **Start menu:** Typ “Resource Monitor”
* **CMD/PowerShell:** `resmon.exe`

![recourse-monitor](/content/7.knowledge-base/4.operating-systems/10.windows/1.fundamentals/5.services-and-processes/resource-monitor.png)

### Resource Monitor Tabs

| Tab          | Beschrijving                                                                              |
| ------------ | ----------------------------------------------------------------------------------------- |
| **Overview** | Combineert alle belangrijke resource categorieën (CPU, Memory, Disk, Network) in één view |
| **CPU**      | Threads, handles, associated modules, context switches, per-process CPU usage             |
| **Memory**   | Werkset, commit, hard faults/sec, shared memory, private memory                           |
| **Disk**     | Disk I/O, read/write latency, queue length, file handles die I/O genereren                |
| **Network**  | TCP connections, listening ports, per-process network throughput, remote IP addresses     |

**Waarom Resource Monitor belangrijk is:**

Het laat zien **wélke processen**:

* Je disk blokkeren
* Je CPU belasten
* Grote hoeveelheden RAM reserveren
* Netwerkverbindingen maken

Dit maakt het een zeer nuttige tool voor:
**forensics, malware detection, performance debugging en red-team opsec monitoring.**




## 🔍 Process Explorer

**Process Explorer** is onderdeel van de Sysinternals tool suite. Deze tool toont welke **handles en DLLs** worden geladen wanneer een programma draait.

### Process Explorer Functionaliteit

**Twee Primary Views:**

1. **Handle View** — Toont welke handles het proces heeft geselecteerd
2. **DLL View** — Toont welke DLLs en memory-swapped files zijn geladen

### Advanced Features

::tip
**Search Functionaliteit:**
Je kunt binnen de tool zoeken om te zien welke processen terugverwijzen naar een **specifieke handle of DLL**.
::

::important
**Parent-Child Process Relationships:**
Process Explorer kan worden gebruikt om:
- **Parent-child process relationships** te analyseren
- Te zien welke **child processen** door een applicatie worden spawned
- **Orphaned processen** te troubleshooten die achterblijven wanneer een proces wordt beëindigd
::
