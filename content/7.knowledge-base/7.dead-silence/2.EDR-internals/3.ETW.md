> ETW (Event Tracing for Windows) is de onderliggende technologie die ook wordt gebruikt door Event Viewer — maar het is veel krachtiger en breder dan alleen wat je daar ziet.

We will review Event Tracing for Windows (ETW), focusing on the architecture, how telemetry is collected, and how events flow through the system.

## Architecture

### Core Components:

* **Controller:**
  Manages ETW sessions by starting or stopping them to begin or end event collection. It enables or disables the capture of telemetry data.

* **Providers:**
  These are sources of event data. Providers control which event sources generate telemetry based on system or application activity. There are two main types:

  * **Kernel-mode providers:** Monitor low-level system events such as process creation, thread activity, file system operations, registry changes, and more.
  * **User-mode providers:** Monitor application-specific activities and generate telemetry based on those.

* **Sessions:**
  Responsible for collecting and managing event data. A session maintains multiple **buffers** where events are temporarily stored before processing. Multiple independent sessions can run simultaneously, collecting events from various providers.

* **Buffers:**
  Temporary in-memory storage for event data before it is either consumed or logged.

* **Consumers:**
  Applications or tools that process the collected event data. There are two types:

  * **Real-time consumers:** Read events directly from session buffers as they are generated.
  * **Logged event consumers:** Read events from ETL (Event Trace Log) files stored on disk for later analysis.

## Anatomy ETW Event

Each **ETW Provider** emits multiple **Events**.

* Every **Event** has:

  * A **unique Event ID** — identifying the specific action or telemetry being recorded.
  * A **version** — newer versions may include **additional or extended properties** for more detailed logging.
  * A set of **Properties** — which are populated by the provider with telemetry data.

* Each **Property** within an event:

  * Has a **name**, **data type** (e.g. integer, string, timestamp), and
  * Holds a **specific value** describing what happened.

::note
A process creation event might have:

* Event ID: `1`
* Version: `2`
* Properties:

  * `ImageFileName` (String)
  * `ProcessId` (UInt32)
  * `ParentProcessId` (UInt32)
  * `CommandLine` (String)

These details are **essential for security tools** to make sense of low-level system activity and recognize patterns of malicious behavior.
::

## Event Flow in ETW

1. **Providers** generate events based on system or application activity.
2. Events are stored in **session buffers** in memory.
3. Depending on configuration, events are either:

   * Consumed in **real-time** from buffers by consumer applications.
   * **Logged** into ETL files on disk for offline analysis.
4. **Consumers** process these events for analysis, monitoring, or alerting.

## Windows Defender and ETW

* Windows Defender utilizes ETW extensively to collect security-related telemetry.

* The core Defender process (`MsSense.exe`) consumes ETW events via multiple sessions:

  * **Defender Audit Logger (Session 1)**
  * **Defender API Logger (Session 2)**

* These sessions capture telemetry such as:

  * Security events
  * System activity
  * Network activity
  * API interactions

* Logs are stored in ETL files located at:
  `C:\Windows\System32\logfiles\WMI\`
  Example files:

  * `EtwDefenderAuditLogger.etl`
  * `EtwDefenderAPILogger.etl`

* The Defender API Logger session is linked to multiple providers, including:

  * Microsoft-Windows-Kernel-General
  * Microsoft-Windows-EventLog
  * Microsoft-Windows-Threat-Intelligence
  * Microsoft-Windows-WMI-Activity

* Each provider emits events with unique IDs and versions that add detail. Events contain properties such as numbers, pointers, or timestamps, which hold useful data for analysis.

## **Why ETW Matters for Security**

* **ETW** provides **real-time telemetry** that security tools like **Windows Defender** use to:

  * **Detect potential malicious behaviors**.
  * **Identify suspicious activities**.
  * **Enhance threat detection capabilities**.

* **Attackers** may try to **evade detection** by **disabling ETW providers** or **stopping sessions**, making **understanding ETW** crucial for **defenders**.


## **Summary**

**Event Tracing for Windows (ETW)** is a **powerful**, **flexible telemetry system** that captures **detailed system and application events**.
Its architecture involves:

* **Controllers**
* **Providers**
* **Sessions**
* **Buffers**
* **Consumers**

...enabling both **real-time** and **logged event processing**.

**Security tools** rely heavily on **ETW** to **monitor** and **respond to threats**.
