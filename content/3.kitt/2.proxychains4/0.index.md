## **0. WSL2**

::caution
Normally, WSL shares the **Windows host's IP address** for all network activity. This means any traffic from WSL will appear as if it originates from your **Windows system's IP**.

However, when using Proxychains, your traffic is routed through the proxies defined in `/etc/proxychains4.conf`. The **final IP** seen by the target website will be the IP of the **last proxy in the chain**, not your Windows or WSL IP.
::

## **1. Installation**

Install Proxychains on Debian/Ubuntu (e.g., in WSL):

```bash
sudo apt update && sudo apt install proxychains4 -y
```

## **2. Edit the Proxychains Configuration**

The default configuration file is located at `/etc/proxychains4.conf`.

1. Open the file:

   ```bash
   sudo nano /etc/proxychains4.conf
   ```

2. **Set the Chaining Mode**:

   - Uncomment one mode:
     - `dynamic_chain` (recommended): Skips unavailable proxies.
     - `strict_chain`: Requires all proxies to work.
     - `random_chain`: Randomly selects proxies.
   - Example:
     ```plaintext
     dynamic_chain
     ```

3. **Enable DNS Resolution through Proxy**:
   Uncomment or add:

   ```plaintext
   proxy_dns
   ```

4. **Add Proxies**:
   Scroll to the `ProxyList` section and add your proxies. Format:
   ```plaintext
   http   <proxy_ip> <port>
   socks4 <proxy_ip> <port>
   socks5 <proxy_ip> <port>
   ```
   Example:
   ```plaintext
   socks5 127.0.0.1 9050  # Tor proxy
   http   154.16.146.41 80
   ```

### **Script: `test_proxies.sh`**

To automate testing, use this script to farm proxies and then validate them.

```bash
#!/bin/bash

# File containing the list of proxies (one per line)
PROXY_LIST="proxies.txt"

# Function to test a single proxy
test_proxy() {
    local proxy=$1
    echo "Testing proxy: $proxy"

    # Use curl to test the proxy
    response=$(curl -s -x http://$proxy https://ipinfo.io --max-time 10)

    if [[ -z "$response" ]]; then
        echo "Proxy $proxy failed or timed out."
    else
        echo "Proxy $proxy succeeded. IP returned: $response"
    fi
    echo "--------------------------------"
}

# Check if the proxy list file exists
if [[ ! -f "$PROXY_LIST" ]]; then
    echo "Error: Proxy list file '$PROXY_LIST' not found."
    exit 1
fi

# Read and test each proxy from the list
while read -r proxy; do
    [[ -z "$proxy" ]] && continue  # Skip empty lines
    test_proxy "$proxy"
done < "$PROXY_LIST"
```

::tip
You can automate this even further to:

- add in new proxies from different sites
- let the script validate them
- add the validated ones to the proxychain4 file
- remove old ones if the list reaches more then 33 proxies
  ::

#### **How to Use the Script**

1. **Create a Proxy List File:**

   - Save all your proxies in a file named `proxies.txt` in the same directory as the script.
   - Each proxy should be on its own line in the format:
     ```
     96.10.115.198:80
     154.16.146.41:8080
     ```

2. **Make the Script Executable:**

   ```bash
   chmod +x test_proxies.sh
   ```

3. **Run the Script:**

   ```bash
   ./test_proxies.sh
   ```

4. **Output Example:**
   ```
   Testing proxy: 96.10.115.198:80
   Proxy 96.10.115.198:80 succeeded. IP returned: 96.10.115.198
   --------------------------------
   Testing proxy: 154.16.146.41:8080
   Proxy 154.16.146.41:8080 failed or timed out.
   --------------------------------
   ```

#### **What the Script Does**

- Reads proxies from the file `proxies.txt`.
- Tests each proxy using `curl` with the `-x` option.
- Outputs whether the proxy worked and, if successful, shows the IP it returned.

Feel free to customize the script to save results to a file or add more advanced features!

## **4. Using Proxychains**

To route a command through the proxy chain:

```bash
proxychains <command>
```

### Examples

- Test your public IP
  ```bash
  proxychains curl http://ifconfig.me
  ```
- Scan a website
  ```bash
  proxychains nmap -sT example.com
  ```

### Debugging

Use verbose mode for detailed output:

```bash
proxychains -f /etc/proxychains4.conf curl https://ipinfo.io
```

### **How to Specify the Proxy Type:**

1. **For HTTP or HTTPS:**

   - Use the `http` scheme when testing with `curl`:
     ```bash
     curl -x http://<proxy_ip>:<proxy_port> https://ipinfo.io
     ```
   - For HTTPS, you can test similarly:
     ```bash
     curl -x http://<proxy_ip>:<proxy_port> https://ipinfo.io
     ```

2. **For SOCKS4 or SOCKS5:**

   - Use the appropriate scheme (`socks4` or `socks5`) with `curl`:

     ```bash
     curl -x socks5h://<proxy_ip>:<proxy_port> https://ipinfo.io
     ```

   - `socks5h` ensures that DNS lookups are routed through the proxy.

## **5. Add IP Tables Rules for Safety**

To ensure traffic never bypasses the proxy (even by accident):

1. Block all outbound traffic:

   ```bash
   sudo iptables -P OUTPUT DROP
   ```

2. Allow traffic only to your proxy:

   ```bash
   sudo iptables -A OUTPUT -d <proxy_ip> -p tcp --dport <proxy_port> -j ACCEPT
   sudo iptables -A OUTPUT -d <proxy_ip> -p udp --dport <proxy_port> -j ACCEPT
   ```

3. (Optional) Allow DNS traffic:

   ```bash
   sudo iptables -A OUTPUT -d 8.8.8.8 -p udp --dport 53 -j ACCEPT
   ```

4. Save rules to persist across reboots:
   ```bash
   sudo apt install iptables-persistent
   sudo iptables-save > /etc/iptables/rules.v4
   ```

## **7. Firefox**

You can use Proxychains to route Firefox traffic through your proxy chain and test it on a website like [whatismyip.com](https://www.whatismyip.com).

### **Steps to Test**

#### **1. Install Firefox**

If you don’t already have Firefox installed in your Linux/WSL:

```bash
sudo apt update && sudo apt install firefox -y
```

#### **2. Open Firefox with Proxychains**

Run Firefox through Proxychains:

```bash
proxychains firefox
```

#### **3. Go to `whatismyip.com`**

- In the Firefox browser, visit [https://www.whatismyip.com](https://www.whatismyip.com).
- The website will display the IP address seen by the server.
- If Proxychains is working correctly, this should match the IP of the last proxy in your chain.---

#### **4. Debugging**

If Firefox doesn't load pages or Proxychains fails:

1. Verify your Proxychains configuration (`/etc/proxychains4.conf`).
2. Ensure proxies are functional using `curl` (as shown earlier).
3. Check Proxychains verbose output:
   ```bash
   proxychains -f /etc/proxychains4.conf firefox
   ```

## **Good Practices**

- **Test All Proxies:** Always verify proxies before adding them to the list.
- **Avoid Leaks:** Block traffic outside the proxy using `iptables` or a downstream firewall.
- **Rotate Proxies Regularly:** Free proxies are unreliable—replace them frequently.
- **Combine with Tor:** Use Tor (`socks5 127.0.0.1 9050`) for a reliable base.

**Note:** _All it takes is one packet to bypass the proxy and reveal your real IP. Be diligent—block all non-proxy traffic using `iptables` or a firewall._
