## ⚙️ Command Execution Modes

Ncat kan **externe commands uitvoeren** en network I/O redirecten naar de command's stdin/stdout. Dit maakt vrijwel elke terminal applicatie **network-accessible**.

**Drie execution modes:**

| Mode | Shell Interpretation | Use Case |
|------|---------------------|----------|
| `--exec` | ❌ Direct execution | Full pathname, geen shell features |
| `--sh-exec` | ✅ Via system shell | Shell scripts, pipes, env vars |
| `--lua-exec` | 🌙 Lua interpreter | Cross-platform scripts, portable |

::note
Command execution werkt in **zowel connect als listen mode** — maar in listen mode met `--keep-open` wordt command **per connection** geforked.
::

---

## 🔧 `--exec` — Direct Execution

### Syntax

```bash
ncat -l --exec "<full-pathname> [args]"
```

**Kenmerken:**
- ✅ Direct execution (geen shell interpretation)
- ❌ Geen shell features (pipes, redirects, wildcards)
- ✅ Full pathname vereist (tenzij in `$PATH`)

### Voorbeelden

**Simple echo server:**
```bash
ncat -l --exec "/bin/echo Hello."
```

**Test:**
```bash
$ ncat localhost 31337
Hello.
```

**Date server:**
```bash
ncat -l --exec "/bin/date"
```

**Multi-connection mode:**
```bash
ncat -l --keep-open --exec "/bin/echo Hello."
```

Nu accepteert de server **meerdere connections** — elke client krijgt "Hello." en disconnect.

::tip
Use `--keep-open` om een **inetd-like service** te maken — blijft draaien tot `Ctrl+C`.
::

---

## 🐚 `--sh-exec` — Shell Execution

### Syntax

```bash
ncat -l --sh-exec "<command-string>"
```

**Kenmerken:**
- ✅ Passed naar system shell (`/bin/sh -c` op Unix, `cmd.exe /C` op Windows)
- ✅ Shell features werken (pipes, redirects, env vars)
- ✅ Geen full pathname nodig (als in `$PATH`)

### Voorbeelden

**Current directory server:**
```bash
ncat -l --sh-exec "echo `pwd`"
```

**Output:**
```
$ ncat localhost 31337
/home/user/directory
```

**System info server:**
```bash
ncat -l --sh-exec "uname -a && uptime"
```

**Pipeline voorbeeld:**
```bash
ncat -l --sh-exec "ps aux | grep bash"
```

::important
**Shell injection risk!** Als user input in command komt, kan dit leiden tot command injection. Use `--exec` voor untrusted scenarios.
::

## 🌙 `--lua-exec` — Lua Script Execution

### Syntax

```bash
ncat -l --lua-exec <script.lua>
```

**Kenmerken:**
- ✅ Cross-platform (zelfde interpreter overal)
- ✅ Built-in Lua interpreter in ncat
- ✅ I/O via `print()`, `io.write()`, `io.read()`

### Simple Lua Server

**hello.lua:**
```lua
print("Hello, world!")
```

**Run:**
```bash
ncat -l --lua-exec hello.lua
```

**Test:**
```bash
$ ncat localhost 31337
Hello, world!
```

---

### Interactive Lua Menu

**menu.lua:**
```lua
io.write("=== Menu ===\n")
io.write("1. Option A\n")
io.write("2. Option B\n")
io.write("3. Quit\n")
io.write("Choose: ")

local choice = io.read()

if choice == "1" then
    print("You chose Option A")
elseif choice == "2" then
    print("You chose Option B")
elseif choice == "3" then
    print("Goodbye!")
else
    print("Invalid choice")
end
```

**Run:**
```bash
ncat -l --lua-exec menu.lua
```

::tip
Voor meer Lua examples, zie Nmap's `ncat/scripts/` directory — inclusiv `httpd.lua` voor een simple HTTP server!
::

## 🌐 Environment Variables

Ncat set automatisch **environment variables** in executed programs:

| Variable | Beschrijving |
|----------|-------------|
| `NCAT_REMOTE_ADDR` | IP address van remote host |
| `NCAT_REMOTE_PORT` | Port number van remote host |
| `NCAT_LOCAL_ADDR` | IP address van local interface |
| `NCAT_LOCAL_PORT` | Port number van local listener |
| `NCAT_PROTO` | Protocol in gebruik (`TCP`, `UDP`, `SCTP`) |

### Voorbeeld Script

**env.sh:**
```bash
#!/bin/bash
echo "Remote: $NCAT_REMOTE_ADDR:$NCAT_REMOTE_PORT"
echo "Local: $NCAT_LOCAL_ADDR:$NCAT_LOCAL_PORT"
echo "Protocol: $NCAT_PROTO"
```

**Run:**
```bash
ncat -l --sh-exec "./env.sh"
```

**Output (client connects):**
```
$ ncat localhost 31337
Remote: 127.0.0.1:54321
Local: 127.0.0.1:31337
Protocol: TCP
```

---

### Lua Environment Access

**env.lua:**
```lua
function env(v)
    print(string.format("%s = %q", v, os.getenv(v)))
end

env("NCAT_REMOTE_ADDR")
env("NCAT_REMOTE_PORT")
env("NCAT_LOCAL_ADDR")
env("NCAT_LOCAL_PORT")
env("NCAT_PROTO")
```

**Output:**
```
NCAT_REMOTE_ADDR = "127.0.0.1"
NCAT_REMOTE_PORT = "60179"
NCAT_LOCAL_ADDR = "127.0.0.1"
NCAT_LOCAL_PORT = "31337"
NCAT_PROTO = "TCP"
```

Command execution werkt **ook in connect mode**:

**Execute command bij connect:**
```bash
ncat server.com 4444 --sh-exec "whoami && hostname"
```

**Use case:** Automated connection met command output naar server.

## 🎛️ Multi-Connection Behavior

### Without `--keep-open` (default)

```bash
ncat -l --exec "/bin/date"
```

**Gedrag:**
1. Accept **one connection**
2. Run command
3. **Quit** na disconnect

### With `--keep-open`

```bash
ncat -l --keep-open --exec "/bin/date"
```

**Gedrag:**
1. Accept connection
2. **Fork new process** voor command
3. Keep listening
4. Accept **multiple concurrent connections**
5. Blijft draaien tot manual stop

::important
**Inetd-like behavior:** Met `--keep-open`, ncat forkt een nieuwe handler per connection — werkt als xinetd/inetd replacement voor simple services.
::

## ⚠️ Buffering Gotchas

**Probleem:** Veel programma's bufferen hun I/O:
- Input buffer moet vol zijn voordat processing
- Output buffer moet vol voordat schrijven
- Kan leiden tot **hangende connections**

**Oplossing:**
- Use programma's die **line-buffered** zijn (zoals shell scripts)
- Flush output expliciet in je script
- Test eerst met kleine bestanden/inputs

::note
Buffers worden **geflushed bij end-of-input/output**, dus zelfs buffered programma's werken wanneer hele file tegelijk processed wordt.
::

## 🛡️ Security Warnings

::caution
**Command execution is gevaarlijk:**
- Connecting applicaties aan netwerk maakt local vulnerabilities **remote exploitable**
- Never run untrusted applications via ncat
- Hostile input kan applicatie crashen of exploiten
- Use met voorzichtigheid en alleen in gecontroleerde omgevingen
::

**Best practices:**
- ✅ Only execute **trusted scripts**
- ✅ Validate input wanneer mogelijk
- ✅ Use `--allow` voor IP restrictions (zie hoofdstuk 8)
- ✅ Avoid `--sh-exec` met user-controlled input
- ✅ Run als **non-privileged user**

## 🎯 Praktische Use Cases

### Simple Web Server (Lua)

**httpd.lua** (simplified):
```lua
-- Read HTTP request
local request = io.read("*l")

-- Send response
io.write("HTTP/1.0 200 OK\r\n")
io.write("Content-Type: text/html\r\n")
io.write("\r\n")
io.write("<html><body><h1>Hello from Ncat!</h1></body></html>\r\n")
```

**Run:**
```bash
ncat -l --keep-open --lua-exec httpd.lua 8080
```

**Test:**
```bash
curl http://localhost:8080/
```

---

### Port Knock Detector

**knock-detector.sh:**
```bash
#!/bin/bash
echo "Connection from $NCAT_REMOTE_ADDR:$NCAT_REMOTE_PORT"
logger "Port knock from $NCAT_REMOTE_ADDR"
echo "Access granted"
```

**Run:**
```bash
ncat -l --sh-exec "./knock-detector.sh" 12345
```

---

### Simple Authentication Wrapper

**auth.sh:**
```bash
#!/bin/bash
echo "Password:"
read -s password

if [ "$password" == "secret123" ]; then
    echo "Access granted"
    /bin/bash -i
else
    echo "Access denied"
    exit 1
fi
```

**Run:**
```bash
ncat -l --sh-exec "./auth.sh" 4444
```

::warning
**Not production-ready!** Dit is een simpel voorbeeld — use proper authentication mechanisms in real scenarios.
::

## 📋 Quick Reference — Command Execution

```bash
# ═══════════════════════════════════════════════════
# --EXEC (DIRECT)
# ═══════════════════════════════════════════════════

ncat -l --exec "/bin/echo Hello"
ncat -l --exec "/usr/bin/date"
ncat -l --keep-open --exec "/bin/bash -i"

# ═══════════════════════════════════════════════════
# --SH-EXEC (SHELL)
# ═══════════════════════════════════════════════════

ncat -l --sh-exec "echo `pwd`"
ncat -l --sh-exec "ps aux | grep nginx"
ncat -l --sh-exec "whoami && hostname"

# ═══════════════════════════════════════════════════
# --LUA-EXEC (LUA SCRIPT)
# ═══════════════════════════════════════════════════

ncat -l --lua-exec script.lua
ncat -l --keep-open --lua-exec httpd.lua 8080

# ═══════════════════════════════════════════════════
# ENVIRONMENT VARIABLES
# ═══════════════════════════════════════════════════

# In scripts:
echo $NCAT_REMOTE_ADDR
echo $NCAT_REMOTE_PORT
echo $NCAT_LOCAL_ADDR
echo $NCAT_LOCAL_PORT
echo $NCAT_PROTO

# In Lua:
os.getenv("NCAT_REMOTE_ADDR")

# ═══════════════════════════════════════════════════
# MULTI-CONNECTION MODE
# ═══════════════════════════════════════════════════

ncat -l --keep-open --exec "/bin/date"      # inetd-like
ncat -l --keep-open --sh-exec "whoami"      # Multiple clients
```

::tip
**Debugging tip:** Use `-vv` (double verbose) om te zien wanneer commands worden executed en connections worden accepted.
::
