## 🛡️ Access Control Met Ncat

Een listening ncat kan **restricteren welke hosts mogen connecten** via IP-based filtering:

| Mechanisme | Beschrijving |
|------------|-------------|
| `--allow` | Whitelist — alleen deze IPs toegestaan |
| `--deny` | Blacklist — deze IPs geblokkeerd |
| `--allowfile` | Whitelist uit bestand |
| `--denyfile` | Blacklist uit bestand |
| `--max-conns` | Maximum aantal concurrent connections |

::important
**Host-based access control is NIET foolproof:**
- Kwetsbaar voor IP spoofing attacks
- Source IP kan via proxies/VPNs veranderd worden
- Niet vertrouwen als enige security layer
::

---

## ✅ Whitelist — `--allow`

**Alleen toegestane hosts kunnen connecten:**

### Syntax

```bash
ncat -l --allow <host-specification> <port>
```

**Host specification formats:**
- IPv4 address: `192.168.0.125`
- IPv6 address: `2001:db8::7d`
- Hostname: `trusted.example.com`
- IPv4 CIDR: `192.168.0.0/24`
- IPv4 range: `192.168.0.0-255`
- IPv6 CIDR: `2001:db8::/32`
- Comma-separated list: `192.168.0.1,192.168.0.2,10.0.0.0/24`

---

### Voorbeelden

**Allow één host:**
```bash
ncat -l --allow 192.168.0.125 4444
ncat -l --allow 2001:db8::7d 4444
ncat -l --allow trusted.example.com 4444
```

**Allow subnet (CIDR):**
```bash
ncat -l --allow 192.168.0.0/24 4444
ncat -l --allow 2001:db8::/32 4444
```

**Allow IP range:**
```bash
ncat -l --allow 192.168.0.0-255 4444
ncat -l --allow 10.0.0.1-50 4444
```

**Allow meerdere hosts:**
```bash
ncat -l --allow 192.168.0.1,192.168.0.2,10.0.0.0/24 4444
```

::tip
**Met whitelist worden ALLE andere IPs automatisch denied** — je hoeft geen extra deny rules toe te voegen.
::

---

## ❌ Blacklist — `--deny`

**Blocked hosts kunnen niet connecten:**

### Syntax

```bash
ncat -l --deny <host-specification> <port>
```

### Voorbeelden

**Deny één host:**
```bash
ncat -l --deny 192.168.0.200 4444
ncat -l --deny 2001:db8::c8 4444
```

**Deny subnet:**
```bash
ncat -l --deny 192.168.100.0/24 4444
```

**Deny meerdere hosts:**
```bash
ncat -l --deny 192.168.0.200,10.0.0.50,172.16.0.0/24 4444
```

::note
**Met blacklist worden ALLE andere IPs toegestaan** behalve de specified hosts.
::

---

## ⚖️ Whitelist vs Blacklist — Wanneer Wat?

| Aspect | Whitelist (`--allow`) | Blacklist (`--deny`) |
|--------|----------------------|---------------------|
| **Security** | ✅ Veiliger | ⚠️ Zwakker |
| **Default behavior** | Deny all, allow specified | Allow all, deny specified |
| **Use case** | Known trusted hosts | Block known bad actors |
| **Maintenance** | Easy met kleine trusted set | Kan oneindig groeien |
| **Recommended** | ✅ Productie gebruik | ⚠️ Alleen voor quick blocks |

::important
**Whitelist is veiliger:**
- Attackers kunnen nieuwe IPs gebruiken (blacklist helpt dan niet)
- Whitelist: alleen known-good IPs toegestaan
- Defense-in-depth: use whitelisting waar mogelijk
::

## 📄 File-Based Rules

Voor grote lijsten: use bestanden met één entry per regel.

### `--allowfile`

**trusted-hosts.txt:**
```
192.168.0.125
192.168.0.126
10.0.0.0/24
trusted.example.com
2001:db8::1
```

**Run:**
```bash
ncat -l --allowfile trusted-hosts.txt 4444
```

---

### `--denyfile`

**blocked-hosts.txt:**
```
192.168.100.200
10.0.0.50
172.16.0.0/16
evil.example.com
```

**Run:**
```bash
ncat -l --denyfile blocked-hosts.txt 4444
```

::tip
File formats zijn compatible met Nmap's `-iL` en `--excludefile` options!
::

## ⚔️ Allow vs Deny — Conflict Resolution

**Wat gebeurt er als een host in BEIDE lijsten staat?**

```bash
ncat -l --allow 192.168.0.0/24 --deny 192.168.0.50 4444
```

**Resultaat:**
- Host `192.168.0.50` wordt **DENIED**
- Andere `192.168.0.x` hosts worden allowed

::important
**Deny wins:** Als een host matcht met zowel `--allow` als `--deny`, wordt deze **denied**.
::

## 🔢 Connection Limits — `--max-conns`

Limiteer **maximum aantal concurrent connections**:

### Syntax

```bash
ncat -l --max-conns <number> <port>
```

### Voorbeelden

**Limit 5 connections:**
```bash
ncat -l --max-conns 5 4444
```

**Single connection only:**
```bash
ncat -l --max-conns 1 4444
```

**Default limits:**
- Linux/Unix: **100** concurrent connections
- Windows: **60** concurrent connections

::note
**Use case:** Prevent resource exhaustion attacks — attacker kan geen 1000+ connections openen om server te overloaden.
::

## 🎯 Real-World Scenarios

### Scenario 1: Pentest Team Collaboration

**Situatie:**
Je draait een reverse shell listener, maar wilt **alleen jouw team** toelaten.

**Setup:**
```bash
# Team IPs: 203.0.113.10, 198.51.100.5
ncat -l --allow 203.0.113.10,198.51.100.5 4444
```

**Resultaat:**
- Alleen team members kunnen connecten
- Random internet users worden geblokkeerd
- Client demo blijft privé

---

### Scenario 2: Block Known Scanners

**Situatie:**
Je ziet agressieve port scans van bekende IPs.

**Setup:**
```bash
ncat -l --deny 192.0.2.50,192.0.2.51 8080
```

**Resultaat:**
- Blocked IPs kunnen niet meer connecten
- Normale traffic gaat door

::caution
**Limitation:** Attacker kan eenvoudig van IP switchen. Combine met andere defensive measures.
::

---

### Scenario 3: Internal Service (Localhost Only)

**Situatie:**
Service mag **alleen lokaal bereikbaar** zijn.

**Setup:**
```bash
ncat -l localhost 8080 --allow 127.0.0.1
```

**Alternatief (bind to localhost only):**
```bash
ncat -l localhost 8080
```

**Resultaat:**
- Alleen connections van localhost worden geaccepteerd
- External access geblokkeerd

## 🔍 Testing Access Control

### Test Allowed Host

```bash
# Server:
ncat -l --allow 192.168.0.100 4444

# Client (allowed):
$ ncat -v 192.168.0.100 4444
Ncat: Connected to 192.168.0.100:4444.
```

### Test Denied Host

```bash
# Client (denied):
$ ncat -v server.ip 4444
Ncat: Connection refused.
```

**Verbose server output:**
```
$ ncat -v -l --deny 192.168.0.200 4444
Ncat: Denied connection from 192.168.0.200:54321.
```

## 🛡️ Defensive Perspective

### Detection Indicators

Defenders kunnen access control usage detecteren:

| Indicator | Wat Te Monitoren |
|-----------|-----------------|
| **Rejected connections** | Firewall/IDS logs met "connection refused" |
| **Pattern analysis** | Same source IP trying multiple ports |
| **Ncat processes** | Processes met `--allow`/`--deny` flags |

### Best Practices

**Voor pentesters:**
- ✅ Use whitelist voor team-only access
- ✅ Combine met `--ssl` voor encryption
- ✅ Rotate IPs wanneer mogelijk
- ✅ Use connection limits om DoS te voorkomen

**Voor defenders:**
- ✅ Monitor voor listening services op unusual ports
- ✅ Inspect process arguments (`ps aux | grep ncat`)
- ✅ Alert op ncat/nc usage from workstations
- ✅ Block outbound connections naar unknown IPs

## 📋 Quick Reference — Access Control

```bash
# ═══════════════════════════════════════════════════
# WHITELIST (--allow)
# ═══════════════════════════════════════════════════

ncat -l --allow 192.168.0.125 4444                  # Single IP
ncat -l --allow 192.168.0.0/24 4444                 # Subnet (CIDR)
ncat -l --allow 192.168.0.0-255 4444                # IP range
ncat -l --allow trusted.example.com 4444            # Hostname
ncat -l --allow 2001:db8::/32 4444                  # IPv6 CIDR
ncat -l --allow 192.168.0.1,10.0.0.0/24 4444        # Multiple

# From file:
ncat -l --allowfile trusted.txt 4444

# ═══════════════════════════════════════════════════
# BLACKLIST (--deny)
# ═══════════════════════════════════════════════════

ncat -l --deny 192.168.0.200 4444                   # Single IP
ncat -l --deny 192.168.100.0/24 4444                # Subnet
ncat -l --deny 192.168.0.200,10.0.0.50 4444         # Multiple

# From file:
ncat -l --denyfile blocked.txt 4444

# ═══════════════════════════════════════════════════
# CONNECTION LIMITS
# ═══════════════════════════════════════════════════

ncat -l --max-conns 5 4444                          # Max 5 connections
ncat -l --max-conns 1 4444                          # Single connection

# ═══════════════════════════════════════════════════
# COMBINED USAGE
# ═══════════════════════════════════════════════════

ncat -l --allow 192.168.0.0/24 --deny 192.168.0.50 --max-conns 10 4444
ncat -l --allowfile trusted.txt --max-conns 5 --ssl 4444
```

::warning
**Security reminder:** IP-based access control is **niet foolproof**:
- Combine met encryption (`--ssl`)
- Use proper authentication waar mogelijk
- Monitor logs voor suspicious activity
- Never rely op IP filtering alleen
::
