## 📝 Output Behavior

Ncat is een **proper pipeline utility** — het leest van stdin en schrijft naar stdout:

```bash
# Input van file:
ncat server.com 80 < input.txt

# Output naar file:
ncat server.com 80 > output.txt

# Pipeline:
echo "GET / HTTP/1.0" | ncat -C server.com 80 | grep "200 OK"
```

**Diagnostic messages** gaan naar **stderr** (niet stdout), zodat data stream clean blijft.

::note
**Exception:** Met `--exec`/`--sh-exec` communiceert ncat met subprocess i.p.v. stdin/stdout.
::

## 🔊 Verbose Output

### Basic Verbose (`-v`)

```bash
ncat -v server.com 80
```

**Output:**
```
Ncat: Version 7.94 ( https://nmap.org/ncat )
Ncat: Connected to 10.10.10.5:80.
```

### Extra Verbose (`-vv`)

```bash
ncat -vv server.com 80
```

**Output:**
```
Ncat: Version 7.94 ( https://nmap.org/ncat )
NCAT DEBUG: Using system default trusted CA certificates and those in /usr/share/ncat/ca-bundle.crt.
NCAT DEBUG: Connecting to 10.10.10.5:80...
Ncat: Connected to 10.10.10.5:80.
NCAT DEBUG: Connection requested from 127.0.0.1
```

::tip
**Use `-vv` voor debugging** — toont internal decision making en connection details.
::

## 📄 Text Logging — `--output`

### Syntax

```bash
ncat --output <logfile> [options] <host> <port>
ncat -o <logfile> [options] <host> <port>
```

### Voorbeelden

**SMTP debugging:**
```bash
ncat -C --output smtp-debug.log mail.example.com 25
```

**Log file contents:**
```
220 mail.example.com ESMTP
HELO client.example.com
250 mail.example.com Hello client.example.com
QUIT
221 mail.example.com closing connection
```

**Reverse shell logging:**
```bash
# Attacker (log alle commands):
ncat -l --output session.log 4444
```

::note
**Log bevat ALLES** — zowel sent als received data, zonder differentiation.
::

## 🔍 Hex Dump Logging — `--hex-dump`

Voor **binary protocols** of wanneer je exact bytes wilt zien:

### Syntax

```bash
ncat --hex-dump <logfile> [options] <host> <port>
ncat -x <logfile> [options] <host> <port>
```

### Voorbeeld

**Test SSH server response:**
```bash
$ ncat -C --hex-dump ssh-hex.log scanme.nmap.org 22
SSH-2.0-OpenSSH_4.3
HELO example.com
Protocol mismatch.
```

**ssh-hex.log contents:**
```
[0000] 53 53 48 2D 32 2E 30 2D 4F 70 65 6E 53 53 48 5F SSH-2.0- OpenSSH_
[0010] 34 2E 33 0A 4.3.
[0000] 48 45 4C 4F 20 65 78 61 6D 70 6C 65 2E 63 6F 6D HELO exa mple.com
[0010] 0D 0A ..
[0000] 50 72 6F 74 6F 63 6F 6C 20 6D 69 73 6D 61 74 63 Protocol mismatc
[0010] 68 2E 0A h..
```

**Format:**
- `[0000]` — Byte offset (reset bij nieuwe transmission)
- Hex bytes — Actual data in hexadecimal
- ASCII representation — Rechts (non-printable = `.`)

::tip
**Perfect voor:**
- Binary protocol analysis
- Debugging custom protocols
- Malware traffic analysis
- Zie precies welke bytes worden verstuurd
::

## 📊 Periodic Status Updates

### `--stats-every`

```bash
ncat --stats-every <interval> <host> <port>
```

**Interval formats:**
- `2s` — 2 seconden
- `500ms` — 500 milliseconds
- `1m` — 1 minuut

**Voorbeeld:**
```bash
ncat --stats-every 2s server.com 80
```

**Output (elke 2 sec):**
```
Ncat: 1024 bytes sent, 2048 bytes received in 2.15 seconds.
Ncat: 2048 bytes sent, 4096 bytes received in 4.30 seconds.
```

---

### On-Demand Stats (tijdens scan)

**Press `SPACE` tijdens active connection:**

```bash
$ ncat server.com 80
# Press SPACE
Ncat: 512 bytes sent, 1024 bytes received in 5.42 seconds.
```

::note
Stats zijn handig bij **lange transfers** om progress te monitoren zonder verbose output pollution.
::

## 🐛 Debugging Techniques

### Combine Verbose + Hex Dump

```bash
ncat -vv --hex-dump debug.log --output text.log server.com 80
```

**Resultaat:**
- Verbose info naar **stderr** (console)
- Text log naar `text.log`
- Hex dump naar `debug.log`

---

### Packet Trace — `--packet-trace`

```bash
ncat --packet-trace server.com 80
```

**Output:**
```
SENT (0.0012s) 12 bytes: GET / HTTP/1.0\r\n\r\n
RECV (0.1523s) 256 bytes: HTTP/1.1 200 OK\r\n...
```

Laat **elk verstuurd en ontvangen packet** zien met timestamps.

::important
`--packet-trace` is **zeer verbose** — use alleen voor debugging!
::

## 🔎 Reason Flag — `--reason`

```bash
ncat --reason server.com 80
```

**In port scanning context:**
```
PORT    STATE    REASON
22/tcp  open     syn-ack
80/tcp  closed   reset
443/tcp filtered no-response
```

Verklaart **waarom** ncat beslist dat een port open/closed/filtered is.

::note
Vergelijkbaar met Nmap's `--reason` flag.
::

## 🎯 Logging Best Practices

### Security & Privacy

**⚠️ Log files kunnen sensitieve data bevatten:**
- Credentials (username/password)
- Session tokens
- Private keys
- Personal data

**Best practices:**
```bash
# Secure permissions:
touch session.log
chmod 600 session.log

# Encrypt logs:
ncat --output - server.com 80 | gpg --encrypt > session.log.gpg

# Delete after analysis:
shred -u session.log
```

---

### Forensics & IR

**Voor incident response:**

```bash
# Timestamp + verbose + hex dump:
ncat -vv --hex-dump incident-$(date +%Y%m%d-%H%M%S).log \
     --output incident-text.log \
     suspicious-server.com 4444
```

**Log metadata:**
```bash
# Add context to log:
echo "=== Incident Response Log ===" > ir.log
echo "Date: $(date)" >> ir.log
echo "Source: $(hostname)" >> ir.log
echo "Target: suspicious-server.com:4444" >> ir.log
echo "===========================" >> ir.log

ncat -vv --output ir.log --append suspicious-server.com 4444
```

## 🛠️ Troubleshooting Met Logs

### Problem: Connection Hangs

**Debug:**
```bash
ncat -vv --packet-trace server.com 80
```

**Look for:**
- Lange delays tussen packets
- Missing ACKs
- Retransmissions

---

### Problem: Unexpected Responses

**Debug:**
```bash
ncat --hex-dump response.log server.com 80
```

**Analyze:**
- Binary data in text protocol?
- Wrong encoding (UTF-8 vs ASCII)?
- Protocol version mismatch?

---

### Problem: SSL Handshake Fails

**Debug:**
```bash
ncat -vv --ssl server.com 443
```

**Output will show:**
```
NCAT DEBUG: SSL connection using TLS_AES_256_GCM_SHA384
NCAT DEBUG: Certificate verification failed
```

## 📋 Quick Reference — Output & Logging

```bash
# ═══════════════════════════════════════════════════
# VERBOSE OUTPUT
# ═══════════════════════════════════════════════════

ncat -v server.com 80                  # Verbose
ncat -vv server.com 80                 # Extra verbose

# ═══════════════════════════════════════════════════
# TEXT LOGGING
# ═══════════════════════════════════════════════════

ncat --output log.txt server.com 80
ncat -o log.txt server.com 80

# ═══════════════════════════════════════════════════
# HEX DUMP LOGGING
# ═══════════════════════════════════════════════════

ncat --hex-dump hex.log server.com 80
ncat -x hex.log server.com 80

# ═══════════════════════════════════════════════════
# PERIODIC STATS
# ═══════════════════════════════════════════════════

ncat --stats-every 2s server.com 80    # Every 2 seconds
ncat --stats-every 500ms server.com 80 # Every 500ms

# During connection: Press SPACE for stats

# ═══════════════════════════════════════════════════
# PACKET TRACE
# ═══════════════════════════════════════════════════

ncat --packet-trace server.com 80

# ═══════════════════════════════════════════════════
# COMBINED LOGGING
# ═══════════════════════════════════════════════════

ncat -vv --output text.log --hex-dump hex.log server.com 80
```

::tip
**Productie tip:** Voor pentest engagements, **log altijd sessions** — essentieel voor reporting en legal documentation.
::
