## 🔀 Proxying Met Ncat

Ncat kan fungeren als **proxy client** (connect via proxy) én als **proxy server** (accept proxy requests).

**Ondersteunde proxy types:**
- SOCKS4
- SOCKS5
- HTTP (CONNECT method)

---

## 🌐 Ncat Als Proxy Client

### Basic Syntax

```bash
ncat --proxy <proxyhost>[:<proxyport>] --proxy-type {http|socks4|socks5} <target> <port>
```

**Parameters:**
- `<proxyhost>` — Proxy server hostname/IP
- `<proxyport>` — Proxy port (optioneel, defaults: 1080 voor SOCKS, 3128 voor HTTP)
- `--proxy-type` — Proxy type (default: `http`)

---

### HTTP Proxy

```bash
ncat --proxy proxy.example.com:3128 --proxy-type http target.com 80
```

**Met authentication:**
```bash
ncat --proxy proxy.example.com:3128 --proxy-type http --proxy-auth user:pass target.com 80
```

::note
HTTP proxies ondersteunen **Basic én Digest authentication** — ncat gebruikt Digest in preference (veiliger).
::

---

### SOCKS5 Proxy

```bash
ncat --proxy localhost:1080 --proxy-type socks5 target.com 22
```

**Met authentication (SOCKS5 only):**
```bash
ncat --proxy localhost:1080 --proxy-type socks5 --proxy-auth user:pass target.com 22
```

---

### SOCKS4 Proxy

```bash
ncat --proxy localhost:1080 --proxy-type socks4 --proxy-auth username target.com 80
```

::caution
**SOCKS4 authentication:** Alleen username, geen password (protocol limitation).
::

---

## 🎧 Ncat Als Proxy Server

Ncat kan luisteren als **HTTP proxy server**:

### Basic HTTP Proxy

```bash
ncat -l 3128 --proxy-type http
```

**Supported methods:**
- `GET`, `HEAD`, `POST` (web browsing)
- `CONNECT` (arbitrary TCP tunneling)

**Clients kunnen nu verbinden:**
```bash
# Configure browser/tool to use localhost:3128
curl --proxy http://localhost:3128 http://example.com
```

---

### Met Authentication

```bash
ncat -l 3128 --proxy-type http --proxy-auth user:pass
```

**Authentication schemes:**
- Basic (credentials in cleartext)
- Digest (hashed, replay window van 10 seconden)

::warning
**Security waarschuwing:**
- HTTP Basic auth stuurt credentials **unencrypted**
- Digest is beter maar nog steeds **replay-vulnerable**
- Use alleen in trusted networks of combine met `--ssl`
::

## 🔄 TCP Relay — Pivot Via Compromised Host

### Wat Is TCP Relay?

**TCP relay = middleman voor network traffic**

```
Attacker ──► Compromised Host (relay) ──► Internal Target
```

**Use case:** Je hebt toegang tot één machine inside een netwerk, maar niet tot andere internal systemen. Gebruik die machine als **relay** om traffic door te sturen.

---

### Scenario — Corporate Network Pivot

```
Internet ────► [Firewall] ────► Internal Network
                                    │
                ┌───────────────────┼────────────────┐
                │                   │                │
           Workstation         File Server      Database
          (compromised)       (10.0.0.10)     (10.0.0.20)
          (10.0.0.5)
```

**Situatie:**
- Je hebt shell op `10.0.0.5` (workstation)
- Je wilt toegang tot `10.0.0.10` (file server)
- File server is **niet direct bereikbaar** van internet

**Oplossing:** TCP relay via `10.0.0.5`

---

### TCP Relay Setup

**Op compromised host (10.0.0.5):**
```bash
# Simple relay: forward localhost:8080 naar 10.0.0.10:445 (SMB)
ncat -l localhost 8080 --sh-exec "ncat 10.0.0.10 445"
```

**Op attacker machine:**
```bash
# SSH tunnel naar compromised host
ssh user@10.0.0.5 -L 8080:localhost:8080

# Nu: localhost:8080 op attacker → relay → 10.0.0.10:445
smbclient //localhost/share -p 8080
```

::tip
**Chain meerdere ncats** om multi-hop pivoting te doen — zie hoofdstuk 11 voor advanced chaining.
::

---

### Named Relay (Multi-target)

**Relay script (relay.sh):**
```bash
#!/bin/bash
# Read target from first line
read target port
ncat $target $port
```

**Run relay:**
```bash
ncat -l 8080 --sh-exec "./relay.sh"
```

**Client specifies target:**
```bash
echo "10.0.0.10 445" | ncat relay-host 8080
# Now connected to 10.0.0.10:445 via relay
```

## 🌉 Connection Brokering Voor NAT Bypass

Wanneer **beide hosts achter NAT** zitten:

```
Host1 (NAT) ◄──► Broker (public IP) ◄──► Host2 (NAT)
```

**Broker (public server):**
```bash
ncat -l --broker 31337
```

**Host2 (connects first):**
```bash
ncat broker.com 31337 > received-file
```

**Host1 (connects second):**
```bash
ncat --send-only broker.com 31337 < file-to-send
```

**Data flow:**
Host1 → Broker → Host2

::important
**Volgorde matters:** Receiver moet **eerst** connecten, anders mist data (broker buffert niet).
::

## 🔐 Encrypted Proxy

Combine proxying met SSL:

**HTTPS proxy server:**
```bash
ncat -l --ssl --ssl-cert cert.pem --ssl-key key.pem 3128 --proxy-type http
```

**Client via encrypted proxy:**
```bash
# Note: Niet alle tools ondersteunen SSL proxies
# Use socat of custom tooling
```

::note
**Limitation:** Ncat's `--proxy` client feature ondersteunt geen `--ssl` voor de proxy connection zelf. Use `socat` of SSH tunnels voor encrypted proxy chains.
::

## 🎯 Real-World Pivoting Example

### Scenario: Multi-Hop Network Access

```
Attacker ──► DMZ Server ──► Internal Network ──► Database
           (compromised)     (pivot point)      (target)
```

**Step 1: Establish relay op DMZ server**
```bash
# Op DMZ server (10.10.10.5):
ncat -l 8888 --sh-exec "ncat 192.168.1.10 3306"
```

**Step 2: SSH tunnel naar DMZ**
```bash
# Op attacker:
ssh user@10.10.10.5 -L 3306:localhost:8888
```

**Step 3: Connect naar database via tunnel**
```bash
# Op attacker:
mysql -h localhost -P 3306 -u dbuser -p
# Traffic: attacker → SSH tunnel → DMZ relay → database
```

## 🛡️ Detection & Defense

### Hoe Defenders Proxies Detecteren

| Indicator | Wat Te Monitoren |
|-----------|-----------------|
| **Listening ports** | Unusual ports in listen state (3128, 1080, 8080) |
| **Process behavior** | ncat/nc spawning met `--proxy` flags |
| **Network patterns** | Single host relaying traffic naar meerdere destinations |
| **SOCKS/HTTP traffic** | CONNECT requests naar internal IPs |

**Defensive tips:**
- Monitor voor listening services op uncommon ports
- Inspect processes met network connections
- Alert op SOCKS/HTTP proxy activity van workstations
- Block outbound proxy ports (1080, 3128) tenzij expliciet toegestaan

## 📋 Quick Reference — Proxying

```bash
# ═══════════════════════════════════════════════════
# NCAT ALS PROXY CLIENT
# ═══════════════════════════════════════════════════

# HTTP proxy:
ncat --proxy proxy.com:3128 --proxy-type http target.com 80
ncat --proxy proxy.com:3128 --proxy-auth user:pass target.com 80

# SOCKS5 proxy:
ncat --proxy localhost:1080 --proxy-type socks5 target.com 22
ncat --proxy localhost:1080 --proxy-type socks5 --proxy-auth user:pass target.com 22

# SOCKS4 proxy:
ncat --proxy localhost:1080 --proxy-type socks4 --proxy-auth username target.com 80

# ═══════════════════════════════════════════════════
# NCAT ALS PROXY SERVER
# ═══════════════════════════════════════════════════

# Basic HTTP proxy:
ncat -l 3128 --proxy-type http

# Met authentication:
ncat -l 3128 --proxy-type http --proxy-auth user:pass

# ═══════════════════════════════════════════════════
# TCP RELAY
# ═══════════════════════════════════════════════════

# Simple relay:
ncat -l 8080 --sh-exec "ncat target.com 80"

# Port forwarding:
ncat -l localhost 8080 --sh-exec "ncat 10.0.0.10 445"

# ═══════════════════════════════════════════════════
# CONNECTION BROKERING (NAT BYPASS)
# ═══════════════════════════════════════════════════

# Broker (public server):
ncat -l --broker 31337

# Receiver (connect first):
ncat broker.com 31337 > file

# Sender:
ncat --send-only broker.com 31337 < file
```

::warning
**Ncat's HTTP proxy is for temporary/testing use only:**
- Niet designed voor production internet exposure
- Zwakke authentication mechanisms
- CONNECT method allows **arbitrary TCP traffic** (niet alleen HTTP)
- Use `--allow` om access te restricten
::
