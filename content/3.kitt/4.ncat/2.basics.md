## 🔌 De Twee Modi Van Ncat

Ncat werkt **altijd** in één van twee basis modes:

| Mode | Beschrijving | Socket analogy |
|------|-------------|----------------|
| **Connect Mode** | Initiates connection naar remote service | Client (`connect()` syscall) |
| **Listen Mode** | Wacht op incoming connection | Server (`bind()` + `listen()` syscalls) |

::note
**Denk aan connect mode als "client mode"** en **listen mode als "server mode"**.
::

---

## 📡 Connect Mode — Als Client Verbinden

### Syntax

```bash
ncat <host> [<port>]
```

**Parameters:**
- `<host>` — Hostname of IP address
- `<port>` — Port number (optioneel, default `31337`)

### Voorbeelden

**Verbind met een web server:**
```bash
ncat -C scanme.nmap.org 80
```

**Interactive HTTP request:**
```bash
$ ncat -C scanme.nmap.org 80
GET / HTTP/1.0
Host: scanme.nmap.org

HTTP/1.1 200 OK
Date: Thu, 09 Dec 2025 12:00:00 GMT
Server: Apache/2.4.41
...
```

::tip
De `-C` flag **enforceert CRLF line endings** (`\r\n`), wat vereist is voor veel protocols zoals HTTP, SMTP, en FTP.
::

**Verbind met custom port:**
```bash
ncat example.com 22      # SSH port
ncat example.com 3389    # RDP port
ncat 10.10.10.5 4444     # Custom listener
```

---

## 🎧 Listen Mode — Als Server Luisteren

### Syntax

```bash
ncat --listen [<host>] [<port>]
ncat -l [<host>] [<port>]
```

**Parameters:**
- `<host>` — Bind address (optioneel, default `0.0.0.0` = all interfaces)
- `<port>` — Port number (optioneel, default `31337`)

### Voorbeelden

**Luister op default port (31337):**
```bash
ncat -l
```

**Luister op custom port:**
```bash
ncat -l 4444
```

**Luister op specifiek interface:**
```bash
ncat -l localhost 8080     # Alleen lokaal bereikbaar
ncat -l 192.168.1.100 4444 # Specifiek IP
```

::important
**Poorten < 1024 vereisen root/admin:**
```bash
sudo ncat -l 80    # HTTP port (privileged)
sudo ncat -l 443   # HTTPS port (privileged)
```
::

## 🔁 Default Gedrag — Single Connection

### Standard Listen Mode

```bash
ncat -l 4444
```

**Wat er gebeurt:**
1. Ncat luistert op port 4444
2. **Accepteert ONE connection**
3. Quit na disconnect

**Voorbeeld:**
```bash
# Terminal 1 (listener):
$ ncat -l 4444
Hello from client!       # Data van client
^C                        # Quit na disconnect

# Terminal 2 (client):
$ ncat localhost 4444
Hello from client!       # Type & send
```

### Multi-Connection Mode (`--keep-open`)

**Probleem:** Default mode quit na één connection.
**Oplossing:** Use `--keep-open` (of `-k`)

```bash
ncat -l -k 4444
```

**Wat er gebeurt:**
1. Accepteert **meerdere concurrent connections** (tot connection limit)
2. Alles van server wordt **naar alle clients** gestuurd
3. Blijft draaien tot `Ctrl+C`

::tip
`--keep-open` maakt ncat als een **simple broadcasting server** — alle connected clients ontvangen dezelfde data.
::

## 🌐 Protocols — TCP, UDP, SCTP

### TCP (Default)

**Transmission Control Protocol** — Reliable, connection-oriented

```bash
ncat -l 4444            # TCP listener (default)
ncat target.com 4444    # TCP client
```

**Kenmerken:**
- ✅ **Reliable** — packets komen aan in volgorde
- ✅ **Connection-oriented** — handshake vereist
- ✅ **Error checking** — corrupt data wordt opnieuw verstuurd

### UDP

**User Datagram Protocol** — Unreliable, connectionless

```bash
ncat -u -l 4444         # UDP listener
ncat -u target.com 4444 # UDP client
```

**Kenmerken:**
- ⚠️ **Unreliable** — packets kunnen verdwijnen
- ⚠️ **No handshake** — gewoon data schieten
- ✅ **Lower overhead** — sneller voor bepaalde use cases

::note
**UDP use cases:** DNS queries, VoIP, streaming, TFTP, SNMP monitoring
::

### SCTP

**Stream Control Transmission Protocol** — Modern reliable protocol

```bash
ncat --sctp -l 4444         # SCTP listener
ncat --sctp target.com 4444 # SCTP client
```

**Kenmerken:**
- ✅ **Reliable** zoals TCP
- ✅ **Multi-streaming** (ncat gebruikt TCP-compatible subset)
- ⚠️ Minder vaak gebruikt (vooral telecom: 4G/5G, SS7)

## 🔢 IPv4 vs IPv6

### Default Behavior

Ncat luistert op **beide IPv4 en IPv6** by default, en connect naar resolved addresses in volgorde van OS.

### Force IPv4 Only

```bash
ncat -4 target.com 80
ncat -4 -l 4444
```

### Force IPv6 Only

```bash
ncat -6 target.com 80
ncat -6 -l 4444
```

**IPv6 voorbeeld:**
```bash
# Listener:
ncat -6 -l 4444

# Client:
ncat -6 fe80::1 4444
ncat -6 2001:db8::1 4444
```

::tip
Force `-4` of `-6` wanneer je wilt vermijden dat ncat automatisch switch tussen address families.
::

## 🎛️ Common Options

### Verbose Output

```bash
ncat -v target.com 4444      # Verbose
ncat -vv target.com 4444     # Extra verbose
```

**Output:**
```
Ncat: Version 7.94 ( https://nmap.org/ncat )
Ncat: Connected to 10.10.10.5:4444.
```

### No DNS Resolution

```bash
ncat -n 192.168.1.100 4444
```

Voorkomt DNS lookups — alleen IP addresses.

::tip
**Gebruik `-n` voor snellere scans** wanneer DNS info niet nodig is (zoals bij Nmap `-n`).
::

### Telnet Mode

```bash
ncat --telnet target.com 23
ncat -t target.com 23
```

**Wat het doet:**
- Respond negatief op Telnet protocol negotiations
- Verwijdert binary negotiations uit stream
- Maakt raw Telnet scripts mogelijk

::note
Gebruik `--telnet` om canned Telnet scripts te draaien zonder binary garbage.
::

## 🕒 Timing Options

Alle timing opties accepteren formaat: `30s`, `500ms`, `2m`, `1h`

### Delay Between Operations

```bash
ncat -l --delay 500ms
```

Wacht 500ms tussen elk read/write operation.

### Idle Timeout (Connect Mode Only)

```bash
ncat -i 10s target.com 4444
```

Drop connection als **geen data** voor 10 seconden.

::caution
`--idle-timeout` werkt **niet** in listen mode!
::

### Connection Wait Timeout

```bash
ncat -w 5s target.com 4444
```

Wacht maximaal 5 seconden om connection te establiseren.

**Default:** 10 seconden


**Client:**
```bash
ncat localhost 8080
```

```bash

...
```

::tip
**Debugging tool:** Gebruik ncat om raw protocol interactions te zien — perfect voor:
- HTTP debugging
- SMTP testing
- Custom protocol development
::

---

### Simple Port Check

```bash
ncat -vz target.com 22
```

**Flags:**
- `-v` — Verbose (show connection status)
- `-z` — Zero-I/O mode (scan zonder data te sturen)

**Output:**
```
Ncat: Connected to target.com:22.
Ncat: 0 bytes sent, 0 bytes received in 0.02 seconds.
```

## 📊 TCP vs UDP — Wanneer Wat Gebruiken?

| Use Case | Protocol | Waarom |
|----------|----------|--------|
| **Reverse shells** | TCP | Reliable, interactive |
| **File transfers** | TCP | Garanteert complete delivery |
| **DNS queries** | UDP | Low overhead, connectionless |
| **Port scanning** | TCP/UDP | TCP voor services, UDP voor SNMP/DNS |
| **VoIP/streaming** | UDP | Low latency, packet loss acceptable |

::important
**Defensive tip:** Firewalls blokkeren vaak **inbound TCP** maar laten **outbound UDP** vaker door. Pentesters gebruiken beide protocols om restrictions te testen.
::

## 🎯 Quick Reference Cheat Sheet

```bash
# CONNECT MODE
ncat <host> <port>              # Basic connection
ncat -4 <host> <port>           # IPv4 only
ncat -6 <host> <port>           # IPv6 only
ncat -C <host> <port>           # CRLF line endings
ncat -v <host> <port>           # Verbose output

# LISTEN MODE
ncat -l <port>                  # Single connection
ncat -l -k <port>               # Multi-connection (keep-open)
ncat -l localhost <port>        # Bind to specific interface

# PROTOCOLS
ncat -u <host> <port>           # UDP
ncat --sctp <host> <port>       # SCTP
# (TCP is default)

# TIMING
ncat -w 5s <host> <port>        # Connection timeout
ncat -i 10s <host> <port>       # Idle timeout
ncat --delay 500ms <host> <port> # Delay between ops

# MISC
ncat -n <host> <port>           # No DNS
ncat -t <host> <port>           # Telnet mode
ncat -vz <host> <port>          # Port scan (zero-I/O)
```
