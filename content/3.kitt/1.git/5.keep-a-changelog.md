**Conventional Changelog** is a standardized approach for creating changelogs based on commit messages. It provides a systematic way to document changes by using consistent formats and conventions. Here’s how it generally works:

- **Commits**: Follow a conventional format (e.g., `feat`, `fix`, `chore`) to categorize changes.
- **Changelog**: Automatically generate or manually maintain a changelog based on these commit messages.
- **Versioning**: Aligns with Semantic Versioning to signal changes and their impact on the software.

## Skeleton

```markdown
# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html) which is made possible by [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) and inpsired by [Nuxt](https://github.com/nuxt/nuxt/releases/tag/v3.13.2).

## [initial-release] - 2024-09-16

### 🔥 Performance

- Description of performance improvements.
- Example: Improved caching mechanisms for faster page loads.

### 🩹 Fixes

- Description of bug fixes.
- Example: Fixed issue with user session persistence (#1234).

### 💅 Refactors

- Description of code refactoring and improvements.
- Example: Refactored components to use new design patterns.

### 📖 Documentation

- Updates or additions to documentation.
- Example: Updated setup instructions in the README.

### 🏡 Chore

- Miscellaneous tasks or maintenance updates.
- Example: Cleaned up obsolete configuration files.

### ✅ Tests

- Changes related to testing, including new tests or test improvements.
- Example: Added unit tests for new feature components.

### 🤖 CI

- Updates related to continuous integration, build, or deployment.
- Example: Improved build pipeline to handle new deployment scenarios.

### 📝 Content

- Updates or additions to website content.
- Example: Updated the FAQ section with new user tips.
- Example: Added a blog post about recent feature updates.
- Example: Revised content on the homepage for clarity and engagement.

### ❤️ Contributors

- List of contributors to this release.
- Example:
  - Daniel Roe (@danielroe)
  - Artem Maslov (@Twocram)
  - Paulo Ricardo Severo (@prsevero)
```

This format provides clear descriptions and examples for each category, making the changelog more informative and user-friendly.
