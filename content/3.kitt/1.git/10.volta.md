Volta, developed by Nuxt Lab, is a web application designed to streamline repository and issue management.

## Kanban Board Types

Volta’s Kanban-style board organizes issues into various stages, making it easy to visualize and manage tasks.

1. `Triage`: 
   - **Purpose**: For new issues or incoming tasks that need initial review.
   - **Usage**: Sort and prioritize issues before moving them to more defined stages.

2. `To Do`: 
   - **Purpose**: For tasks that are identified and ready to be worked on but haven’t started yet.
   - **Usage**: Move issues here once they are planned and scheduled for action.

3. `Backlog`: 
   - **Purpose**: For tasks that are identified but not yet prioritized for immediate action.
   - **Usage**: Use this column for issues that are important but not urgent, to be addressed later.

4. `In Progress`: 
   - **Purpose**: For tasks that are currently being worked on.
   - **Usage**: Move issues here when work has started. This helps track ongoing efforts.

5. `Review`: 
   - **Purpose**: For tasks that have been completed and are awaiting review or approval.
   - **Usage**: Use this column for issues that need to be reviewed by team members or stakeholders before finalizing.

6. `Done`: 
   - **Purpose**: For tasks that have been completed and require no further action.
   - **Usage**: Move issues here once they are fully resolved and no longer need attention.

7. `Released`: 
   - **Purpose**: For tasks that have been completed and deployed or released to production.
   - **Usage**: Use this column to track issues that are done and live, indicating they’ve been rolled out.

8. `Cancelled`: 
   - **Purpose**: For tasks that are no longer relevant or have been intentionally stopped.
   - **Usage**: Move issues here if they are no longer needed or if they were mistakenly added.

Each column on Volta’s Kanban board helps you track the progress of tasks and issues from inception to completion, enhancing your project management and workflow efficiency.
