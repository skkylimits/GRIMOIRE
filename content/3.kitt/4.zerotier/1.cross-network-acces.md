## Setting Up ZeroTier and HTTP Server Access between Windows and WSL

### 1. **Install and Configure ZeroTier on Both Machines**

1. **Download and Install ZeroTier** on both the Windows machine and the other device (e.g., another PC or server).
   - Go to [ZeroTier](https://www.zerotier.com/download/) and install the software.

2. **Create a ZeroTier Network**:
   - Log in to your ZeroTier account, create a new network, and note the network ID.

3. **Join Both Devices to the ZeroTier Network**:
   - On both the Windows machine and the second device (or WSL), join the network by running the following command:
     ```shell [terminal]
     zerotier-cli join <network-id>
     ```
   - Authorize both devices in the ZeroTier web admin interface.

4. **Verify Connectivity**:
   - **Test ping** between the two devices using the ZeroTier-assigned IPs.
     - Example:
       ```shell [terminal]
       ping <zerotier-ip-of-other-device>
       ```
   - If both machines respond to pings, the ZeroTier network is correctly set up.

### 2. **Set Up an HTTP Server in WSL**

1. **Start the HTTP Server**:
   - Install and start `http-server` in your WSL environment:
     ```shell [terminal]
     npm install -g http-server
     ```
   - Start the server and bind it to all interfaces:
     ```shell [terminal]
     http-server -a 0.0.0.0 -p 8080
     ```

2. **Verify Local Access**:
   - Test if the HTTP server is accessible from **within** WSL:
     ```shell [terminal]
     curl http://localhost:8080
     ```

### 3. **Set Up Port Forwarding to Access WSL from Windows**

Because WSL is a virtualized environment with its own network stack, you need to forward ports from your Windows host to the WSL instance:

1. **Forward the Port using `netsh`**:
   - Run this command on your Windows host to forward traffic from port `8080` to WSL’s IP address:
     ```shell [terminal]
     netsh interface portproxy add v4tov4 listenport=8080 listenaddress=0.0.0.0 connectport=8080 connectaddress=<wsl-ip-address>
     ```
   - Replace `<wsl-ip-address>` with the actual IP address of WSL. You can get this with:
     ```shell [terminal]
     ip addr show eth0
     ```

2. **Check Port Forwarding**:
   - To ensure the forwarding rule is applied correctly, run:
     ```shell [terminal]
     netsh interface portproxy show all
     ```

3. **Test Access from Windows**:
   - Now try to access the HTTP server from the Windows browser:
     ```shell [terminal]
     http://localhost:8080
     ```

### 4. **Test Access from Other Devices via ZeroTier**

1. **Test Access from Another Device**:
   - Use the ZeroTier IP of the Windows machine and try accessing the HTTP server from another machine on the ZeroTier network:
     ```shell [terminal]
     curl http://<zerotier-ip-of-windows-machine>:8080
     ```

2. **Troubleshoot Access**:
   - If you cannot access the server, proceed with the next steps to check firewall settings and ensure there are no blocks.

### 5. **Configure Firewall Settings on Windows**

1. **Temporarily Disable the Firewall**:
   - To identify if the Windows firewall is causing issues, temporarily disable it:
     ```shell [terminal]
     netsh advfirewall set allprofiles state off
     ```
   - Retry accessing the HTTP server from the other machine. If it works with the firewall off, you’ll need to configure the firewall to allow traffic on port 8080.

2. **Turn the Firewall Back On**:
   - Re-enable the firewall once you've identified the problem:
     ```shell [terminal]
     netsh advfirewall set allprofiles state on
     ```

3. **Create a Firewall Rule to Allow HTTP Traffic**:
   - Create a new inbound firewall rule to allow traffic on port `8080`:
     1. Open **Windows Defender Firewall with Advanced Security**.
     2. Go to **Inbound Rules** and click **New Rule**.
     3. Select **Port** and click **Next**.
     4. Choose **TCP** and specify **8080** for the port.
     5. Select **Allow the connection** and ensure the rule applies to all profiles (Domain, Private, Public).
     6. Name the rule (e.g., `Allow HTTP 8080`) and finish the setup.

4. **Test Access with Firewall Rule**:
   - Now test access again to the HTTP server from the other device using the ZeroTier IP of the Windows machine.

### 6. **Final Verification**

1. **Verify Everything Works**:
   - With the firewall rule in place, confirm that:
     - You can access the HTTP server locally on the Windows machine.
     - You can access the HTTP server from the second machine on the ZeroTier network.

2. **Check for Network Stability**:
   - Ensure that both machines stay connected to the ZeroTier network and remain reachable via their ZeroTier IPs.

## Troubleshooting Tips:

- **Ping Tests**: If you can ping the machines but cannot access the HTTP server, check that the correct ports are open and listening. Run `netstat -an` to verify that `0.0.0.0:8080` is listening on all interfaces in powershell.
    - if `netstat -an` gives `TCP    127.0.0.1:8080         0.0.0.0:0              LISTENING`, you can see from the `127.0.0.1:8080` that it's only listening on locahost.
- **Firewall Rules**: Double-check firewall rules if access is inconsistent. Be sure to allow the specific ZeroTier IP range or the specific port across all profiles. Turn the firewall off to see if that's the problem `netsh advfirewall set allprofiles state off`
