## 🌍 Wat is GeoIP?

**GeoIP** maakt het mogelijk om IP-adressen te koppelen aan geografische locaties. In Wireshark kun je hiermee:

- **Locatie bepalen** - Land, stad, coördinaten van IP-adressen
- **ASN info** - Autonomous System Number (ISP/organisatie)
- **Visualisatie** - IP-locaties op een wereldkaart projecteren
- **Filteren** - Verkeer filteren op geografische criteria

::note
**Use cases voor security:**
- Malware C2 server locaties identificeren
- Brute force aanvallen geografisch traceren
- DDoS attack sources visualiseren
- Data exfiltration naar verdachte landen detecteren
::

## 🛠️ Installeren

### 📦 **1. Download de juiste MaxMind GeoLite2 databases**

1. Ga naar
   **[https://maxmind.com](https://maxmind.com)**
2. Maak een gratis account aan
3. Download deze drie databases:

   * `GeoLite2-Country.mmdb`
   * `GeoLite2-City.mmdb`
   * `GeoLite2-ASN.mmdb`

### 🎯 Belangrijk

**De bestandsnamen moeten exact zo blijven.**
Dus NIET: `GeoLite2-City_20241212.mmdb` → werkt NIET.

### 📁 **2. Zet de databases in een lokale map**

Bijv.:

```
D:\GeoIP\
    ├── GeoLite2-City.mmdb
    ├── GeoLite2-Country.mmdb
    └── GeoLite2-ASN.mmdb
```

Unzippen → `.mmdb` bestanden moeten **los** in de map staan.

### ⚙️ **3. Wireshark configureren voor GeoIP**

Open Wireshark:

### Ga naar:

**Edit → Preferences → Name Resolution**

Scroll naar **MaxMind database directories**
Klik op **Edit → + (Add)**
Selecteer *alleen de map*, dus:

✔️ `D:\GeoIP\`

Niet de bestanden zelf.

::caution
Klik **OK** → sluit het venster → herstart Wireshark.
::

## 🔍 GeoIP Informatie Bekijken

### Packet Details Venster

Open een capture met publieke IP-adressen (private IPs zoals 192.168.x.x hebben geen GeoIP data).

Klik op een packet → expand **Internet Protocol Version 4** of **IPv6**:

```
Internet Protocol Version 4
  ├── Source: 95.213.143.12
  │   ├── Source GeoIP: Russia
  │   │   ├── City: Moscow
  │   │   ├── Country: Russia (RU)
  │   │   ├── Latitude: 55.7522
  │   │   ├── Longitude: 37.6156
  │   │   └── AS Number: 12389 (ROSTELECOM)
  └── Destination: 192.168.1.5
      └── Destination GeoIP: Unknown (private IP)
```

### Beschikbare GeoIP Fields

| Field | Beschrijving | Voorbeeld |
|-------|--------------|-----------|
| `ip.geoip.country` | Land (ISO code) | `RU`, `NL`, `US` |
| `ip.geoip.country_name` | Land (volledige naam) | `Russia`, `Netherlands` |
| `ip.geoip.city` | Stad | `Moscow`, `Amsterdam` |
| `ip.geoip.asnum` | AS Number | `15169` (Google) |
| `ip.geoip.as_org` | AS Organisatie | `GOOGLE`, `AMAZON-02` |
| `ip.geoip.latitude` | Latitude | `55.7522` |
| `ip.geoip.longitude` | Longitude | `37.6156` |

## 🔍 Filteren op GeoIP

### Display Filters

**Filter op land:**
```wireshark
ip.geoip.country == "RU"              # Russische IP-adressen
ip.geoip.country == "CN"              # Chinese IP-adressen
ip.geoip.country == "NL"              # Nederlandse IP-adressen
```

**Filter op stad:**
```wireshark
ip.geoip.city == "Moscow"
ip.geoip.city == "Amsterdam"
```

**Filter op ASN:**
```wireshark
ip.geoip.asnum == 15169               # Google
ip.geoip.asnum == 16509               # Amazon AWS
ip.geoip.asnum == 8075                # Microsoft Azure
```

**Combinatie filters:**
```wireshark
ip.geoip.country == "RU" and tcp.port == 22        # SSH from Russia
ip.geoip.country == "CN" and http                  # HTTP from China
ip.geoip.asnum == 15169 and dns                    # DNS via Google
```

**Exclude bepaalde landen:**
```wireshark
!(ip.geoip.country == "NL")                        # Niet-Nederlandse IPs
!(ip.geoip.country == "US" or ip.geoip.country == "NL")  # Exclude US + NL
```

::note
Je kunt prepared filters ook combineren met `and` / `or` / `not` voor complexe queries.
::

## 📊 GeoIP Kolommen Toevoegen

### Kolommen Maken

Dit is **zeer handig** voor quick analysis - je ziet geografische info direct in de packet list.

**Stappen:**

1. Klik op een packet
2. Expand `Internet Protocol` in packet details
3. Rechterklik op het gewenste field:
   - `ip.geoip.city` → **Apply as Column**
   - `ip.geoip.country` → **Apply as Column**
   - `ip.geoip.asnum` → **Apply as Column**

### Aanbevolen Kolom Setup

Voeg deze kolommen toe voor complete overzicht:

| Kolom Naam | Field | Doel |
|------------|-------|------|
| **Src Country** | `ip.src.geoip.country` | Source land |
| **Src City** | `ip.src.geoip.city` | Source stad |
| **Dst Country** | `ip.dst.geoip.country` | Destination land |
| **ASN** | `ip.geoip.asnum` | AS Number |

**Resultaat in packet list:**

| No. | Time | Source | Destination | Src Country | Src City | Protocol |
|-----|------|--------|-------------|-------------|----------|----------|
| 12 | 0.523 | 95.213.143.12 | 192.168.1.5 | RU | Moscow | TCP |
| 15 | 1.234 | 8.8.8.8 | 192.168.1.5 | US | Mountain View | DNS |
| 23 | 2.567 | 185.220.101.5 | 192.168.1.5 | DE | Frankfurt | HTTPS |

::tip
Kolommen kunnen worden:
- **Herordend** (drag & drop)
- **Hernoemd** (Edit → Preferences → Columns)
- **Opgeslagen** in profiles voor hergebruik
::

## 🎯 Forensics

### Malware C2 Detectie

```wireshark
# Zoek ongebruikelijke landen voor uitgaande connecties
ip.src == 192.168.0.0/16 and !(ip.geoip.dst_country == "The Netherlands" or ip.geoip.dst_country == "United States")
```

* Detecteert **uitgaand verkeer van jouw LAN**
* Dat **niet naar legit / whitelisted landen** gaat
* Perf voor C2 detection, beacons, exfiltration, malware outbound infra

**Waarom:** Als je een Nederlands bedrijf bent en traffic ziet naar Rusland/China zonder goede reden, kan dit C2 communicatie zijn.

### Brute Force Attack Sources

```wireshark
# SSH brute force from not whitelisted countries.
tcp.port == 22 
and tcp.flags.syn == 1 
and !(ip.geoip.dst_country == "The Netherlands" 
      or ip.geoip.dst_country == "United States" 
      or ip.geoip.dst_country == "United Kingdom" 
      or ip.geoip.dst_country == "Germany")
```

* Nieuwe SSH-verbinding pogingen (tcp.flags.syn == 1)
* Naar jouw SSH server (tcp.port == 22)
* Vanuit landen die **niet op jouw whitelist staan**

**Analyse:** Tel het aantal unieke source IPs de wereld die SSH SYN packets sturen.

::caution
Let op!

* **SSH brute force attempts**
* **Botnet scanning**
* **Credential stuffing**
* **Automated attacks tegen jouw server**
* **Masscan / ZMap probes**
* **C2 probing**
::

### Data Exfiltration Detection

```wireshark
# Large uploads naar verdachte landen
ip.geoip.country == "CN" and tcp.len > 1000 and ip.src == 192.168.0.0/16
```

* **Uitgaand LAN-verkeer** (192.168.0.0/16)
* Naar **China** (ip.geoip.country == "CN")
* Met **grote TCP payloads** (meer dan 1000 bytes)


**Waarom:** Grote uploads vanuit je netwerk naar China kunnen data exfiltration zijn.

### Cloud Service Identificatie

```wireshark
# AWS traffic
ip.geoip.as_org contains "AMAZON"

# Google Cloud traffic
ip.geoip.asnum == 15169

# Azure traffic
ip.geoip.asnum == 8075
```

### Tor Exit Node Detection

```wireshark
# Veel Tor exit nodes zitten in specifieke ASNs
ip.geoip.as_org contains "OVH" or ip.geoip.as_org contains "HETZNER"
```

::warning
Dit is geen 100% accurate Tor detection - gebruik dedicated Tor IP lists voor betere resultaten.
::

### Export GeoIP Data

**Via tshark (command-line):**

```bash
tshark -r capture.pcap -T fields \
  -e ip.src \
  -e ip.geoip.src_country \
  -e ip.geoip.src_city \
  -e ip.dst \
  -e ip.geoip.dst_country \
  -E header=y -E separator=, > geoip_export.csv
```

**Resultaat:** CSV met IP + GeoIP info voor verdere analyse (Excel, Python, etc.).
