## Open, save, close, export packets

**🗂️ Open a Capture File**

To open a previously saved capture file:

* Click **File > Open**
* Or press **Ctrl + O**
* Select a `.pcap`, `.pcapng`, or similar file

**💾 Save a Capture File**

To save the current capture:

* Click **File > Save As...**
* Choose a location and filename
* Select file format (usually `.pcapng` or `.pcap`)
* Click **Save**

💡 You can also just hit **Ctrl + S** to quickly save.

**❌ Close a Capture**

To close the current capture file:

* Click **File > Close**
* This clears the capture window but doesn't close Wireshark.

**🎯 Export Packets**

To export packets:

1. Apply a filter (e.g., `ip.addr == 192.168.1.5`)
2. Go to **File > Export Specified Packets**
3. Choose:
   * All packets
   * Displayed packets (filtered)
   * Marked packets only
4. Pick a format and filename
5. Click **Save**

::tip
💡 Handy for sharing only relevant traffic with others.
::

## Compare pcap vs pcapng

::tip
**Good for**: Compatibility with older tools, simple captures
::

📂 **PCAP** (Packet Capture)

* **Older format** (used since the early days of Wireshark/tcpdump)
* Stores packets **sequentially**, with limited metadata
* Does **not support**:

  * Multiple interfaces in one file
  * Comments on packets
  * Time zone info or interface statistics
* **File extension:** `.pcap`

::caution
**Limitations**: No advanced metadata, no multi-interface support
::

📂 **PCAPNG** (PCAP Next Generation)

::tip
**Good for**: Advanced analysis, modern captures, saving extra context
::

* **Newer and more flexible** format (now the default in Wireshark)
* Supports:

  * **Multiple interfaces** in one file
  * **Packet-level comments**
  * **Custom metadata**
  * **Name resolution info**
  * **Interface statistics**
* **File extension:** `.pcapng`

::caution
**Limitations**: Not supported by some older tools
::

**🆚 Summary**

| Feature             | PCAP        | PCAPNG            |
| ------------------- | ----------- | ----------------- |
| Multiple interfaces | ❌ No        | ✅ Yes             |
| Packet comments     | ❌ No        | ✅ Yes             |
| Interface stats     | ❌ No        | ✅ Yes             |
| Time resolution     | Basic       | High              |
| Tool compatibility  | ✅ Very high | ⚠️ Slightly lower |
| File extension      | `.pcap`     | `.pcapng`         |

---

::tip
💡 **Pro Tip**: Unless you need compatibility with very old tools, stick with **pcapng** — it gives you richer data and more analysis power.
::

## Export capture objects

Wireshark can extract certain types of files (like images, documents, executables) that were transferred over the network — for example, via HTTP, SMB, or FTP.

1. **Open Your Capture in Wireshark**

Load your `.pcap` or `.pcapng` file.

2. **Go to:**

* **File** > **Export Objects**
  Then select a protocol:

  * **HTTP** – for web downloads
  * **SMB** – for Windows file shares
  * **FTP** – for file transfers
  * **DICOM**, **TFTP**, etc., depending on the capture

3. **Browse the Export Window**

* You'll see a list of files detected by Wireshark.
* Each entry shows:

  * File name
  * Size
  * Host
  * Content type

4. **Select & Save**

* Click on any file you want to save.
* Press **Save As** to export it to your machine.

**📂 Example:**

Captured a file downloaded over HTTP? You can extract `.jpg`, `.pdf`, `.zip`, etc., if they were fully transferred in the capture.

::caution
You can only extract files **fully captured** in the trace.

* If the capture started mid-transfer or the file is fragmented and not fully reassembled, extraction might not work.
::

## Find specific packets

The **Find Packet** feature helps you search for packets that match specific text, addresses, values, or display filters. Super helpful when you're working with large captures!

🛠️ **Steps to Use It:**

1. **Open your capture file** in Wireshark.

2. Go to the menu:
   **Edit** > **Find Packet**
   Or press **Ctrl + F** (Cmd + F on macOS).

3. A **search bar** appears at the bottom of the window.

🔧 **What You Can Search For:**

* **Display Filter** – use Wireshark filters like `http`, `ip.addr == 192.168.1.1`, `tcp.port == 443`, etc.
* **String** – search for specific text (like a URL or username) in packet content.
* **Hex** – search for specific byte sequences.
* **Packet List / Details / Bytes** – choose where to search (header, detailed view, raw bytes).

✅ **Example Searches:**

| Search Type | Example                   | Use Case                            |
| ----------- | ------------------------- | ----------------------------------- |
| Display     | `http.request`            | Find HTTP request packets           |
| String      | `"example.com"`           | Find packets containing that domain |
| Hex         | `0x16 03 01`              | Find specific TLS handshake bytes   |
| IP Address  | `ip.addr == 192.168.0.10` | Traffic to/from a host              |

::note
**Pro Tip:**

Once you find a matching packet, you can right-click and:

* Follow TCP/HTTP stream
* Mark the packet
* Set time reference
::

## Add packet/file comments

Wireshark lets you add comments to individual packets or the entire capture file. This is super useful for:

📝 **How to Add a Comment to a Packet:**

1. **Right-click** on the packet you want to comment on.
2. Choose **"Packet Comment"**.
3. A text box pops up – type your comment there.
4. Click **OK** – you'll now see a little **speech bubble icon** in the Info column next to the packet.

You can also go to:
**Packet Details pane > Right-click > Packet Comment**

📁 **How to Add a File Comment:**

1. Go to **File** > **File Properties**.
2. In the **“Comment”** field at the bottom, type your notes about the entire capture.
3. Click **OK**.

This comment will be saved with the capture file.

::caution
**Why Use Comments?**

* Keeping notes while analyzing packets
* Useful when sharing captures with others
* Marking important packets for later
::

::tip
Comments are only saved if you use the **PCAPNG** file format – not plain PCAP!
::

## Set/unset time reference

**Time references** help you measure time between packets by setting one as "time zero."

**✅ Set Time Reference:**

* Right-click a packet → **Set Time Reference (toggle)**
* Or select the packet + press **Ctrl+T**
* All other packet times become **relative** to this one

**❌ Unset Time Reference:**

* Right-click the same packet → **Set Time Reference (toggle)** again
* Or go to **Edit > Time Reference > Clear All Time References**

::caution
**Why use it?**

* Measure delays or time between events
* Useful for performance or protocol analysis
::

::tip
Tip: Use **pcapng** format to save time references with your file.
::

## Apply time formats

You can change how **timestamps** appear in the packet list — super helpful for different kinds of analysis.

📍 To Change Time Format:

1. Go to **View > Time Display Format**
2. Choose from formats like:

   * **Date and Time of Day** (`2025-06-05 13:45:22.123`)
   * **Seconds Since Beginning of Capture** (default)
   * **Seconds Since Previous Packet**
   * **Seconds Since Previous Displayed Packet**
   * **Seconds Since First Displayed Packet**

::caution
**Why use different formats?**

* **Date & time**: Useful for correlating with logs
* **Since previous packet**: Great for spotting delays
* **Since start**: Good for absolute timing
::

::tip
These only change how times are displayed — the capture data itself stays the same.
::

## Configure name resolution

**🌐 Name Resolution in Wireshark**

Wireshark can **translate raw addresses (like IPs, MACs, ports)** into **human-readable names** using different types of name resolution:


**🔍 Types of Name Resolution**

| Type                 | What It Resolves            | Example                        |
| -------------------- | --------------------------- | ------------------------------ |
| **Network (IP)**     | IP address → Hostname       | `192.168.1.10 → printer.local` |
| **Transport (Port)** | Port number → Protocol name | `443 → HTTPS`                  |
| **MAC (Ethernet)**   | MAC address → Manufacturer  | `00:1A:2B → Cisco`             |


**⚙️ How to Enable or Disable Name Resolution**

**Temporarily (just in the session):**

* Go to **View > Name Resolution** and check/uncheck:

  * **Resolve Network (IP) addresses**
  * **Resolve Transport names**
  * **Resolve MAC addresses**

**Permanently (via Preferences):**

1. Go to **Edit > Preferences > Name Resolution**
2. Customize resolution options:

   * Enable/disable DNS lookups
   * Use external resolvers or hosts files
   * Control whether name resolution happens during capture or display

::note
* **Name resolution can slow down loading** if you have large captures or slow DNS.
* You can turn it off during analysis to speed things up.
::

## Use 'Decode as'

Sometimes Wireshark doesn’t automatically recognize what protocol a certain packet or port is using. The **“Decode As”** feature lets you **manually tell Wireshark** how to interpret that traffic.

::caution
**🔧 When to Use “Decode As”:**

* A custom application uses **non-standard ports**

  > e.g., HTTPS running on port `8081` instead of `443`
* Protocol detection failed or is ambiguous
* You're analyzing encapsulated or tunneled traffic
::

**✅ How to Use It:**

1. Right-click a packet in the packet list.
2. Select **“Decode As...”**
3. Choose the **correct protocol** from the dropdown menu.
4. Apply your changes — Wireshark will re-parse matching packets accordingly.


**📝 Example:**

* TCP port `8888` is actually carrying **HTTP**.
* Use “Decode As” to force Wireshark to treat traffic on port 8888 as HTTP, so you get proper HTTP analysis.

::tip
**Reversible:**

* You can **clear or modify** your “Decode As” rules at any time.
::

## View capture file properties

Capture File Properties shows a summary of important details about your capture file.

🤔 **How to access:**

* Go to **File** > **Capture File Properties** in Wireshark.

🔍 **What you’ll find:**

* **File Name and Location**
* **Capture Start and End Time**
* **Duration of the capture**
* **Number of Packets captured**
* **Packet size statistics** (min, max, average)
* **Protocols involved**
* **Capture interface details**
* **Capture filter used** (if any)
* **Comments** (if added)

::caution
**Why it’s useful:**

* Quickly understand the scope and context of the capture.
* Verify capture completeness and duration.
* See what kind of traffic was recorded and which interfaces were involved.
* Helps troubleshoot capture issues or prepare for deeper analysis.
::

## Analyze protocol hierarchy


Shows a **breakdown of all protocols** seen in a capture file, organized in a **tree structure by protocol layers**.
It helps you quickly **identify which protocols are present**, see **how much traffic each generates**, and understand the overall **data distribution**.

🤔 **How to access:**

  1. Open your capture file in Wireshark.
  2. Go to the menu: **Statistics** > **Protocol Hierarchy**.
  3. A window opens showing a detailed summary of all protocols seen in the capture.

👁️ **What you’ll see:**

  * Percentage and count of packets per protocol.
  * Nested protocol layers (for example, Ethernet > IP > TCP > HTTP).
  * Helps you spot unexpected protocols or traffic types.

::caution
**Why is this useful?**

* Quickly understand the **network traffic mix**.
* Identify if the expected protocols are actually running.
* Detect unusual or suspicious protocols.
* Get insights for troubleshooting or performance analysis.
::

## Identify conversations

Conversations displays the **main communication pairs** within your capture — for example, the IP addresses or MAC addresses that are talking to each other, along with the protocols like TCP or UDP.


🤔 **How to access it:**

* Go to **Statistics** > **Conversations** in the Wireshark menu.
* Select the tab for the type of conversation you want to see (Ethernet, IPv4, IPv6, TCP, UDP, etc.).
* You’ll get a list of conversations with details like packet count, bytes transferred, duration, etc.

 **How it helps:**

* Spot heavy traffic between specific devices.
* Detect unusual communication patterns that might indicate issues or attacks.
* Focus your analysis on important conversations instead of sifting through all packets.

::caution
**Why use it:**

- It helps you quickly see **who is communicating with whom**, how much data is being sent back and forth, and identify the key traffic flows on the network.
::

## Analyze endpoints traffic


Endpoints lists **all the devices (hosts)** seen in the capture, identified by their IP addresses, MAC addresses, or other protocol-specific identifiers.


🤔 **How to access it:**

* Go to **Statistics** > **Endpoints** in the Wireshark menu.
* Choose the protocol tab you want (Ethernet, IPv4, IPv6, TCP, UDP, etc.).
* You’ll see a list of devices, with details like packet counts, bytes sent/received, and more.

✅ **How it helps:**

* Identify which hosts are the busiest or most active.
* Spot devices that might be causing network issues.
* Analyze traffic patterns per device to focus troubleshooting or security investigations.

::caution
**Why use it:**

- It helps you understand **how much traffic each device is sending or receiving**, giving you a clear overview of all active participants on the network.
::
