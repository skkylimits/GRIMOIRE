## 📥 Installatie van Nmap (Linux / Kali)

### Waarom vanaf source installeren?

* Je krijgt **altijd de nieuwste versie**
* Direct toegang tot **bugfixes en nieuwe features**
* Je kunt functies **aan/uit zetten** voor maatwerk

### Kies folder

De Nmap-source directory bevat waardevolle build-scripts, NSE-engine code en referentiebestanden — handig voor updates en scriptontwikkeling.
Kies daarom **bewust een vaste, logische locatie** voor jouw source repository.

### Vereisten installeren

```bash
sudo apt update && sudo apt install -y \
    subversion \
    build-essential \
    gcc \
    make \
    autoconf \
    automake \
    libtool \
    pkg-config \
    openssl \
    libssl-dev \
    libpcap-dev \
    libssh2-1-dev \
    libpcre2-dev \
    zlib1g-dev
```


::note
OpenSSL is optioneel maar wordt sterk aanbevolen (service detection, hashing, SSL detectie, enz.)
::

### Source Code downloaden

```bash
svn co https://svn.nmap.org/nmap
```

Dit maakt een directory **./nmap/** aan met de volledige broncode.

### Compileren & installeren

```bash
cd nmap
./configure
make
sudo make install
```

```bash
NPING SUCCESSFULLY INSTALLED
make[1]: Leaving directory '/home/skkylimits/nmap/nping'
NMAP SUCCESSFULLY INSTALLED
```

Als je de fout krijgt bij setuptools:

```bash
pip install --upgrade setuptools==57.5.0
```

Daarna opnieuw:

```bash
sudo make install
```


### Which

Wanneer je zowel **apt-Nmap** als een **source build** op je systeem hebt staan, kun je eenvoudig verifiëren welke versie daadwerkelijk wordt uitgevoerd.

#### **Check actieve binary**

```bash
which nmap
```

#### **Bekijk build-details**

```bash
nmap -V
```

#### **Source build — typische output**

```
Nmap version 7.95 ( https://nmap.org )
Compiled with: liblua libpcre2 openssl libssh2
Compiled on: Linux x86_64-unknown-linux-gnu
```

#### **APT-build — typische output**

```
Nmap version 7.93 ( https://nmap.org )
Platform: x86_64-pc-linux-gnu
Compiled with: liblua, openssl, pcre…
```

#### **Waar op letten**

* **Versienummer** → meestal nieuwer bij source
* **Compiled on / Platform** → apt-versies tonen OS-distributie info
* **Gebruikte libraries** → source build vaak vollediger
* **Pad naar binary** → `/usr/local/bin` overheerst `/usr/bin`

### Zenmap starten (als je deze ook wil gebruiken)

```bash
cd zenmap
sudo ./zenmap
```
