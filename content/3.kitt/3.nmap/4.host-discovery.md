## 🔍 Host Discovery (Ping Types)

Het is geen "poortscan", maar slechts een check of de host leef.

### Ping volledig uitschakelen

```bash
nmap -Pn target
```

::tip
Essentieel voor hosts achter firewalls die ICMP blokkeren.
::

### Legacy Ping Scan (`-sP`) — *deprecated*

```bash
nmap -sP target               # Oude host-discovery (deprecated)
nmap -sP target/cidr          # Hele subnetten scannen (verouderde syntaxis)
```

::important
`-sP` is een **legacy-optie**. Sinds Nmap 5.21 is het vervangen door `-sn` (No port scan).
Gebruik **-sn** voor moderne host discovery.
::

### Host Discovery (No Port Scan)
```bash
nmap -sn target               # Moderne vervanger voor -sP
```

**Doet het volgende automatisch:**

* Probeert **meerdere ping-methodes**:

  * ARP (LAN)
  * ICMP Echo
  * ICMP Timestamp
  * TCP SYN ping
  * TCP ACK ping
* *Geen* poortscan daarna

**Het is een *smart*, automatische host discovery.**

::note
Nmap heeft host discovery volledig herbouwd

* `-sn` is duidelijker én consistenter met andere scan types
::

### ICMP Ping

Effectief tegen niet goed geconfigureerde firewalls.

```bash
nmap -PE target              # ICMP Echo → type 8
nmap -PP target              # ICMP Timestamp → type 13
nmap -PM target              # ICMP Netmask → type 17
```

Effectief op lokaal netwerk waar icmp open staat. Veel internet host zijn geconfigureerd om niet te antwoorden op ping vanwege veiligheid.

::warning
Voor ICMP-hostdiscovery zijn root-rechten vereist; zonder root valt Nmap terug op minder betrouwbare TCP-pings.
::

::note
Deze ICMP-opties (-PE, -PP, -PM) zijn de moderne host-discovery methoden en werken voor zowel losse hosts als volledige subnets via CIDR-notatie (bijv. 10.0.0.0/24).
::

#title
🚦 Wat bedoelen we met “ICMP types”?

#content
ICMP-berichten zijn **gestandaardiseerde packetsoorten**, elk met een eigen *type-nummer*.
Dit nummer bepaalt **welk soort ICMP-packet** wordt verstuurd en hoe een host moet reageren.

Bijvoorbeeld:

| ICMP Type | Betekenis            |
| --------- | -------------------- |
| **8**     | Echo Request (ping)  |
| **0**     | Echo Reply           |
| **13**    | Timestamp Request    |
| **14**    | Timestamp Reply      |
| **17**    | Address Mask Request |
| **18**    | Address Mask Reply   |
::

### TCP SYN Ping

```bash
nmap -PS target               # SYN-ping op default poorten (80 en 443)
nmap -PS 80 target            # SYN ping op poort 80
nmap -PS 80,443 target        # SYN-ping op meerdere poorten
nmap -PS 1-1024 target        # SYN-ping op een volledig poortbereik
```

### TCP ACK Ping

```bash
nmap -PA target               # ACK-ping op default poorten (80 en 443)
nmap -PA 80 target            # ACK-ping op poort 80
nmap -PA 80,443 target        # ACK-ping op meerdere poorten
```

### UDP Ping

```bash
nmap -PU target               # UDP-ping op default poort 40125
nmap -PU 53 target            # UDP-ping op DNS-poort
nmap -PU 53,123,161 target    # Typische UDP-diensten: DNS, NTP, SNMP
```

::note
Veel firewalls blokkeren UDP, maar slecht geconfigureerde systems laten 53/161/123 door.
::

### SCTP INIT Ping

```bash
nmap -PY target               # SCTP INIT-ping op default poort 80
nmap -PY 443 target           # INIT-ping op poort 443
nmap -PY 2905 target          # Typische SCTP telecom/SS7 poort
```
::note
SCTP is een protocol naast TCP/UDP, vooral gebruikt in telecom in **4G/5G-telecom, SS7, Diameter, VoIP-infrastructuur**.
::

### ARP Ping (LAN)

```bash
nmap -PR target               # Betrouwbare ARP-ping op lokale netwerken
```

ARP werkt alleen in LAN’s, niet over internet.

::important
ARP scans zijn **altijd accurate** op het lokale netwerk - ARP kan niet geblokkeerd worden door host-firewalls.
::

### IP Protocol Ping

```bash
nmap -PO target               # Probeert standaard: ICMP(1), IGMP(2), IP-in-IP(4)
nmap -PO 1,2,4 target         # Specificeer zelf de IP-protocollen
nmap -PO 6,17 target          # Voorbeeld: TCP(6) & UDP(17)
```
::important
Met `-PO` kun je hosts ontdekken door **IP-protocolpakketten** te sturen in plaats van TCP/UDP of ICMP.
Sommige routers/firewalls filteren **ICMP, TCP en UDP**, maar laten minder gebruikte IP-protocollen ongefilterd door.

* Veel systemen reageren op onbekende of onverwachte protocollen met een **ICMP Unreachable**, wat voldoende is om te bevestigen dat de host *online* is.
* Dit maakt `-PO` een uitstekende methode voor **stealthy host discovery** op streng gefilterde netwerken.
::

### Trace Route

```bash
nmap --traceroute target
```

Nmap’s **traceroute** is krachtiger dan de standaard traceroute omdat het gebruikmaakt van dezelfde scan-technieken die Nmap ook gebruikt (SYN, ACK, ICMP, UDP, etc.).

### Reverse DNS forcing

```bash
nmap -R target               # Always resolve
nmap -n target               # Never resolve (faster). 
```

::tip
Gebruik `-n` voor snellere scans als je DNS info niet nodig hebt.
::

### System DNS Resolver

```bash
nmap --system-dns target
```

::caution
Met `--system-dns` forceert Nmap het gebruik van de **DNS-resolver van het lokale systeem** (zoals `/etc/resolv.conf`) in plaats van Nmap’s eigen, snellere DNS-implementatie.

- **Troubleshooting DNS-problemen**
- **Consistente resolutie binnen één omgeving**
- **Debuggin in enterprise-omgevingen met split DNS, interne zones of resolvers die query’s manipuleren.**
- **Bypasses en detectie.** Soms levert de systeemresolver meer info op.
::

### Custom DNS Servers

```bash
nmap --dns-servers 8.8.8.8 target
nmap --dns-servers 1.1.1.1,8.8.8.8 target
nmap --dns-servers 8.8.8.8 10.0.2.0/24   # Scannen + eigen DNS-resolver
```

::caution
Met `--dns-servers` forceer je Nmap om **specifieke DNS-servers** te gebruiken voor hostname-resolutie, in plaats van de systeemresolver.
Dit is ideaal voor nauwkeurige scans in netwerken waar DNS cruciaal is.

* **Tijdens interne audits waarbij je een interne DNS wil gebruiken**
* **Bij scans waar DNS-resolutie inconsistent of verdacht is**
* **Wanneer je performantieproblemen ziet bij hostname lookups**
* **Stealth of OPSEC**
  Queries gaan **niet langs de standaard DNS-servers**, waardoor jouw scans minder snel traceerbaar zijn
::
