## 🛡️ Firewall Evasion & Obfuscation

### MTU aanpassen

```bash
nmap --mtu 16 target         # Must be multiple of 8
```

::note
`--mtu` forceert Nmap om pakketten op te splitsen in een **specifieke Maximum Transmission Unit (MTU)**.
De waarde *moet een veelvoud van 8 zijn*, omdat IP-fragmentatie werkt in blokken van 8 bytes.

* **Evasion** → Kleine fragmenten kunnen sommige firewalls/IDS moeilijker inspecteren.
* **Protocol testing** → Controleren hoe een host omgaat met IP-fragmentatie.
* **Bypass filters** → Sommige apparaten herkennen een scan niet wanneer pakketten gefragmenteerd zijn.
::

### Decoys (verhult jouw bron-IP)

```bash
nmap -D RND:5 target                      # 5 random decoys
nmap -D 192.168.1.10,192.168.1.20,target  # Specific decoys
```

::caution
Decoys werken niet als:
- ISP filtert spoofed traffic
- Target heeft ingress filtering
- Te veel decoys → network congestion
::

### Idle / Zombie Scan

https://nmap.org/book/idlescan.html

```bash
nmap -sI zombieIP target
```

**Requirements:**
- Zombie host must be truly idle (no other traffic)
- Predictable IP ID sequence generation

::note
Idle scans sturen **geen** packets van jouw IP naar de target - zeer stealth!
::

### Source port spoofing

```bash
nmap --source-port 53 target  # Spoof DNS port
nmap -g 20 target             # Synonym (FTP-data)
```

**Common bypass ports:** 20 (FTP), 53 (DNS), 67 (DHCP), 88 (Kerberos)

### Packet payload vullen

```bash
nmap --data-length 25 target # Append random data
```

::note
Dit verandert **niet** de scanmethode — alleen de *grootte* van het pakket.

* **Evasion** → Anders uitziende pakketgroottes kunnen IDS/IPS-regels ontwijken die op vaste packet sizes matchen.
* **Fingerprint obfuscation** → Maakt jouw scan minder herkenbaar voor defensieve systemen.
::

### Random host order

```bash
nmap --randomize-hosts target
```

::note
Deze optie laat Nmap de scanvolgorde van hosts willekeurig maken in plaats van sequentieel (1, 2, 3, 4…).

* **IDS-evasion** → Intrusion Detection Systems letten op scans die hosts in nette volgorde afgaan. Random volgorde maakt patroonherkenning veel moeilijker.
* **Stealth** → Vermindert de kans dat een beveiligingssysteem jouw scan als een enkele campagne detecteert.
* **Load balancing** → Handig wanneer sommige hosts trager reageren; random distributie voorkomt vertraging door slechte hosts vroeg of laat in de reeks.
::

### MAC Spoofing

```bash
nmap --spoof-mac 0 target                  # Random MAC
nmap --spoof-mac AA:BB:CC:DD:EE:FF target  # Specific MAC
nmap --spoof-mac Apple target              # Vendor name
```
::note
Met `--spoof-mac` laat je Nmap pakketten verzenden alsof ze afkomstig zijn van een **ander MAC-adres**.
::

### Bad checksums

```bash
nmap --badsum target
```

::important
**Je gebruikt `--badsum` om verborgen firewalls/IDS te ontmaskeren die anders niet zichtbaar zouden zijn.**
::
