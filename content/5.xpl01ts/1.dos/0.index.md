## Types of DoS Attacks

### **Network Layer Attacks**

1. **Single-source DoS Attack**

   - A **basic DoS attack** originates from a single source. This attacker overwhelms a system with traffic or requests, preventing normal access to legitimate users.

   - _Example:_ **Ping Flood**, where the attacker sends an overwhelming number of ICMP echo request packets (ping) to a target system to exhaust its resources.

2. [**Distributed Denial of Service (DDoS) Attack**](/xpl01ts/dos/ddos)

   - A **DDoS attack** is a more powerful version of a DoS attack where the traffic comes from multiple sources (often thousands of compromised machines forming a botnet).

   - _Example:_ **DNS Amplification Attack**, where attackers use DNS servers to amplify the amount of traffic sent to the target.

3. **Amplification Attacks**

   - These attacks leverage open servers (e.g., DNS or NTP) to send a large volume of traffic to a victim. Attackers send small queries to these servers, which return large responses to the victim, amplifying the traffic.

   - _Example:_ **NTP Amplification Attack**, where attackers use the Network Time Protocol to amplify traffic toward the victim.

4. **Reflection Attacks**

   - In these attacks, the attacker sends forged requests to a server that then reflects the response to the victim. The attack appears as if the victim is making the request, but in reality, it's the attacker redirecting the traffic.

   - _Example:_ **DNS Reflection Attack**, where a DNS server sends responses to a spoofed victim’s IP address, resulting in overwhelming traffic.

### **Volumetric Attacks**

1. **UDP Flood**

   - Floods the target with User Datagram Protocol (UDP) packets, consuming bandwidth and resources at the network layer (Layer 4).

2. **ICMP/Ping Flood**

   - Overwhelms the system with ICMP echo request (ping) packets at the network layer (Layer 3), consuming bandwidth and resources.

3. **DNS Amplification**
   - Uses DNS servers at Layer 4 to increase traffic volume by sending small queries and receiving large responses to overwhelm the network.

### **Protocol-based Attacks**

1. **SYN Flood**

   - Exploits the TCP handshake process at Layer 4 by sending a barrage of SYN (synchronize) packets without completing the handshake, overwhelming the network resources.

2. **Smurf Attack**

   - Spoofs the target's IP address and sends ICMP requests to a network broadcast address at Layer 3, flooding the target with the replies.

3. **Fragmentation Attack**
   - Sends fragmented packets at the network layer that cannot be reassembled, consuming network resources and causing a denial of service.

### **Application Layer Attacks**

1. **HTTP Flood**

   - Sends numerous HTTP requests to the web server at Layer 7, consuming server resources and bandwidth.

2. **Slowloris Attack**

   - Opens many slow connections to a web server at Layer 7, consuming server resources without closing them.

3. **R-U-Dead-Yet (RUDY)**
   - Slowly sends POST requests with large payloads at Layer 7 to exhaust the server's resources.

## Additional Types of DoS Attacks:

1. **Ping of Death**

   - The attacker sends malformed or oversized ping packets to a target, causing a buffer overflow and crashing the system. _(Network Layer - Layer 3)_

2. **Teardrop Attack**

   - The attacker sends fragmented packets that can't be reassembled properly, causing the system to crash due to the mismanaged memory. _(Network Layer - Layer 3)_

3. **LAND Attack**

   - The attacker sends a spoofed packet where the source and destination addresses are the same, confusing the target system and causing it to crash or become unresponsive. _(Network Layer - Layer 3)_

4. **Resource Exhaustion**

   - The attacker uses up the target’s CPU, memory, or disk resources through sustained, intensive requests, leaving the system unable to process legitimate traffic. _(Layer 7 and Network Layer)_

5. **Zero-day DDoS Attack**

   - These attacks exploit previously unknown vulnerabilities in the target system or software to cause a denial of service. Zero-day exploits are particularly dangerous because they have no immediate patch or defense. _(Any Layer)_

6. **Sockstress Attack**

   - An attack that keeps TCP connections in a half-open state at Layer 4, consuming the server's memory and connection tables until it runs out of resources and cannot respond to legitimate requests. _(Protocol Layer - Layer 4)_

7. **Flooding with Spoofed Requests**

   - The attacker uses spoofed IP addresses to send massive numbers of fake requests at the network and protocol layers, overloading the target while making it difficult to block specific addresses. _(Network and Protocol Layer - Layer 3/4)_

8. **Application DDoS with CAPTCHA Bypass**

   - Attackers automate bots to bypass CAPTCHA systems, overwhelming a target application (e.g., a website login) at Layer 7 by brute-forcing legitimate access systems, causing resource exhaustion. _(Application Layer - Layer 7)_

9. **SSL/TLS-based Attacks**
   - Attacks like **SSL renegotiation attacks** or **HTTPS floods** target the cryptographic handshake in SSL/TLS sessions, overloading a web server's encryption-decryption capacity at Layer 7. _(Application Layer - Layer 7)_
