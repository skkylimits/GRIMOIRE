# 1 вҖ” Wat is *binary* (bits & bytes)

* **Bit** = binary digit, waarde `0` of `1`.
* **Byte** = 8 bits. Meest gebruikte eenheid.
* Grotere: kilobyte (KB), megabyte (MB), etc. (contextueel: 1024 vs 1000).

### Waarom belangrijk?

Alles in een computer is uiteindelijk een reeks bits. In security / reverse engineering moet je vaak binaire representaties lezen (netwerkheaders, file formats, machine code).

# 2 вҖ” Binair вҶ’ Decimaal (stap-voor-stap)

Elke positie is een macht van 2, beginnend rechts bij (2^0).

Voorbeeld: binaire `1101011`

Reken stap-voor-stap uit:

* bit posities (van rechts naar links): 0,1,2,3,4,5,6
* waarden: (1\cdot2^6 + 1\cdot2^5 + 0\cdot2^4 + 1\cdot2^3 + 0\cdot2^2 + 1\cdot2^1 + 1\cdot2^0)
* bereken: (1\cdot64 + 1\cdot32 + 0 + 1\cdot8 + 0 + 1\cdot2 + 1\cdot1 = 64+32+8+2+1 = 107)

Dus `1101011вӮӮ = 107вӮҒвӮҖ`.

---

# 3 вҖ” Decimaal вҶ’ Binair (division-by-2 methode)

Voor 107:

* 107 /2 = 53 remainder 1 (LSB)
* 53 /2 = 26 remainder 1
* 26 /2 = 13 remainder 0
* 13 /2 = 6 remainder 1
* 6 /2 = 3 remainder 0
* 3 /2 = 1 remainder 1
* 1 /2 = 0 remainder 1 (MSB)

Lees remainder van onder naar boven: `1101011`.

---

# 4 вҖ” Hexadecimal (hex) вҖ” handig bij binaire weergave

* Hex gebruikt 4 bits per hex-digit (nibble).
* 1 hex-digit = waarden 0..15 вҶ’ `0..9, A..F`.
* Voorbeeld: `0x6B` = `6` is `0110`, `B` is `1011` вҶ’ gezamenlijke bits `01101011` = 107.

Gebruik hex omdat het compact binaire data toont en makkelijk naar bytes te lezen is.

---

# 5 вҖ” Twos-complement (negatieve getallen)

Veel systemen gebruiken two's-complement om negatieve integers te representeren.

8-bit voorbeeld:

* `+5` = `00000101`
* Neem bij `-5`: invert bits вҶ’ `11111010` then add 1 вҶ’ `11111011`.
* `11111011` (8-bit) = -5 in two's-complement.

Belangrijk: teken bit is impliciet in two's-complement; overflow regels volgen vaste patronen.

---

# 6 вҖ” Endianness (byte order)

* **Little-endian**: laagste significante byte eerst (Intel x86).

  * e.g. 32-bit waarde `0x12345678` in memory (bytes): `78 56 34 12`.
* **Big-endian**: hoogste byte eerst (network byte order).

  * `12 34 56 78`.

Network protocols gebruiken **network byte order = big-endian**. In C moet je soms `htonl()` / `ntohl()` gebruiken.

---

# 7 вҖ” Bitwise operators вҖ” praktische hulpmiddelen

* AND `&` вҖ” maskeren: `x & mask`.
* OR `|` вҖ” zetten van bits.
* XOR `^` вҖ” togglen.
* NOT `~` вҖ” invert bits.
* Shift `<<` `>>` вҖ” vermenigvuldigen/delen door 2^n of bit-plaatsing.

Voorbeeld: zet bit 3 (index vanaf 0) van `flags`:

```c
flags |= (1 << 3);     // set
flags &= ~(1 << 3);    // clear
flags ^= (1 << 3);     // toggle
```

# 8 вҖ” Binary en C: waarom C leren?

* C geeft directe controle over geheugen en bytes вҖ” essentieel om binaire structuren te maken/lezen (headers, network packets, structs in files).
* Veel low-level tools en OS-componenten zijn in C geschreven вҶ’ begrijpen ervan helpt bij reverse engineering.


# 10 вҖ” IP-adressen & netwerken вҖ” bits first!

IP (IPv4) is eenvoudig om in binaire termen te begrijpen: het is 32 bits, meestal weergegeven als 4 octets (dotted decimal).

Voorbeeld IP: `192.168.10.5`

Converteer elk octet:

* 192 вҶ’ `11000000`
* 168 вҶ’ `10101000`
* 10  вҶ’ `00001010`
* 5   вҶ’ `00000101`

Samen: `11000000.10101000.00001010.00000101`

---

## Subnetmasker & CIDR

* `/24` betekent dat de eerste 24 bits zijn **netwerk**. Mask: `255.255.255.0` вҶ’ binair `11111111.11111111.11111111.00000000`.
* Network address = IP & mask
* Broadcast address = network_address | (~mask)

### Voorbeeld berekening

IP: `192.168.10.5` (/24)

* Mask `/24` = `255.255.255.0`
* Network = `192.168.10.0`
* Broadcast = `192.168.10.255`
* Host range = `192.168.10.1` вҖҰ `192.168.10.254`

### Oefening: /26

IP `10.0.0.73/26`

* /26 вҶ’ mask `255.255.255.192` вҶ’ laatste 6 bits host.
* Network: behoud eerste 26 bits van IP, host bits zero:

  * 10.0.0.73 binair (last octet 73 = `01001001`)
  * mask last octet = `11000000` (192)
  * Network last octet = `01000000` = 64 вҶ’ Network = `10.0.0.64`
* Broadcast = network | (~mask last octet) = `01000000 | 00111111` = `01111111` = 127 вҶ’ broadcast `10.0.0.127`
* Hosts: `10.0.0.65` вҖҰ `10.0.0.126` (64 = network, 127 = broadcast)

# 11 вҖ” IP in C: parse & bit-ops (voorbeelden)

Gebruik `inet_pton` / `inet_ntop` voor conversies netwerk <-> presentation.

```c
#include <stdio.h>
#include <arpa/inet.h>
#include <stdint.h>

int main() {
    const char *ip_str = "10.0.0.73";
    uint32_t ip_bin;
    inet_pton(AF_INET, ip_str, &ip_bin); // ip_bin in network byte order (big-endian)

    uint32_t ip_host = ntohl(ip_bin); // convert to host order so shifts make sense
    printf("IP host-order hex: 0x%x\n", ip_host);

    uint32_t mask = 0xFFFFFFFF << (32 - 26); // /26 mask in host order
    uint32_t network = ip_host & mask;
    uint32_t broadcast = network | ~mask;

    struct in_addr tmp;
    tmp.s_addr = htonl(network);
    printf("Network: %s\n", inet_ntoa(tmp));
    tmp.s_addr = htonl(broadcast);
    printf("Broadcast: %s\n", inet_ntoa(tmp));
    return 0;
}
```

## рҹ§© Waarom 8 bits вҖңgenoegвҖқ leek

## рҹ§  Waarom 2вҒ° = 1 (en niet 0)

Er is een belangrijk verschil tussen:

* **вҖңmacht nulвҖқ** вҶ’ `2вҒ°`
* **вҖңkeer nulвҖқ** вҶ’ `2 Г— 0`

Dat zijn **totaal verschillende dingen.**

---

### рҹ”№ 1. Herhaling van vermenigvuldiging

Een macht betekent вҖңhoe vaak je iets met zichzelf vermenigvuldigtвҖқ.

Bijvoorbeeld:

| Macht | Berekening | Resultaat |
| :---- | :--------- | :-------- |
| 2В№    | 2          | 2         |
| 2ВІ    | 2 Г— 2      | 4         |
| 2Ві    | 2 Г— 2 Г— 2  | 8         |

Elke keer dat de exponent stijgt met 1,
vermenigvuldig je het vorige getal met 2.

---

### рҹ”№ 2. Wat gebeurt er als we omlaag gaan?

Kijk nu wat er gebeurt als we *delen* i.p.v. vermenigvuldigen:

| Macht | Berekening | Resultaat |
| :---- | :--------- | :-------- |
| 2Ві    | 8          | 8         |
| 2ВІ    | 4          | 8 Г· 2     |
| 2В№    | 2          | 4 Г· 2     |
| 2вҒ°    | ?          | 2 Г· 2 = 1 |

Zie je het patroon?
Elke stap omlaag = delen door 2 вҶ’ dus 2вҒ° = **1**.

---

### рҹ”№ 3. En negatief dan?

Hetzelfde patroon gaat verder:

| Macht | Berekening | Resultaat |
| :---- | :--------- | :-------- |
| 2вҒ°    | 1          |           |
| 2вҒ»В№   | 1 Г· 2      | 0.5       |
| 2вҒ»ВІ   | 1 Г· 4      | 0.25      |

---

### рҹ”№ 4. In woorden

> `2вҒ° = 1`
> omdat als je telkens deelt door 2 (in plaats van vermenigvuldigt),
> je uiteindelijk op **1** uitkomt вҖ” het neutrale startpunt van vermenigvuldiging.

En alleen bij `2 Г— 0` krijg je natuurlijk `0`,
maar dat is iets hГ©Г©l anders: dat is gewoon een gewone vermenigvuldiging, geen macht.

# Binary вҖ” complete study note (based on *Binary Explained in 01100100 Seconds*)

## Quick overview (what the video teaches)

* **Binary** is the base-2 positional numbering system (only digits `0` and `1`). Computers use binary because physical devices (transistors) naturally have two stable states (on/off). ([YouTube][1])
* A **bit** is a single binary digit; a **byte** is 8 bits (the most common grouping). Multi-byte widths (16, 32, 64 bits) extend the numeric range. ([YouTube][1])
* Text is stored by mapping characters to numeric codes вҖ” e.g. **ASCII** maps letters to byte values (e.g. `'A'` вҶ’ `01000001`). ([ChatTube][2])

---

## 1 вҖ” Number systems & positional notation (short)

* Decimal (base 10): positions represent powers of 10.
* Binary (base 2): positions represent powers of 2.
* Example: binary `1011вӮӮ` = (1В·2^3 + 0В·2^2 + 1В·2^1 + 1В·2^0 = 8 + 0 + 2 + 1 = 11_{10}). ([PBS][3])

---

## 2 вҖ” Why computers use binary (physical/machine reason)

* Transistors are switches: theyвҖҷre naturally ON or OFF вҶ’ map easily to `1` and `0`.
* Building logic (gates) and memory from two-state devices is robust and scalable; this is why digital electronics use binary. ([YouTube][1])

---

## 3 вҖ” Bits, bytes & sizes

* 1 bit = 0 / 1
* 1 byte = 8 bits вҶ’ can represent `0..255` unsigned.
* Common sizes: 16-bit (0..65535), 32-bit, 64-bit. Older systems were 8/16-bit; modern CPUs use 64-bit registers. ([YouTube][1])

**Exercise:** what is the maximum unsigned value of 12 bits?
Answer: (2^{12}-1 = 4095).

---

## 4 вҖ” Binary вҶ” Decimal conversion (procedures + examples)

### Binary вҶ’ Decimal (weights)

Write out bit weights and sum where bits = 1.

Example: `1101011вӮӮ`
Positions (rightвҶ’left): (2^0,2^1,2^2,2^3,2^4,2^5,2^6)
Value: (1В·2^6 +1В·2^5 +0В·2^4 +1В·2^3 +0В·2^2 +1В·2^1 +1В·2^0 = 107).

### Decimal вҶ’ Binary (divide by 2 method)

Take 107:

* 107 /2 вҶ’ q=53 r=1 (LSB)
* 53 /2 вҶ’ q=26 r=1
* 26 /2 вҶ’ q=13 r=0
* 13 /2 вҶ’ q=6 r=1
* 6 /2 вҶ’ q=3 r=0
* 3 /2 вҶ’ q=1 r=1
* 1 /2 вҶ’ q=0 r=1 (MSB)
  Read remainders bottomвҶ’top: `1101011`.

## 5 вҖ” Hexadecimal вҖ” compact binary representation

* 1 hex digit = 4 bits (nibble).
* Example: `0x6B` вҶ’ `6` = `0110`, `B` = `1011` вҶ’ byte = `01101011` вҶ’ decimal 107.
  Hex is used everywhere in debugging, reversing and network dumps. ([Instructables][4])

---

## 6 вҖ” ASCII and storing text

* ASCII assigns numbers to characters: `'A'` = 65 = `01000001вӮӮ`.
* Text strings in files are sequences of bytes, each byte representing a char per an encoding (ASCII, UTF-8 uses multiple bytes per codepoint for non-ASCII). ([Khan Academy][5])

**Quick example:** the string `"Hi"` вҶ’ bytes `0x48 0x69` вҶ’ binary `01001000 01101001`.

---

## 7 вҖ” Signed integers & twoвҖҷs complement (why used)

* TwoвҖҷs complement is standard to represent signed integers. For an N-bit value:

  * Range signed: (-2^{N-1}) .. (2^{N-1}-1).
* To get negative: invert bits and add 1.

Example: `-5` in 8-bit:

* `+5` = `00000101`
* invert вҶ’ `11111010`
* add 1 вҶ’ `11111011` (twoвҖҷs complement representation). ([PBS][3])

---

## 8 вҖ” Endianness (byte order) вҖ” why it matters

* **Little-endian** (Intel): least significant byte stored first. e.g. `0x12345678` вҶ’ bytes `78 56 34 12`.
* **Big-endian** (network order): most significant byte first (used by many network protocols).
  When parsing binary protocols or file formats you must be aware which byte order is used. ([PBS][3])

---

## 9 вҖ” Bits in networks: IPv4 example (ties to your earlier net work study)

* IPv4 = 32 bits. Dotted decimal is four octets.
  Example `192.168.10.5` вҶ’ binary `11000000.10101000.00001010.00000101`.
  CIDR `/24` means first 24 bits network (mask `255.255.255.0`). Calculate network & broadcast by bitwise AND/OR with mask. (This ties into earlier C/Python examples you asked for.) ([YouTube][1])

---

## 10 вҖ” How binary is used inside encryption, hashing & encoding (short recap)

* AES, SHA, XOR-based obfuscation etc. all operate on bits/blocks (e.g. AES operates on 128-bit blocks; SHA-256 on 512-bit blocks internally). Understanding bitwise ops (XOR, shifts, rotations) is essential for analyzing crypto primitives and for reversing. ([YouTube][1])

## 11 вҖ” Practical tools & quick commands

* `xxd -g 1 file` or `hexdump -C file` вҖ” inspect raw bytes.
* `python` quick experiments:

```python

```
