## Wat is Hex Encoding?

`Hex encoding` is **geen wiskundige bewerking**,  
maar een **weergavevorm** van binaire data in base-16.  

Het is gemaakt om **bytes leesbaar te maken voor mensen**.  
Elke byte (8 bits) wordt opgedeeld in **twee hex-symbolen**:

```text
Byte → Hex
00000000 → 00
11111111 → FF
````

Dat is alles — geen compressie, geen encryptie — enkel een visuele vertaling.

> 💡 Hex encoding verandert *niet de data zelf*,
> alleen *hoe wij die data zien*.

---

## Waarom hex encoding?

Computers denken in bits, mensen niet.
Een reeks van 10 miljoen nullen en enen is onmogelijk te lezen.
Daarom gebruiken we hex encoding als **compacte representatie** van ruwe bytes.

```text
Binary: 01001000 01100101 01101100 01101100 01101111
Hex:    48 65 6C 6C 6F
Text:   Hello
```

Elke hex-digit representeert 4 bits → 2 hex = 1 byte.
Zo kan elke byte als 2 tekens worden weergegeven.

---

## Hex vs Binary vs Text

| Weergave          | Bits     | Leesbaar voor mens | Gebruik           |
| ----------------- | -------- | ------------------ | ----------------- |
| **Binary**        | 01001000 | Nee                | CPU / Machinecode |
| **Hex**           | 0x48     | Ja (technisch)     | Debugging / Dumps |
| **ASCII / UTF-8** | H        | Ja (taal)          | Tekstvoorstelling |

> 🧠 **Hex encoding** is dus de *tussenlaag* tussen binaire data en tekst:
> een brug tussen wat de machine ziet en wat wij begrijpen.

---

## Hoe werkt hex encoding

Stel we willen het woord `"Hi"` omzetten naar hex:

1️⃣ Schrijf het in **bytes** via ASCII of UTF-8:

| Karakter | ASCII (dec) | Binair   | Hex |
| -------- | ----------- | -------- | --- |
| H        | 72          | 01001000 | 48  |
| i        | 105         | 01101001 | 69  |

2️⃣ Combineer de bytes:

```text
Hex output → 48 69
```

3️⃣ Optioneel prefix `0x` bij elk byte:

```text
0x48 0x69
```

## In de praktijk

### 🔹 In shellcode

```c
unsigned char Shellcode[] = {
  0xFC, 0x48, 0x83, 0xE4, 0xF0, 0xE8, 0xC0, 0x00, 0x00, 0x00
};
```

Elke `0x..` is één byte in **hexadecimale vorm**.
De CPU ziet het als binaire machine-instructies,
maar wij lezen het als hex zodat we patronen herkennen.

> 💡 Shellcode ≠ tekst.
> De hex-encoding is alleen een **menselijke notatie van ruwe bytes**.

---

### 🔹 In files

Open eender welk bestand met een hex-viewer (zoals `xxd` of `hexdump`):

```bash
xxd -g 1 file.bin | head
```

Je ziet dan:

```
00000000: 89 50 4E 47 0D 0A 1A 0A  PNG....
00000008: 00 00 00 0D 49 48 44 52  ....IHDR
```

De bytes aan de linkerkant zijn hex —
de tekst rechts is de ASCII-interpretatie (indien afdrukbaar).

---

### 🔹 In netwerken

Data wordt vaak ge-encoded voor transmissie:
maar bij **packet-capture tools** (zoals Wireshark) zie je weer hex.

```
0000  00 11 22 33 44 55 08 00 27 66 88 99 08 00 45 00
0010  00 28 6F 22 40 00 40 06 A6 EC C0 A8 00 02 C0 A8
```

Elke hex-paar = één byte uit het TCP/IP-frame.

---

## Hex encoding ≠ Base64

Veel mensen halen hex en Base64 door elkaar — maar ze zijn totaal verschillend.

| Eigenschap  | Hex Encoding                | Base64                        |
| ----------- | --------------------------- | ----------------------------- |
| **Basis**   | 16                          | 64                            |
| **Ratio**   | 2 hex = 1 byte              | 4 base64 = 3 bytes            |
| **Doel**    | Leesbare byte-representatie | Teksttransport via e-mail/web |
| **Gebruik** | Debugging, shellcode, dumps | SMTP, JSON, MIME, URLs        |

> 💡 Hex is *ruw en exact*.
> Base64 is *compact en transport-vriendelijk*.

---

## Endianness en hex

Wanneer je hex gebruikt om **meerdere bytes** te representeren (bijv. een 32-bit integer),
moet je weten in welke **bytevolgorde** (endianness) ze staan.

Voorbeeld: het getal `0x12345678`

| Volgorde          | Hex bytes   |
| ----------------- | ----------- |
| **Big endian**    | 12 34 56 78 |
| **Little endian** | 78 56 34 12 |

x86-processoren (Windows, Intel, AMD) gebruiken **little endian**,
terwijl netwerkprotocollen (TCP/IP) **big endian** gebruiken.

## Tools & conversie

### 🔹 Command-line (Linux/macOS/WSL)

```bash
xxd -p file.bin         # print raw hex
xxd -r -p hex.txt > out.bin   # convert hex → bytes
hexdump -C file.bin
```

### 🔹 PowerShell

```powershell
[BitConverter]::ToString([System.Text.Encoding]::UTF8.GetBytes("Hi"))
# 48-69
```

### 🔹 Python

```python
b = b"Hello"
b.hex()                  # '48656c6c6f'
bytes.fromhex('48656c6c6f')  # b'Hello'
```

## In malware & reverse engineering

🔸 **Strings** – herkend als hexpatronen (`41 42 43 00` → “ABC”)
🔸 **File headers (magic bytes)** – `89 50 4E 47` → PNG
🔸 **Signatures** – antivirussen matchen hex-reeksen
🔸 **Patchen** – bytes aanpassen via hex editors (`BE EF DE AD`)
🔸 **Hashes** – resultaten als hex (`SHA256: 9f2b…`)

```text
41 00 42 00 43 00 → UTF-16LE wide string "ABC"
25 50 44 46 → "%PDF"
```

> 🧬 Hex is de microscoop van reverse engineering:
> je kijkt rechtstreeks in het DNA van binaire data.
