## Wat is een word size?

Een `word size` is de **natuurlijke datagrootte** waarmee een processor werkt.
Het bepaalt hoeveel bytes de CPU in één keer kan lezen, schrijven of berekenen.

* `8-bit` → 1 byte tegelijk (oude microcontrollers, early consoles)
* `16-bit` → 2 bytes tegelijk (eerste PC’s)
* `32-bit` → 4 bytes (standaard tot ±2010)
* `64-bit` → 8 bytes (moderne CPU’s en besturingssystemen)

::important
Elke CPU heeft interne **registers** — kleine geheugencellen die één woord groot zijn.  
Hun breedte bepaalt hoe “breed” de CPU denkt.
::

## Waarom het belangrijk is

* **Geheugenbereik:** een 32-bit CPU kan `2^32` adressen aan (≈ 4 GB).
* **Snelheid:** grotere woorden → minder leescycli voor grote getallen.
* **Compatibiliteit:** software en OS moeten op dezelfde woordgrootte gebouwd zijn.

## Geheugenlimieten

| **Architectuur** | **Wiskundige formule**                    | **Maximaal adresseerbaar geheugen**    |
| ---------------: | :---------------------------------------- | :------------------------------------- |
|          `8-bit` | `2^16 = 65.536 bytes`                     | **≈ 64 KB**                            |
|         `16-bit` | `2^20–2^24`                               | **≈ 1–16 MB**                          |
|         `32-bit` | `2^32 = 4.294.967.296 bytes`              | **≈ 4 GB**                             |
|         `64-bit` | `2^64 = 18.446.744.073.709.551.616 bytes` | **≈ 16 exabyte (16 EB)**               |
|        `128-bit` | `2^128 ≈ 3.4 × 10^38 bytes`               | **Onvoorstelbaar groot (theoretisch)** |

---

::caution

**Observatie**

- Zelfs de krachtigste AI-servers van vandaag gebruiken **1–12 TB RAM**,
terwijl 64-bit systemen **16 miljoen terabytes** kunnen adresseren —
we benutten dus nog niet eens **één miljoenste** van hun theoretische capaciteit.
::

## 📖 Analogie

Stel je een snelweg voor:

* De **breedte van de weg** = *word size* (hoeveel auto’s naast elkaar kunnen rijden).
* De **lengte van de weg** = *adresruimte* (hoe ver de weg loopt → hoeveel geheugen bereikbaar is).

Een 64-bit CPU heeft dus een **8-byte brede snelweg**,
maar die snelweg kan **tot 16 exabyte ver** doorlopen. 🚗🚗🚗

## Waarom niet meteen 64-bit (of 128-bit)?

Elke sprong (8 → 16 → 32 → 64 bit) vereist **volledige systeemvernieuwing** —
CPU, geheugen, moederbord, OS, drivers én software moeten meeschalen.

* **`8-bit`** → genoeg voor eenvoudige apparaten, games en tekst.
* **`16-bit`** → bracht meer geheugen en eerste besturingssystemen (MS-DOS, vroege Windows).
* **`32-bit`** → maakte multitasking, grafische interfaces — standaard in de jaren ’90–2000.
* **`64-bit`** → werd pas nodig toen RAM > 4 GB ging (video, 3D, virtualisatie).

::important
**Waarom nog geen `128-bit`?**
- Omdat 64-bit systemen al **ver boven huidige behoeften** liggen —
`2^64` adressen = **16 exabyte geheugenruimte**,
terwijl de grootste servers vandaag amper terabytes gebruiken.
::

**Te vroeg:** hardware zwaarder, software groter, geen winst.  
**Te laat:** geheugenlimieten.

::note
De overstap naar 64-bit kwam toen het nodig was —
de overstap naar 128-bit komt pas als we écht meer dan 16 exabyte nodig hebben.
::
