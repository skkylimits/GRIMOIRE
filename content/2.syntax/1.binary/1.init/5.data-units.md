> “Before we can encode information, we must first measure it.”

Computers leven in een wereld van `bits`, `bytes` en `woorden`.  

Zodra we data gaan opslaan of verzenden, hebben we **eenheden van grootte** nodig —  
en daar begint de verwarring:  
🧮 is 1 kilobyte nou 1000 bytes of 1024 bytes?

Laten we dat één keer goed helder maken.

## Alles begint met bits

De kleinste eenheid van informatie is de **bit**.  
Een bit heeft maar twee mogelijke waarden: `0` of `1`.

Elke extra bit verdubbelt het aantal mogelijke combinaties:

| Bits | Mogelijke waarden |
|------|--------------------|
| 1 | 2 |
| 2 | 4 |
| 3 | 8 |
| 4 | 16 |
| 5 | 32 |
| 6 | 64 |
| 7 | 128 |
| 8 | 256 |

Formule:  
`n bits → 2ⁿ mogelijke waarden`

## Bits en Bytes

- `1 bit` → kleinste informatie-eenheid (`0` of `1`)
- `1 byte` → groep van **8 bits**

```

8 bits = 1 byte

```

De byte werd de standaard omdat computers meestal rekenen met blokken van 8 bits  
(past bij CPU-registers, geheugenadressering en bestandsstructuren).

## Van Bytes naar Grotere Blokken

Een **byte** is handig om één karakter of klein stukje data op te slaan —  
maar computers werken met **gigantische hoeveelheden bytes**.

Om dat begrijpelijk te maken, groepeerden ingenieurs bytes in **vaste blokken**.

| Term | Betekent | Uitleg |
|------|-----------|--------|
| `KiB` | Kibibyte | 1024 bytes — een klein blok data |
| `MiB` | Mebibyte | 1024 × 1024 bytes — een groter blok |
| `GiB` | Gibibyte | 1024 × 1024 × 1024 bytes — nog groter |
| `TiB` | Tebibyte | 1024 × 1024 × 1024 x 1024 bytes — nog een stap groter |

> Zo ontstond een hiërarchie:  
> **bit → byte → kibibyte → mebibyte → gibibyte → tebibyte...**

## Waarom precies 10 bits = 1024

Computers denken niet in tientallen, maar in **machten van 2**.

Kijk wat er gebeurt bij 10 bits:

```

1111111111

```

Omgezet naar decimaal:

```

(2¹⁰) - 1 = 1023

```

Er zijn dus **1024 combinaties (0–1023)**.  
Daarom staat `2¹⁰ = 1024` voor één binaire “kilostap” — het dichtst bij 1000.


### Waarom niet 1000?

Omdat een computer geen begrip heeft van “tientallen”.  
Hij verdubbelt steeds:

```

2 → 4 → 8 → 16 → 32 → 64 → 128 → 256 → 512 → 1024

```

## Hoe computers tellen

Een CPU, geheugencontroller of netwerkinterface werkt puur met **bits en bytes**.  
Voor de hardware is het getal gewoon “aantal bytes”.

```

Er is geen ‘KiB’ of ‘MiB’ in de hardware.
Alleen: ‘je krijgt 4194304 bytes.’

```

::important
Computers tellen in bytes omdat dat hun **adreseenheid** is:  

- Een `geheugenadres` wijst naar één `byte`.
- Een `blok` bestaat uit meerdere opeenvolgende geheugenadressen.

Dus: **veel adressen samen = één blok.**
::

### Waarom groeperen in blokken?

Hoewel computers in bytes tellen, verwerken ze data bijna nooit één byte tegelijk.  
Dat zou te traag zijn.  
Daarom werken ze in **vaste blokken** — groepen bytes die in één keer gelezen of geschreven worden.

Je kunt het vergelijken met het lezen van een boek:
> Je leest niet letter voor letter, maar hele woorden of zinnen tegelijk.

Zo doet een computer dat ook:
- Meerdere bytes samen = **één “woord”**  
- Meerdere woorden samen = **één “blok”**

## Waarom steeds ×1024?

Elke keer dat je ×1024 doet, voeg je 10 binaire bits toe aan je adresruimte.  
Dat sluit perfect aan bij hoe hardware werkt.

| Schaal | Macht van 2 | Bytes | Betekenis |
|--------|--------------|--------|------------|
| Byte | 2⁰ | 1 | Basiseenheid |
| KiB | 2¹⁰ | 1,024 | 10 bits extra |
| MiB | 2²⁰ | 1,048,576 | 20 bits extra |
| GiB | 2³⁰ | 1,073,741,824 | 30 bits extra |
| TiB | 2⁴⁰ | 1,099,511,627,776 | 40 bits extra |

Elke stap omhoog = 10 extra binaire bits.  
De **machten van 2** sluiten exact aan op de binaire logica van adressen en registers.

## Verschillende niveaus van data

| Eenheid | Betekent | Rol |
|----------|-----------|-----|
| `1 bit` | één binaire toestand (`0` of `1`) | atoom van data |
| `1 byte` | 8 bits samen | bouwsteen van data |
| `1 KiB` | 1024 bytes samen | blok van data |

### Mentale beeldspraak
- Een **byte** is één letter (zoals `A`) → 8 bits.  
- Een **KiB** is een pagina tekst → 1024 letters.  

Een byte beschrijft *inhoud binnenin*,  
een KiB beschrijft *hoeveelheid buitenom*.

## Twee meetsystemen: SI vs Binary

Er bestaan twee “talen” om datagroottes uit te drukken:

| Systeem | Gebruikt door | Basis | Vermenigvuldiger |
|----------|----------------|--------|------------------|
| **SI (decimaal)** | opslagfabrikanten, marketing | 10 | `1 kB = 1000 bytes` |
| **Binaire (IEC)** | besturingssystemen, techniek | 2 | `1 KiB = 1024 bytes` |

De SI-standaard zegt:  
> “Kilo = 1000, Mega = 1 000 000.”

Opslagfabrikanten gebruiken dat, want hun cijfers lijken groter.  
Besturingssystemen rekenen binaire (macht van 2).

## De “verloren megabytes”

Je koopt een `1 TB`-schijf.
De fabrikant rekent in SI:

```
1 TB = 1000^4 bytes = 1,000,000,000,000 bytes
```

Maar jouw besturingssysteem (bijv. Windows of Linux) rekent in binaire stappen:

```
1 TiB = 1024^4 bytes = 1,099,511,627,776 bytes
```

Geen data verdwenen — alleen een andere **rekenbasis**.
