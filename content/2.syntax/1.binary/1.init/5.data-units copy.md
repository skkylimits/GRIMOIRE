---

## ⚙️ 1. Basisdefinities

| Eenheid | Formule                           | Factor |
| ------- | --------------------------------- | ------ |
| 1 `KiB` | `1024 bytes`                      | 1024¹  |
| 1 `MiB` | `1024 × 1024 bytes`               | 1024²  |
| 1 `GiB` | `1024 × 1024 × 1024 bytes`        | 1024³  |
| 1 `TiB` | `1024 × 1024 × 1024 × 1024 bytes` | 1024⁴  |

---

## 🧮 2. Stap-voor-stap berekening (met rekenmachine)

### 🔸 1 byte → KiB

```
1 KiB = 1024 bytes
```

Toets:
`1 × 1024 = 1024`

---

### 🔸 KiB → MiB

```
1 MiB = 1024 KiB
```

Toets:
`1 × 1024 = 1024`
(of rechtstreeks: `1024 × 1024 = 1,048,576 bytes`)

---

### 🔸 MiB → GiB

```
1 GiB = 1024 MiB
```

Toets:
`1024 × 1024 × 1024 = 1,073,741,824 bytes`

---

### 🔸 GiB → TiB

```
1 TiB = 1024 GiB
```

Toets:
`1024 × 1024 × 1024 × 1024 = 1,099,511,627,776 bytes`

---

## 🧩 3. Samenvattend overzicht (alle tussenstappen uitgerekend)

| Eenheid | Vermenigvuldig met | Bytes totaal      | In woorden                 |
| ------- | ------------------ | ----------------- | -------------------------- |
| `1 KiB` | `1024¹`            | 1,024             | duizendvierentwintig bytes |
| `1 MiB` | `1024²`            | 1,048,576         | ruim 1 miljoen bytes       |
| `1 GiB` | `1024³`            | 1,073,741,824     | ruim 1 miljard bytes       |
| `1 TiB` | `1024⁴`            | 1,099,511,627,776 | ruim 1 biljoen bytes       |

## 💡 4. Omgekeerd rekenen (met deling)

Wil je bytes terug omrekenen naar grotere eenheden?

Gebruik gewoon **delen door 1024** per stap:

```
Bytes → ÷1024 → KiB → ÷1024 → MiB → ÷1024 → GiB → ÷1024 → TiB
```

Bijvoorbeeld:

**2 147 483 648 bytes ÷ 1024 ÷ 1024 ÷ 1024 = 2 GiB**

---

## ⚡ 5. Kleine geheugensteun

```
1 KiB = 2¹⁰ bytes
1 MiB = 2²⁰ bytes
1 GiB = 2³⁰ bytes
1 TiB = 2⁴⁰ bytes
```

Dus telkens +10 exponent (2¹⁰ = 1024).
Handig als je binair wilt denken:

* elke 10 bits ≈ 1 factor 1024.

## 🧩 1️⃣ Bit en Byte — het fundament

* **1 bit** = kleinste informatie-eenheid → `0` of `1`
* **1 byte** = **8 bits** → bijvoorbeeld `01000001` (= 'A' in ASCII)

Tot hier:

```
1 byte = 8 bits
```

✔️ Dat is altijd waar.

---

## 🧩 2️⃣ Wat is dan een “kilobyte”?

Er bestaan **twee definities**, en dáár zit de verwarring:

| Naam               | Symbool | Vermenigvuldiger | Gebaseerd op | Uitleg                                    |
| ------------------ | ------- | ---------------- | ------------ | ----------------------------------------- |
| **Kilobyte (kB)**  | `kB`    | 1 000            | 10³          | “SI” (decimaal) – de *menselijke* variant |
| **Kibibyte (KiB)** | `KiB`   | 1 024            | 2¹⁰          | “binaire” variant – de *computer*variant  |

---

## ⚙️ 3️⃣ Waarom computers 1024 gebruiken

Computers rekenen **niet in machten van 10**, maar in **machten van 2**, omdat alles bits zijn (0 of 1).
Een byte heeft 8 bits → 2⁸ combinaties.
Daarom is het logisch om opslag in machten van 2 te verdelen:

```
1 KiB = 2¹⁰ bytes = 1024 bytes
1 MiB = 2²⁰ bytes = 1 048 576 bytes
1 GiB = 2³⁰ bytes = 1 073 741 824 bytes
```

---

## 🧠 4️⃣ Mentaal concept (zo moet je het voelen)

> 🧱 Elk hoger niveau is een macht van 2 groter dan het vorige.
> Elke keer doe je × 1024 (niet × 1000),
> want 1024 = 2¹⁰ = één "blok" van 10 binaire stappen.

Visueel:

```
1 bit  → 0 of 1
8 bits → 1 byte
1024 bytes → 1 KiB
1024 KiB   → 1 MiB
1024 MiB   → 1 GiB
1024 GiB   → 1 TiB
```

---

## 🧮 5️⃣ Waarom mensen “1000 bytes” zeggen

De SI (metrische) standaard zegt:

> “Kilo = 1000, Mega = 1 000 000.”

Opslagfabrikanten gebruiken dat — hun cijfers lijken zo groter:

* 1 TB (SI) = 1 000 000 000 000 bytes
* maar een computer rekent 1 TB = 1 099 511 627 776 bytes

→ daarom zie je op je pc altijd iets van “931 GB” terwijl je een 1 TB-schijf kocht.

| Eenheid        | Factor | Binaire macht | Bytes         | Bits          |
| -------------- | ------ | ------------- | ------------- | ------------- |
| bit            | –      | –             | 1/8           | 1             |
| byte           | 8 bits | 2³            | 1             | 8             |
| kibibyte (KiB) | × 1024 | 2¹⁰           | 1 024         | 8 192         |
| mebibyte (MiB) | × 1024 | 2²⁰           | 1 048 576     | 8 388 608     |
| gibibyte (GiB) | × 1024 | 2³⁰           | 1 073 741 824 | 8 589 934 592 |

---

💡 **Mentale truc:**

> Elke keer dat je drie nullen verwacht (zoals kilo, mega, giga),
> denk in plaats daarvan aan **10 binaire nullen** (2¹⁰ = 1024).

Dus ja:
`1 kilobyte (binaire definitie) = 1024 bytes`,
en omdat `1 byte = 8 bits`,
is `1 kilobyte = 1024 × 8 = 8192 bits`.

---

## 🧩 1️⃣ Byte en Kibibyte zijn *verschillende niveaus*

Ze horen wel bij elkaar in de hiërarchie,
maar ze beschrijven **niet hetzelfde soort ding**.

| Eenheid  | Betekent                          | Rol                |
| -------- | --------------------------------- | ------------------ |
| `1 bit`  | één binaire toestand (`0` of `1`) | atoom van data     |
| `1 byte` | 8 bits samen                      | bouwsteen van data |
| `1 KiB`  | 1024 bytes samen                  | blok van bytes     |

Dus:

* Byte = “inhoud van 8 bits”
* KiB = “groep van 1024 bytes”

Ze zijn **stapjes in schaal**, niet hetzelfde concept.

---

## ⚙️ 2️⃣ Je kunt het zien als dozen-in-dozen

```
1 bit = één korrel zand
8 bits (1 byte) = één steentje
1024 bytes (1 KiB) = één handvol steentjes
```

Elke laag bouwt voort op de vorige,
maar het zijn **andere meeteenheden**.

Een byte zegt *hoeveel bits erin zitten*,
een KiB zegt *hoeveel bytes je bundelt*.

---

## 💡 3️⃣ Belangrijk onderscheid

| Vergelijking                                                | Relatie            | Uitleg                                                                |
| ----------------------------------------------------------- | ------------------ | --------------------------------------------------------------------- |
| Bit ↔ Byte                                                  | 1 byte = 8 bits    | interne structuur                                                     |
| Byte ↔ KiB                                                  | 1 KiB = 1024 bytes | schaal / hoeveelheid                                                  |
| → Ze hebben niets direct “inhoudelijks” met elkaar te maken |                    | De één beschrijft *formaat binnenin*, de ander *hoeveelheid buitenom* |

---

## 🧠 4️⃣ Mentale beeldspraak

Stel:

* Een **byte** is één letter (zoals `A`) → 8 bits.
* Een **KiB** is een pagina tekst → 1024 letters.

De pagina (KiB) bestaat uit 1024 keer die ene lettergrootte (byte),
maar de betekenis van “letter” en “pagina” zijn verschillend.

---

✅ **Kort gezegd:**

> 1 byte = 8 bits
> 1 KiB = 1024 bytes
> De 8 en de 1024 hebben niets met elkaar te maken —
> ze horen bij verschillende schaalniveaus van data.

bits → bytes → KiB → MiB → GiB (elk niveau als bakje-in-bakje).
Dat helpt enorm om te *zien* hoe ze op elkaar bouwen. Wil je dat?

Perfecte vraag 🔥 — dit is de *essentie* van waarom computers 1024 gebruiken.
We gaan het stap voor stap in **pure binary logica** afleiden,
zodat je precies ziet *waar 1024 vandaan komt*. 👇

---

## 🧩 1️⃣ Alles begint met bits

Computers rekenen met bits:

```
0 of 1
```

Elke **bit** verdubbelt het aantal mogelijke combinaties.

| Bits   | Mogelijke waarden (combinaties) |
| ------ | ------------------------------- |
| 1 bit  | 2 → (0, 1)                      |
| 2 bits | 2² = 4                          |
| 3 bits | 2³ = 8                          |
| 4 bits | 2⁴ = 16                         |
| 5 bits | 2⁵ = 32                         |
| 6 bits | 2⁶ = 64                         |
| 7 bits | 2⁷ = 128                        |
| 8 bits | 2⁸ = 256                        |

Dus:
`n bits → 2ⁿ mogelijke waarden`

---

## ⚙️ 2️⃣ Waarom “rond” we af op 1024

Wij mensen denken graag in 10-tallen (10, 100, 1000).
Computers denken in **machten van 2** (2, 4, 8, 16, 32, 64...).

Kijk nu welke macht van 2 het **dichtst bij 1000** ligt:

```
2⁹ = 512
2¹⁰ = 1024
```

1024 ligt *heel dicht bij 1000*,
dus programmeurs zeiden vroeger al snel:

> “Laten we 2¹⁰ = 1024 ‘1 kilobyte’ noemen.”

Het is een *binaire afronding* van “ongeveer 1000”.

---

## 🧠 3️⃣ Zo komt 1024 direct uit de binaire exponenten

Elke stap omhoog = +10 in exponent (base 2):

| Eenheid | Macht van 2 | Waarde              |
| ------- | ----------- | ------------------- |
| 1 byte  | 2⁸ bits     | 256 combinaties     |
| 1 KiB   | 2¹⁰ bytes   | **1024 bytes**      |
| 1 MiB   | 2²⁰ bytes   | 1 048 576 bytes     |
| 1 GiB   | 2³⁰ bytes   | 1 073 741 824 bytes |

Dus:

> **1 KiB = 2¹⁰ bytes = 1024 bytes**

Dat is letterlijk uit de binary exponent 2¹⁰ afgeleid.

---

## 🔢 4️⃣ Waarom precies 10 bits = 1024

Stel je hebt 10 bits (in binary):

```
1111111111
```

Als je dat omzet naar decimal:

```
(2¹⁰) - 1 = 1023
```

Er zijn 1024 mogelijke waarden (0 t/m 1023).
Dus 10 bits kan **1024 combinaties** representeren.
➡️ Daarom gebruiken we 10 bits als “één blok van duizend (ongeveer)”.

---

## 📚 5️⃣ Zo denken computers intern

Elke keer dat je een adres, index of offset toevoegt:

* 10 binaire posities = 1024 mogelijkheden
* 20 posities = 1 048 576 mogelijkheden
* 30 posities = 1 073 741 824 mogelijkheden

Daarom zijn geheugen, opslag en netwerken allemaal in machten van 2 gestructureerd — het sluit exact aan op **hoe bits tellen**.

---

## ✅ Samenvatting

| Concept               | Formule                         | Betekenis                 |
| --------------------- | ------------------------------- | ------------------------- |
| 1 stap in binary = ×2 | 2ⁿ                              | elke extra bit verdubbelt |
| 2¹⁰ = 1024            | basisblok in binaire schaal     | dicht bij 1000            |
| 1 KiB = 2¹⁰ bytes     | fundamentele binaire “kilobyte” | precies 1024 bytes        |
| 1 MiB = 2²⁰ bytes     | 1 024 × 1 024                   | 1 048 576 bytes           |

---

💡 **Kort gezegd:**

> 1024 komt van `2¹⁰` —
> omdat 10 binaire posities precies 1024 combinaties opleveren.
> Het is de natuurlijke “ronde” grens in de wereld van bits.

zodat je het visueel kunt zien groeien — van 1 → 2 → 4 → 8 → … → 1024?

💥 Ja — héle goeie vraag.
En dit is precies het punt waar je echt begint te **denken als een systeemarchitect** in plaats van een gebruiker.
Je ziet nu namelijk dat *1024, 2048, 4096, …* voor een computer gewoon getallen zijn — dus waarom praten we dan over “KiB”, “MiB”, “GiB”?
Laten we dit ontleden in 3 lagen: wat de **machine** doet, wat de **software** doet, en wat **mensen** nodig hebben. 👇

---

## 🧠 1️⃣ Wat de computer zélf doet

Een CPU, geheugencontroller of netwerkinterface werkt puur met **bytes en bits**.
Voor de hardware is het getal gewoon een **aantal bytes**.

```
Er is geen “KiB” of “MiB” in de hardware.
Alleen: “je krijgt 4194304 bytes.”
```

➡️ Computers tellen in bytes omdat dat hun **adreseenheid** is:
elk geheugenadres wijst naar één byte.

---

## ⚙️ 2️⃣ Wat de software doet

Software gebruikt hogere eenheden (KiB, MiB, GiB) alleen om **leesbaarheid** te verbeteren.

Stel, een bestand is 10485760 bytes.

Voor een mens is dat getal lelijk: `10485760`
Voor een systeembeheerder is het fijner als je zegt:

```
10485760 bytes ÷ 1024 ÷ 1024 = 10 MiB
```

Dus:

> De schaal (KiB/MiB/GiB) is **een representatie**, geen nieuwe grootheid.
> Het verandert niets aan de onderliggende bytes.

---

## 📏 3️⃣ Waarom steeds ×1024?

Omdat het handig is om een **vaste sprong** te hebben tussen “orde van grootte”.

Elke keer dat je ×1024 doet, voeg je 10 binaire bits aan ruimte toe.
Dat past perfect bij hoe adressen en registers werken:

* 10 extra bits = 1024× meer bereik.

Dus een 10-bit verhoging in adresruimte = “1 stap hoger in schaal”.

| Schaal | Macht van 2 | Bytes             | Binaire betekenis |
| ------ | ----------- | ----------------- | ----------------- |
| Byte   | 2⁰          | 1                 | basiseenheid      |
| KiB    | 2¹⁰         | 1,024             | 10 bits extra     |
| MiB    | 2²⁰         | 1,048,576         | 20 bits extra     |
| GiB    | 2³⁰         | 1,073,741,824     | 30 bits extra     |
| TiB    | 2⁴⁰         | 1,099,511,627,776 | 40 bits extra     |

➡️ Dus elke stap omhoog = 10 extra binaire bits.
Die **macht van 2** sluit direct aan bij de logica van adressering in hardware.

---

## 🧩 4️⃣ Waarom het voor mensen “1000” lijkt

Omdat wij het fijn vinden om grote getallen te groeperen:
duizendtallen, miljoenen, miljarden.

1024 ligt dicht bij 1000, dus KiB, MiB, GiB zijn gewoon de **binaire equivalenten** van kilo, mega, giga.
We gebruiken de namen alleen om de orde van grootte te herkennen.

> Voor de computer maakt 1024 vs 1000 niks uit.
> Voor ons is het een manier om “grote hoeveelheden bytes” overzichtelijk te benoemen.

---

## 🧮 5️⃣ Hoe je het echt moet zien

Een computer kent alleen:

```
adres 0000000000000000 → byte
adres 0000000000000001 → byte
...
adres 0000000011111111 → byte
```

Wij zeggen dan:

> “Dat zijn 256 bytes, dus ongeveer een kwart kilobyte.”

De **schaal** (KiB, MiB, GiB) is puur mensentaal bovenop de binaire werkelijkheid.
De machine blijft altijd tellen in *bytes* (of bits bij transmissie).

| Niveaulaag | Wat het doet                  | Telt in              | Waarom 1024?                               |

| Hardware   | Adressering, geheugen, opslag | Bytes                | Elk adres = 1 byte; 2ⁿ bits bepalen bereik |
| Software   | Schaalbare presentatie        | KiB, MiB, GiB        | 10 bits per schaalstap (×1024)             |
| Mensen     | Begrip & communicatie         | “kilobyte, megabyte” | makkelijker lezen dan miljoenen bytes      |

💡 **Kort gezegd:**

> Computers rekenen in bytes (pure aantallen).
> Wij drukken dat uit in machten van 2 (KiB/MiB/GiB) zodat het leesbaar wordt.
> 1024 is gewoon de natuurlijke stap in binary, omdat 2¹⁰ precies 1024 is.
