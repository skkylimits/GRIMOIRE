ASCII, stands for American Standard Code for Information Interchange. It is a 7-bit character code where each individual bit represents a unique character.
::

> From Bits to Letters

## Wat is ASCII

`ASCII` (American Standard Code for Information Interchange) is het eerste wijdverspreide systeem waarmee computers tekst konden begrijpen en uitwisselen.

Uitgevonden in `1963`, gebruikt het een `7-bit` code — genoeg voor `128` unieke waarden (`0–127`).

Elke waarde staat voor een letter, cijfer of symbool.

| Type | Voorbeelden | Decimaal bereik |
|:----:|:-------------|:----------------|
| Letters | `A–Z`, `a–z` | 65–90, 97–122 |
| Cijfers | `0–9` | 48–57 |
| Symbolen | `@`, `#`, `!`, `?`, `.` | 32–64 |
| Control codes | `NUL`, `TAB`, `LF` (newline), `CR` | 0–31 |

### Voorbeelden

```text
'A' → 65₁₀ → 01000001₂
'a' → 97₁₀ → 01100001₂
')' → 41₁₀ → 00101001₂
':' → 58₁₀ → 00111010₂
```

Elke letter of symbool is dus niets meer dan een unieke `binaire code`.

## Control characters

De eerste 32 ASCII-codes (`0–31`) waren niet bedoeld voor letters, maar voor **besturing**.
Bijvoorbeeld:

| Code | Afkorting | Betekenis                     |
| :--: | :-------: | :---------------------------- |
|  `0` |   `NUL`   | Einde van data                |
|  `9` |   `TAB`   | Horizontale tab               |
| `10` |    `LF`   | Line Feed (nieuwe regel)      |
| `13` |    `CR`   | Carriage Return (regel terug) |

Vroeger — toen beeldschermen nog typemachines waren — moest je echt een `CR` en `LF` sturen om naar de volgende regel te gaan.
Zonder die codes liep tekst letterlijk van het scherm af.

## Waarom ASCII belangrijk was

Voor het eerst konden computers van **verschillende fabrikanten** tekst met elkaar uitwisselen.
Dit noem je `interoperabiliteit`.

Een bestand geschreven op een IBM-computer kon worden gelezen door een DEC, HP of Apple — omdat ze allemaal dezelfde tabel gebruikten.

::important
ASCII was dus de **eerste wereldtaal van computers**.
::

## De beperking van ASCII

ASCII werkte goed voor Engels, maar niet voor andere talen.

Er zijn immers maar `128` codes in 7 bits — geen ruimte voor `é`, `ß`, `ø`, `Ж`, of `漢`.

Toen begon iedereen het laatste (achtste) bit te gebruiken (`128–255`) voor **nationale varianten**:

* In de VS → extra symbolen en accenten
* In Rusland → Cyrillisch
* In Griekenland → Grieks
* In Duitsland → Umlauts (`ä`, `ö`, `ü`)

Dat werkte… totdat iemand een Frans e-mailtje op een Japanse computer probeerde te lezen 💥

## Het probleem: Mojibake

Omdat elk land zijn eigen “uitbreiding” had, begrepen computers elkaar niet meer.
Dezelfde bytes gaven totaal andere tekens weer.

```text
01100101 01101110 → “en” in Engels
01100101 01101110 → “何” in Japans (?!)
```

Het resultaat was willekeurige symbolen en chaos op het scherm —
de Japanners gaven het zelfs een naam: `mojibake` (文字化け) → *vervormde tekst*.

## De opvolger: Unicode

Om dit op te lossen werd in `1992` `Unicode` ontwikkeld — één universeel systeem voor *alle talen*.

Unicode gebruikt minimaal `16 bits`, met ruimte voor meer dan `1 miljoen` tekens,
waaronder alle letters, symbolen, emoji en historische scripts.

::important
ASCII leeft nog steeds voort — de eerste 128 tekens van Unicode **zijn identiek aan ASCII**.
Daardoor is alle oude tekst nog steeds leesbaar in moderne systemen.
::
