> From Data to Representation

## Wat is Encoding

`Encoding` betekent letterlijk: **iets omzetten naar een andere vorm** zodat het kan worden **opgeslagen, verzonden of begrepen**.

Computers gebruiken encoding om **binaire data** (nullen en enen) te vertalen naar vormen die mensen, netwerken of andere systemen kunnen lezen.

Bijvoorbeeld:
* tekst → bytes (zoals ASCII of UTF-8)  
* bestanden → Base64  
* URL’s → %20 voor spaties  

Kort gezegd:
> Encoding is de taal die bits begrijpelijk maakt — zonder hun betekenis te veranderen.

## Waarom we encoden

Er zijn drie hoofdredenen waarom encoding bestaat:

1️⃣ **Transmissie** — niet elk systeem accepteert alle tekens.  
→ Een spatie (` `) mag niet in een URL, dus wordt het `%20`.

2️⃣ **Opslag** — harde schijven, netwerken en protocollen werken met bits, niet met letters.  
   → Encoding vertaalt mensentaal naar binaire patronen.

3️⃣ **Compatibiliteit** — verschillende systemen moeten dezelfde data op dezelfde manier kunnen interpreteren.  
   → “A” in Windows moet hetzelfde zijn als “A” in Linux of op een webserver.


## Encoding is omkeerbaar

Encoding is **reversible** — je kunt altijd terug naar de originele data.  
Dat onderscheidt encoding van encryptie of hashing.

| Proces | Richting | Doel |
|:-------|:----------|:-----|
| **Encoding** | ↔ bidirectioneel | Data correct opslaan / verzenden |
| **Encryptie** | ↔ bidirectioneel (met sleutel) | Data beveiligen |
| **Hashing** | → eenrichtingsverkeer | Data verifiëren / identificeren |

Voorbeeld:
```text
'Hello World'  →  48656C6C6F20576F726C64  (hex encoding)
48656C6C6F20576F726C64  →  'Hello World'  ✅ terug te decoderen
```

> Encoding verandert **de vorm** van data,
> niet **de betekenis** of **de inhoud**.


## Soorten Encoding

Er bestaan talloze encodingmethoden, elk met een specifiek doel:

| Type                 | Voorbeeld               | Doel                                  |
| -------------------- | ----------------------- | ------------------------------------- |
| **Tekstencoding**    | ASCII, UTF-8, UTF-16    | Taaltekens omzetten naar bytes        |
| **Bestandencoding**  | Base64                  | Binaire data in tekstvorm stoppen     |
| **Netwerkencoding**  | URL-encoding, MIME      | Data veilig door protocollen sturen   |
| **Controleencoding** | Pariteitsbit, checksums | Fouten detecteren tijdens transmissie |

Elke encoding heeft zijn eigen regels — maar het principe blijft hetzelfde:
**bits vertalen naar een geldig, herkenbaar formaat**.

## URL Encoding

Stel je zoekt op Google naar:

```
how to learn c++
```

De browser kan geen spaties of `+`-tekens direct gebruiken in een URL,
dus zet hij ze om naar een geldige representatie:

```
how%20to%20learn%20c%2B%2B
```

| Symbool      | Encoded | Waarom                  |
| :----------- | :------ | :---------------------- |
| ` ` (spatie) | `%20`   | spaties niet toegestaan |
| `+`          | `%2B`   | gereserveerd teken      |
| `?`          | `%3F`   | scheiding in URL-query  |

> Encoding zorgt ervoor dat tekst *veilig door het web reist* zonder betekenis te verliezen.

## Encoding in de praktijk (Frontend & Web)

Encoding gebeurt niet alleen bij tekst of netwerken —
het zit diep verweven in alles wat je op het web ziet.

### 🎨 In CSS – kleuren in hexadecimaal

Kleuren in CSS worden vaak *ge-encodeerd* in **hexadecimale vorm** omdat dat compacter is dan decimaal.

```css
color: #FF5733;  /* RGB(255, 87, 51) */
```

In decimaal zou dat veel langer zijn —
hex maakt de waarden kort, exact en binaire-vriendelijk.

---

### 🖼️ In HTML – afbeeldingen als Base64

Kleine afbeeldingen kunnen direct in HTML of CSS worden *ingebed* met **Base64 encoding**:

```html
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAA..." />
```

Zo hoef je geen apart bestand op te halen —
de afbeelding reist *mee als tekst* binnen de HTML zelf.

> 💡 Encoding maakt binaire data (zoals een afbeelding) tijdelijk *tekstueel* zodat het via webprotocollen kan reizen.

## Encoding ≠ Encryptie

Een veelgemaakte fout:
**Encoding beschermt niets.**
Iedereen kan de data weer decoderen.

Encryptie daarentegen **versleutelt** de data zodat alleen iemand met de juiste sleutel het terug kan halen.

```text
"hello" → Base64 → "aGVsbG8="   (iedereen kan decoderen)
"hello" → AES-256 → "3ad77bb40d7a..."   (alleen met sleutel te ontcijferen)
```

> Encoding = verpakking
> Encryptie = kluis

## Encoding in actie

Encoding is overal:

* Wanneer je een bestand mailt (Base64)
* Wanneer je een URL deelt (`%20`)
* Wanneer je tekst opslaat in een database (UTF-8)
* Wanneer je netwerkpakketten verstuurt (binary framing)
* Wanneer je een CSS kleurcode of inline afbeelding gebruikt

Alles wat je digitaal ziet of verstuurt is op een of andere manier **gecodeerd**.

```text
01001000 01100101 01101100 01101100 01101111
↓
ASCII / UTF-8  →  "Hello"
↓
Base64         →  "SGVsbG8="
↓
URL-safe       →  "SGVsbG8%3D"
```
