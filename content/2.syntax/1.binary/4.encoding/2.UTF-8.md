> From ASCII to a Universal Alphabet

Zonder `UTF-8` zouden talen, emoji en symbolen nog steeds in aparte werelden leven.

## Het probleem dat Unicode oplost

`ASCII` werkte goed voor Engels, maar de wereld is groter dan 128 tekens.
Talen als Chinees, Arabisch of Hindi hebben **duizenden symbolen**.

Om dat te fixen werd in `1992` `Unicode` ontworpen:
één systeem dat elk karakter op aarde een uniek nummer geeft — een `code point`.

```text
U+0041 → 'A'
U+03A9 → 'Ω'
U+4E2D → '中'
U+1F600 → 😀
```

Elk karakter heeft dus een **Unicode-nummer**, onafhankelijk van hoe het wordt opgeslagen.

## Unicode vs Encoding

`Unicode` is geen bestandsformaat — het is een **concept**.

Het zegt:

> “Dit karakter = codepoint U+0041.”

Maar om dat karakter **in bytes op te slaan**, heb je een **encoding** nodig.

Er bestaan meerdere manieren om Unicode in bits te zetten:

* `UTF-8`
* `UTF-16`
* `UTF-32`

## Wat doet UTF-8 precies

`UTF-8` (*Unicode Transformation Format 8-bit*) zet elk Unicode-codepoint om naar **1 tot 4 bytes**.
Die bytes beginnen altijd met speciale “startbits” die aangeven **hoeveel bytes** bij dat karakter horen.

💡 De eerste `128` tekens (`0–127`) zijn identiek aan `ASCII`.
Dat betekent: elke ASCII-tekst **is automatisch geldige UTF-8**.

| Code range (Unicode) | Bytes |           Startbits patroon           | Voorbeeld          |
| :------------------: | :---: | :-----------------------------------: | :----------------- |
|   `U+0000 – U+007F`  |   1   |               `0xxxxxxx`              | `A → 41`           |
|   `U+0080 – U+07FF`  |   2   |          `110xxxxx 10xxxxxx`          | `é → C3 A9`        |
|   `U+0800 – U+FFFF`  |   3   |      `1110xxxx 10xxxxxx 10xxxxxx`     | `中 → E4 B8 AD`     |
| `U+10000 – U+10FFFF` |   4   | `11110xxx 10xxxxxx 10xxxxxx 10xxxxxx` | `😀 → F0 9F 98 80` |

## Zo codeert UTF-8 een code point

Laten we `é (U+00E9)` als voorbeeld nemen.

1️⃣ Het Unicode-nummer `00E9₁₆` in binair is:

```text
0000 0000 1110 1001
```

2️⃣ UTF-8 kijkt naar de waarde: `U+00E9` ligt tussen `0080–07FF`,
dus het moet **2 bytes** worden:

**Patroon voor 2 bytes:**

```
110xxxxx 10xxxxxx
```

3️⃣ Die bits worden ingevuld van rechts naar links:

```
11000011 10101001
```

4️⃣ In hex:

```
C3 A9
```

💡 Zo weet UTF-8 altijd hoeveel bytes een teken gebruikt:
de **eerste bits van de eerste byte** zeggen alles.

---

## Waarom ASCII een subset is

ASCII gebruikt maar 7 bits (`0–127`), wat past in het eerste UTF-8 patroon:

```
0xxxxxxx
```

Daarom is:

```text
'A' = 65₁₀ = 01000001₂ = 41₁₆
```

geldige ASCII én geldige UTF-8 —
de rest van de bytes zijn nooit nodig.

## Voorbeelden in context

```text
"A"   → U+0041 → 01000001 → 41
"é"   → U+00E9 → C3 A9
"中"  → U+4E2D → E4 B8 AD
"😀"  → U+1F600 → F0 9F 98 80
```

In binaire vorm:

```text
A  = 01000001
é  = 11000011 10101001
中 = 11100100 10111000 10101101
😀 = 11110000 10011111 10011000 10000000
```

## Waarom UTF-8 won

* `Backwards compatible` met `ASCII`
* Ondersteunt alle `Unicode`-tekens
* Compact — Engelse tekst = 1 byte per karakter
* Universeel — websites, JSON, e-mails, terminals → allemaal UTF-8

## Is Unicode base16?

Een veelvoorkomende misvatting is dat `Unicode` een `base16` (hexadecimaal) systeem zou zijn.
Dat klopt niet helemaal.

`Unicode` zelf is **geen getalsysteem** — het is een **concept** dat elk karakter op aarde
een uniek nummer geeft, het zogeheten `code point`.

Wat wél waar is:
Unicode-codepoints worden **bijna altijd weergegeven in hexadecimale notatie**,
omdat dat korter en leesbaarder is dan decimale of binaire getallen.

```text
A   → 65₁₀ → U+0041₁₆
é   → 233₁₀ → U+00E9₁₆
中  → 20013₁₀ → U+4E2D₁₆
😀  → 128512₁₀ → U+1F600₁₆
```

Die `U+XXXX`-notatie is dus enkel een **schrijfwijze in hex**,
niet een aanduiding dat Unicode zelf “base16” is.

### 🧩 Vergelijking

| Term              | Betekenis                                                      | Base      |
| :---------------- | :------------------------------------------------------------- | :-------- |
| `Unicode`         | Concept dat elk karakter een uniek nummer geeft                | Geen base |
| `U+XXXX`          | Hexadecimale notatie van dat nummer                            | Base 16   |
| `UTF-8 / 16 / 32` | De binaire encodings waarmee Unicode in bytes wordt opgeslagen | Base 2    |


::important
**Kort gezegd:**
- Unicode = de kaart met alle karakters
- Hex (`U+1F600`) = hoe wij die nummers opschrijven
- UTF-8/16/32 = hoe computers ze als bits bewaren
::
