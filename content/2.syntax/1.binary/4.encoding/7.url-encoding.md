Percent-encoding is a mechanism to encode 8-bit characters that have specific meaning in the context of URLs
::

> From Text to Safe URLs

## Wat is URL-encoding?

Browsers en webservers mogen niet elk teken ongefilterd in een URL sturen.  
`URL-encoding` (ook: *percent-encoding*) zet niet-toegestane tekens om naar een veilige vorm: `%HH`, waarbij `HH` de hex-waarde van de byte is.

` ` (spatie) → `%20`,  
`?` → `%3F`,  
`+` → `%2B`.

## Waarom het nodig is

Een URL heeft onderdelen (scheme, host, path, query, fragment) en elk deel heeft een eigen reeks toegestane tekens.  
Als je daar tekens stopt die verwarring of parsing-fouten veroorzaken, moet je ze encoden zodat de ontvanger precies weet wat je bedoelt.

Voorbeeld (zoekquery):
```

[https://www.google.com/search?q=how](https://www.google.com/search?q=how) to learn c++

```
wordt:
```

[https://www.google.com/search?q=how%20to%20learn%20c%2B%2B](https://www.google.com/search?q=how%20to%20learn%20c%2B%2B)

```

---

## Percent-encoding: de regels (kort)

- Een byte wordt weergegeven als `%` gevolgd door twee hex-cijfers (uppercase of lowercase is OK): `%2B` of `%2b`.
- Alfanumerieke tekens (`A–Z`, `a–z`, `0–9`) en een paar extra tekens (`- _ . ~`) blijven ongewijzigd (veilig).
- Andere tekens moeten gecodeerd worden — vooral in `path`, `query` en `fragment` kunnen regels verschillen per context.

**Veilige set (unreserved):**
```

A–Z a–z 0–9 - _ . ~

```

**Gereserveerde tekens (moeten soms gecodeerd worden):**
```

: / ? # [ ] @ ! $ & ' ( ) * + , ; =

```
Sommige van deze hebben een speciale betekenis (bijv. `?` scheidt path van query), dus encoderen voorkomt ambiguïteit.

## Veelvoorkomende voorbeelden

| Origineel | Encoded |
|----------:|:--------|
| ` ` (spatie) | `%20` |
| `+` | `%2B` |
| `?` | `%3F` |
| `#` | `%23` |
| `/` (in query) | `%2F` |
| `€` (UTF-8 bytes `E2 82 AC`) | `%E2%82%AC` |

::warning
Let op: voor non-ASCII tekens wordt eerst UTF-8 gebruikt, daarna percent-encoding per byte.  

> Bijvoorbeeld `€` → UTF-8 `E2 82 AC` → `%E2%82%AC`.
::

## Path vs Query — subtle verschil

- `path`-deel: `/foo/bar baz` → `/foo/bar%20baz` (spaties encoden)
- `query`-deel: `q=a+b` — in sommige formaten (application/x-www-form-urlencoded) staat `+` voor spatie; in raw URL percent-encoding is `+` gewoon `+` (dus veilige aanpak: gebruik `%20` voor spatie).

Praktisch:
- Voor webformulieren (HTML forms) wordt vaak `application/x-www-form-urlencoded` gebruikt:  
  spatie → `+`, en andere bytes → `%HH`.
- Voor JavaScript `encodeURIComponent()` gebruik je percent-encoding (`+` blijft `+` tenzij je replace gebruikt).

## Dubbele encoding & fouten

Veel bugs ontstaan door **double-encoding**:

1. `+` → `%2B`
2. Encode opnieuw → `%252B`  (`%` → `%25`)

Resultaat: server decodeert één stap en krijgt nog steeds `%2B` in plaats van `+`.  
Voorkomen: encodeer precies éénmaal net vóór het versturen; decodeer aan serverkant precies eenmaal.

## Voorbeelden in praktijk

1. Zoekterm:
```

q = "C# tips & tricks"
→ q = "C%23%20tips%20%26%20tricks"
URL: [https://example.com/search?q=C%23%20tips%20%26%20tricks](https://example.com/search?q=C%23%20tips%20%26%20tricks)

```

2. Inline data (safe name):
```

filename = "rapport 2025.pdf"
→ filename=rapport%202025.pdf

```

3. Unicode:
```

woord = "café"
UTF-8 bytes: 63 61 66 C3 A9
Encoded: cafe%CC%81  (if decomposed) OR caf%C3%A9 (normal case)

```
(gebruik normalisatie — zie NFC — zodat alle partijen hetzelfde encoden)


## Tools & tips (praktisch)

- JavaScript: `encodeURIComponent(str)` (voor query-waarden), `encodeURI(str)` (voor volledige URI).
- Python: `urllib.parse.quote()` / `unquote()`.
- CLI quick check:
```bash
python -c "import urllib.parse; print(urllib.parse.quote('café'))"
# → caf%C3%A9
```

* Browser devtools: network tab laat gecodeerde URL's en decodering zien.

## Valideren en veilig parsen (defensie)

* **Valideer** URLs op server: decodeer exact één keer en controleer op ongewenste inhoud (null bytes, control chars).
* **Normaliseer** input (NFC) voordat je encodeert of vergelijkt.
* **Log both**: raw URL bytes en de gedecodeerde waarde — helpt bij forensics.
* **Wees voorzichtig met filenames**: decodeer niet blind naar filesystem-paths (directory traversal `../` kan verborgen worden via encodering).
