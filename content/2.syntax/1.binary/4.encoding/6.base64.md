> Van bits naar tekst

## Wat is Base64

`Base64` is een **coderingssysteem** (geen encryptie!) dat binaire data omzet in **leesbare ASCII-tekst**.

Het doel is om data — die normaal uit nullen en enen bestaat — door te sturen via systemen die alleen tekst begrijpen, zoals:

- HTTP-headers  
- E-mail (MIME)  
- JSON of XML  
- HTML `img` tags  

Base64 doet dit door gebruik te maken van **64 symbolen**, die elk **6 bits** aan informatie voorstellen.

```text
2⁶ = 64 → elke Base64-teken = 6 bits
```

Drie bytes (24 bits) worden dus netjes omgezet in vier Base64-tekens (4 × 6 bits = 24 bits).

## Waarom Base64?

Veel communicatieprotocollen zijn ontworpen voor **tekst**, niet voor ruwe bytes.
Als je binaire data (zoals een afbeelding of wachtwoordhash) rechtstreeks doorstuurt, kan die beschadigd raken.

Base64 vertaalt die binaire inhoud naar een **veilige tekstreeks** met uitsluitend **printbare ASCII-karakters**.

| Eigenschap                              | Uitleg                     |
| --------------------------------------- | -------------------------- |
| ✅ Veilig in tekstgebaseerde protocollen | (HTTP, SMTP, JSON)         |
| ✅ Vermijdt speciale tekens              | Geen spaties of quotes     |
| ✅ Universeel ondersteund                | In browsers, talen en APIs |
| ⚠️ Iets groter                          | Data groeit ±33%           |

## De Base64 tekenset

Base64 gebruikt precies 64 karakters:

```text
A–Z  → 26 hoofdletters
a–z  → 26 kleine letters
0–9  → 10 cijfers
+ /  → 2 symbolen
---------------------
Totaal: 64 tekens
```

Daarbij komt soms het `=`-teken voor als **padding** (vulling aan het eind).

## The Trade-Off

Hoeveel groter wordt het?

Omdat 3 bytes (24 bits) worden omgezet naar 4 tekens (32 bits):

```text
(4 / 3) × 100% = 133%
```

Dat betekent een **33% grotere bestandsgrootte**.

```text
3 MB → 4 MB na Base64-encoding
```

## Hoe Base64 werkt

### Stap 1 – Binaire invoer

Elke tekst, afbeelding of bestand bestaat uiteindelijk uit bytes.
Bijvoorbeeld de string `"xyz"`:

```text
x → 01111000
y → 01111001
z → 01111010
```

Samen vormen ze:

```text
01111000 01111001 01111010  (3 bytes = 24 bits)
```

### Stap 2 – Splitsen in 6-bits blokken

Base64 leest steeds **6 bits per keer**:

```text
011110 000111 100101 111010
```

Elke groep van 6 bits wordt omgezet naar een decimale waarde van `0–63`:

```text
011110 → 30
000111 → 7
100101 → 37
111010 → 58
```

### Stap 3 – Opzoeken in de tabel

De Base64-index wordt gebruikt om deze waarden te vertalen:

| Decimaal | Base64-teken |
| -------- | ------------ |
| 0        | A            |
| 7        | H            |
| 30       | e            |
| 37       | l            |
| 58       | 6            |

Zo krijgen we de uitvoer `"eHls"` (voorbeeldwaarden).

### Stap 4 – Resultaat

De vier Base64-tekens vormen de uiteindelijke gecodeerde output:

```text
"xyz" → "eHl6"
```

3 bytes (24 bits) worden 4 karakters (24 bits).
Omdat elk Base64-teken in ASCII 1 byte groot is, groeit de tekst met **ongeveer 33%**.

## Padding (`=`)

Base64 werkt in blokken van 3 bytes.
Als de invoer niet precies deelbaar is door 3, wordt `=` toegevoegd om de lengte uit te lijnen.

| Overgebleven bytes | Outputlengte | Padding | Voorbeeld |
| :----------------: | :----------- | :------ | :-------- |
|          3         | 4            | Geen    | `YW5k`    |
|          2         | 3 + `=`      | `=`     | `YW4=`    |
|          1         | 2 + `==`     | `==`    | `YQ==`    |

### Voorbeeld 1 — Eén byte

```text
Invoer: "a"
Binary: 01100001
→ 011000 01 + 0000 (pad)
→ 24 (Y), 16 (Q)
→ "YQ=="
```

### Voorbeeld 2 — Twee bytes

```text
Invoer: "aa"
Binary: 01100001 01100001
→ 011000 010110 000100
→ 24 (Y), 22 (W), 4 (E)
→ "YWE="
```

## Decoderen

Het decoderen is exact het omgekeerde proces:

1. Neem elk Base64-teken en vertaal het terug naar zijn 6-bit waarde.
2. Combineer de bits tot blokken van 8 bits.
3. Zet elk 8-bit blok terug naar een ASCII-teken.

Voorbeeld:

```text
SGk= → S (18), G (6), k (36)
Binary: 010010 000110 100100
→ 01001000 01101001
→ "Hi"
```

## Realistische toepassingen

### 🧾 HTTP Basic Auth

In een HTTP-header worden gebruikersnaam en wachtwoord samengevoegd met een dubbele punt, daarna Base64-gecodeerd:

```text
user:password → dXNlcjpwYXNzd29yZA==
```

**Let op:** dit is *geen* beveiliging — Base64 kan eenvoudig worden terugvertaald.

### 🔐 JSON Web Tokens (JWT)

Een JWT bestaat uit drie delen:

```text
<header>.<payload>.<signature>
```

Elk deel is afzonderlijk **Base64 URL-gecodeerd** (zie verderop), en samengevoegd met puntjes.

### 🖼️ Inline afbeeldingen (HTML)

Je kunt kleine afbeeldingen rechtstreeks in HTML opnemen:

```html
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUg..." />
```

Zo hoef je geen extern bestand te laden — handig voor icoontjes of embedded dashboards.

## Base64 vs Hexadecimaal

| Eigenschap        | Base64                           | Hexadecimaal          |
| ----------------- | -------------------------------- | --------------------- |
| Basis             | 64 (2⁶)                          | 16 (2⁴)               |
| Bits per karakter | 6                                | 4                     |
| Data-expansie     | +33%                             | +100%                 |
| Leesbaarheid      | Matig                            | Goed (menselijk)      |
| Doel              | Transport / tekstcompatibiliteit | Inspectie / debugging |

Hex wordt vaak gebruikt om *te lezen*, Base64 om *te verzenden*.

## Base64 URL Encoding

Sommige Base64-symbolen (`+`, `/`, `=`) veroorzaken problemen in URLs.
Daarom bestaat de **URL-veilige variant**.

| Standaard | URL-Veilig   |
| --------- | ------------ |
| `+`       | `-`          |
| `/`       | `_`          |
| `=`       | (weggelaten) |

Padding wordt in de URL-variant vaak **weggelaten**, omdat de decoder het aantal bits zelf kan afleiden.

```text
"SGk=" → "SGk"
"+/Y=" → "-_Y"
```

Zo kun je Base64 veilig gebruiken in tokens, querystrings of bestandsnamen.

## Base64 in één oogopslag

| Concept              | Uitleg                                   |
| -------------------- | ---------------------------------------- |
| **Doel**             | Binaire data omzetten naar veilige tekst |
| **Karakters**        | `A–Z`, `a–z`, `0–9`, `+`, `/`            |
| **Bits per symbool** | 6                                        |
| **Invoerblok**       | 3 bytes → 4 tekens                       |
| **Padding**          | `=` om lengte te vullen                  |
| **Toename**          | +33%                                     |
| **Omkeerbaar**       | Ja                                       |
| **Beveiliging**      | Nee (geen encryptie)                     |

## Waarom dit belangrijk is

Base64 is één van de meest fundamentele concepten in **cybersecurity en webontwikkeling**.

Het komt terug in:

* E-mailbijlagen (MIME)
* API-tokens
* JWTs
* PowerShell of malware payloads
* Configbestanden

## Karakterset & padding

* Tekens: `A–Z a–z 0–9 + /` (standaard)
* Padding: `=` of `==` (afhankelijk van resterende bytes)
* URL-safe variant: `-` en `_` in plaats van `+` en `/` (geen padding vaak)

## CLI voorbeelden

```bash
# Encode (Linux/macOS)
echo -n "Hello World" | base64
# Decode
echo "SGVsbG8gV29ybGQ=" | base64 -d

# OpenSSL (Windows/older systems)
echo -n "data" | openssl base64
echo "SGVsbG8=" | openssl base64 -d -A
```
