Bitmasking is een fundamentele techniek om individuele bits in een getal te lezen, zetten, wissen of om te schakelen. Onmisbaar bij low-level programmeren, netwerkprotocols, permissies en reverse engineering.
::

> Bits controleren, zetten en manipuleren — efficiënt en deterministisch

## Wat is bitmasking

`Bitmasking` = het gebruiken van een `mask` (meestal een getal met specifieke bits gezet) samen met bitwise-operaties om individuele bits binnen een byte/word te manipuleren of te controleren.

Belangrijkste bitwise-operaties:
- `AND`  (`&`) — mask gebruikt om bits te testen / behouden
- `OR`   (`|`) — bits zetten
- `XOR`  (`^`) — bits togglen (wisselen)
- `NOT`  (`~`) — bits inverteren
- `<<` / `>>` — bitshifts (links/rechts schuiven)

---

## Waarom bitmasking gebruiken?

- Compacte representatie van meerdere booleans in één register (flags).
- Efficiënt (1 CPU-instructie vs meerdere).
- Onmisbaar in hardware-interface, netwerkheaders, bestandsformaten, permissies (Unix), en exploit-development.
- Vaak gebruikt in C/assembly en in config/feature flags.

---

## Basisconcepten & notatie

- `bit` — 0 of 1
- `nibble` — 4 bits
- `byte` — 8 bits
- `mask` — een getal waarvan geselecteerde bits `1` zijn (rest `0`)
- `flag` — betekenis gegeven aan één bit binnen een mask

Voorbeeld: 8-bit byte `0b01010110`  
Index (LSB = bit0):

```

bits:  7 6 5 4 3 2 1 0
value: 0 1 0 1 0 1 1 0

````

---

## Bitposities & conventies

- **LSB (Least Significant Bit)** = bit0 (rechtse bit)  
- **MSB (Most Significant Bit)** = bitN-1 (linkse bit)
- Endianness betreft byte-volgorde, niet bit-volgorde binnen een byte (bits lezen blijft LSB→MSB)

---

## Veelvoorkomende patronen / masks

- `BIT(n)` = `(1 << n)` — mask met enkel bit n gezet
- `LOWER_N_BITS(n)` = `(1 << n) - 1` — mask voor laagste n bits
- `BYTE_MASK` = `0xFF` — 8 bit mask
- `WORD_MASK` = `0xFFFF` — 16 bit mask

Voorbeelden:
```c
BIT(0)  = 0x01  = 00000001
BIT(3)  = 0x08  = 00001000
BIT(7)  = 0x80  = 10000000
````

## Basisoperaties — concept + voorbeelden

### Test of een bit gezet is (check)

Gebruik `AND` met een mask.

```c
// C
if (value & (1 << 3)) {
    // bit 3 is 1
}
```

```python
# Python
if value & (1 << 3):
    # bit 3 is 1
```

Voorbeeld concreet:

```text
value = 0b00001010  (10)
mask  = 0b00001000  (1<<3)
value & mask = 0b00001000  => truthy  => bit 3 is set
```

---

### Een bit zetten (set)

Gebruik `OR` met een mask.

```c
value |= (1 << 2); // zet bit2
```

```python
value = value | (1 << 2)
```

Voorbeeld:

```
value = 0b00000001
mask  = 0b00000100  (1<<2)
value | mask = 0b00000101
```

---

### Een bit wissen (clear)

Gebruik `AND` met de inverse van het mask.

```c
value &= ~(1 << 2); // wis bit2
```

```python
value = value & ~(1 << 2)
```

Voorbeeld:

```
value = 0b00000101
~(1<<2) = 11111011
AND => 00000101 & 11111011 = 00000001
```

---

### Bit togglen (flip)

Gebruik `XOR`.

```c
value ^= (1 << 1); // toggle bit1
```

```python
value = value ^ (1 << 1)
```

Als bit=0 → wordt 1; als bit=1 → wordt 0.

---

### Extract (isoleren) van meerdere bits (shift + mask)

Wil je n bits vanaf positie p halen:

```c
(value >> p) & ((1 << n) - 1)
```

Voorbeeld: 3 bits vanaf bit 4:

```
(value >> 4) & 0b111
```

## Concrete voorbeelden & use-cases

### 1) Unix-style permissies (rwx)

Unix permissions per bestand: `rwxrwxrwx` → 9 bits.

* `r` = 4, `w` = 2, `x` = 1 (per trio)

Voorbeeld: `0755` → `rwx r-x r-x`

```bash
# check owner execute
if (mode & 0100) != 0: # owner execute (octal)
```

In hex/bitmask:

```
0o755 = 0b111101101
```

---

### 2) TCP flags (SYN, ACK, FIN, RST, PSH, URG)

TCP header contains een 6-bit flags field. Gebruik masks:

```c
#define TCP_FIN  0x01
#define TCP_SYN  0x02
#define TCP_RST  0x04
#define TCP_PSH  0x08
#define TCP_ACK  0x10
#define TCP_URG  0x20

if (tcp_flags & TCP_SYN) { ... } // nieuw verbinding verzoek
```

---

### 3) Feature flags / config bits

Sla meerdere boolean features op in één integer:

```c
#define FEATURE_A (1<<0)
#define FEATURE_B (1<<1)
#define FEATURE_C (1<<2)

int flags = FEATURE_A | FEATURE_C;

if (flags & FEATURE_B) { /* ingeschakeld */ }
```

---

### 4) Bitfields vs masks (C-structs)

Bitfields in struct:

```c
struct {
    unsigned int flagA:1;
    unsigned int flagB:1;
    unsigned int reserved:6;
} s;
```

Voordeel: duidelijke fields; nadeel: compiler/endianness/packing issues — masks geven meer controle.

## Praktische voorbeelden in meerdere talen

### C — set/clear/toggle/check

```c
#include <stdio.h>

int main() {
    unsigned char v = 0x00; // 00000000

    // set bit 2
    v |= (1 << 2);          // v = 00000100

    // test bit2
    if (v & (1 << 2)) printf("bit2 set\n");

    // clear bit2
    v &= ~(1 << 2);         // v = 00000000

    // toggle bit0
    v ^= (1 << 0);          // v = 00000001

    printf("v = 0x%02X\n", v);
}
```

### Python — uitleg-friendly

```python
v = 0b0000_0000

# set bit 3
v |= (1 << 3)

# test
is_set = bool(v & (1 << 3))

# clear
v &= ~(1 << 3)

# extract bits 4..6 (3 bits from pos 4)
val = (v >> 4) & 0b111
```

### PowerShell (useful voor sysadmins)

```powershell
# set bit 2
$value = $value -bor (1 -shl 2)

# test bit
if ($value -band (1 -shl 2)) { "set" }

# clear bit
$value = $value -band -bnot (1 -shl 2)

# toggle
$value = $value -bxor (1 -shl 2)
```

## Masks & ranges — voorbeelden met getallen

### Mask voor laagste n bits:

`(1 << n) - 1`

* `n=4` → `00001111` → mask `0x0F`
* `n=8` → `0xFF`

Voorbeeld: laagste 5 bits isoleren:

```c
mask = (1 << 5) - 1; // 0b00011111
result = value & mask;
```

## Bit-velden in netwerkheaders (praktisch voorbeeld)

Stel een 16-bit field met meerdere subvelden:

```
bits 15..12 = version (4b)
bits 11..8  = type    (4b)
bits 7..0   = length  (8b)
```

Extract version:

```c
version = (field >> 12) & 0xF; // 4 bits
type    = (field >> 8)  & 0xF;
length  = field & 0xFF;
```

## Endianness reminder (byte order ≠ bit order)

* Bitmasking binnen een byte blijft altijd bit0 = LSB, bit7 = MSB.
* Endianness beïnvloedt **byte** volgorde bij multi-byte waarden:

  * Little endian 32-bit `0x12345678` in geheugen: `78 56 34 12`
  * Als je maskt op byte-offsets, houd endianness in gedachten.

---

## Veelgemaakte fouten en valkuilen

* Verwarring tussen bit-index en byte-offset (bit 0 van welk byte?). Wees expliciet: `bit 0 of totaal value` vs `bit 0 van byte i`.
* Onjuiste prioritéit bij mixen van signed types en shifts in C (gebruik unsigned voor shifts).
* Vergeten `()` rond shifts en masks bij berekeningen.
* Verwachtte endianess niet checken bij parsing van netwerk/bestandsformaten.

## Oefeningen met antwoorden

Oefening A:

```
value = 0b10110100
mask  = 1<<2
Vraag: is bit2 gezet? Wat is value na v |= (1<<0) ?
```

Oefening B:

```
field = 0x9A3C  (hex)
Extract versie = bits 15..12, type = bits 11..8, length = bits 7..0
```

Oefening C (praktisch):

```
Je hebt flags = 0x05 (00000101). 
Welke bits zet je en wis je om flags -> 0x0A ?
```

**Antwoorden**
A: `value & mask` → bit2 (waarde 4) is 1? 0b10110100 & 0b00000100 = 0b00000100 → ja. Na `v |= (1<<0)` → set bit0: 0b10110100 | 0b00000001 = 0b10110101.
B: 0x9A3C = 1001 1010 0011 1100

* versie = 1001 = 9
* type = 1010 = 10 (0xA)
* length = 0x3C = 60
  C: flags = 0x05 = 00000101; doel 0x0A = 00001010
* Welke zetten? set bit3 and clear bit0 and clear bit2. sequence:

  * flags |= (1<<3) -> 00001101
  * flags &= ~(1<<2) -> 00001001
  * flags &= ~(1<<0) -> 00001000
  * maybe toggle bit1 to get 00001010 (set bit1): flags |= (1<<1) => 00001010

## Cheat-sheet (snel referentie)

```
BIT(n)        = (1 << n)
SET bit n     = x |= BIT(n)
CLEAR bit n   = x &= ~BIT(n)
TOGGLE bit n  = x ^= BIT(n)
TEST bit n    = (x & BIT(n)) != 0
EXTRACT n@p   = (x >> p) & ((1<<n)-1)
LOWER_N_MASK  = (1<<n) - 1
```

## Samenvatting

Bitmasking is een compacte en efficiënte manier om individuele bits te beheren.
Leer de basis-operaties (`AND`, `OR`, `XOR`, `NOT`, `SHL`, `SHR`), onthoud je mask-patronen (`BIT(n)`, `LOWER_N_BITS`) en wees voorzichtig met endianness en signed/unsigned behaviours in hogere talen.

Bitmasking is deterministisch en snel — perfect voor low-level debugging, protocolparsing en performance-kritische code. Maar document altijd goed welke bit-indexen wat betekenen.
