# Binary ‚Äî complete study note (based on *Binary Explained in 01100100 Seconds*)

## Quick overview (what the video teaches)

* **Binary** is the base-2 positional numbering system (only digits `0` and `1`). Computers use binary because physical devices (transistors) naturally have two stable states (on/off). ([YouTube][1])
* A **bit** is a single binary digit; a **byte** is 8 bits (the most common grouping). Multi-byte widths (16, 32, 64 bits) extend the numeric range. ([YouTube][1])
* Text is stored by mapping characters to numeric codes ‚Äî e.g. **ASCII** maps letters to byte values (e.g. `'A'` ‚Üí `01000001`). ([ChatTube][2])

---

## 1 ‚Äî Number systems & positional notation (short)

* Decimal (base 10): positions represent powers of 10.
* Binary (base 2): positions represent powers of 2.
* Example: binary `1011‚ÇÇ` = (1¬∑2^3 + 0¬∑2^2 + 1¬∑2^1 + 1¬∑2^0 = 8 + 0 + 2 + 1 = 11_{10}). ([PBS][3])

---

## 2 ‚Äî Why computers use binary (physical/machine reason)

* Transistors are switches: they‚Äôre naturally ON or OFF ‚Üí map easily to `1` and `0`.
* Building logic (gates) and memory from two-state devices is robust and scalable; this is why digital electronics use binary. ([YouTube][1])

---

## 3 ‚Äî Bits, bytes & sizes

* 1 bit = 0 / 1
* 1 byte = 8 bits ‚Üí can represent `0..255` unsigned.
* Common sizes: 16-bit (0..65535), 32-bit, 64-bit. Older systems were 8/16-bit; modern CPUs use 64-bit registers. ([YouTube][1])

**Exercise:** what is the maximum unsigned value of 12 bits?
Answer: (2^{12}-1 = 4095).

## 4 ‚Äî Binary ‚Üî Decimal conversion (procedures + examples)

### Binary ‚Üí Decimal (weights)

Write out bit weights and sum where bits = 1.

Example: `1101011‚ÇÇ`
Positions (right‚Üíleft): (2^0,2^1,2^2,2^3,2^4,2^5,2^6)
Value: (1¬∑2^6 +1¬∑2^5 +0¬∑2^4 +1¬∑2^3 +0¬∑2^2 +1¬∑2^1 +1¬∑2^0 = 107).

### Decimal ‚Üí Binary (divide by 2 method)

Take 107:

* 107 /2 ‚Üí q=53 r=1 (LSB)
* 53 /2 ‚Üí q=26 r=1
* 26 /2 ‚Üí q=13 r=0
* 13 /2 ‚Üí q=6 r=1
* 6 /2 ‚Üí q=3 r=0
* 3 /2 ‚Üí q=1 r=1
* 1 /2 ‚Üí q=0 r=1 (MSB)
  Read remainders bottom‚Üítop: `1101011`.

## 5 ‚Äî Hexadecimal ‚Äî compact binary representation

* 1 hex digit = 4 bits (nibble).
* Example: `0x6B` ‚Üí `6` = `0110`, `B` = `1011` ‚Üí byte = `01101011` ‚Üí decimal 107.
  Hex is used everywhere in debugging, reversing and network dumps. ([Instructables][4])

---

## 6 ‚Äî ASCII and storing text

* ASCII assigns numbers to characters: `'A'` = 65 = `01000001‚ÇÇ`.
* Text strings in files are sequences of bytes, each byte representing a char per an encoding (ASCII, UTF-8 uses multiple bytes per codepoint for non-ASCII). ([Khan Academy][5])

**Quick example:** the string `"Hi"` ‚Üí bytes `0x48 0x69` ‚Üí binary `01001000 01101001`.

---

## 7 ‚Äî Signed integers & two‚Äôs complement (why used)

* Two‚Äôs complement is standard to represent signed integers. For an N-bit value:

  * Range signed: (-2^{N-1}) .. (2^{N-1}-1).
* To get negative: invert bits and add 1.

Example: `-5` in 8-bit:

* `+5` = `00000101`
* invert ‚Üí `11111010`
* add 1 ‚Üí `11111011` (two‚Äôs complement representation). ([PBS][3])

---

## 8 ‚Äî Endianness (byte order) ‚Äî why it matters

* **Little-endian** (Intel): least significant byte stored first. e.g. `0x12345678` ‚Üí bytes `78 56 34 12`.
* **Big-endian** (network order): most significant byte first (used by many network protocols).
  When parsing binary protocols or file formats you must be aware which byte order is used. ([PBS][3])

---

## 9 ‚Äî Bits in networks: IPv4 example (ties to your earlier net work study)

* IPv4 = 32 bits. Dotted decimal is four octets.
  Example `192.168.10.5` ‚Üí binary `11000000.10101000.00001010.00000101`.
  CIDR `/24` means first 24 bits network (mask `255.255.255.0`). Calculate network & broadcast by bitwise AND/OR with mask. (This ties into earlier C/Python examples you asked for.) ([YouTube][1])

---

## 10 ‚Äî How binary is used inside encryption, hashing & encoding (short recap)

* AES, SHA, XOR-based obfuscation etc. all operate on bits/blocks (e.g. AES operates on 128-bit blocks; SHA-256 on 512-bit blocks internally). Understanding bitwise ops (XOR, shifts, rotations) is essential for analyzing crypto primitives and for reversing. ([YouTube][1])

## 11 ‚Äî Practical tools & quick commands

* `xxd -g 1 file` or `hexdump -C file` ‚Äî inspect raw bytes.
* `python` quick experiments:

```python
# print binary of byte
b = 107
print(f"{b:08b}")  # '01101011'
# IP to binary
'.'.join(f"{int(o):08b}" for o in "192.168.10.5".split('.'))
```

* Use `inet_pton` / `ntohl` in C for IP conversions (I can provide ready C snippets if you want). ([Instructables][4])

## 13 ‚Äî Study path & how to apply this to reversing / malware analysis

1. Master bit/byte conversions and hex ‚Äî daily drill.
2. Read/write C code that manipulates bytes/structs/pointers. (Practice `memcpy`, bit masks, reading little vs big endian.)
3. Learn to use `xxd`, `strings`, `objdump`, `radare2`/`ghidra` to inspect binaries and identify ascii, headers, patterns.
4. Practice network packet inspection (Wireshark) and map headers to binary fields.
5. Study simple crypto primitives at bit level (XOR streams, rotations used by SHA/AES round functions). ([PBS][3])
