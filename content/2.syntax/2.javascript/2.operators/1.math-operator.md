## <3 Everybody loves math

We know many operators from school. They are things like addition +, multiplication *, subtraction -, and so on.

We’ll start with simple operators, then concentrate on JavaScript-specific aspects, not covered by school arithmetic.

## 🧠 Terminology

### operand

Think of an _operand_ as a **thing you do something to**.

In this:
```js
5 * 2
```
- `*` is the **operator** (it multiplies)
- `5` and `2` are the **operands** (they’re the things getting multiplied)

It’s just like in real life:
> If I say "Double 3", then 3 is the thing I'm doing something to → the **operand**.


### unary

Means it acts on just **one thing**.

```js
let x = 5;
x = -x;
```

Here:
- `-` is a _unary_ operator (it flips the sign)
- `x` is the **only** operand

🔍 Think of it like: “Negate this number.”


### binary

It needs **two things** to work with.

```js
let a = 10;
let b = 3;
let result = a - b;
```

Here:
- `-` is now a _binary_ operator (subtracting)
- `a` and `b` are both operands

🔍 Think of it like: “Take `a`, subtract `b` from it.”

## 📐 Maths

The following math operations are supported:

- **Addition** `+`
- **Subtraction** `-`
- **Multiplication** `*`
- **Division** `/`
- **Remainder** `%`
- **Exponentiation** `**`

The first four are straightforward, while `%` and `**` need a few words about them.

### 🔸 Remainder `%`

The **remainder operator** `%`, despite its appearance, is not related to percents.

The result of `a % b` is the **remainder** of the integer division of `a` by `b`.

```js
alert(5 % 2); // 1, the remainder of 5 divided by 2
alert(8 % 3); // 2, the remainder of 8 divided by 3
alert(8 % 4); // 0, the remainder of 8 divided by 4
```

### 🔹 Exponentiation `**`

The **exponentiation operator** `a ** b` raises `a` to the power of `b`.

In school maths, we write that as `aᵇ`.

```js
alert(2 ** 2); // 2² = 4
alert(2 ** 3); // 2³ = 8
alert(2 ** 4); // 2⁴ = 16
```

Just like in maths, the exponentiation operator works with **non-integer numbers** as well.

For example, a **square root** is an exponentiation by `1/2`:

```js
alert(4 ** (1/2)); // 2 (power of 1/2 is the same as a square root)
alert(8 ** (1/3)); // 2 (power of 1/3 is the same as a cubic root)
alert(16 ** (1/4)); // 2 (power of 1/4 is the same as a cubic root)
```

### **Step-by-Step Breakdown:**

#### 1. **Understanding Exponents in General:**
Exponents are shorthand for repeated multiplication. Here’s a refresher:
- **a²** means **a × a** (multiply a by itself once).
- **a³** means **a × a × a** (multiply a by itself twice).
- **a⁴** means **a × a × a × a** (multiply a by itself three times), and so on.

#### 2. **What Do Fractional Exponents Mean?**
When you have a fraction as an exponent, it tells us:
- **Numerator (top part)**: How many times we multiply the number.
- **Denominator (bottom part)**: The root we are taking (square root, cube root, etc.).

For example:
- **a^(1/2)** means the **square root** of **a** because the denominator is **2** (2nd root).
- **a^(1/3)** means the **cube root** of **a** because the denominator is **3** (3rd root).
- **a^(1/4)** means the **fourth root** of **a** because the denominator is **4** (4th root).

#### 3. **Why is 1/2 Equal to the Square Root?**
- **a^(1/2)** is asking, "What number, when multiplied by itself, gives me **a**?"
- This is the definition of a square root! So **a^(1/2)** is a fancy way of saying **√a**.

---

### **Square, Cube, and Fourth Roots**

Roots are numbers that, when multiplied by themselves a certain number of times, give us the original number. Here’s how it works:

- **Square root (√)**: Find a number that, when multiplied by itself, equals the original number.
  - Example: **√9 = 3** because **3 × 3 = 9**.

- **Cube root (∛)**: Find a number that, when multiplied by itself **three times**, equals the original number.
  - Example: **∛8 = 2** because **2 × 2 × 2 = 8**.

- **Fourth root (∜)**: Find a number that, when multiplied by itself **four times**, equals the original number.
  - Example: **∜16 = 2** because **2 × 2 × 2 × 2 = 16**.

---

### **How to Calculate Them:**

- **Square root**: Use `√a` or `a ** (1/2)`.
- **Cube root**: Use `∛a` or `a ** (1/3)`.
- **Fourth root**: Use `∜a` or `a ** (1/4)`.

For exact roots, use a **calculator** or estimate between perfect squares, cubes, or fourths.

---

### **Example 1: Finding the Square Root of 9**

To find the square root of **9**, we know that we’re looking for a number that, when multiplied by itself, gives us **9**.

We start at 1 and go up to test numbers. Here's how:

- **1 × 1 = 1** (not 9, so keep going)
- **2 × 2 = 4** (still not 9)
- **3 × 3 = 9** (Aha! This is the answer!)

So, the square root of **9** is **3** because **3 × 3 = 9**.

---

### **Example 2: Finding the Cube Root of 8**

To find the cube root of **8**, we know that we’re looking for a number that, when multiplied by itself **three times**, gives us **8**.

We start at 1 and go up to test numbers. Here's how:

- **1 × 1 × 1 = 1** (not 8, so keep going)
- **2 × 2 × 2 = 8** (Aha! This is the answer!)

So, the cube root of **8** is **2** because **2 × 2 × 2 = 8**.

---

### **Example 3: Finding the Fourth Root of 16**

To find the fourth root of **16**, we know that we’re looking for a number that, when multiplied by itself **four times**, gives us **16**.

We start at 1 and go up to test numbers. Here's how:

- **1 × 1 × 1 × 1 = 1** (not 16, so keep going)
- **2 × 2 × 2 × 2 = 16** (Aha! This is the answer!)

So, the fourth root of **16** is **2** because **2 × 2 × 2 × 2 = 16**.

---

### **Non-Perfect Squares, Cubes, or Fourths:**
For numbers that aren’t perfect squares, cubes, or fourths, use a **calculator** to find an accurate root. For example, the square root of **7** or the cube root of **30** will not be whole numbers, so a calculator is needed for precise answers.
::

## String concatenation with binary `+`

In JavaScript, the `+` operator usually adds numbers, but if applied to strings, it **concatenates** them.

```js
let s = "my" + "string";
alert(s); // "mystring"
```

If one operand is a string, the other is converted to a string:

```js
alert('1' + 2); // "12"
alert(2 + '1'); // "21"
```

**Important:** When both numbers and strings are involved, operations happen left to right:

```js
alert(2 + 2 + '1'); // "41"
alert('1' + 2 + 2); // "122"
```

Unlike the `+` operator, other arithmetic operators (like `-`, `/`, `*`) convert operands to numbers:

```js
alert(6 - '2'); // 4
alert('6' / '2'); // 3
```

The `+` operator is the only one that works with strings in this way.

## Numeric Conversion with Unary `+`

The unary `+` converts non-numbers to numbers, but has no effect on numbers themselves:

```js
// No effect on numbers
let x = 1;
alert(+x); // 1

let y = -2;
alert(+y); // -2

// Converts non-numbers
alert(+true); // 1
alert(+"");   // 0
```

When handling form input, values are usually strings. To sum them as numbers:

```js
let apples = "2";
let oranges = "3";

// Converts strings to numbers and sums
alert(+apples + +oranges); // 5

// the longer variant
// alert( Number(apples) + Number(oranges) ); // 5
```

It’s equivalent to `Number()`, but shorter. `alert( Number(apples) + Number(oranges) ); // 5`

> The unary `+` is applied before the binary `+` due to its higher precedence, converting values before adding.

## Operator Precedence

_Operator precedence_ determines the order of operations in an `expression`. Higher-precedence operators are evaluated first.

For example, in `1 + 2 * 2`, multiplication (`*`) has a higher precedence than addition (`+`), so the multiplication is performed first.

You can change the default order using parentheses: `(1 + 2) * 2`.

Here’s a simplified part of the [precedence table](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Operator_precedence) (no need to memorize it, but note that unary operators have higher precedence than their binary counterparts):

| Precedence | Operator        | Example         |
|------------|-----------------|-----------------|
| 14         | Unary plus (+)  | `+x`            |
| 14         | Unary negation (-) | `-x`          |
| 13         | Exponentiation (**) | `2 ** 3`    |
| 12         | Multiplication (*) | `2 * 3`      |
| 12         | Division (/)    | `6 / 2`         |
| 11         | Addition (+)    | `2 + 3`         |
| 11         | Subtraction (-) | `5 - 3`         |
| 2          | Assignment (=)  | `x = 5`         |

## Assignment `=`

The **assignment operator (`=`)** in JavaScript is used to assign a value to a variable. It's listed with a low priority of **2** in the operator precedence table, meaning it is evaluated **after** arithmetic operations.

Example:

```javascript
let x = 2 * 2 + 1;
alert(x); // 5
```
Here, multiplication and addition happen first, then the result (5) is assigned to `x`.

### Assignment Returns a Value

Unlike many other operators, the _assignment_ operator returns the value that is assigned.

The call `x = value` writes the `value` into `x` and then returns it.

This means the result of an assignment can be used in further expressions.

```javascript
let a = 1;
let b = 2;

let c = 3 - (a = b + 1);

alert(a); // 3
alert(c); // 0
```

In this example, `(a = b + 1)` assigns 3 to `a`, and this value (3) is then used in the rest of the expression to compute `c`.

::caution
Funny code, isn’t it? We should understand how it works, because sometimes we see it in JavaScript libraries.

Although, please don’t write the code like that. Such tricks definitely don’t make code clearer or readable.
::

### Chaining Assignments

You can also chain assignments together, where the rightmost value is assigned to multiple variables:

Example:

```javascript
let a, b, c;
a = b = c = 2 + 2;

alert(a); // 4
alert(b); // 4
alert(c); // 4
```

Chained assignments are evaluated **from right to left**. So, `c` is assigned 4 first, then `b` and `a` also get 4.

For readability, it’s better to split chained assignments into multiple lines:

```javascript
c = 2 + 2;
b = c;
a = c;
```

## Modify-in-place

We often need to apply an operator to a variable and store the new result in that same variable.

For example:

```javascript
let n = 2;
n = n + 5;
n = n * 2;
```

This notation can be shortened using the operators `+=` and `*=`, like so:

```javascript
let n = 2;
n += 5; // now n = 7 (same as n = n + 5)
n *= 2; // now n = 14 (same as n = n * 2)

alert(n); // 14
```

Short “modify-and-assign” operators exist for all arithmetic and bitwise operators: `/=`, `-=`, etc.

Such operators have the same precedence as a normal assignment, so they run after most other calculations:

```javascript
let n = 2;

n *= 3 + 5; // right part evaluated first, same as n *= 8

alert(n); // 16
```

## Increment/Decrement

Increasing or decreasing a number by one is a common operation, and JavaScript has special operators for this:

- **Increment (`++`)**: Increases a variable by 1.

```javascript
let counter = 2
counter++ // same as counter = counter + 1
alert(counter) // 3
```

- **Decrement (`--`)**: Decreases a variable by 1.

```javascript
let counter = 2
counter-- // same as counter = counter - 1
alert(counter) // 1
```

### Prefix vs Postfix

::caution
**Increment/decrement can only be applied to variables**. Using it on a value like `5++` will throw an error.
::

- Both `++` and `--` can be used in **prefix** or **postfix** forms:
  - **Postfix** (`counter++`): Increments after returning the original value.
  - **Prefix** (`++counter`): Increments and returns the new value.

::warning
The difference between `++counter` or `counter++` is only when one of these expressions is included in an other expression(`alert()` for example)

- **Prefix:**

```javascript
let counter = 1
let a = ++counter // increments and assigns the new value
alert(a) // 2
```

- **Postfix:**

```javascript
let counter = 1
let a = counter++ // returns the old value before incrementing
alert(a) // 1
```
::


### Readability

The `++` and `--` operators have higher precedence than most arithmetic operators and can be used inside expressions as well:

```javascript
let counter = 1
alert(2 * ++counter) // 4 (prefix)
alert(2 * counter++) // 2 (postfix)
```
::tip
Combining them with other operators can reduce readability. It's better to follow a `"one line – one action"` style:

```javascript
let counter = 1
alert(2 * counter) // 2
counter++
```
::

## Bitwise `^`

Bitwise operators treat numbers as 32-bit integers and perform operations on their binary representation. These operators are common in many languages but are rarely needed in everyday web development. They're more useful in fields like cryptography.

**The operators are:**
- **AND (`&`)**
- **OR (`|`)**
- **XOR (`^`)**
- **NOT (`~`)**
- **Left Shift (`<<`)**
- **Right Shift (`>>`)**
- **Zero-fill Right Shift (`>>>`)**

For detailed information, you can explore the [Bitwise Operators](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Expressions_and_Operators#bitwise_operators) chapter on MDN when needed.

## Comma `,`

The **comma operator** is one of the rarest and unusual operators. It allows multiple expressions to be evaluated in one line, but only the result of the last expression is returned.

```javascript
// With parentheses
a = (1 + 2, 3 + 4); // a = 7
alert(a); // 7

// Without parentheses
a = 1 + 2, 3 + 4; // a = 3, the rest is ignored
alert(a); // 3
```
Here, `1 + 2` is evaluated and discarded, while `3 + 4` is returned.

::note
Comma has a very low precedence

- **Low Precedence:** Comma has very low precedence (lower than `=`), so parentheses are needed to control the evaluation order.
- **In Practice:** Often used in complex constructs, like in `for` loops:
```javascript
for (a = 1, b = 3, c = a * b; a < 10; a++) {
  // operations
}
```
Though useful in some frameworks, it usually makes code less readable, so use sparingly.
