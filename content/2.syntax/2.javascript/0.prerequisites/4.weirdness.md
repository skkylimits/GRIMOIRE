## 🔄 `==` vs `===`
- `==` does type coercion → `'0' == 0` is `true`
- `===` is strict → `'0' === 0` is `false`

## 💥 `NaN` weirdness
- `NaN === NaN` → `false`
- Best way to check: `Number.isNaN(value)`

## 🤯 `typeof null`
```js
typeof null // "object" ← historical bug, never fixed
```

## 🔍 Falsy values
These all become `false` in `if(...)`:
```js
false, 0, '', null, undefined, Number.NaN
```
Everything else is truthy (even `"false"` and `"0"` strings).

## 🧮 Weird math
```js
0.1 + 0.2 === 0.3 // false 🤯 (because of floating point precision)
```

## 💡 `[] + []` and friends
```js
[] + []           // ""
[] + {}           // "[object Object]"
{} + []           // 0 (depends on context)
```
