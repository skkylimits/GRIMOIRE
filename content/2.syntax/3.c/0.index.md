## Pointer to a buffer

### ⭐ Wat betekent “pointer to a buffer”? (Generieke C-uitleg)

In C betekent *“pointer to a buffer”* simpelweg:

> **Je geeft een functie een *stukje geheugen* waar zij informatie mag opslaan.**

C-functies maken dat geheugen dus **niet zelf aan** —
JIJ moet het eerst maken, en daarna het *adres* ervan doorgeven.

---

### ⭐ Stap 1 — Jij maakt de buffer

```c
char buffer[100];
```

Dit is een **doos** met plek voor 100 characters.

* `buffer` = de doos
* `&buffer[0]` = het adres (pointer)
* De inhoud is in het begin nog leeg

---

### ⭐ Stap 2 — Je geeft de buffer aan een functie

```c
readSomethingIntoBuffer(buffer, 100);
```

Wat gebeurt hier?

* `buffer` wordt automatisch omgezet naar **een pointer**
* De functie krijgt het adres van jouw doos
* De functie kan nu *in jouw geheugen schrijven*

---

### ⭐ Stap 3 — De functie vult de buffer

Bijvoorbeeld:

```c
void readSomethingIntoBuffer(char *buf, int size) {
    snprintf(buf, size, "Hello World!");
}
```

Nu staat in `buffer`:

```
H e l l o   W o r l d ! \0
```

---

### ⭐ Belangrijke regels bij buffers

#### ✔ De buffer moet groot genoeg zijn

Als de tekst langer is dan `size`, wordt hij afgekapt:

```
Original: "This is too long"
Buffer:   "This is to\0"
```

#### ✔ De buffer moet bestaan voordat je hem doorgeeft

Deze werkt dus NIET:

```c
char *buf;
writeInto(buf, 20); // ❌ fout, buf wijst nergens naartoe
```

Maar deze wel:

```c
char buf[20];
writeInto(buf, 20); // ✔ buf wijst naar 20 bytes geheugen
```

---

### ⭐ TL;DR

> **Een buffer is een doos.
> Een pointer naar een buffer is het adres van die doos.
> De functie krijgt het adres en stopt er iets in.
> Jij bepaalt hoe groot de doos is.**

---

### ⭐ Tutorial

```c
#include <stdio.h>

void fillMessage(char *buffer, int size) {
    snprintf(buffer, size, "This came from the function!");
}

int main() {
    char message[50]; // buffer = 50 characters
    fillMessage(message, sizeof(message));
    printf("Buffer contains: %s\n", message);
    return 0;
}
```

Output:

```
Buffer contains: This came from the function!
```

# 📦 **C Buffers in Praktijk — Wat doet `WCHAR szFileName[MAX_PATH * 2] = { 0x00 };`?**

In C zie je vaak code die er simpel uitziet, maar veel meer doet dan je denkt.
Dit is een perfect voorbeeld:

```c
WCHAR szFileName[MAX_PATH * 2] = { 0x00 };
```

Laten we dit stap voor stap ontleden.

---

## **1️⃣ We maken een buffer aan**

```c
WCHAR szFileName[MAX_PATH * 2];
```

Dit betekent:

* **Maak een array** van `WCHAR` (Unicode characters).
* Grootte = `MAX_PATH × 2` → een ruim blok geheugen.
* Deze array is onze **buffer**: een doos waar tekst in past.

Een `WCHAR` is 2 bytes, dus dit is genoeg voor lange UTF-16 strings.

---

## **2️⃣ De buffer wordt gevuld met nullen**

```c
= { 0x00 };
```

Dit is een speciale C-init die betekent:

> "Zet elk element in de array op nul."

In geheugen ziet de buffer er zo uit:

```
[0x0000][0x0000][0x0000][0x0000] ... (520 keer)
```

Dit geeft je:

* een **lege Unicode-string**
* met correcte null-terminator (`L"\0"`)
* en **géén oude rotzooi** uit RAM

Het is dezelfde werking als:

```c
memset(szFileName, 0, sizeof(szFileName));
```

---

## **3️⃣ Waarom is dit belangrijk?**

### ✔ **Veiligheid**

Zonder deze init bevat de buffer **random data** uit RAM → crashes, corrupte strings, undefined behavior.

### ✔ **Windows & C verwachten een null-terminated string**

Een lege wide string moet beginnen met `0x0000`.
Deze initialisatie garandeert dat.

### ✔ **Klaar voor gebruik als "out buffer"**

Veel functies willen een **leeg blok geheugen** waar ze info in mogen schrijven.
Bijvoorbeeld:

```c
GetModuleFileNameW(NULL, szFileName, MAX_PATH * 2);
```

Dit werkt alleen goed als de buffer schoon is.

---

## **4️⃣ Voorbeeld uit het dagelijks leven**

Stel je een **grote lege doos** voor waar iemand anders iets in moet leggen.

* De array → de doos
* `{0x00}` → de doos is volledig leeg
* Windows API → de persoon die er iets in stopt

Een gevulde doos werkt prima.
Een doos vol rommel werkt **niet**.

---

## **5️⃣ Mini-voorbeeld (algemeen C, geen Windows):**

```c
char message[100] = {0}; // alles nul → lege string

snprintf(message, 100, "Hello!"); // buffer wordt gevuld

printf("%s\n", message); // Hello!
```

```c

```

Betekent:

> **“Maak een grote Unicode-stringbuffer en vul hem helemaal met nullen zodat hij veilig en leeg is voordat we hem gebruiken.”**
