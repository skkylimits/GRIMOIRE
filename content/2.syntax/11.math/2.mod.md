🧠 De echte betekenis van `mod 256`

### 1️⃣ “mod” betekent: **rest na deling**

Niet “delen door”, maar:

> “Wat blijft er over als je deelt?”

Bijvoorbeeld:

```
14 ÷ 8 = 1 rest 6
→ 14 mod 8 = 6
```

Dat is de **rest** — wat overblijft na dat je 8 er één keer uit gehaald hebt.

### 2️⃣ Wat betekent dat voor RC4?

In RC4 heb je **256 stoelen**.
Ze zijn genummerd van `0` tot `255`.

```
[0, 1, 2, 3, ... 254, 255]
```

En dan komt `mod 256` om de hoek kijken:

> het zorgt dat je **nooit van de tafel valt**.

## 🎨 Denkbeeld: de kring van stoelen

Stel je voor:
je hebt een kring van **256 stoelen**.
Je begint bij stoel **0**.
Elke stap die je zet, ga je **één stoel verder**.

Als je **256 stappen** doet —
dan ben je **weer terug bij stoel 0**.

Dat is wat `mod 256` doet.
Hij zegt:

> “Elke keer dat je 256 rondjes maakt, begin je opnieuw bij 0.”

## ⚙️ Voorbeeld 1 — Klein getal

```
j = (0 + 0 + 75) mod 256
```

Dat is:

```
j = 75
```

Omdat 75 kleiner is dan 256,
past het prima op de tafel → stoel 75.


## ⚙️ Voorbeeld 2 — Groter getal

```
j = (236 + 3 + 75) mod 256
```

Tel dat op:

```
236 + 3 + 75 = 314
```

Nu komt de vraag:

> “Waar op de tafel met 256 stoelen kom ik terecht na 314 stappen?”

Je deelt 314 door 256:

```
314 ÷ 256 = 1 rest 58
```

👉 Dat betekent:

* Je hebt **1 volledige ronde van 256 stoelen** gelopen
* En je bent **58 stoelen verder** in de volgende ronde

Dus:

```
314 mod 256 = 58
```

oftewel: **je eindigt op stoel 58** 🪑

## 🌀 Zo kun je het altijd zien

| Wat je berekent | Uitleg in woorden                                                     |
| --------------- | --------------------------------------------------------------------- |
| `x mod 256`     | “Waar kom ik uit als ik x stappen doe op een cirkel van 256 stoelen?” |
| `0 mod 256`     | stoel 0                                                               |
| `255 mod 256`   | stoel 255                                                             |
| `256 mod 256`   | terug bij stoel 0                                                     |
| `257 mod 256`   | stoel 1                                                               |
| `314 mod 256`   | stoel 58                                                              |
| `512 mod 256`   | weer stoel 0 (2 rondjes)                                              |

## 🧩 Waarom 256 specifiek?

Omdat 256 = 2⁸ → dat past precies in **één byte**
(en RC4 werkt byte voor byte).

Dus elk getal dat groter wordt dan 255 “wrapt around” —
het **rolt over naar nul** in een kring.

## 🪄 Samenvatting in babytaal

```
mod = "de rest nadat je steeds 256 hebt afgetrokken"
```

Bijvoorbeeld:

```
314 mod 256
→ 314 - 256 = 58
→ resultaat: 58
```

Of als zin:

> “Ik heb 256 stoelen, ik loop 314 stappen,
> na één volledig rondje sta ik op stoel 58.”
