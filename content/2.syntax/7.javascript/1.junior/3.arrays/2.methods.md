## Add/Remove Items

We already know methods that add and remove items from the beginning or the end:

- `arr.push(...items)` ‚Äì adds items to the end,
- `arr.pop() ‚Äì extracts` ‚Äì an item from the end,
- `arr.shift() ‚Äì extracts` ‚Äì an item from the beginning,
- `arr.unshift(...items)` ‚Äì adds items to the beginning.

## Splice

::warning
How to delete an element from the array?
::

### üìö **Bookshelf**

`Arrays` are like shelves with labeled slots (indexes). If you use `delete`, it only clears the slot (leaves it `undefined`), but **the slot still exists** ‚Äî the array doesn‚Äôt shrink.

```js
let arr = ["I", "go", "home"];

delete arr[1]; // remove "go"

alert(arr[1]); // undefined
alert(arr.length); // 3
```

Picture this array as a 3-slot bookshelf:
```
[ "I" ] [ "go" ] [ "home" ]
   0       1         2
```

#### Now you do this:
```js
delete arr[1];
```

It‚Äôs like you just **remove the book from slot 1**, but you **leave the empty slot there**!
So now it looks like:
```
[ "I" ] [     ] [ "home" ]
   0       1       2
```

- üì¶ Slot 1 still exists ‚Äî it just has **`undefined`** in it.
- üßÆ The arr.length is still `3`, because you didn‚Äôt shrink the shelf ‚Äî just cleared one spot.
- ‚ùå Not very useful if you want the array to shift and adjust.

### üî™ Swiss army knife for arrays

The `splice()` method in JavaScript a and a way to **surgically edit arrays**. You can:
- ‚úÇÔ∏è **cut stuff out** (remove elements)
- ü™Ñ **put new stuff in** (insert elements)
- üîÅ **swap stuff around** (replace elements)

All with one powerful method:
```js
arr.splice(start, deleteCount, item1, item2, ...)
```

#### ‚úÖ The Right Way: Use `splice()` Instead

If you actually want to **remove** the item and shift everything, use this:
```js
arr.splice(1, 1);
```

Now your shelf adjusts:
```
[ "I" ] [ "home" ]
   0        1
```

- The "go" book is gone.
- The remaining books moved left.
- The shelf is now only 2 slots long ‚Äî just how we like it!

That‚Äôs why it‚Äôs preferred when you want to **truly change** the array, not just blank out values.

### **üìÑ Examples**

#### 1. **Remove an element**
```js
let arr = ['I', 'study', 'JavaScript']
arr.splice(1, 1)
```
üìç Start at index 1, üóë remove 1 item ‚Üí "study"
‚úÖ Result: `["I", "JavaScript"]`

#### 2. **Replace some elements**
```js
let arr = ['I', 'study', 'JavaScript', 'right', 'now']
arr.splice(0, 3, 'Let\'s', 'dance')
```
üìç Start at index 0
üóë Remove 3 items: "I", "study", "JavaScript"
‚ûï Add "Let's", "dance"
‚úÖ Result: `["Let's", "dance", "right", "now"]`

#### 3. **Just insert ‚Äî don‚Äôt delete**
```js
let arr = ['I', 'study', 'JavaScript']
arr.splice(2, 0, 'complex', 'language')
```
üìç Start at index 2
üóë Delete 0 items
‚ûï Insert "complex", "language"
‚úÖ Result: `["I", "study", "complex", "language", "JavaScript"]`

#### 4. **Use negative indexes (count from the end!)**
```js
let arr = [1, 2, 5]
arr.splice(-1, 0, 3, 4)
```
üìç Start at index -1 ‚Üí which means *one step from the end*
üóë Delete 0 items
‚ûï Insert 3, 4
‚úÖ Result: `[1, 2, 3, 4, 5]`

::tip
üßÝ **Tips to Remember**
- `splice()` **changes the original array**.
- You can both **remove and add** items in one go.
- The return value is an array of the items you removed.
- Negative index = count from the end (just like Python or slicing in some other languages).
::

::warning
**Delete vs Splice:**
- `delete` is like ripping the label off a slot ‚Üí the slot is still there but empty.
- `splice` is like cleanly pulling the book out and sliding the rest over ‚ú®
::

## Slice

The `slice()` method lets you **copy part of an array** into a **new array**.

> ‚ÄúHey computer, copy this chunk of the array for me. I‚Äôll tell you where to start and stop, and I don‚Äôt want to break the original.‚Äù


::tip
The original array stays **unchanged** ‚Äî slice doesn‚Äôt mess with the original array.
::

### üß± Imagine This Array:

```js
let arr = ["t", "e", "s", "t"];
// Indexes:   0    1    2    3
```

Every letter has a number (called an index). `arr[0]` is `"t"`, `arr[1]` is `"e"`, and so on.

---

#### üì¶ What `slice(start, end)` Does

It‚Äôs like saying:
> ‚ÄúGive me a *copy* of the stuff from position `start` up to, but **not including**, position `end`.‚Äù

Let‚Äôs try:

```js
arr.slice(1, 3);
```

This means:
- Start at index 1 ‚Üí that‚Äôs `"e"`
- Stop **before** index 3 ‚Üí so you don‚Äôt include the second `"t"`
- So you get: `["e", "s"]`

> It **copies** that part. It does *not* delete it. It does *not* change the original.

---

#### üåÄ Okay, What About Negative Numbers?

```js
arr.slice(-2);
```

That‚Äôs like saying:
> ‚ÄúStart 2 from the end, and give me everything from there.‚Äù

So:
- The last 2 letters are `"s"` and `"t"`
- You get: `["s", "t"]`

---

#### üëê And `arr.slice()` With No Numbers?

This is like:
> ‚ÄúCopy the whole array.‚Äù

So you get:
```js
["t", "e", "s", "t"]
```

That‚Äôs often used to obtain a copy for further transformations that should not affect the original array.

## Concat

`arr.concat(...)` makes a **new array** by adding things to the end of an existing array.
It **doesn‚Äôt change the original**, just gives you the combined result.

> Think of `concat()` as your **friendly glue gun** üîó It *sticks arrays and values together into a new one*, without messing up the originals.

```js
let arr = [1, 2];
```

Now if you do:

```js
arr.concat([3, 4])
```

You get:

```js
[1, 2, 3, 4]
```

Because it‚Äôs just:
‚Üí `[original stuff] + [new stuff]`

Same idea with more pieces:

```js
arr.concat([3, 4], [5, 6]) // [1, 2, 3, 4, 5, 6]
```

And yes, you can even throw in plain values:

```js
arr.concat([3, 4], 5, 6) // [1, 2, 3, 4, 5, 6]
```

### üßÉAdd a Weird Object?

```js
let arrayLike = {
  0: "something",
  length: 1
};

arr.concat(arrayLike) // [1, 2, {0: "something", length: 1}]
```

‚öÝÔ∏è That object just gets added as-is ‚Äî it‚Äôs not a real array, so it doesn‚Äôt get ‚Äúunpacked.‚Äù


#### üßô‚Äç‚ôÇÔ∏è But‚Ä¶ Magic Exists

If you give that object a special symbol like this:

```js
let arrayLike = {
  0: "something",
  1: "else",
  length: 2,
  [Symbol.isConcatSpreadable]: true
};
```

Now `concat()` treats it like a real array and spreads its elements:

```js
arr.concat(arrayLike) // [1, 2, "something", "else"]
```

‚ú® The Symbol.isConcatSpreadable tells concat: ‚ÄúYo, open me up like an array.‚Äù

---

::tip
Real-Life Use

- Combining lists of data (`[search results] + [more results]`)
- Appending things without modifying the original array
- Merging pieces together in one clean shot
::

## forEach()

> `forEach` is like saying:
> **‚ÄúHey, do this thing for every item in the array.‚Äù**

It's a *loop*, but it sounds nicer and doesn‚Äôt make you write a bunch of boilerplate.

### üì¶ The Setup

```js
arr.forEach(function(item, index, array) {
  // Do something with each item
});
```

What those things mean:
- `item`: the current element
- `index`: where you are in the array (0, 1, 2‚Ä¶)
- `array`: the whole array you‚Äôre looping through (in case you wanna reference it)

#### üßô‚Äç‚ôÇÔ∏è Example 1: The Simple One

```js
["Bilbo", "Gandalf", "Nazgul"].forEach(alert);
```

This is like saying:
> ‚ÄúHey alert(), show each of these names one by one.‚Äù

Behind the scenes, it does:
```js
alert("Bilbo");
alert("Gandalf");
alert("Nazgul");
```

---

#### üîç Example 2: With Some Extra Info

```js
["Bilbo", "Gandalf", "Nazgul"].forEach((item, index, array) => {
  alert(`${item} is at index ${index} in ${array}`);
});
```

It‚Äôs saying:
- Show the item (`Bilbo`)
- Where it is in the array (index `0`)
- And show the full array too

So the alert will say:
```
Bilbo is at index 0 in Bilbo,Gandalf,Nazgul
```
...and so on for the rest.

#### üí° Real-World Use

```js
let total = 0;
[1, 2, 3].forEach(num => total += num);
// total is 6
```

You‚Äôre just using it to *do something* (add up, display, store, etc) ‚Äî not to return anything.

::note
Note

- It loops automatically. You don‚Äôt need `for`, `while`, or counters.
- It **ignores return values**. You can‚Äôt use `forEach` to build a new array ‚Äî just to *do something* with each item (like show it, log it, change an outside variable, etc.)
- It **doesn‚Äôt stop early** ‚Äî it‚Äôll run the function on *every* item no matter what.
::

## Searching In Arrays

The methods we're talking about let you search for items in an array in different ways. Some methods just check if an item exists, others return the item or its index, and others even let you search in a more complex way. Here‚Äôs a quick rundown:

### 1. `indexOf` and `includes` üìç
- **`indexOf(item)`**: Finds the first occurrence of `item` and returns its index. If it doesn't find it, it returns `-1`.
- **`includes(item)`**: Checks if `item` exists in the array, and returns `true` or `false`.

**Why use them?**
- If you only care about whether the item exists, `includes` is simpler.
- If you need the index of where the item is, use `indexOf`.

#### Example:
```javascript
let arr = [1, 0, false];
alert(arr.indexOf(0)); // 1 (position of 0)
alert(arr.includes(false)); // true (false exists in the array)
```

::important
strict equality (===)

- Both methods use *strict equality (===)*, meaning `false` isn‚Äôt equal to `0`, and `NaN` isn't equal to `NaN` (which is a weird quirk in JavaScript).
::

::note
NaN

- A minor but noteworthy feature of the `includes` method is that it handles `NaN` correctly, unlike `indexOf`.

```javascript
const arr = [NaN];

alert(arr.indexOf(NaN));   // -1 (incorrect)
alert(arr.includes(NaN));  // true (correct)
```

**Why this happens**:
The reason `includes` works correctly with `NaN` is that it was added to JavaScript later and uses a more up-to-date comparison algorithm internally. On the other hand, `indexOf` uses the older algorithm, which doesn't consider `NaN` equal to itself.

::

### 2. `lastIndexOf` üï∞Ô∏è
This is like `indexOf`, but it searches from the right end of the array.

#### Example:
```javascript
let fruits = ['Apple', 'Orange', 'Apple'];
alert(fruits.indexOf('Apple')); // 0 (first 'Apple')
alert(fruits.lastIndexOf('Apple')); // 2 (last 'Apple')
```

### 3. `find` üßê
This method is super useful when you're searching for an item based on a condition (like an object with a specific property).

The syntax is:

```js
let result = arr.find(function(item, index, array) {
  // if true is returned, item is returned and iteration is stopped
  // for falsy scenario returns undefined
});
```
The function is called for elements of the array, one after another:

- item is the element.
- index is its index.
- array is the array itself.

::caution
Warning

If it returns `true`, the search is stopped, the item is returned. If nothing is found, `undefined` is returned.
::

#### Example:
```javascript
let users = [
  {id: 1, name: "John"},
  {id: 2, name: "Pete"},
  {id: 3, name: "Mary"}
];
let user = users.find(item => item.id === 1);
alert(user.name); // John
```
::note
**Why this is cool**: You can search for an object with a certain property. So if you have an array of people, you can easily find the first person with a specific `id`.
::

### 4. `findIndex` & `findLastIndex` üèÉ‚Äç‚ôÄÔ∏è
Both are similar to `find`, but instead of returning the item, they return **the index** of the item that matches the condition.

- **`findIndex(fn)`**: Returns the index of the first item that matches.
- **`findLastIndex(fn)`**: Like `findIndex`, but searches from **right to left**.

#### Example:
```javascript
let users = [
  {id: 1, name: "John"},
  {id: 2, name: "Pete"},
  {id: 3, name: "Mary"},
  {id: 4, name: "John"}
];

// Find the first index of 'John'
alert(users.findIndex(user => user.name == 'John')); // 0

// Find the last index of 'John'
alert(users.findLastIndex(user => user.name == 'John')); // 3
```

### 5. `filter` üßπ
If you need to **find multiple** items that match a condition, use `filter`. It returns an **array** of all matching items, not just the first one.

#### Example:
```javascript
let users = [
  {id: 1, name: "John"},
  {id: 2, name: "Pete"},
  {id: 3, name: "Mary"}
];

let someUsers = users.filter(item => item.id < 3);
alert(someUsers.length); // 2
```

::note
**Why use `filter`?** If you expect multiple results and want them all, `filter` is the way to go. It doesn‚Äôt stop after finding the first match like `find`.
::

## Transform An Array

In JavaScript, we have some awesome methods to `transform`, `reorder`, or `manipulate arrays`. These methods let you change data in arrays in smart ways, like `mapping values`, `sorting` them, or even `joining` them back together. Here's the rundown of each method:

- **Use `map`** to transform array elements.
- **Use `sort`** to reorder elements (just remember it defaults to string sorting).
- **Use `reverse`** to flip the array around.
- **Use `split` and `join`** to convert between strings and arrays.
- **Use `reduce`** to combine elements into a single value.

### 1. `map` - Transform Each Item ‚ú®

The arr.[map()](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/map) method lets you **transform** each item in the array by applying a function to it. It returns a **new array** with the results of that transformation.

The syntax is:

```js
let result = arr.map(function(item, index, array) {
  // returns the new value instead of item
});
```

For instance, here we transform each element into its length:

```javascript
let lengths = ["Bilbo", "Gandalf", "Nazgul"].map(item => item.length);
alert(lengths); // [5, 7, 6]
```
::note
In the example, we're taking each name in the array and replacing it with the length of that name (i.e., how many characters it has). Super handy for transforming data!
::

### 2. `sort` - Reorder the Array üîÄ
The arr.[sort()](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/sort) method rearranges the elements in an array, but there's a trick. By default, it **sorts as strings**, so numbers might not behave the way you expect!

**Example:**
```javascript
let arr = [ 1, 2, 15 ];
arr.sort(); // sorts as strings
alert(arr); // [1, 15, 2]
```

#### Why does this happen? ü§î
JavaScript converts all elements to strings for comparison. So, "2" is considered bigger than "15" because of **lexicographic order** (like dictionary order).

**Fixing it:**
You can **provide a comparison function** to tell JavaScript how to sort the elements. For numbers, the function looks like this:

```javascript
function compareNumeric(a, b) {
  if (a > b) return 1;
  if (a < b) return -1;
  return 0;
}

let arr = [ 1, 2, 15 ];
arr.sort(compareNumeric);
alert(arr); // [1, 2, 15]
```

Or even shorter with an **arrow function**:
```javascript
arr.sort((a, b) => a - b);
alert(arr); // [1, 2, 15]
```

Use `localeCompare` for **strings**:

::note
Remember strings comparison algorithm? It compares letters by their codes by default.

For many alphabets, it‚Äôs better to use str.localeCompare method to correctly sort letters, such as √ñ.

::

```javascript
// For example, let‚Äôs sort a few countries in German:
let countries = ['√ñsterreich', 'Andorra', 'Vietnam'];

alert( countries.sort( (a, b) => a > b ? 1 : -1) ); // Andorra, Vietnam, √ñsterreich (wrong)

alert( countries.sort( (a, b) => a.localeCompare(b) ) ); // Andorra,√ñsterreich,Vietnam (correct!)
```

### 3. `reverse` - Flip the Array üîÑ
If you want to **reverse the order** of elements, `reverse` is your friend. It flips the entire array around.

**Example:**
```javascript
let arr = [1, 2, 3, 4, 5];
arr.reverse();
alert(arr); // [5, 4, 3, 2, 1]
```

### 4. `split` and `join` - Convert Between Strings and Arrays üîó
These two methods let you convert between a string and an array.

- **`split(delim)`**: Splits a string into an array based on a delimiter (e.g., a comma or space).
- **`join(glue)`**: Takes an array and joins its elements back into a string, with a separator between them.

**Example (`split`):**
```javascript
let names = 'Bilbo, Gandalf, Nazgul';
let arr = names.split(', ');
alert(arr); // ["Bilbo", "Gandalf", "Nazgul"]
```

**Example (`join`):**
```javascript
let arr = ['Bilbo', 'Gandalf', 'Nazgul'];
let str = arr.join('; ');
alert(str); // "Bilbo; Gandalf; Nazgul"
```

### 5. `reduce` and `reduceRight` - Combine Array Values üî¢
These methods are powerful tools for **accumulating** or **reducing** the values in an array into a single value (like a sum, or concatenated string).

#### Syntax:
```javascript
let result = arr.reduce(function(accumulator, item) {
  // do something with item
}, initialValue);
```

- **`accumulator`**: Keeps track of the accumulated result (starts as `initialValue`).
- **`item`**: The current item in the array.

#### Example:
```javascript
let arr = [1, 2, 3, 4, 5];
let sum = arr.reduce((acc, num) => acc + num, 0);
alert(sum); // 15
```

Here‚Äôs what happens:
- First, `acc` is 0 (the initial value).
- Then, the function adds each number in the array to `acc`.
- The final result is the sum of all numbers.

::note
Key Notes üìù

- **`reduceRight`** works like `reduce`, but it goes from **right to left** through the array.
- You can also **omit the initial value** in `reduce`. If you do that, it will use the first element of the array as the starting point. Just be careful with empty arrays ‚Äî if you don't provide an initial value, calling `reduce` on an empty array will throw an error!
::

## `Array.isArray`


Arrays in JavaScript are based on objects, which means the `typeof` operator can't distinguish between a plain object and an array:

**Example:**
```javascript
alert(typeof {}); // "object"
alert(typeof []); // "object" (same as an object)
```

Since arrays are so commonly used, JavaScript provides a special method to check if a value is an array: `Array.isArray(value)`. This method returns `true` if the value is an array, and `false` otherwise.

**Example:**
```javascript
alert(Array.isArray({})); // false
alert(Array.isArray([])); // true
```

## Most methods support `thisArg`

Almost all array methods that call functions ‚Äì like `find`, `filter`, `map`, with a notable exception of `sort`, accept an optional additional parameter `thisArg`.

::caution
That parameter is not explained in the sections above, because it‚Äôs rarely used. But for completeness, we have to cover it.
::

Imagine we're building a simple video game and we have a **hero** character. The hero has a **name** and a **health** level. We want to perform some actions with the hero, but sometimes we need a special instruction (or "context") to make sure the right things happen.

### **What is `thisArg`?**

**`thisArg`** is like saying, *"Hey, use this object instead of the default one when doing something!"* Think of it as passing a friend to help your function know who it should be talking to. Without `thisArg`, a function might get confused about who it‚Äôs supposed to interact with.

#### Here's a simple example:

Imagine a **Hero** object and a **task** function:

```javascript
let hero = {
  name: 'Knight',
  health: 100,
  attack: function() {
    console.log(`${this.name} attacks!`);
  }
};
```

In the example above, `this` inside `attack()` refers to the **hero** object, because the function is inside the hero object. If we call the function using `hero.attack()`, it works because `this` is the hero.


### Without `thisArg` (problem):

Now, let's say we have an array of heroes, and we want to tell each hero to attack. If we just use `.map()` or `.forEach()` without passing `thisArg`, it won't work because the function will get confused about which `this` to use.

```javascript
let heroes = [hero, hero];  // Array with two heroes

heroes.forEach(hero => hero.attack());  // Works fine because the function is inside the hero object.
```

But what if the function isn‚Äôt directly inside the object? It gets trickier. ü§î

### Introducing `thisArg`:

Let‚Äôs say we have a **game** object, and we want to tell a list of heroes to attack, but we want the `attack` function to know it's from the **game**, not just any random object.

Here‚Äôs the **game** object:

```javascript
let game = {
  name: 'Battle Royale',
  heroAttack: function(hero) {
    console.log(`${this.name} tells ${hero.name} to attack!`);
  }
};
```

Now, if we use `.forEach()` to loop over the heroes and call `heroAttack()`, the function will refer to the **game** object (because `this` will be the game, not the hero). To fix that, we use `thisArg`.

### How `thisArg` fixes the problem:

```javascript
let heroes = [
  { name: 'Knight', health: 100 },
  { name: 'Archer', health: 80 }
];

game.heroAttack = function(hero) {
  console.log(`${this.name} tells ${hero.name} to attack!`);
};

// Using `thisArg` to fix the issue
heroes.forEach(function(hero) {
  game.heroAttack.call(game, hero);  // Manually specify 'this' as the 'game' object
}, game);  // Here, 'game' is the 'thisArg'
```

**What‚Äôs happening here:**
- We call `heroAttack()` on each hero, but since it's not inside the game object, we need to tell it that `this` should refer to **game**, not just the global object.
- We use `.call(game, hero)` to pass `thisArg` (the **game** object), ensuring that inside `heroAttack()`, `this` refers to the game, and we can use `this.name` (the game's name).

### Super Simple Example with `thisArg`:

Let‚Äôs simplify this even more. Imagine you‚Äôre telling a **group of friends** to do something, but you need to remind them who‚Äôs giving the command. You‚Äôd pass yourself as `thisArg` to remind them. üíÅ‚Äç‚ôÇÔ∏è

```javascript
let teacher = {
  name: 'Mr. Smith',
  tellStudents: function(student) {
    console.log(`${this.name} says: Hello, ${student.name}!`);
  }
};

let students = [
  { name: 'Alice' },
  { name: 'Bob' }
];

// This will throw an error because `this` inside `tellStudents` will be undefined, not 'Mr. Smith'.
students.forEach(student => teacher.tellStudents(student));

// We fix that by passing `thisArg` (the teacher) as the context:
students.forEach(function(student) {
  teacher.tellStudents.call(teacher, student);  // 'this' is explicitly set to 'teacher'
});
```

**In this case**:
- `teacher.tellStudents.call(teacher, student)` makes sure that when `tellStudents()` runs, it knows to refer to `teacher` (Mr. Smith) and not some other random object.

### Why Use `thisArg`?

You don‚Äôt always need to use `thisArg`, but it‚Äôs useful when you have a function that relies on an object‚Äôs properties (like `name` or `age`), and you want to make sure it uses the correct object (or "context") when it runs.

#### Simple Breakdown:

1. **Without `thisArg`**: JavaScript will automatically use a default object as `this` (which might be undefined or the global object). This can lead to errors or unexpected behavior.

2. **With `thisArg`**: You can specify which object JavaScript should use for `this`, making sure the function behaves as you expect.
