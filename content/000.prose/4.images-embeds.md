## Image

### Using SVG

> In your use case, if you want more specific control over how the image is presented (like in your detailed Git workflow document), the HTML `<img>` tag would give you the flexibility needed.

::code-preview
<img
	src="/DAN-DAN-DAN.jpg"
	alt="DAN-DAN-DAN"
	style="border: 1px solid #ff5859"
	loading="lazy"
/>

#code
```markdown
<img
	src="/DAN-DAN-DAN.jpg"
	alt="DAN-DAN-DAN"
	style="border: 1px solid #ff5859"
	loading="lazy"
/>
```
::

::note
If [`@nuxt/image`](https://image.nuxt.com/get-started/installation) is installed, the `<NuxtImg>` component will be used instead of the native `img` tag.
::

### Using Markdown

Display images using standard Markdown syntax. Markdown images are simple to implement for basic image display.

::code-preview
![DAN-DAN-DAN](/content/1.the-lab/1.the-lab/13.prose/DAN-DAN-DAN-BW.webp)

#code
```mdc
![img](/link/to/img)
```
::

::note
If [`@nuxt/image`](https://image.nuxt.com/get-started/installation) is installed, the `<NuxtImg>` component will be used instead of the native `img` tag.
::

### Using Embeds

Use embeds for more image customization. Embeds offer greater control over image attributes like size and styling.

::code-preview

## SLIDES

> The URL Path/Endpoint:/embed part of the URL is called the "embed mode" or "embed URL". It is a specific URL path used by Google to provide an embeddable version of its documents, including Google Slides, Google Docs, Google Sheets, etc.

```html
<iframe
	src="https://docs.google.com/presentation/d/1QdRlmAGxmm4StYfHMSn-oLFmkR5kiceQfAmkIZXAjOA/embed"
	width="100%"
	height="400"
>
</iframe>
```
