## Why

It unifies on-premises and cloud identities, letting users sign in once for both environments. It simplifies deployment, supports the latest features, and replaces older tools like DirSync and Azure AD Sync.

## Hybrid Identity

Today, businesses use a mix of on-premises and cloud apps, requiring users to access both. Managing these users is complex.

Microsoft’s hybrid identity solutions provide a common user identity for authentication and authorization across on-premises and cloud resources.

![scenario-1](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/0.entra-connect/scenario-1.png)

## Provisioning

Businesses now use a mix of on-premises and cloud apps, requiring a single identity across both.

**Provisioning automates creating, updating, and deleting user accounts based on conditions**.

> For example, when a new employee is added in HR, provisioning creates their accounts in Active Directory, cloud, and apps, ensuring access from day one.

![cloud-1](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/0.entra-connect/cloud-1.png)

### HR-driven Provisioning

HR provisioning automatically creates, updates, disables, or reactivates user accounts in Active Directory, Microsoft Entra ID, and cloud apps based on HR data.

![cloud-2](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/0.entra-connect/cloud-2.png)

### App Provisioning

In Microsoft Entra ID, app provisioning automatically creates, updates, and removes user identities and roles in cloud apps like Dropbox, Salesforce, and ServiceNow based on user status and roles.

![cloud-3](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/0.entra-connect/cloud-3.png)

### (Inter)-directory Provisioning

**Directory provisioning** syncs users from on-prem sources (like Active Directory) to Microsoft Entra ID using tools like `Entra Connect Sync`, `Cloud Provisioning`, or `Microsoft Identity Manager`.

**Inter-directory provisioning** refers more broadly to syncing identities between different directory systems—commonly from Active Directory to Entra ID, but potentially between other systems too.

![cloud-4](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/0.entra-connect/cloud-4.png)

## AuthN Options

Strong authentication is key for securing cloud access and protecting Microsoft Entra ID resources. It ensures safe user access and forms the foundation of modern IT security.

::caution
Choosing the right method, set up via Microsoft Entra Connect, is a key first step. Consider factors like time, infrastructure, complexity, and cost, which vary by organization and may change over time.
::

### Out of scope

This article excludes organizations without on-premises directories. Such businesses use cloud-only identities that don’t require hybrid identity solutions.

### Cloud

**Cloud Authentication** lets Microsoft Entra ID handle user sign-ins with single sign-on for seamless access. It offers two options:

* **Password Hash Sync**: Syncs on-premises password hashes to Entra ID, allowing users to sign in with the same credentials without extra infrastructure `validating passwords in the cloud`. Required for some premium features.

* **Pass-through Authentication**: `Validates passwords directly on-premises` AD via an agent, keeping validation off the cloud. Ideal for enforcing on-premises policies instantly.

### Federated

**Federated Authentication** hands off user sign-in to a trusted system like on-premises AD FS, which can support advanced features like third-party MFA.

* Microsoft Entra ID can either:
	- `Validate sign-in itself in the cloud`
	- `Or delegate to on-premises providers.`
* Use federated authentication if you need on-premises policies (account lockout, expiration, sign-in hours) enforced at sign-in.
* Some features, like third-party auth or multi-site on-premises auth, require federation.

::warning
Even with federated authentication, **Password Hash** Sync is still needed for Identity Protection reports and as a backup if federation fails which requires Microsoft Entra ID P2 licenses.
::

## Sync

### MIM

**Microsoft Identity Manager** (MIM) is a tool that helps you manage users, passwords, rules, and access in your organization. It also helps connect and sync identities between your on-premises Active Directory, Microsoft Entra ID, and other directories to support hybrid identity setups

### Entra Connect Sync

**Microsoft Entra Connect Sync** is a comprehensive hybrid identity solution that synchronizes users, groups, and passwords from your on-premises Active Directory to Microsoft Entra ID. It provides deep integration and supports features like password hash synchronization, pass-through authentication, and federation. Key features include:

* **Ideal for single-forest or simple multi-forest environments with connected AD infrastructure**
* Supports rich sign-in options including:
	- **Password Hash Sync**: Syncs password hashes from on-prem AD to Entra ID.
	- **Pass-through Auth**: Uses on-prem password without federated setup.
	- **Federation**: Optional AD FS integration for hybrid setups and management.
* **Synchronization**: Syncs users, groups, and passwords between on-prem and cloud.
* **Health Monitoring**: Tracks sync and auth health in Entra admin center.
* **Centralized configuration and management** via a single tool for deployment and updates

![microsoft-entra-cloud-hybrid-446945fc-4b860688](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/5.synchronization/microsoft-entra-cloud-hybrid-446945fc-4b860688.png)

### Entra Cloud Sync

**Microsoft Entra Cloud Sync** is a new solution for hybrid identity that syncs users, groups, and contacts to Microsoft Entra ID using lightweight cloud provisioning agents instead of the traditional Entra Connect app. It can work alongside Entra Connect Sync and offers:

* **Sync from multi-forest, disconnected AD environments (e.g., mergers, acquisitions)**
* **Easy installation** with cloud-managed sync configuration
* **Multiple agents** for high availability, especially for password hash sync
* **Support for large groups** (up to 50,000 members) using OU scoping filters for better performance

![architecture-2](/content/7.knowledge-base/6.operating-systems/5.entra-id/6.entra-connect/5.synchronization/architecture-2.png)

### Connect Vs Cloud

* **Entra Connect** is ideal for **single-forest** environments or simpler setups where you want deep on-premises integration and features like password hash sync, pass-through auth, or federation.
* **Cloud Sync** shines in **multi-forest** or complex, disconnected AD environments, offering lightweight, cloud-managed sync agents that handle multiple forests easily.

::tip
So, single forest → Entra Connect, multi-forest → Cloud Sync — that’s a solid rule of thumb!
::
