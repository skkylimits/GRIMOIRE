### 🤔 What is `requestAnimationFrame`?

It’s a **built-in browser function** that tells the browser:

> “Hey, run this function just before the next repaint/frame.”

This makes it **perfect** for animations and updates that need to stay in sync with the screen refresh rate (usually 60 frames per second).

---

### 📦 How it's used in this project (in `Time.js`)

```js
window.requestAnimationFrame(() => {
	this.RAF()
})
```

This is what starts the **loop**. The method `RAF()` (short for Request Animation Frame) will:

1. Measure time elapsed since the last frame.
2. Emit the `'RAF'` event to tell everyone: “a frame just passed.”
3. Call itself again to repeat.

So `Time.js` becomes this never-ending **heartbeat**:

```js
RAF() {
  // ⏱ Measure timing
  this.delta = ...
  this.elapsed = ...

  // 📣 Trigger the 'RAF' event
  this.trigger('RAF')

  // 🔁 Loop again on next frame
  window.requestAnimationFrame(() => {
    this.RAF()
  })
}
```

### 🔁 What Happens When `'RAF'` Triggers?

Inside `Experience.js`, you’ve got this:

```js
this.time.on('RAF', () => {
	this.update()
})
```

So **every frame**, `Time` says:

> "It’s time to update!" 👏

And the `Experience` responds with:

```js
update() {
  this.camera.update()
  this.renderer.update()
  this.world.update()
  this.ui.update()
}
```

Which makes:

* the camera move 🎥
* the scene render again 🖼️
* any animations play out 🎭
* UI or world interactions happen in real time 🧙‍♂️

### 🔄 Why Not Just Use `setInterval`?

Because `requestAnimationFrame` is **better optimized**:

| Feature                        | `setInterval()`    | `requestAnimationFrame()` |
| ------------------------------ | ------------------ | ------------------------- |
| Syncs with screen              | ❌ No               | ✅ Yes                     |
| Skips frames when tab inactive | ❌ Keeps running    | ✅ Pauses automatically    |
| Consistent framerate           | ❌ Sometimes choppy | ✅ Smoother                |

So `requestAnimationFrame` makes your animations **look better** and **run smarter**.

### 💡 The Standard Pattern (Used in Many Projects)

This pattern is seen all over modern 3D/animation setups:

```js
class Time {
	constructor() {
		this.raf = this.raf.bind(this)
		window.requestAnimationFrame(this.raf)
	}

	raf() {
		// update time calculations

		this.trigger('tick') // or 'RAF', 'update', etc.

		window.requestAnimationFrame(this.raf)
	}
}
```

Then in your main experience:

```js
this.time.on('tick', () => {
	this.update()
})
```

It’s clean, modular, and keeps everything **in sync** — which is key for real-time graphics or immersive UI.

### 🧩 Bonus: It’s Decoupled (thanks to EventEmitter)

Notice how the `Time` class **doesn’t need to know** who’s listening. It just goes:

> "Hey, a frame passed!" every time.

Anyone (like the camera, renderer, or world) can listen in, making it super modular and flexible.
