**Access Control Mechanisms** are systems and protocols that manage and restrict access to resources based on predefined policies and criteria, ensuring only authorized users can access sensitive data and systems.

## **Core Elements**

1. **Role-Based Access Control (RBAC)**:
   - **Definition**: Controls access by assigning roles with specific permissions to users.
   - **Implementation**: Managed in directories like AD or cloud platforms like Azure AD.

2. **Attribute-Based Access Control (ABAC)**:
   - **Definition**: Grants access based on user and resource attributes.
   - **Implementation**: Utilized in cloud services like AWS IAM and security management tools.

3. **Access Control Lists (ACLs)**:
   - **Definition**: Lists specifying user permissions for resources.
   - **Implementation**: Configured at the resource level in operating systems.

4. **Policy-Based Access Control (PBAC)**:
   - **Definition**: Manages access through predefined policies.
   - **Implementation**: Administered via centralized systems or application settings.
