## Installation

### **1. Install PHP**
   - **Download PHP** from the [official PHP website](https://windows.php.net/download/). Be sure to select the **VS17 x64 Non-Thread Safe** version (as of 2025-Feb-11 16:29:52).

   - After downloading the ZIP archive, **extract the contents** to a folder named `C:\php`. You should have the PHP files inside this folder, such as `php.exe`, `php.ini`, and others.

   - **Make PHP Available Globally in PowerShell:**
     To run PHP from any terminal (e.g., `php -v`), you need to add the `C:\php` folder to your system's environment variables:

     1. Open **PowerShell** as Administrator.
     2. Run the following command to append `C:\php` to your **System Path**:
        ```powershell
        [System.Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\php", [System.EnvironmentVariableTarget]::Machine)
        ```
     3. To confirm PHP is correctly installed and available globally, run:
        ```powershell
        php -v
        ```
        You should see the PHP version details printed to the screen.

### **2. Configure XAMPP**

::steps{level="4"}

#### Download and Install XAMPP
If you haven’t installed XAMPP yet, download it from [Apache Friends](https://www.apachefriends.org/index.html) and install it on your system.

#### Modify Apache SSL Configuration
To change the default ports, open the **Apache SSL Configuration** file:

📂 **File Location:**
- `xampp\apache\conf\extra\httpd-ssl.conf` (or `https-ssl.config`, depending on your XAMPP version).

🔧 **Update the following lines:**

1️⃣ **Change the default HTTP port (from 80 to 8080):**
   ```apache [httpd.conf]
   Listen 8080
   ServerName localhost:8080
   ```

2️⃣ **Change the default HTTPS port (from 443 to 4433):**
   ```apache [httpd-ssl.conf]
   Listen 4433
   <VirtualHost _default_:4433>
       ServerName localhost:4433
   </VirtualHost>
   ```

Save the file and restart Apache from the XAMPP control panel.

#### Enable SQLite3 in Apache
By default, SQLite3 may be disabled in PHP. To enable it:

📂 **Open PHP Configuration:**
1. In the XAMPP Control Panel, click **Config** under Apache.
2. Select **PHP (php.ini)** to open the configuration file.
3. Find this line:
   ```ini [php.ini]
   ;extension=sqlite3
   ```
4. Remove the `;` at the beginning to uncomment it:
   ```ini
   extension=sqlite3
   ```
5. Save the file and restart Apache.

#### Debug Port Issues
If Apache fails to start, another application may be using ports **80** or **443**. Use these commands to check and free up the ports:

1️⃣ **Find processes using port 80:**
   ```sh
   netstat -ano | findstr :80
   ```
2️⃣ **Find processes using port 443:**
   ```sh
   netstat -ano | findstr :443
   ```
3️⃣ **Kill the process using the port (replace `<PID>` with the actual process ID from the previous command):**
   ```sh
   taskkill /PID <PID> /F
   ```

After stopping the conflicting process, restart Apache in XAMPP.

✅ Your XAMPP server is now configured with ports `8080` (HTTP) and `4433` (HTTPS), with SQLite3 enabled and port conflicts resolved. 🚀

::

### **3. Access Phishing Server**

1. Clone the **365-Stealer** repository:

   ```bash
   git clone https://github.com/AlteredSecurity/365-Stealer.git .
   rm -rf .git
   ```

2. Once Apache is configured and running, open your browser and navigate to:
   `https://localhost:4433/`.

3. You will likely see a security warning because of the self-signed certificate. To bypass it, click **Advanced** and select **Proceed to localhost (unsafe)**.

4. Navigate to the `/yourVictims` directory to access your phishing page.

   ![yourVictims](/content/7.knowledge-base/8.cyber-crusades/6.cloud/4.enumeration/6.365-stealer/yourVictims.png)

### **4. Run 365-stealer.py**

### **4. Start Capturing Tokens**

To begin capturing tokens, open a terminal and navigate to the **365-Stealer** directory, which should be located in `C:\xampp\htdocs`. Then run the following command:

```cmd[C:\xampp\htdocs]
python 365-Stealer.py --run-app --no-ssl
```

Alternatively, you can start the script directly through the management portal by navigating to:

```http
https://localhost:4433/yourVictims
```

![run-365-stealer](/content/7.knowledge-base/8.cyber-crusades/6.cloud/4.enumeration/6.365-stealer/run-365-stealer.png)

### **5. (Optional) CA**
   - Created certificate. For added security, you can create a **self-signed SSL certificate** for your local server. This is optional, but it's recommended for secure interactions. Here's a helpful [guide](https://www.sslshopper.com/article-how-to-create-a-self-signed-certificate-for-a-localhost-server.html) on generating one.

### **6.Phishing Link**
   - Once the server is running, copy the phishing link (generated by the script) and paste it into an email or share it with your target. When the victim grants consent, their tokens will be captured.

