title: Using Tokens
description: Use them wisely.
---

When working with **Azure REST APIs**, authentication is done using **access tokens**. The two primary REST API endpoints used are:
- **Azure Resource Manager (ARM)**: `https://management.azure.com/`
- **Microsoft Graph (MS Graph)**: `https://graph.microsoft.com/` *(Note: The deprecated AAD Graph API was `https://graph.windows.net/`.)*

This document provides simple PowerShell examples to authenticate and interact with these APIs.

## Requesting Token

**Azure PowerShell**

```powershell
$passwd = ConvertTo-SecureString "Inloggen01!" -AsPlainText -Force
$creds = New-Object System.Management.Automation.PSCredential ("test@qhps.nl", $passwd)
Connect-AzAccount -Credential $creds
```

**Using an existing session (ARM):**
- If a user is logged in, retrieve their access token via:
```powershell
$token = (Get-AzAccessToken).Token
```

**For Microsoft Graph (Entra ID Access):**
```powershell
$msgraphtoken = (Get-AzAccessToken -ResourceTypeName MSGraph).Token
```

**Use multiple access tokens for different services:**
```powershell
Connect-AzAccount -AccountId test@qhps.nl -AccessToken $token -MicrosoftGraphAccessToken $msgraphtoken
```

**Supported Endpoints**

| **Azure Service**           | **Command** |
|----------------------------|------------|
| **Microsoft Graph (MSGraph)** | `Get-AzAccessToken -ResourceTypeName MSGraph` |
| **Azure AD Graph (Deprecated)** | `Get-AzAccessToken -Resource "https://graph.windows.net"` |
| **Azure Resource Manager (ARM)** | `Get-AzAccessToken -ResourceTypeName Arm` |
| **Analysis Services** | `Get-AzAccessToken -ResourceTypeName AnalysisServices` |
| **Key Vault** | `Get-AzAccessToken -ResourceTypeName KeyVault`  |
| **Data Lake** | `Get-AzAccessToken -ResourceTypeName DataLake`  |
| **Batch Services** | `Get-AzAccessToken -ResourceTypeName Batch` |
| **Attestation** | `Get-AzAccessToken -ResourceTypeName Attestation` |
| **Azure Storage** | `Get-AzAccessToken -ResourceTypeName Storage` |
| **Synapse Analytics** | `Get-AzAccessToken -ResourceTypeName Synapse` |

> You can use the full resource url aswell, like https://graph.microsoft.com instead of MSGraph

## **Microsoft Graph API**
To interact with **Microsoft Graph**, obtain an **MS Graph access token** and pass it in the request headers.

- **List All Users**

```powershell
$Token = $msgraphtoken
$URI = "https://graph.microsoft.com/v1.0/users"

$RequestParams = @{
    Method  = "GET"
    Uri     = $URI
    Headers = @{
        "Authorization" = "Bearer $Token"
    }
}

(Invoke-RestMethod @RequestParams).value
```

## **Azure Resource Manager**
To interact with **Azure Resource Manager**, obtain an **access token** and pass it in the request headers.

::note
Make sure the token has the right permissions. Else you'd get no results using the following call.
::

- **List All Subscriptions**

```powershell
$Token = $token
$URI = "https://management.azure.com/subscriptions?api-version=2020-01-01"

$RequestParams = @{
    Method  = "GET"
    Uri     = $URI
    Headers = @{
        "Authorization" = "Bearer $Token"
    }
}

(Invoke-RestMethod @RequestParams).value
```
