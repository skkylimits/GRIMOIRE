![grc-framework-v3-inline](/content/1.the-lab/2.core-principles/13.governance-risk-compliance/grc-framework-v3-inline.png)

## **Governance**

Governance is all about making rules and setting up processes to control what people in the company can do. It decides who can use what resources, when they can use them, and what permissions they have.

## **Risk**

Risk management is about finding and dealing with things that could harm the company or its customers. Risks can come from outside the company, like storms or hacking, or inside, like mistakes or fraud.

## **Compliance**

Compliance means following laws and rules set by governments. It explains what data needs protection, how to handle it, and what happens if you don’t follow the rules.

* **Data Residency**: Rules about where data can be stored and moved, based on the country or region.
* **Data Sovereignty**: Laws about data depending on where it’s collected, stored, or processed.
* **Data Privacy**: Laws that require companies to be clear about how they collect, use, and share personal information.

::note
Compliance isn't the same as security, but security is key to meeting compliance standards.
::

## ID Protection

ID Protection helps organizations detect, investigate, and remediate identity-based risks. It leverages advanced machine learning to analyze user and sign-in patterns based on signals from various sources such as IP address, geolocations, devices, and more.

## Access Reviews

::caution
Excessive access rights are a known security risk. However, when people move between teams, or take on or relinquish responsibilities, access rights can be difficult to control.
::

Access reviews help make sure only the right people can access apps and groups. This keeps your data safe by removing access from people who don’t need it anymore.

Entra lets both your employees and guests use apps and devices, so it’s important to check access regularly.

Common reasons to do access reviews:

* **Check admins:** Make sure only the right users have admin access.
* **Protect important data:** Ask users to confirm they still need access.
* **Manage exceptions:** Keep track of any special access permissions.
* **Review guests:** Group owners check if guests still need access.
* **Set regular reviews:** Schedule reviews that happen automatically, with reminders and easy approvals.

### Reviews

**Manage access with reviews**

Access reviews let you check if users and guests still need access. Reviewers confirm or remove access based on Microsoft Entra ID suggestions.

Admins track progress, and changes apply only after reviews finish. Some groups need manual updates.

### Multi-stage

**Multi-stage access reviews**

Microsoft Entra supports up to three review stages, letting different reviewers check access in sequence. This helps meet audit needs and makes the review process easier by sharing decisions among reviewers.

## Entitlement Management

Automates user access requests, approvals, reviews, and expiration.

* Manages external users automatically.
* Prevents unnecessary long-term access.
* Users may not know what access they should have, and even if they do, they might have difficulty locating the right individuals to approve it.

Delegated managers create access packages with rules for approval and expiration. External users are invited and removed when access ends.

### Terms of Use

**Microsoft Entra Terms of Use**

Terms of use show important info to users before they access apps or data, ensuring legal and compliance checks.

Use cases include:

* Before accessing sensitive data or apps.
* Recurring reminders.
* Based on user roles.
* For all users.

![terms-of-use-inline](/content/1.the-lab/2.core-principles/16.governance-risk-compliance/terms-of-use-inline.png)

Terms are shown as PDFs and work on mobile devices. Conditional Access policies require users to accept terms before access. Admins can track who agrees or declines.
::

## GSA

Microsoft Entra Global Secure Access unifies Microsoft Entra Internet Access and Private Access to provide secure access to SaaS, internet apps, and private corporate resources.

* **Internet Access** protects users, devices, and data from internet threats while accessing public and Microsoft services.
* **Private Access** secures remote and office user access to private corporate resources.

### SSE

Together, they form a Security Service Edge (SSE) solution that integrates Zero Trust network, identity, and endpoint controls, enabling secure access from any location or device.

::important
This addresses challenges like reducing lateral movement risk via VPNs, securing internet assets, and improving service for remote offices.
::

> A client on user devices controls traffic, sending it through Entra Internet or Private Access, and uses real-time policies to keep resources safe.

![global-secure-access-v3](/content/1.the-lab/2.core-principles/16.governance-risk-compliance/global-secure-access-v3.png)

### Microsoft Entra Private Access

::caution
Traditional VPNs often give broad network access once connected, increasing security risks due to over-permissioned users and devices.
::

Microsoft Entra Private Access replaces VPNs to block lateral attacks, limit excessive access, and securely connect users—whether remote or in-office—to private corporate resources.

**Here’s how it works:**

* You create an enterprise application that acts as a container for your private resources.
* A network connector links this app to the private resources users need to access.
* You can control access to these resources in two ways:

1. **Quick Access:** Add private resources by defining their address (FQDN, IP, ports) into a single “Quick Access” app. Conditional Access policies can then be applied to this app.

	![quick-access-diagram](/content/1.the-lab/2.core-principles/16.governance-risk-compliance/quick-access-diagram.png)

2. **Global Secure Access app:** Offers finer control by creating multiple apps, each with specific private resources, user groups, and tailored conditional access policies — for example, different rules based on access method or time.

	![per-app-access-diagram](/content/1.the-lab/2.core-principles/16.governance-risk-compliance/per-app-access-diagram.png)

> This setup provides secure, flexible access tailored to your organization’s needs.

### Microsoft Entra Internet Acess

Microsoft Entra Internet Access is an identity-focused `Secure Web Gateway` (SWG) that protects users and devices from web threats when accessing `SaaS` apps and `internet resources`.

* **Protection against user identity or token theft:**
  Uses Conditional Access policies to perform compliant network checks at both authentication and data planes.

  * *Authentication plane:* Enforcement happens during user sign-in via Microsoft Entra ID.
  * *Data plane:* Works with services supporting Continuous Access Evaluation (CAE).
  * **Continuous Access Evaluation (CAE):** Apps and Entra constantly communicate to keep user access secure and up-to-date, adjusting or blocking access in near real-time if risks arise (like location changes or threats).

* **Tenant restrictions:** Prevent data leaks to other tenants, personal accounts, or anonymous users.

* **Traffic forwarding policies:** Control which internet sites remote workers can access to ensure secure and managed internet connections.

* **Web content filtering:** Regulate access to websites based on content categories and domain names.

* **And many more advanced security controls.**
