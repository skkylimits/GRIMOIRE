If **[authentication](https://learn.microsoft.com/en-us/entra/identity/authentication/)** is the key that opens the door, **authorization** decides where you can go and what you can see inside.

> OAuth 2.0 handles authorization — it gives you permission and defines what you can do with access tokens and refresh token (What can you do?)

## Authorization

When you authenticate a user, you need to decide where they can go, and what they're allowed to see and touch. This process is called authorization.

> Suppose you want to spend the night in a hotel. The first thing you do is go to reception to start the "authentication process." After the receptionist has verified who you are, you're given a keycard and can go to your room. Think of the keycard as the authorization process. The keycard will only let you open the doors and elevators you're permitted to access, such as for your hotel room.

## Authorization Types

These describe **how you define and organize access** — the **structure** or **logic** behind the decisions. They’re about *what rules you use to decide who gets access* and *how those rules are expressed.*

* **ACLs** — Explicit lists of allowed/denied users or groups.
* **RBAC** — Access based on assigned roles
* **ABAC** — Access based on attributes of users, resources, environment
* **PBAC** — Combines roles with policies (broader rules)

Think of these as **the frameworks or blueprints** that guide *who* can do *what*.

## Authorization Components

* **Entitlements:** Define *who* has access to *what* resources, assigned via groups, roles (RBAC), attributes (ABAC), or policies (PBAC).
* **Access Policies:** The rules that govern *what* users can do with resources (the how and when).
* **Enforcement:** Describes *where* and *how* these access decisions are implemented—commonly within applications, through network proxies, or centralized policy engines.

## Authorization Security Techniques

Authorization controls what an authenticated user can access.

### Zero Trust

Is a broad security model/philosophy that incorporates techniques like `Conditional Access` and `Least Privileged Access`.

- **Verify Explicitly:** Every request is authenticated and authorized before access.
- **Use Least Privileged Access:** Limit permissions minimaly. Only access to what they need to.
- **Assume Breach:** Plan for breaches and minimize attacker movement within the network.

### Conditional Access

Is a specific `enforcement tool` to apply rules based on conditions like device state, location with if/then logic.

> Let’s see how this would work in an IT scenario. Increasingly, people are working from home. Because of this, they might be using their personal computer to access work-related content. With conditional access, an organization might grant access for an authenticated user to a confidential system, such as payroll, only if it's made through secure corporate computers located at their headquarters. If the authenticated user tries to access the payroll system from a personal computer at home, they would be blocked.

### Least Privileged Access

**Give only minimal necessary rights.**

Typically enforced through `RBAC` and tools like `PIM` that control role assignments and privileges.

> For example, when you board an airplane, you have access to the main cabin area to get to your seat, but no passenger is allowed in to the cockpit. Also, if you're traveling with a coach-class ticket, you only have access to that section. To improve security, each person can only access the areas they need to.

### PIM

`JIT` is a feature within PIM that allows temporary privilege access.

> Imagine you need to fix an urgent issue on a secure server. Instead of having permanent access, JIT allows you to request access just for the duration of the task. After the time ends, your access is revoked automatically, minimizing exposure.

### Lateral Movement Prevention

**Limit how attackers can move after breach.**

Is more of a network and architecture design approach—often implemented using segmentation, firewalls, endpoint protection, etc., rather than a single authorization tool.

> Think of an office building where a criminal gets past the security of the main reception area. They can then generally move around the rest of the building, accessing different floors and offices. It's important to provide added layers of security to protect against intrusion in sensitive areas.



**Implementing these techniques helps safeguard data and resources from unauthorized access.**
